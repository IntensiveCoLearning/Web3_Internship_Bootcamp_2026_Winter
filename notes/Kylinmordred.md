# 交易技术分析核心笔记

## 一、 核心交易工具概念

1.  **闪兑**
    
    -   定义：免挂单，平台实时定价一键兑换加密货币，支持跨链。
        
    -   特点：即时性、操作简单，汇率含手续费。
        
    -   适用场景：新手换币、紧急调仓、跨链资产转移。
        
2.  **跨链**
    
    -   定义：实现不同区块链网络资产 / 数据互通的技术，解决 “链孤岛” 问题。
        
    -   实现方式：公证人机制、侧链 / 中继链机制、哈希锁定。
        
    -   应用场景：跨链兑换、跨链 DeFi 操作、跨链数据共享。
        
3.  **冰山单**
    
    -   定义：大额订单拆分为 “可见 + 隐藏” 部分，仅展示小部分，降低市场冲击。
        
    -   逻辑：成交后自动补充可见量，直至全部成交。
        
    -   适用场景：机构 / 大户大额建仓 / 减仓、浅深度币种交易。
        
4.  **只做 Maker（Post Only）**
    
    -   定义：订单仅挂入订单簿不立即成交，确保成为流动性提供者。
        
    -   优势：享受更低手续费，订单若会即时成交则被取消 / 调整。
        
5.  **啄仓（追逐的啄啄仓）**
    
    -   定义：小额、多次、持续交易，试探市场或逐步建仓 / 减仓。
        
6.  **美国线**
    
    -   构成：竖线（最高 - 最低）+ 左右短横线（开盘 - 收盘）。
        
    -   特点：简洁，适合多周期叠加分析，无涨跌视觉强化。
        
7.  **K 线（蜡烛图）**
    
    -   构成：实体（开盘 - 收盘，阳线涨 / 阴线跌）+ 上下影线（最高 - 最低）。
        
    -   经典形态：光头光脚线、十字星、锤子线、上吊线。
        
    -   作用：判断趋势、支撑阻力、多空博弈及反转信号。
        

## 二、 常用技术分析指标

### （一） 趋势类指标（判断价格方向）

| 指标名称 | 核心原理 | 实战用法 |
| --- | --- | --- |
| 移动平均线（MA/EMA） | 平均价格平滑波动，EMA 加权近期价格更灵敏 | 多头排列（短 > 长）看涨；空头排列看跌；金叉看涨 / 死叉看跌 |
| 布林带（BOLL） | 中轨（MA20）±2 倍标准差，界定波动区间 | 价格破上轨超买、破下轨超卖；收口待突破、开口趋势延续 |
| MACD | 短期 EMA - 长期 EMA，含 DIF、DEA、柱状图 | 金叉看涨 / 死叉看跌；顶背离（价高指标低）看跌；底背离看涨 |

### （二） 震荡类指标（判断超买超卖，适用于横盘）

| 指标名称 | 取值范围 | 实战用法 |
| --- | --- | --- |
| RSI（相对强弱指数） | 0-100，默认参数 14 | >70 超买、<30 超卖；背离提示趋势反转 |
| KDJ（随机指标） | 0-100，默认参数 9,3,3 | K/D>80 超买、<20 超卖；金叉 / 死叉信号；J>100 或 < 0 预警 |
| WR（威廉指标） | 0- -100（部分平台 0-100） | <-80 超卖、> -20 超买 |

### （三） 量能类指标（判断资金热度，验证趋势）

| 指标名称 | 核心原理 | 实战用法 |
| --- | --- | --- |
| VOL（成交量） | 展示周期内交易总量 | 量价同步（涨放量 / 跌缩量）趋势健康；量价背离趋势乏力 |
| OBV（量价趋势指标） | 累计成交量判断资金流向 | OBV 上升资金流入看涨；下降资金流出看跌；背离提示反转 |

## 三、 指标使用注意事项

1.  单一指标有局限，建议**趋势 + 震荡 + 量能**组合使用（如 MACD+RSI+VOL）。
    
2.  趋势行情中，震荡类指标超买超卖信号易失效，优先顺势操作。
    
3.  指标参数可灵活调整，适配不同品种（股票 / 加密货币）和周期。

# Web3 与 Web2 就业市场对比笔记

核心结论：Web3 与 Web2 就业市场基础逻辑类似，但岗位方向、能力要求、发展路径差异显著 ——Web2 晋升赛道拥挤，Web3 机会更集中但对经验门槛要求高，合规相关岗位为核心刚需方向。

## 一、整体市场概况对比

-   **共性**：均以技术岗为核心需求，产品、运营、市场等非技术岗为支撑，核心围绕 “产品落地、用户服务、商业增长” 展开。
    
-   **差异核心**：Web2 市场成熟饱和，Web3 处于快速发展期，岗位增量与人才缺口并存，但受行业周期影响明显。
    

## 二、核心维度详细对比

### 1\. 岗位方向

-   **Web2**：聚焦传统互联网场景，岗位类型成熟，包括前端 / 后端开发、产品经理、运营（用户 / 内容 / 活动）、市场推广、运维等，技能要求偏向通用型。
    
-   **Web3**：岗位围绕区块链生态，新增大量专属方向，技术岗以智能合约开发、区块链底层协议开发（Rust/Go）、安全审计为主；非技术岗重点突出合规（合规官、KYC/AML 专员、加密资产律师）、Tokenomics 设计、社群运营等，且合规岗位跻身需求 Top10。
    

### 2\. 晋升与竞争

-   **Web2**：晋升渠道固定（初级→中级→高级→管理），行业成熟导致人才供给过剩，初中高级岗位竞争激烈，“内卷” 明显。
    
-   **Web3**：晋升路径更灵活（技术专精 / 生态资源整合双路线），岗位增量显著，但 67%“初级岗位” 要求 3 年以上经验，中高级岗位需求占主导，新人需靠实战项目积累竞争力。
    

### 3\. 能力与经验要求

-   **Web2**：重视学历背景与通用技能，应届生友好度高，入门后可逐步积累经验晋升。
    
-   **Web3**：更看重实战经验与行业认知，学历门槛宽松但能力要求复合 —— 技术岗需掌握区块链底层逻辑（共识机制、加密算法），非技术岗需懂链上经济模型，合规岗需兼具国际法规与本地政策理解能力。
    

### 4\. 核心关注点

-   **Web2**：聚焦用户增长、业务规模化、商业变现效率，对合规的关注度集中在传统数据安全与行业规范。
    
-   **Web3**：合规为核心生存前提，同时重视安全审计、链上性能优化、生态协同，所有岗位均需适配去中心化特性。
    

### 5\. 薪资与福利

-   **Web2**：薪资梯度稳定，初级年薪 3-4.6 万，高级 7-9.8 万，福利体系完善但弹性小。
    
-   **Web3**：薪资溢价明显，技术岗薪资为 Web2 的 2-4 倍，非技术岗月薪普遍翻倍，且含 Token 激励（占收入 20%-40%），90% 岗位支持远程办公。
    

## 三、关键行业趋势

1.  **Web3 特有趋势**：AI 技能已成为必选项，牛熊周期影响岗位波动（基础设施、安全、合规岗受影响最小），头部企业贡献 40% 岗位需求，马太效应突出。
    
2.  **Web2 转型趋势**：部分从业者向 Web3、AI 等赛道转型，核心竞争力从 “通用技能” 向 “跨领域复合能力” 迁移。

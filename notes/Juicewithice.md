# 2026-01-22
<!-- DAILY_CHECKIN_2026-01-22_START -->



DApp：

-   DApp 是由前端、智能合约和区块链网络共同组成的去中心化应用，而不仅仅是合约本身。  
    区块链主要负责状态存储与共识，智能合约负责业务逻辑的确定性执行。  
    钱包在 DApp 中承担身份认证和交易签名的角色，而不是传统账号体系。  
    用户与 DApp 的交互流程本质是前端触发请求、钱包签名交易、合约执行并更新链上状态。  
    Gas 是链上计算和存储资源的定价方式，每一次合约调用都会消耗 Gas。  
    主网、测试网和本地链在安全性、成本和用途上不同，开发阶段应优先使用测试网或本地链。  
    DApp 的安全风险不仅存在于合约代码，也可能来自前端逻辑、私钥管理和交互设计。
    

Hardhat：

-   Hardhat 是一个以 JavaScript 和 TypeScript 生态为核心的以太坊开发框架。  
    Hardhat 提供本地 EVM 网络，便于开发者进行快速调试和测试。  
    Hardhat 支持合约的编译、部署、测试和调试的一体化开发流程。  
    Hardhat 的测试体系通常基于 Mocha 和 Chai，更贴近传统 Web 开发者习惯。  
    插件机制使 Hardhat 可以快速集成 ethers 等主流 Web3 工具。  
    Hardhat 更适合中大型项目、多人协作以及工程结构较复杂的场景。  
    Hardhat 注重工程可维护性，但在执行速度和底层控制上相对保守。
    

Foundry：

-   Foundry 是一套基于 Rust 实现的以太坊开发工具链，强调性能和底层控制能力。  
    Foundry 使用 Solidity 本身编写测试，使测试逻辑与合约逻辑高度一致。  
    Forge 是 Foundry 中用于合约编译、测试和调试的核心工具。  
    Foundry 的测试执行速度非常快，适合高频迭代和复杂逻辑验证。  
    Cheatcode 允许在测试中直接操控区块时间、高度和账户状态。  
    Foundry 对开发者的 Solidity 和 EVM 理解要求更高。  
    Foundry 特别适合合约安全研究和审计相关工作。
    

三者适用场景：

-   DApp 基础解决的是区块链应用如何运行的问题。  
    Hardhat 解决的是 Web3 项目的工程化和团队协作问题。  
    Foundry 解决的是合约层面的深度验证和性能效率问题。

## 一、Sepolia 是什么？

Sepolia 是 **以太坊官方推出的一条测试网络（Testnet）**，主要用于 **智能合约和 DApp 的开发与测试**。  
在 Sepolia 上使用的是 **测试 ETH（Sepolia ETH）**，不具备任何经济价值。

* * *

## 二、为什么会有 Sepolia？

早期以太坊测试网（Ropsten、Rinkeby、Goerli）存在以下问题：

-   部分采用 PoW，维护成本高
    
-   测试币被滥用、倒卖
    
-   链上历史数据膨胀，状态不干净
    

随着以太坊转向 **PoS 共识机制**，官方需要一条：

-   干净
    
-   稳定
    
-   长期维护  
    的新测试链，因此诞生了 **Sepolia**。
    

* * *

## 三、Sepolia 链是怎么来的？

Sepolia **不是主网分叉**，而是：

-   由以太坊核心开发者
    
-   通过 **重新创建创世区块（Genesis Block）**
    
-   独立启动的一条新链
    

Sepolia 具有：

-   全新的链 ID
    
-   全新的账户状态
    
-   独立的验证者集合
    

* * *

## 四、Sepolia 的启动过程（简化）

1.  编写 Sepolia 的 Genesis 配置文件
    
2.  配置初始验证者节点
    
3.  使用以太坊客户端启动网络
    
4.  生成创世区块，链开始运行
    

* * *

## 五、Sepolia 使用的共识机制

-   采用 **PoS（权益证明）**
    
-   由官方和社区验证者维护
    
-   不支持挖矿
    

因此：

-   Sepolia ETH 不能挖
    
-   只能通过 Faucet 领取
    

* * *

## 六、Sepolia ETH 是什么？

Sepolia ETH 是 **测试币**，特点：

-   无实际价值
    
-   用于支付 Gas
    
-   用于测试转账和合约部署
    

来源：

-   创世区块分配
    
-   官方 Faucet 发放
    

* * *

## 七、Sepolia 与以太坊主网的关系

-   两者 **完全独立**
    
-   数据不互通
    
-   Sepolia 上的操作不会影响主网
    
-   合约需重新部署到主网
    

* * *

## 八、为什么开发者常用 Sepolia？

-   链状态干净
    
-   出块快、稳定
    
-   运行环境接近主网
    
-   官方长期维护
    

主要用途：

-   智能合约开发
    
-   DApp 功能测试
    
-   教学与实验

# 2026-01-16
<!-- DAILY_CHECKIN_2026-01-16_START -->









ETH安全依赖节点数量和节点分散程度。

**全节点（Full Node）**

-   验证所有区块
    
-   保存当前状态
    
-   常用于开发、RPC 服务
    

**归档节点（Archive Node）**

-   保存所有历史状态
    
-   查询能力最强
    
-   成本最高
    

**轻节点（Light Node）**

-   只保存区块头
    
-   依赖全节点验证
    
-   适合钱包、移动端

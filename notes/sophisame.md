## 加密哈希函数

-   可以将任何字符串作为输入，产生固定长度输出（本节课采用固定256bit作为输出）
    
-   三个属性
    
    -   collision-free碰撞自由性：即不存在 两个不同的输入会产生相同的输出。（无碰撞）
        
        -   意味着没人可以找到x和y
            
        -   x和y是不同的，但是x和y的哈希值是相同的（ H(x) =H(y) )
            
        -   解释：因为当两个不通过的输入产生相同的输出时，说明碰撞是存在的，而加密哈希函数需要“没人能发现碰撞“（因为事实上碰撞是存在的），所以具有collision-free
            
            -   碰撞为什么一定存在
                
                -   因为我们有很多可能性的输入，但是输出确实固定长度的，意味着输入大于输出的量，所以存在两个不同的输入可能产生相同的输出
                    
        -   Q：如何找到碰撞
            
            -   随机选择130个输入，就会发现至少有两个它们之间会发生碰撞的概率有99.8%
                
                -   这必须算出130个数的哈希值，然后找出每两个之间相同的概率（很麻烦）
                    
        -   因为没有人证明出哪个加密哈希函数不存在碰撞，所以我们假定我们有一个无碰撞（冲突）的哈希函数。那无碰撞（冲突）有什么意义呢？
            
            -   假设我们记住一个文件的哈希，之后又有一个新文件，我们需比较这两个文件哈希是否一致，一致的话，我们得出结论，这两个文件是相同的（因为我们的哈希函数是无碰撞的，意味着这两个文件x和y不能是不同的）。这有利于我们知道旧文件的摘要
                
    -   hiding隐藏性：即使知道输出，也无法推算出输入x。
        
        -   为了让别人很难通过h（x）找到输入值x，我们必须引入隐形值r，r是从具有高最小熵的分布中选择的随机值（即r是从真正分散的发行版中选择的，则r没有特定的值）。则把x和r串联起来将h（x)变为 h(r| x)则意味着取出r的所有数加上x所有数。这样就很难找到x了
            
        -   应用
            
            -   承诺

固定总量 + 线性发行：HOOK 总量固定为 5 亿，后续无「减半」事件。

销毁机制：当用户在 Hooked 生态中使用 HOOK 购买某些应用内物品时，  这部分被花费的 HOOK 会被永久销毁，以对抗通货膨胀、并在长  期内为代币增加一定的通缩压力。

随着 Hooked 生态中 NFT 与 Learn & Earn 模块活跃度提升，  对质押与交易手续费的需求同步上升，推动 HOOK 内在价值增长。

# pendle协议

我们将资产通过协议转化为生息代币aUSDC，再将aUSDC包装为SY-aUSDC，将每个SY-aUSDC拆分成一个PT-aUSDC+一个YT-aUSDC。YT代表的就是从今天开始到到期日这段时间里，整个SY产生的收益（利息）

一个SY可以拆分成一个PT+一个YT。相应的一个YT+一个PT也能总换成SY（直接合并拆分）

详细介绍

Underlying APY：过去七天里stETH在Lido的实际年化，实际资产能产生的收益 Implied APY:市场认为这个资产的年化是多少(收益多少)

Long Yield APY:因为 Implied APY和Underlying APY的差异，根据这两个差异，协议计算出来的静态收益

解释：你现在买YT，价格可能是年化6.4%，而过去七天实际只有5%，所以意味着买完YT后可能会亏19.17%

Fixed APY：市场认为年化是6.4%，如果你现在买PT，持续到到期日结束会得到5.8的年化

## PT详细信息

Liquidity市场流动性，24h Volume最近成交量，Underlying APY：过去七天里stETH在Lido的的实际年化，实际资产能产生的收益 。Fixed APY：你现在买入PT并持有到到期日能得到的年化(意味着你的PT不在Lido里)

# uniswap协议：LP挖矿：AMM恒定乘积做市函数

uniswap协议是以池子的方式进行交易的形式。这是一种做市函数。我们是和资金池做交易，所以Dex里需要引入新角色：LP的提供者，他们维持了整个交易的流动性。LP提供者就是普通用户的对手。

```
Uniswap和中心化交易所的区别
```

```
区别于其他中心化交易所，中心化交易所中每个人的对手都是另一个人，而在AMM池子里我们的对手是整个协议，和资金池做交易原因是中心化交易所是以账单的形式记录交易，而这么庞大的数据无法全部放在区块链，他们一般使用了中心化的服务器。所以对于uniswap中只能选择退而其次的办法
```

LP提供者提供流动性。

## AMM恒定乘积做市函数

### V2池子

-   x\*y = k = L^{2}
    

x是一个代币的数量，y是另一个代币的数量，k为常数，L就是流动性

所以只有x和y改变了，之间成反比

-   如果想添加流动性，池子两边的价值都是50%：50% ：两边相等
    

x/y =P(汇率） ： 假如x相当于稳定币的数量，y当成以太坊数量，就可以得到一个汇率

\==提供流动性可以得到交易手续费==，因为一般交易是需要自己提交手续费的，而uniswap中不用。但这种好处也 是有成本的。

对于LP来说是有无常损失的，无常损失 = 持币不动价值 - 当前LP仓位价值

T0时间是添加流动性的时候，T1,T2分别为两次变化。并且假如y是稳定币。一开始在T0的时候，30000✖️2=6000 和y值一样，因为池子两边是相等的。之后经过一轮做市后得到T1的数据。持币不动数据如何得来：假设在T0时刻不做市，那么最终会得到：4000✖️2+6000=14000。意味着如果你不做市，你的仓位将价值14000，这比做市后的13856高，所以意味着你亏了14（41.02%）

这就是无常损失

为什么到了T1的时候x价格变高，并且x数量变低：因为在T0-T1之间，有人在不断的买x，所以x的价格被拉高了，而我们则在卖x，在比4000低的价格，3000-4000之间我们一直在卖x。所以导致了无常损失。

# 概述

区块链是一种分布式账本技术，通过加密链式结构和共识机制，实现去中心化、数据不可篡改和高度透明的记录存储。它不仅支撑数字货币，还广泛应用于供应链、金融、医疗、政务等领域，并正随着智能合约与 Layer 2 方案的发展进入新阶段。以下章节将详细阐述区块链的各方面内容。

# 基本概念

定义：区块链是一种分布式数字账本，记录所有网络参与者之间的交易或状态变更，各节点共享同一账本副本，具有防篡改和可追溯特性​

特征总结：它融合了分布式存储、加密算法、共识协议和点对点网络，构成一个无需信任第三方的可信系统。

# 核心特性

去中心化：数据不依赖单一服务器或机构，而是在所有节点间同步，消除单点故障与托管风险。

不可篡改：每个区块包含上一区块的哈希值，一旦写入就难以篡改，保证历史记录的完整性和可审计性​。

透明性与可追溯性：链上交易对所有参与者可见，任何修改都会留有痕迹，便于审计和监管。

# 架构组成

区块（Block）：包含若干已验证的交易数据、时间戳、前一区块哈希等内容，构成账本的基本单位。

在区块中我们把数据分成了块高，Nonce，Data，需将这三个进行哈希运算得到哈希值。当我们提供了这三个值放入哈希函数再”Mine”后会得到一个哈希值：此时的Nonce值就会改变。早起矿工挖矿就是解决这个问题：矿工需找到一个Nonce，这个Nonce需和块高和Data一起进行哈希运算，结果必须是以4个0开头的哈希值。

节点（Node）：运行区块链软件的计算机，存储账本副本并参与交易验证，分为全节点和轻节点两类。

点对点网络（P2P）：节点之间通过对等网络协议广播交易和区块，实现信息传播与同步​。

智能合约（Smart Contract）：部署在链上的自动执行代码，用于实现去中心化的业务逻辑和资产管理。

\## 共识机制

\*为什么要使用POW ：

比特币区块链就是所有用户均可查看的公共交易数据库（账本）。试想一下，某人和其他三位好友共用一本记事本记录自己的比特币交易。任何时候，其中一人要进行价值转移时需要记下上一笔交易的信息，这样我们就有办法追踪以上BTC金额，但这是这三个人相互信任的前提下。但当人数增大时没人会信任陌生人掌管账本。而工作量证明确保用户不得支付无权动用的资金。工作量证明算法结合密码学，根据系统规则，让所有人都能更新区块链。

Proof‑of‑Stake（PoS）：节点通过抵押代币获得打包权，降低能耗并提高吞吐，已被以太坊等主流网络采用。

在权益证明系统中，区块是“锻造”而非“挖矿”而来。参与锻造过程的用户必须将一定数量的代币锁定到网络中，作为他们的权益而存在。权益的大小决定了选择一个节点作为下一个验证者的机会，权益越大，机会越大。为确保该过程不仅仅偏向网络中最富有的节点，这个选择过程添加了很多特殊的方式。最常用的两种方法是“\_随机区块选择\_(Randomized Block Selection)”和“\_币龄选择\_(Coin Age Selection)”。

其他机制：包括Delegated PoS、Practical Byzantine Fault Tolerance (PBFT)、Hashgraph 等，适用于不同信任与性能需求。

# 智能合约与代币设计

智能合约：是指在区块链上自动执行、控制和记录合同条款的程序。它的执行是完全自动化的，无需依赖中介或第三方。

### 智能合约的特点：

自动执行：一旦满足特定条件，智能合约会自动执行。

不可篡改性：合约一旦部署到区块链上，任何人都无法更改其中的代码。

透明性：所有的操作、条款和交易都是公开的，可以被任何人查看。

去中心化：不依赖任何中心化机构。

### 智能合约如何工作：

智能合约的代码通常在 以太坊 这样的智能合约平台上执行。以太坊的智能合约是用 Solidity 语言编写的。

执行过程：当合约条件满足时（如收到某个交易），智能合约会被触发。

合约会根据预设规则进行计算，并自动执行，完成后更新区块链上的数据。

### 智能合约的实际应用：

ICO（首次代币发行） ：通过智能合约实现投资者的资金管理和代币分发。

DeFi 协议：比如借贷协议、去中心化交易所（DEX）等。

NFT（非同质化代币） ：智能合约用于管理 NFT 的铸造和交易。

### Defi介绍

Defi的特点：

**无许可**：任何人都可以参与，且不需要传统的信用认证。

**去中心化**：无需传统的金融机构，所有交易通过智能合约执行。

**透明性**：所有的交易记录在区块链上，任何人都可以查看。

\##### NFT介绍

**NFT（Non-Fungible Token，非同质化代币）** 是一种基于区块链技术的数字资产，具有\*\*唯一性\*\*和\*\*不可替代性\*\*。\*\*与普通加密货币（如 BTC、ETH）不同，NFT 不能被拆分或相互替换\*\*，每个 NFT 都是独特的，就像现实世界中的收藏品或艺术品。

**NFT 的核心特征**：

\- **唯一性**：每个 NFT 都有独特的元数据，无法复制或篡改。

\- **不可分割性**：不像 BTC 或 ETH 可以拆分，NFT 通常必须整体交易。

\- **可验证性**：NFT 所有权可以在区块链上公开验证。

\- **可交易性**：NFT 可在各种 NFT 市场（如 **OpenSea、Blur、Rarible**）上买卖。

**NFT 的主要应用场景**：

\- **数字艺术**：艺术家可以创建和出售数字艺术作品，如 Beeple 的 NFT 作品在佳士得拍卖行卖出 6900 万美元。

\- **游戏资产**：如 Axie Infinity、The Sandbox，玩家可以拥有、交易游戏内资产。

\- **虚拟房地产**：如 Decentraland、The Sandbox 中的虚拟土地 NFT。

\- **音乐和娱乐**：艺术家可通过 NFT 直接向粉丝出售音乐、门票等。

\- **身份认证**：NFT 可作为数字身份凭证，如 **ENS（Ethereum Name Service）** 。

**NFT 生态**：

\- **NFT 发行平台**：Rarible、Zora、Foundation

\- **NFT 交易市场**：OpenSea、Blur、Magic Eden（Solana 生态）

\- **NFT 基础设施**：Arweave（去中心化存储）、IPFS（文件存储系统）

**NFT 市场趋势**

\- 蓝筹 NFT（如 CryptoPunks、BAYC）仍然是市场标杆。

\- **NFT-Fi（NFT+DeFi）** ：NFT 质押借贷、碎片化交易等新玩法兴起。

\- **品牌 NFT 化**：Nike、Adidas 等公司推出 NFT 相关产品。

\##### DAO\*\*（去中心化自治组织）\*\*

**DAO（Decentralized Autonomous Organization，去中心化自治组织）** 是基于智能合约运行的组织，管理权力由代币持有者投票决定，而不是由传统的公司管理层控制。

**DAO 的核心特点**：

\- **去中心化**：所有决策由社区成员投票决定，而非中心化机构。

\- **自治性**：DAO 通过智能合约自动执行组织规则，减少人为干预。

\- **透明性**：所有治理活动和资金流动都记录在区块链上，公开透明。

\- **激励机制**：成员通常通过治理代币（如 ENS、\*\*AAVE\*\*）参与投票，并获得奖励。

**DAO 的运作方式**：

1\. **代币治理**：持有 DAO 的治理代币，即可在组织内投票决定重要事务。

2\. **社区提案**：成员可提交治理提案，如资金分配、项目合作等。

3\. **投票机制**：多数 DAO 采用链上投票（如 Snapshot、Tally）决定提案是否通过。

4\. **资金管理**：DAO 资金通常存放在多签钱包（如 Gnosis Safe）中，需多人授权才能执行交易。

# 交易与签名

## 交易（tx）

-   做的事+手续费+防重放序号
    
    -   做的事：转账，调用的函数，投票
        
    -   手续费：网络资源付费
        
-   钱包
    
    -   组装交易内容
        
    -   在交易时用私钥签名：data中包含基本信息
        
    -   广播到区块链网络
        
-   数字签名
    
    -   私钥推出地址
        
    -   通过密码学不暴露私钥的情况下，验证是你的私钥推出的交易
        
-   gas fee
    
    -   三层作用
        
        -   防垃圾交易淹没网络（提高作恶成本）
            
        -   激励打包者/验证者

# 共识机制：POW vs POS

-   POW(工作量证明）BTC
    
    -   算力竞争写账
        
    -   更有算力资源的人更有优势
        

-   Pos（权益证明）ETH
    
    -   质押+随机选人写账（更节能）
        
    -   如果造假，押金会被没收，每个人都可验证
        
    -   交更多押金的人更有优势

-   学习了 DeFi 是一类建立在区块链上的金融系统，用智能合约替代银行和中介，提供借贷、交易、理财等功能。
    
-   了解到它的核心特征是无需许可、自动执行、资金自托管，用户直接用钱包和合约交互。
    
-   记下常见模块包括：去中心化交易所（DEX）、借贷协议、稳定币、收益农场等。
    
-   学习到利率、清算、分红规则都写在合约里，由程序自动触发，而不是人工审批。
    
-   也意识到主要风险来自：智能合约漏洞、预言机错误、清算机制失效，以及系统性连锁风险。

# 2026-01-26
<!-- DAILY_CHECKIN_2026-01-26_START -->

-   预言机是把链下数据引入区块链的中间层，比如价格、利率、天气、比赛结果等。
    
-   智能合约本身不能访问外部世界，所有外部信息必须通过预言机提供。
    
-   常见模式是由多个节点同时喂价，通过聚合算法得到最终结果，降低单点作恶风险。
    
-   预言机直接影响 DeFi 的清算、借贷和衍生品定价，是系统中的高风险基础设施。
    
-   如果预言机数据被操纵，合约逻辑本身是正确的，也可能导致大规模资金损失。
<!-- DAILY_CHECKIN_2026-01-26_END -->

# 2026-01-25
<!-- DAILY_CHECKIN_2026-01-25_START -->


-   Layer 2 是在主链之外处理交易，再把结果提交回主链，用来缓解以太坊拥堵、降低 Gas。
    
-   目标是在不牺牲主链安全性的前提下，提高吞吐量和交易确认速度。
    
-   主要实现方式是 Rollup，把多笔交易打包后一次性上链，其中分为 Optimistic Rollup 和 ZK Rollup 两类。
    
-   资产在 L2 上使用前，需要先通过跨链桥从主链转入，安全性很依赖桥和验证机制。

# 2026-01-23
<!-- DAILY_CHECKIN_2026-01-23_START -->



-   弄懂了智能合约不是“合同文件”，而是部署在区块链上的一段程序，条件满足就自动执行，不需要人工确认。
    
-   明白它在链上的作用：转账、发币、投票、分红，其实都是通过调用智能合约函数来完成的。
    
-   开始理解一个关键点：合约一旦部署基本不能随便改，所以代码写错，后果会直接在链上放大。

# 2026-01-22
<!-- DAILY_CHECKIN_2026-01-22_START -->




-   弄懂了 DAO 就是一种在区块链上运行的组织，它没有 CEO 或董事会，规则全部写在智能合约里，大家按照合约来行动。所有操作和钱的动向都能在链上看到，不容易被篡改。
    
-   DAO 的运作基本靠治理代币和投票：持有代币的人可以发起提案、参与投票，投票结果一旦达到预设条件，智能合约就会自动执行决策（比如动用资金、改规则）。
    
-   核心逻辑是“规则是代码、代码是宪法”：成员的权力大小通常跟持币数量有关，大家一起决定组织方向，不需要中心化的管理层。
    
-   也意识到一些局限：因为智能合约自动执行，所以代码有漏洞可能会出问题，而且如果大户掌握太多代币，投票可能会偏向少数人。

-   今天对 Uniswap v3 的整体思路更清楚了：它不是像以前那样把资金铺在 0 到无穷，而是让 LP 自己选一个价格区间去做市，价格跑出区间就不再赚手续费。

-   个人感受：v3 把做市从“被动放钱”变成了“需要持续管理的策略”，资金效率高了，但对普通 LP 的门槛也明显提高了。

# 2026-01-15
<!-- DAILY_CHECKIN_2026-01-15_START -->









-   听了 rich 老师的分享，了解到为什么 web3 对于 ai 时代的存在更具有重要意义
    
-   AI智能体就像有“大脑、记忆、工具”的组合体：大脑是GPT-5.2、DeepSeek V3.2这些LLM模型，负责推理和规划；记忆靠向量数据库，通过RAG流程存数据；工具是API接口，能连外部世界执行代码、查数据库。传统LLM是“无状态”的，而智能体靠向量数据库和API有了持久记忆和执行身份
    
-   未来机器经济的崛起，agent 会更加普遍，机器间的交易会更小而频繁， 交易需要更简化，绕开中心化机构或平台，避免多次不要的手续费
    
-   web3的基础设施更能承载灵活性要求强的交易方式，x402协议是目前 agent 调用 api 绕开中性化平台的一个演示
    

区块链给AI智能体提供了不可篡改的身份证明和原生银行账户，协议栈分三层：

• 应用框架层：SpoonOS Framework，是开发者操作系统

• 身份层：ERC-8004标准，相当于智能体的“链上护照”，基于ERC-721有唯一链上ID，还有声誉系统记录评分、TEE/zkML验证机制解决“是谁、可不可信”的问题；

• 交易层：X402协议，是HTTP原生的机器支付协议，支持按文件、按算力付费，不用订阅

最后，分享会还是要重复观看，边看边查，搞懂自己没听懂的部分

# Web3 运行原理

## 一、Web3 到底在解决什么问题？

**核心问题：**

在没有中心化机构的情况下，

如何确认“你是谁”

如何保证“账本可信”

如何让“规则自动执行”

**Web3 的答案：**

-   私钥 & 数字签名 → 身份与授权
    
-   去中心化网络 + 共识 → 可信账本
    
-   智能合约 → 自动执行规则
    

* * *

## 二、钱包、私钥、助记词：Web3 的身份系统

### 1\. 三者关系

-   **私钥（Private Key）**
    
    -   资产和权限的唯一控制权
        
    -   区块链世界的“终极印章”
        
    -   丢了 ≈ 永久丢失（无法重置）
        
-   **助记词（Seed Phrase）**
    
    -   私钥的可读备份
        
    -   可以派生多个账户
        
-   **地址（Address）**
    
    -   公钥哈希后的字符串
        
    -   不是隐私，只是公开收款号
        
    -   行为可被链上分析关联
        

### 2\. 核心认知

-   区块链 **不认识人**
    
-   只认识：**谁能签名**
    
-   谁掌握私钥，谁就拥有资产与权限
    

### 3\. 私钥的社会学意义

-   任何人都可以随机生成私钥
    
-   不需要审批、身份、国家、机构
    
-   **个人主权的起点**
    
-   权利不是被“发放”的，而是自己生成的
    

* * *

## 三、交易（Transaction）是什么？

### 1\. 一笔交易包含什么？

交易 = 要做的事 + Gas Fee + Nonce

-   要做的事：转账 / 调用合约 / mint / 投票
    
-   Gas Fee：为网络资源付费 + 激励
    
-   Nonce：防止同一交易被重复执行
    

### 2\. 钱包真正做的事

-   组装交易数据
    
-   用私钥签名（授权）
    
-   广播到区块链网络
    

钱包不是“记账者”，只是“签名器 + 发信人”

* * *

## 四、数字签名：Web3 信任的根基

### 数字签名能保证什么？

-   任何人都可以验证交易是你发的
    
-   任何人都无法伪造你的签名（除非有私钥）
    

信任来自 **密码学**，不是机构

* * *

## 五、Gas Fee：为什么必须存在？

### Gas 的三层作用

1.  **防垃圾交易**
    
    -   提高作恶成本
        
2.  **激励节点/验证者**
    
    -   提供算力、带宽、维护网络
        
3.  **经济闭环**
    
    -   使用资源 → 付费 → 网络可持续
        

没有激励的去中心化系统无法长期运行

* * *

## 六、一笔交易是如何“跑起来”的？

### 从交易到出块的完整流程

Wallet（签名）  
→ RPC / Node（传播）  
→ Mempool（排队）  
→ Builder / Validator（挑选）  
→ Block（出块）  
→ Explorer（可查）

### 为什么需要“确认时间”？

-   区块像账本的一页
    
-   新区块引用旧区块 hash
    
-   越往后，历史越难被推翻
    
-   PoS 下有 Finality（最终确认），约 12–13 分钟
    

**不可篡改 ≠ 立刻不可改**

* * *

## 七、共识机制：全球陌生人如何记同一本账？

### PoW（工作量证明｜BTC）

-   拼算力
    
-   能耗高
    
-   规则简单、历史最久
    

### PoS（权益证明｜ETH）

-   质押 + 随机选验证者
    
-   更节能
    
-   作恶会被罚没质押资产
    

共识的本质：让作恶成本远高于收益

* * *

## 八、钱包 ≠ 网络：RPC 是现实中的风险点

### 真实关系

-   钱包通常只连一个 RPC
    
-   RPC 背后是节点或节点集群
    
-   RPC 是常见的“中心化入口”
    

### 抗审查来自哪里？

-   可换钱包
    
-   可换前端
    
-   可换 RPC
    
-   可换节点
    

**替换性 = 抗审查能力**

* * *

## 九、智能合约：链上的“自动执行规则”

### 智能合约是什么？

-   写在区块链里的可执行代码
    
-   在 EVM 中运行
    
-   由交易触发
    
-   自动改变链上状态
    
-   一旦部署，难以随意更改
    

### Code is Law 的含义

-   规则透明、可验证
    
-   执行历史可追溯
    
-   减少中介与信任成本
    

从“信人 / 信机构”  
转向“信代码 + 共识 + 治理”

* * *

## 十、区块链协议如何升级？（以太坊）

### EIP 流程

1.  社区讨论（论坛）
    
2.  形成提案文档（EIP）
    
3.  社区共识
    
4.  硬分叉升级（每年 1–2 次）
    

关键点：

-   没有 CEO
    
-   没有拍板人
    
-   协议是被治理的代码

## Web3 行业全局介绍 & 岗位概览

### 一、行业阶段与趋势

-   **Web3 四阶段演进**：  
    1）BTC 极客实验期 →  
    2）ETH + ICO 造富期 →  
    3）NFT / DeFi / L2 应用爆发 →  
    4）**合规 + ETF + RWA + DePIN + AI 融合（当前）**
    
-   **2024–2026 核心变化**：监管转暖、BTC/ETH 现货 ETF 通过、机构与传统资本大规模入场，行业从投机走向基础设施与实体结合
    

### 二、就业市场现状

-   **人效极高**：10 人团队可管理数亿–数十亿美金 TVL
    
-   **开发者稀缺**：全球活跃开发者仅约 2.5–3 万
    
-   **2025 招聘反弹**：新增岗位约 6.6 万（+47%）
    
-   **结构性失衡**：
    
    -   初级岗位极度内卷（≈450:1）
        
    -   资深 Rust / 合规 / Research 人才严重短缺
        

### 三、岗位与薪酬梯队

-   **第一梯队**：智能合约 / 安全审计 / 合规（高壁垒 + 高薪）
    
-   **第二梯队**：Research / 核心产品（懂 Tokenomics）
    
-   **第三梯队**：运营 / 增长（普通岗位低，但增长高手天花板高）
    
-   **区域分工**：
    
    -   技术核心：北美
        
    -   交易所 / 运营 / 资管：亚洲 & 中东
        

### 四、求职关键认知

-   **Web3 不看“简历”，看“链上履历”**：  
    GitHub、DAO 治理、Snapshot 投票、Mirror 输出
    
-   **核心渠道**：Web3 垂直平台 + Twitter/X + Discord + DAO
    
-   **必备能力**：
    
    -   Dev：Solidity → Rust / Move / Cairo + 安全意识
        
    -   非技术：英语 + Tokenomics + 链上交互经验
        
-   **避坑原则**：远离“三无项目”（无融资 / 无代码 / 全匿名）
    

### 五、个人反思

Web3 本质是**高杠杆、低人数、强结果导向**的行业，真正的竞争力来自可验证的链上行为与长期输出，而不是传统投递逻辑。

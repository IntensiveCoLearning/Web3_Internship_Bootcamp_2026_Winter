# **Web3运行原理**

**资产安全与钱包原理**

**1\. 安全警示**

 - 物理隔离： 助记词（Seed Phrase）

 - 防截屏/录屏：

 - 授权钓鱼： 在网页点击“授权”（Approve/Sign）时，必须核对网址和交易内容，防止被钓鱼网站骗取权限。

**2\. EOA (Externally Owned Account)：**用户的“身份证”和“银行卡”。Permissionless/个人主权

助记词 vs 私钥：

是“种子”或“根”。一个助记词可以派生出无限个私钥（一对多关系）。

私钥 (Private Key)： 对应唯一的公钥和地址（一对一关系）。

**交易**

**1\. 交易的构成要素**

当你在钱包中点击“发送”时，钱包在后台为你组装了一个数据包，包含：From/To，Value，Data，Gas Fee：Gas Price： 油价（网络越拥堵越贵）/Gas Limit： 油箱上限（防止死循环消耗无限费用），Nonce (随机数/序号)，Signature (数字签名)：使用私钥对上述所有信息进行加密签名/都可以用公钥解密签名，但无法反推出你的私钥。

**2\. 交易的生命周期**

1.  构造与签名 - 广播 (Broadcast) - 进入 Mempool (内存池) - 排序与打包 - 共识与上链 - 最终性 (Finality)：区块变为“不可逆”状态。
    

**区块链网络架构 (The Network)**

**1\. RPC (Remote Procedure Call) ：**钱包连接区块链的“桥梁”

 - 去中心化的悖论： 区块链本身是去中心化的，但用户访问链通常依赖中心化的 RPC 服务商（如 Infura, Alchemy）。

 - 风险： RPC 服务商可能审查你的交易、记录你的 IP、或者因服务宕机导致你无法转账。

 - 解决方案： 运行自己的节点，或者使用抗审查的去中心化 RPC 协议。

**2\. 共识机制的演进：POW vs POS**

 - POW (工作量证明 - Proof of Work)：比特币，谁算得快谁记账，极度耗能

 - POS (权益证明 - Proof of Stake)：以太坊，质押资产（32 ETH）作为押金，引入Validator (验证者)：=记账员，乱记账系统会触发Slashing (罚没) 机制，扣除你的押金。节能（99.9%），经济安全性更高。

**3\. 节点的分布与抗审查**

 **-** 全球分布

 - 抗审查性： 只要全球还有一个节点在运行，网络就不会停止。即使某个国家断网，其他国家的节点依然可以维护账本。

 - 客户端多样性 (Client Diversity)：如果一种软件出现 Bug，只要不是所有节点都用它，网络就不会崩溃（容错性）。

**智能合约 (Smart Contract)**

**1\. 概念理解**

 **-** 运行在区块链上的“自动售货机”。代码即法律 (Code is Law)。

 **-** 一旦部署，代码无法篡改（除非预留了升级接口），自动执行预设逻辑。

**2\. My First NFT**

 - 合约地址： 智能合约在链上的唯一标识。

 - Mint (铸造) 逻辑：用户调用 mint 函数 - 该地址是否已经 Mint 过？如果是新用户 -> 生成 NFT 数据（Token ID 映射到 Owner 地址） -> 写入链上状态/如果是老用户 -> 报错拒绝。

 - 数据存储： NFT 的本质是在合约的账本上记录了 TokenID : 0xUserAddress 的对应关系。

**3\. 合约升级**

 \- 不可篡改性： 传统软件可以随意更新，智能合约通常不能。

 - 升级方案：

代理模式 (Proxy Pattern)： 用户访问代理合约，代理合约将指令转发给逻辑合约。升级时，只需修改代理合约指向的逻辑合约地址。

数据迁移： 部署新合约，让用户把资产迁移过去。

  

# 行业法律与安全风险学习笔记

1.     中国监管环境：以“全面封堵金融属性”为核心

自2017年起，中国对虚拟货币及相关业务实施了一系列严格限制，形成了以“全面封堵金融属性，有限容忍技术创新”为核心的监管基调。政策明确禁止ICO、虚拟货币交易所运营、将虚拟货币作为支付工具，并打击利用其进行的非法集资、传销、洗钱等活动。

2.     核心法律风险点

\-   代币发行与交易的风险

任何形式的代币发行融资（如ICO、IEO、IDO）均属非法。即便以“积分”、“治理Token”等名义进行，只要具备融资或流通功能，就可能构成非法金融行为。项目方及相关技术人员（如合约开发者）均可能被追究法律责任，如非法吸收公众存款罪。

 -  高发的刑事犯罪风险

开设赌场罪：链游等项目中若存在“充值-随机收益-提现”的闭环结构

组织、领导传销活动罪：采用“多级推广”、“团队计酬”模式的NFT、DAO或挖矿项目风险

非法经营罪：利用虚拟货币进行外汇兑换或支付结算

非法吸收公众存款罪：以“挖矿”投资、虚拟币交易等名义向公众募集资金

洗钱罪：虚拟货币的匿名性使其易成为洗钱工具——反洗钱的重要性 

\-     民商事争议与维权困境

涉及虚拟货币买卖、委托投资的合同，因违反监管政策，在司法实践中通常被认定为无效，导致投资者“风险自担”

3.     全球监管趋势：合规化与趋严并行

全球主要司法辖区正监管框架：

欧盟：推出全面的MiCA法规，对稳定币提出严格储备要求。

美国：由SEC、CFTC等多机构监管，倾向于将多数代币视为证券进行管理。

香港：实施VASP牌照制度，允许零售交易，但配套严格的KYC/AML要求。

全球统一化：FATF制定的“旅行规则”等国际标准正被广泛采纳，推动全球监管协同。

4.     个人风险防范指南

 - 雇佣关系与薪酬风险

 - 劳动关系模糊

 - 薪酬结构隐患

 - 出金风险

 - 项目连带责任

5.     网络安全

 - 主要攻击类型：

钓鱼攻击：伪造官网、邮件、社群身份，以面试、空投、奖学金为诱饵，诱导连接钱包、签名或输入私钥。

恶意软件/木马：伪装成面试软件、工具插件，植入后实施剪贴板劫持（替换转账地址）、窃取文件、远程控制。

社交工程：冒充HR、好友、官方客服，利用信任实施诈骗。

供应链攻击：在浏览器插件、开源代码库中植入后门，造成广泛影响。

 - 关键防护措施

软件与链接：只从官方渠道下载；对任何要求安装软件、点击链接的请求保持警惕。

钱包与转账：助记词/私钥绝不触网；转账前多次核对地址；定期检查并取消不必要的钱包授权。

账号安全：使用密码管理器并为每个平台设置独立强密码；重要账户开启双重认证（2FA）。

信息核实：对任何“官方”通知、福利活动，通过多源头进行核实。

Web2 到 Web3 学习笔记：Intro

区块链核心概念解析：

1.        区块链的本质

 - 去中心化网络：点对点（P2P）网状结构，无中心控制

 - 抗审查性：即使部分节点失效，网络仍可运行

 - 无许可参与：不需要传统银行账户，只需私钥即可创建地址

2.        智能合约：自动执行的“托管机器”

核心功能：在互不信任的各方之间建立可编程的协作规则

特点：部署后无法更改（不可变性），代码即规则（信任来源于代码而非人），全球冗余：每个节点保存所有合约副本

3.        激励机制

  - Web2：依赖中心化服务器执行定时任务

  - Web3：通过经济激励让任何人执：行任务并获得报酬

以太坊生态系统与应用领域

\-              主要应用方向：去中心化金融（DeFi）流动性提供（LP）年化收益（APY）策略

\-              去中心化自治组织（DAO）=基于代币的治理投票

\-              NFT 与数字资产：收藏品，代币门槛社区（Token Gated）SVG NFT（完全在合约内渲染）

当前局限性：

 - 成本高：Gas 费用可能昂贵

 - 速度慢：交易确认需要时间

 - 异步性：与传统即时系统不同

 - 私钥管理风险：丢失助记词/私钥 = 永久丢失资产

  

钱包设置

1.        钱包：理解两种密钥

·      助记词 (Mnemonic / Secret Recovery Phrase)

由12个（或24个）英文单词组成的序列，是生成所有密钥的主种子（Master Seed）丢失即永久失去所有资产。

安全备份：物理备份/分散备份/密码管理器+测试

·      本地密码 (Local Password)

解锁本地MetaMask扩展的密码，仅加密存储在你设备上的数据，忘记后可凭助记词恢复钱包。防止他人直接操作钱包。

2.        账户与地址

无需许可：生成助记词即创建账户，无需任何机构批准。

地址生成：助记词 → (通过确定算法) → 私钥 → (通过椭圆曲线加密) → 公钥 → (哈希) → 以太坊地址 (0x...)。

不可逆性：交易一旦被网络确认（打包），将永久存在，无法撤销。没有客服。

3.        区块浏览器：Etherscan

区块链的“公共搜索引擎”和“中立第三方”。任何人都可查看所有公开交易。

\-              通过交易哈希（Transaction Hash） 查询交易详情（发送方、接收方、金额、状态）。

\-              通过地址查看账户余额和交易历史。

提供透明、可验证的交易记录，信任问题。

4.        确定性钱包

一个助记词，多个账户：通过助记词+不同的“索引号”（Index），可以生成无数个确定性账户（如 Account 1, Account 2...）

恢复特性：只要助记词相同，无论何时何地恢复钱包，生成的账户地址序列完全一致。  

5.        托管 vs. 非托管

特性：托管 (如 Coinbase) 非托管 (如 MetaMask)

控制权：交易所控制私钥，你是数据库中的一条记录/个人完全控制私钥。

责任 “他们负责安全，你信任他们。”

风险交易所可能被黑、跑路、冻结你的账户

6.        进阶解决方案：智能合约钱包

\-              解决痛点：传统钱包（EOA）的“单点故障”问题（助记词丢失=资产丢失）。Gnosis Safe, Argent。

\-              核心功能：多重签名 (Multisig)：需要多个预设私钥中的一定数量（如3选2）批准才能执行交易，适合团队或高额资产。社交恢复/守护人 (Guardians)：可指定可信地址为“守护人”。若丢失设备，可通过守护人投票来恢复钱包访问权，无需泄露助记词。

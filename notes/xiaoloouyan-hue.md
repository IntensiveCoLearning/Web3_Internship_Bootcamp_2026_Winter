### NFTs!!! ERC20 vs ERC721, IPFS, Metadata

**“NFT Day” 的实操与原理结合课程**，目标是让初学者：

-   亲手 **买 / mint / 查看 / 转移 NFT**
    
-   理解 **NFT 在技术上到底是什么**
    
-   看清 **Web3 的理想 vs 现实（去中心化 vs 平台化）**
    

## **二、NFT 核心概念**

### **1\. NFT 是“所有权记录”，不是图片本身**

-   区块链只负责记录：**谁拥有哪个 token｜**真正的内容（图片 / 元数据）通常：
    

-   要么 **存在 IPFS**
    
-   要么 **直接写在智能合约里（on-chain SVG）**
    

👉 真正“拥有”的是：**一个指向内容的不可篡改凭证（receipt）**

### **2\. ENS、本身就是 NFT**

### ENS 域名是 ERC-721 NFT

-   谁持有这个 NFT，谁就控制这个名字
    
-   把 ENS 转走 = 把控制权转走
    

### **3\. NFT 的价值来自三件事**

1.  **出处 / provenance**（谁 mint 的、历史如何）
    
2.  **稀缺性**（合约里写死的 total supply）
    
3.  **社区与共识**（市场是否承认这是“正品”）
    

👉 就算复制一模一样的图片，只要不是原合约 mint，**市场不会认**

## **三、NFT 的几种形态（很重要）**

### **1️⃣ Marketplace 买的 NFT（如 OpenSea）**

-   二级市场
    
-   平台展示 ≠ 链上真实存在
    
-   平台有 **强 Web2 式的筛选 / 审查**
    

### **2️⃣ Mint 网站直接 mint**

-   直接与智能合约交互
    
-   风险更高（要“信任代码”）
    
-   更贴近 Web3 本质
    

### **3️⃣ On-chain NFT（高级玩法）**

-   SVG / metadata 直接写在合约里极贵，但：
    

-   **完全不可篡改**
    
-   高度可组合（适合游戏、机制设计）
    
-   **4️⃣ IPFS NFT（最常见）**
    

-   合约只存 token → URI
    
-   URI 指向 IPFS 上的 metadata
    
-   Metadata 再指向 image
    

## **四、IPFS 的关键思想**

### **Content-addressable（内容寻址）**

-   文件的 **hash = 地址**
    
-   内容改 1 个字节 → 地址就变
    

所以：

-   ✅ 不可篡改
    
-   ✅ 没有“假副本“
    

⚠️ 但现实问题：

-   如果没人 pin，内容可能消失
    
-   需要 Pinata / Arweave 等服务
    

## **五、Web3 的“理想 vs 现实”**

### **理想：**

-   NFT 遵循标准 → 应该在哪都能看到 / 交易
    
-   去中心化、抗审查
    

### **现实：**

-   OpenSea / Rarible / Foundation 都是 **Web2 公司**
    

**Web2 和 Web3 并不是非此即彼，而是强烈混合**

## **六、Fungible vs Non-fungible（技术本质）**

### **Fungible（ERC-20）**

-   记录的是：**地址 → 余额**
    
-   转账 = 数值加减
    

### **Non-fungible（ERC-721）**

-   记录的是：**tokenID → owner**
    
-   每个 token 独一无二
    
-   转移 = 改 owner
    

👉 NFT ≠ “在网上传文件”

👉 NFT = **全网同步更新的一本“所有权账本”**

## **七、艺术家 & 创作者的意义**

NFT 让艺术家可以：

-   直接面对支持者（而不是平台）
    
-   写入 **版税（royalties）**
    

用 NFT 做：

-   门票
    
-   投票权
    
-   社群准入
    
-   游戏资产
    

👉 从「百万粉丝」转向「百个超级支持者」

## **八、重要风险提示**

-   不看 tokenURI = 不知道你买了什么
    
-   NFT 指向个人服务器 = **可被随意替换（rug）**
    
-   未验证合约 = **极大风险**
    
-   发错地址 / 发到合约地址 = **永远找不回**

**1\. 全球监管大趋势**

-   各国对加密货币和区块链的监管持续加强，形成更明确的合规框架，例如欧盟的 _MiCA_（加密资产市场法规）、美国的稳定币监管法案，以及FATF（金融行动特别工作组）的反洗钱/反恐融资要求。合规可以降低风险、提升用户信心并带来发展机会。
    

_Travel Rule_（旅行规则）要求虚拟资产服务提供商在转账中收集并传递发送方和接收方信息，这对去中心化交易协议提出挑战。

### 安全与合规在 Web3 领域既包括 **法律合规风险**（如代币发行、场外交易、合同效力等），也包括 **网络安全风险**（如钓鱼、木马、社交工程等）。对 Web3 从业者、求职者和参与者来说：

-   理解国内外监管趋势并评估自身行为是否合法是必要前提。
    

**非法金融活动风险**

-   代币发行（如 ICO 等）被认定为非法融资行为；即便以“积分”或“凭证”方式发行也可能触犯法律。
    

**刑事风险类别**

-   使用闭环经济模型（如链游抽奖兑币）可能构成非法赌博；多层级推广则可能构成传销罪。
    

**洗钱与非法经营风险**

-   场外交易和跨境资金绕开监管可能被认定为非法外汇经营、洗钱或协助犯罪。
    

**合同与民商事风险**

-   由于虚拟货币不具备法偿性，与其相关的合同在法律上可能被视为无效
    

三、Web3从业者与求职者需注意的合规事项

雇佣与薪酬风险

•由于 Web3项目常在境外注册，劳动合同、社保、税务等法律保障可能缺失。

•薪酬中包含 Token 或 USDT 有可能不被法律认可为工资，影响社保和个人权利。

出金与资金流通风险

•若采用非正规渠道将数字资产兑换成法币，可能卷入非法资金流、洗钱等法律问题。

项目合法性审查

•加入项目前应核实其注册地、合法性和合规情况，避免参与非法经营、未经许可的金融活动等。

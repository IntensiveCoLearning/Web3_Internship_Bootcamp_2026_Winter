Web3 AI

**AI 与 Web3 基础设施及 SpoonOS 框架** 的详细总结笔记：

### **1\. AI 的演进：从聊天机器人到自主智能体 (Agent)**

演讲者 Rick 首先阐述了 AI 发展的两个阶段。2023 年是“大语言模型（LLM）”的爆发期，本质是基于概率的文本生成工具，主要功能是陪聊或回答问题，但缺乏记忆和行动能力。这类传统 LLM 存在三大局限：**无状态**（每次对话从零开始，不记得历史）、**无工具**（无法联网或调用 API）、**无身份**（不能签署协议或持有资产）。  
展望 2026 年，AI 将进化为 **Agent（智能体）**。它不仅是“大脑”，还拥有“手脚”（工具/API）和“身份”。它不再仅仅是生成文本，而是能够替人类打工、做决策、甚至赚钱。为了实现这一跨越，AI 需要从单纯的信息处理转向具备经济行为能力的实体。

### **2\. 现有的基础设施缺口与 Web3 的必要性**

目前的互联网（Web2）架构无法支撑 AI Agent 的大规模经济活动，主要存在三大痛点：

-   **身份验证（Identity）：** 在 HTTP 协议下，无法区分请求来自真人还是 AI，也无法验证 AI 的归属权。这导致了安全隐患，如商业间谍伪装成合法 Agent 窃取数据。
    
-   **审计与追责（Auditability）：** AI 的决策日志若存储在中心化服务器上，可以被篡改或删除。一旦 AI 签署错误合同造成损失，缺乏不可篡改的证据链来追责。
    
-   **支付与结算（Payment）：** 传统金融体系（信用卡/银行卡）对 AI 不友好。
    
    -   **KYC 障碍：** AI 无法进行人脸识别或实名认证。
        
    -   **微支付成本：** AI 调用 API 可能只需支付 0.001 美元，但传统支付的手续费远超交易额。
        
    -   **结算速度：** 银行跨行/跨境转账需要 T+1 甚至 T+3，而 AI 的高频交互需要毫秒级或秒级结算。
        

因此，**Web3 与 AI 的结合是刚需**。区块链提供了不可篡改的账本（审计）、去中心化的身份（ID）、以及可编程的货币（无须许可的支付），成为了 AI Agent 的金融操作系统。

### **3\. 核心协议标准：ERC-8004 与 HTTP 402**

为了解决上述问题，Rick 介绍了两个关键的协议标准：

-   **ERC-8004（AI 链上身份标准）：** 被称为“AI 的护照”。它基于 NFT（ERC-721）为每个 Agent 铸造唯一的链上身份。
    
    -   **功能：** 包含元数据（Agent 类型、能力声明、模型哈希）、所有者地址以及声誉注册表。
        
    -   **声誉系统：** 防止刷分。只有发生过链上交互（支付过 Gas 费）的地址才能评价，评价不可篡改且具有时间衰减性（近期的评价权重更高）。
        
    -   **可验证性：** 结合 TEE（可信执行环境）或 ZK（零知识证明），确保 AI 确实运行了特定的模型代码。
        
-   **HTTP 402（支付协议）：** 复活了 HTTP 协议中保留的 "402 Payment Required" 状态码。它允许在 API 调用层直接嵌入支付逻辑。Agent 可以在一次交互中自动完成“请求资源 -> 支付 -> 获取数据”的闭环，支持微支付和批量结算。
    

### **4\. SpoonOS：AI Agent 的 Web3 框架**

Rick 团队开发了 **Spoon OS**，这是一个类似于“AI 安卓系统”的开发框架，旨在降低开发者将 AI 接入 Web3 的门槛。它将复杂的底层技术（私钥管理、签名、链上交互）封装成简单的 API。  
Spoon OS 的架构分为四层：

-   **数据层：** 负责数据的采集、清洗、向量化存储（支持 Vector DB），为 AI 提供记忆。
    
-   **内核层（Kernel）：** 统一的 LLM 接口（适配 OpenAI、Claude 等），包含任务编排引擎和上下文管理。
    
-   **协议层（Protocol）：** 核心层。实现了 ERC-8004 身份管理和 HTTP 402 支付网关。提供私钥托管（支持 TEE 安全存储），确保开发者无需直接触碰私钥即可让 AI 拥有钱包功能。
    
-   **行动层（Action）：** 负责与外部世界交互，连接 DeFi 协议（如 Uniswap）、跨链桥或其他 Web2 API。
    

### **5\. 未来展望：Agent 经济**

Rick 预测，未来的经济主体将从人类转向 AI Agent。Coinbase 的数据显示，稳定币的月交易量已超过 Visa，其中大量交易来自 Bot 和 DeFi 自动化。随着 Agent 渗透率从 2024 年的不到 1% 增长到 2028 年预期的 33%，机器与机器之间（M2M）的高频、微额交易将爆发。区块链将成为承载这一海量交易的各种 Agent 的基础设施。

### **6\. 问答环节重点 (Q&A)**

-   **AI 交易是否必须上链？**
    
    -   如果是个人自用的量化策略，可以使用中心化交易所托管。但如果是面向公众募资或需要建立信任的基金（如链上对冲基金），则必须拥有链上身份以实现资金透明和逻辑可验证。
        
-   **Web3 的效率问题：**
    
    -   虽然区块链 TPS 可能低于支付宝，但在**跨境结算**和**去中介化**方面效率极高。Web2 依赖中心化服务器和繁琐的银行合规流程，而 Web3 实现了全天候、无国界的自动化结算。
        
-   **声誉系统防作恶：**
    
    -   当前声誉绑定在 Agent 的 NFT ID 上。虽然开发者可以放弃坏名声的 Agent 重新部署，但重新积累声誉和链上交互历史的成本很高（Gas 费）。
        
-   **Spoon OS 的定位：**
    
    -   它目前是一个 SDK/开发框架，帮助开发者快速构建具备 Web3 能力的 Agent。未来目标是发展成一个 Agent 网络生态。

以下是021第三章《账户类型与结构》的详细学习总结：

**一、 以太坊账户的“双轨制”：EOA 与合约账户**  
以太坊的世界里只有两种账户，它们的本质区别在于“控制权”的归属。第一种是**外部拥有账户（EOA）**，也就是我们常用的 MetaMask 钱包地址。它的核心特征是“有人”，由私钥/助记词直接控制。EOA 没有代码，也不存状态，它唯一的超能力是“发起交易”——在以太坊协议层，所有的操作必须由 EOA 签名并发起，它是资金流动的起点，像一张由人持有的银行卡。第二种是**合约账户（Contract Account）**，它没有私钥，没人能直接“登录”它。它是由部署在链上的代码（EVM 字节码）控制的“自动售货机”。它不能主动发起交易，只能被动等待别人调用，一旦被触发，就严格按照写死的代码逻辑执行（比如转账、记录数据）。

**二、 账户在底层的“四元组”结构**  
虽然 EOA 和合约账户功能天差地别，但在以太坊的“世界状态”数据库里，它们的数据结构其实是一模一样的，都包含四个字段，只是填充内容不同。

1.  **Nonce（计数器）：** 对 EOA 来说，它记录你发了多少笔交易（防重放攻击）；对合约来说，它记录这个合约创建了多少个子合约。
    
2.  **Balance（余额）：** 记录账户里有多少 Wei（ETH）。
    
3.  **StorageRoot（存储根）：** 这是合约专属的“档案柜”，存放合约的所有变量数据（如代币余额表）。EOA 的这个字段是空的。
    
4.  **CodeHash（代码哈希）：** 合约账户这里存的是代码的哈希值（指向逻辑代码）；EOA 这里是空的，代表没有代码。  
    所以，判断一个地址是人还是合约，本质上就是看它有没有 Code。
    

**三、 智能合约的“出生”与“地址由来”**  
合约账户不是凭空产生的，它必须由 EOA（或者另一个合约）发起一笔“创建交易”来部署。有趣的是，合约的地址不是随机生成的，而是**计算出来**的。  
目前主要有两种生成方式：

-   **传统 CREATE：** 地址由 部署者地址 + Nonce 计算得出。这意味着只要你知道是谁部署的、是第几笔交易，你就能预测出合约地址。
    
-   **CREATE2（EIP-1014）：** 地址由 部署者地址 + 自定义盐值(salt) + 合约代码 计算得出。这让开发者可以在合约还没部署前，就确切地知道它将来会部署在哪（“反事实部署”），这对交易所充值地址、Layer2 账户等场景非常重要。
    

**四、 交易的“主动权”与账户抽象**  
笔记反复强调了一个铁律：**只有 EOA 能主动发起交易并支付 Gas**。合约账户无论多智能，它都是“被动”的。合约想要做操作（比如 A 合约调用 B 合约），必须是有一个 EOA 先推了 A 一把（发交易），A 才能在执行过程中去调用 B（这叫内部交易/消息调用）。  
虽然现在有了 **ERC-4337（账户抽象）** 和未来的 **EIP-7702**，让合约钱包看起来像在“自动运行”或“批量操作”，但从协议底层看，依然需要一个 Bundler（打包者）用 EOA 账号把这些用户意图打包上链。所以，私钥依然是控制链上交互的唯一物理入口。

**五、 代币的本质：不是在你钱包里，而是在合约的账本里**  
这是一个非常重要的认知纠正。当你觉得“我的钱包里有 100 个 USDT”时，其实你的 EOA 地址里什么都没有（除了 ETH）。  
真实的逻辑是：USDT 的合约里有一个巨大的账本（Mapping 结构），里面写着 你的地址 = 100。

-   **转账操作**，不是把代币发出去，而是你去调用 USDT 合约的 transfer 函数，让合约修改它内部账本上的数字（把你名下的减去，给对方加上）。
    
-   **查余额**，是钱包去读取合约里的存储数据并展示给你看。  
    所以，代币本质上就是智能合约数据库里的一行记录。
    

**六、 部署后的宿命：不可篡改与“假”删除**  
合约一旦部署，代码就是**不可修改**的（Immutable）。CodeHash 写死在链上，谁也改不了。如果想升级功能，只能用“代理模式（Proxy Pattern）”——让一个壳合约存数据，把逻辑指向新的实现合约。  
另外，关于“删除合约”，笔记特别提到了 **Dencun 升级（EIP-6780）** 后的重大变化：以前可以用 SELFDESTRUCT 指令销毁合约并清空状态，但现在这个指令已经被“阉割”了。除了在合约创建的同一笔交易内（极其罕见的情况），现在的 SELFDESTRUCT 只会把合约里的 ETH 转走，**代码和存储空间不再被删除**。这意味着“自毁合约”这种操作在现代以太坊上已经基本失效，开发者不能再依赖它来清理垃圾或重置状态。

**总结来说：**  
这一章不仅讲了账户分类，更揭示了以太坊是一个\*\*“由 EOA 驱动的状态机”\*\*。EOA 提供动力（Gas 和签名），合约账户提供逻辑（Code）和记忆（Storage）。所有的应用（DeFi、NFT、DAO）都是建立在这套账户交互规则之上的。

**今日学习《021学习以太坊》《Web3实习手册入门导读》《Uniswap v2 核心白皮书》《**[**Day 2: Becoming a Power User - Wallets, Mnemonics, Keypairs》**](https://www.youtube.com/watch?v=_GjPeRLCREA&list=PLJz1HruEnenAf80uOfDwBPqaliJkjKg69&index=2)

**021 第二章学习笔记：以太坊的骨架与脉络**

一、 现在的节点是“双核”驱动  
读完这一章，首先要更新的一个重要概念是：自 2022 年“合并（The Merge）”之后，运行一个以太坊全节点不再是启动一个软件那么简单了。现在的架构是**执行层（EL）与共识层（CL）的分离与协作**。可以把它们想象成同一个办公室里的两个部门：执行客户端（如 Geth）是“干活的”，负责处理交易、运行智能合约、维护账本状态；而共识客户端（如 Prysm）是“做决策的”，负责通过 PoS 机制投票、确立链的最新区块。两者必须同时运行，并通过本地的 **Engine API** 接口像专线电话一样实时沟通，才能组成一个完整的、能同步数据的以太坊节点。

二、 节点类型的选择逻辑  
在存储数据方面，书中用“图书馆”做比喻非常精准，这决定了我们在实际业务中该部署哪种节点。  
最常用的是**全节点（Full Node）**，它像一个公共图书馆，保存了最新的账本状态和所有的区块数据，但会定期“修剪”那些过时的中间状态以节省空间，这对绝大多数开发者和钱包服务来说已经足够安全且高效。  
如果是做深度历史数据分析或司法审计的，那就需要**归档节点（Archive Node）**，它像国家档案馆一样，不仅有现在的数据，还保留了从创世块以来每一个时间切片的所有历史状态，代价就是需要巨大的硬盘空间（TB 级别）。  
而对于普通用户或手机设备轻节点（Light Node）则像一个只存了目录的检索机，它不存具体数据，只验证区块头，需要数据时再去向全节点查询，主打轻量和便捷。

三、 数据的传播：像“传八卦”一样  
以太坊作为一个去中心化网络，没有中央服务器来分发数据，而是依靠 **P2P 网络和 Gossip 协议**。这个机制非常有意思，它模仿了人类社会的谣言传播：当你发出一笔交易，你的节点不会把它广播给全网所有人（那样网络会瘫痪），而是随机传给身边的几个“邻居节点”，邻居再传给邻居。这种指数级的扩散方式，保证了信息能在几秒钟内覆盖全球成千上万个节点，同时又具有极强的容错性——哪怕某些节点挂了，消息也能通过其他路径传达。

四、 为什么我们要自己跑节点？  
这一章花了很大篇幅讨论“自建节点 vs 使用公共 RPC（如 Infura）”的权衡。核心逻辑在于“Don’t trust, verify”（不要信任，要验证）**。虽然使用 Infura 这种第三方服务很方便，但你本质上是在信任它们给你的数据是真的，而且你的隐私（IP 地址与钱包的关联）也暴露给了服务商。对于开发者或机构而言，运行自己的全节点意味着**拥有“主权”：你不受制于人的限速，不用担心被审查，且能获得最低的内网延迟。

五、 实操层面的建议  
最后，在实际动手层面，现在的节点同步技术已经很成熟。比如 **Snap Sync（快照同步）** 技术，让新节点不需要从 2015 年开始重跑一遍历史（那太慢了），而是直接下载最近的状态快照，几天甚至几小时就能追上进度。如果你的需求仅仅是查查数据，而不想买硬盘跑节点，书中也指了条明路：利用 **Dune** 或 **Google BigQuery** 这种数据分析平台，或者用 Etherscan 的 API，这属于“借别人的车赶路”，在学习初期是非常经济实惠的选择。

**关于 Uniswap v2 核心白皮书的阅读笔记**

Uniswap v2 本质上是对 v1 版本的全面升级，核心算法依然沿用了经典的恒定乘积公式（x⋅y=kx⋅y=k），但在灵活性和基础设施层面做了很多关键改进。最直观的变化是去除了 ETH 作为中介货币的限制。在 v1 中，任何代币兑换都必须通过 ETH 中转（比如 DAI 换 USDC，必须是 DAI->ETH->USDC），这导致了双重滑点和双倍手续费。v2 允许任意两个 ERC-20 代币直接建立交易对（Pair），这让稳定币之间的互换变得更高效，虽然理论上这可能导致流动性碎片化，但可以通过链下的路由算法来寻找最优路径。

在预言机（Oracle）方面，v2 做了一个非常硬核的改进，试图解决链上价格容易被操控的问题。v1 的价格是即时的，攻击者可以在同一个区块内先砸盘、利用低价从其他协议获利、再买回，从而零成本操纵价格。v2 引入了“时间加权平均价格”（TWAP）的概念，它只记录每个区块“第一笔交易前”的价格（也就是上个区块的收盘价）。合约会在链上累积“价格时间”的值，外部使用者只需选取两个时间点相减并除以时间差，就能得到一段时间内的平均价格。这种机制让攻击者必须持续多个区块操纵价格才能影响预言机，成本高到几乎不可行。

另外一个重大的功能创新是“闪电换”（Flash Swaps）。这有点像借贷协议里的闪电贷，允许用户在支付费用之前就先把代币借出来使用。只要在同一笔交易结束前，用户能把相应的资产（加上手续费）还回去，或者支付等值的另一种代币，交易就能成功；否则整个交易回滚。这对于套利者来说是神器，意味着他们不需要持有大量本金，只要发现 Uniswap 和其他地方有价差，就可以空手套白狼，这也反过来帮助 Uniswap 的价格更紧跟市场。

在合约架构和安全性上，团队也做了很多深思熟虑的调整。代码从 Vyper 换回了更通用的 Solidity。架构上采用了“Core”和“Periphery”分离的模式：Core 合约只负责最核心的资金存储和安全逻辑，越简单越好，以减少攻击面；复杂的路由和交互逻辑则交给外围合约处理。为了防止某些非标准代币（比如有重入风险的 ERC-777）搞破坏，v2 引入了防重入锁。此外，针对初始流动性注入时可能发生的“份额极其昂贵”的攻击，v2 会在第一次铸造流动性代币时，强制销毁最初的 1000 wei 份额，虽然这对正常用户来说微不足道，但能有效杜绝极端的数学攻击。

最后是关于手续费和 ETH 的处理。v2 依然收取 0.3% 的交易费，但代码里埋了一个“协议费开关”。这个开关一旦由治理层打开，将会从这 0.3% 中分出 0.05% 给协议方，流动性提供者（LP）则剩下 0.25%。目前这个开关默认是关闭的。另外，为了简化核心代码逻辑，v2 的 Core 合约不再直接处理原生 ETH，而是全部统一使用 WETH（封装的 ETH），这意味着所有涉及 ETH 的交易在底层都变成了 ERC-20 对 ERC-20 的标准交互。

视频深入探讨了区块链的核心逻辑，特别是对于开发者而言的思维转变：

**_区块链本质_**：不仅仅是链表，更是一个**P2P 网状网络**。所有节点遵循同一套规则，具有抗审查性。

**_Trustless与 Escrow_**：

核心比喻：智能合约就像一个自动售货机或托管账户。

场景：Alice 和 Bob 互不信任（甚至可能是敌对国家，如朝鲜和美国），但可以通过代码设定的规则进行协作。

机制：只有当双方都满足条件（投入钱和货）时，交易才会执行。你不需要信任对方，只需要信任代码逻辑。

**_Incentives_**：

Cron Job 的区别：在 Web2 中，你设置服务器定时运行任务。在 Web3 中，你编写规则并提供Payment，让网络中的任何人（只要有利可图）在特定时间以此触发任务。

通过编写正确的规则和激励，引导网络参与者的行为。

**_Unstoppable Apps_**：一旦合约部署且代码逻辑正确，没有人（包括开发者自己）可以停止它。

**冗余性**：整个网络就是你的冗余备份，所有节点都保存状态。

**_Use Cases：_**

DeFi ：投资、赚取收益、流动性提供 (LP)。

DAO ：治理结构，使用代币进行投票决策。

NFT：数字收藏品、代币门控聊天 (Token-gated chats)、SVG NFT（直接在链上渲染图像）。

**_The Gotchas：_**

用户体验：交易昂贵、速度慢、异步处理。

密钥管理：丢失私钥/助记词 = 永久丢失资产。

解决方案方向：智能合约钱包、多重签名、社交恢复。

不可逆性：代码即法律，Bug 可能会导致资金被黑且无法追回。

**_学习建议与心态_**

非“快乐路径” ：讲师们在视频中展示了真实的开发过程，包括犯错、调试和解决 Gas 问题，而不仅仅是完美的演示。

学习层级：Level 1: 成为超级用户，理解机制。Level 2: 动手构建小项目，并在 Twitter 上分享（Build in public）。Level 3: 成为审计员 (Auditor)，深入研究合约安全攻击与防御。

**021学习以太坊第一章学习总结**

这一章主要从宏观角度介绍了以太坊的定义、历史、核心机制以及它与比特币的区别。

### 第一章：认识以太坊 (学习笔记)

1\. 核心定义与愿景

-   **什么是以太坊？**以太坊是一个**去中心化、开源、具备智能合约功能**的公共区块链平台。
    
-   它不仅仅是像比特币那样的“记账系统”，更是一台“全球可编程的计算机”（World Computer）。
    
-   **核心组件：** EVM（以太坊虚拟机），它是图灵完备的，允许运行任意逻辑的程序。
    
-   **愿景：** 建立在抗审查、去中心化基础设施上的全球共享状态机，任何人都可以在上面构建应用（dApps）。
    

2\. 核心特性与概念

-   **智能合约 (Smart Contracts)：**
    
    -   **比喻：** 自动售货机。投入资金（Gas/ETH），按下按钮（调用函数），机器自动执行逻辑（出货/找零），无需中介（店员）干预。
        
    -   **特点：** 代码即法律，自动执行，不可篡改。
        
-   **去中心化应用 (dApps)：**
    
    -   后端逻辑运行在区块链上，永不宕机，抗审查。
        
-   **Ether (ETH) 的职能：**
    
    -   **燃料 (Gas)：** 支付网络计算费用（防止滥用资源）。
        
    -   **质押资产 (Staking)：** 在 PoS 机制下维护网络安全的保证金。
        
    -   **货币属性：** 价值存储、DeFi 中的抵押品、NFT 的计价单位。
        
-   **代币标准：**
    
    -   **ERC-20：** 同质化代币（如稳定币、治理代币），类似货币，可互换。
        
    -   **ERC-721/1155：** 非同质化代币（NFT），代表独特的数字资产（如艺术品、游戏道具）。
        

3\. 共识机制的演变 (PoW → PoS)

-   **历史转折：** 2022 年 9 月完成“合并”（The Merge），从工作量证明（PoW）切换到权益证明（PoS）。
    
-   **PoS (权益证明) 的优势：**
    
    -   **节能：** 能耗降低约 99.95%。
        
    -   **安全性：** 依靠质押 ETH 和**削减机制 (Slashing)**（没收作恶者的资金）来保障安全，而非依靠算力。
        
    -   **发行量：** 相比 PoW 时代，ETH 的增发量大幅减少，配合 EIP-1559 的销毁机制，ETH 具有了通缩潜力。
        

4\. 扩容路线图 (Layer 2 & Sharding)

以太坊不再追求在主网（L1）上做所有事情，而是转向“以 Rollup 为中心”的路线：

-   **Layer 1 (L1)：** 负责安全、共识和数据可用性（结算层）。
    
-   **Layer 2 (L2)：** 负责执行交易，速度快、费用低（如 Arbitrum, Optimism）。
    
-   **关键升级：**
    
    -   **Dencun 升级 (EIP-4844)：** 引入了 **Blob**（临时数据块），类似给以太坊挂了一个“小挂车”，专门便宜地存储 L2 的数据，大幅降低了 L2 的 Gas 费。
        

5\. 以太坊 vs. 比特币

| 特征 | 比特币 (Bitcoin) | 以太坊 (Ethereum) | | 定位 | 数字黄金，价值存储 | 世界计算机，应用平台 | | 功能 | 简单的价值转移 | 图灵完备，智能合约，DeFi/NFT | | 共识 | PoW (算力挖矿) | PoS (质押验证) | | 发行 | 2100 万上限，四年减半 | 无硬顶，但有销毁机制，动态平衡 | | 扩展性 | 侧重闪电网络 (支付) | 侧重 Rollup (智能合约扩容) |

**总结一句：** 比特币是极简可靠的“专用计算器”（或黄金），以太坊是可安装各种 APP 的“通用智能手机”（或操作系统）。

6\. 生态系统应用

-   **DeFi (去中心化金融)：** 无需银行的借贷（Aave）、交易（Uniswap）、稳定币（MakerDAO/Sky）。
    
-   **NFT & 游戏：** 数字资产的确权，元宇宙（Decentraland）和链游资产。
    
-   **DAO (去中心化自治组织)：** 基于代码规则和代币投票的组织治理，无中心化董事会。
    

7\. 社区文化

-   **无限花园 (Infinite Garden)：** 以太坊不是由一家公司控制的产品，而是一个由全球开发者共同维护、修剪和培育的生态系统。
    
-   **多客户端：** 为了去中心化，以太坊有多种客户端实现，避免单点故障。
    

* * *

**第一章核心总结**

“以太坊的网络结构 = 理论上对任何人开放（permissionless），现实中受限于硬件和带宽，但通过 Layer 2 + 数据分片等方案，把‘更多人参与’和‘系统还能跑得动’两件事尽量同时做到。”

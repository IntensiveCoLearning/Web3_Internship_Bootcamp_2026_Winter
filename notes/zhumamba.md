# Web3新人要注意的安全与合规tips

## 一、安全方面：钓鱼攻防挑战中的部分案例

-   **不点！不签！不装！不转！**
    
-   最危险的不是明显的骗局，而是“看起来一切正常”的那一步操作
    
-   使用苹果全家桶设备可以避免大部分安全问题
    

### 例子 1：Uniswap 空投钓鱼

-   **表象**：收到 UNI 空投邮件，页面显示 _ERC-20: app.uniswap.org (UNI)_
    
-   **陷阱点**：
    
    -   页面、交易记录看起来都“很像真的”
        
    -   实际访问的 **不是官方域名**
        
-   **核心教训**：
    
    -   看交易记录 ≠ 安全
        
    -   **只认官方主域名**，不要只看页面文案或 Token 名字
        

### 例子 2：Trezor Punycode 域名钓鱼

-   **表象**：收到“官方安全更新”邮件，域名显示为  
    `trẹzor.com`
    
-   **真实情况**：
    
    -   这个 **ẹ 不是字母 e**
        
    -   实际域名是：`xn--trzor-o51b.com`
        
-   **核心教训**：
    
    -   域名里出现“看着像但不完全一样的字符”＝高危
        
    -   Punycode 是高级钓鱼常用手段
        

### 例子 3：剪贴板地址劫持

-   **表象**：
    
    -   好友发来 ETH 地址
        
    -   点击 Copy → 粘贴进转账框
        
-   **陷阱点**：
    
    -   粘贴后的地址 **和原地址不一致**
        
    -   用户不仔细核对就会直接转账
        
-   **核心教训**：
    
    -   转账前必须 **人工核对地址**
        
    -   不要完全信任“复制粘贴”
        

### 例子 4：Microsoft Teams 钓鱼

-   **表象**：
    
    -   收到“Teams 安全更新 / 会议邀请”
        
    -   页面长得像微软
        
    -   有明显的 **“Join on the Teams app”** 按钮
        
-   **陷阱点**：
    
    -   域名是：`microsoft-meet.com`
        
    -   **不是** `teams.microsoft.com`
        
    -   点击按钮会下载恶意程序
        
-   **可能后果**：
    
    -   账号凭证被盗
        
    -   本地文件泄露
        
    -   钱包被扫描 → 资产被盗
        
-   **核心教训**：
    
    -   官方会议工具 **不会让你装奇怪更新包**
        
    -   域名不对 = 一票否决
        

### 例子 5：0 ETH 交易 + 合约调用

-   **表象**：
    
    -   Value = 0 ETH
        
    -   Gas 很低
        
    -   状态显示 Success
        
-   **陷阱点**：
    
    -   实际是 **Call Function**
        
    -   执行的是授权 / 转账逻辑
        
-   **核心教训**：
    
    -   **0 ETH ≠ 没风险**
        
    -   要看“你在调用谁的合约、干了什么事”
        

## 二、合规方面：在中国大陆语境下如何避免踩红线？

-   **不发币、不募资、不走资金通道，是最重要的安全底线**
    

### 1.绝对不能碰的红线

**凡是形成「钱 → 币 → 再变钱」闭环的行为，在大陆都极高风险**

-   任何形式的 **发币 + 融资**（ICO / 私募 / 变相空投筹资）
    
-   在境内 **做虚拟币兑换、撮合、资金通道（U 商 / 法币对敲）**
    
-   **合约、永续、期权等衍生品** 的运营、推广、返佣
    
-   **拉人头、多级返佣、团队计酬**（不管叫 DAO / 社区 / 节点）
    
-   以“技术中立 / 去中心化”为理由规避监管
    

### 2.个人出入金的现实风险

-   不明来源的币 / 钱 **不要动、不要换 U、不要出金**
    
-   OTC、私下换汇 **极易踩洗钱雷**
    
-   **借账户、代收代付、帮忙过账** ≈ 高危行为
    
-   大额或频繁出金，极易触发 **冻结 / 风控 / 调查**
    

### 3.在 Web3 行业工作的相对安全做法

-   岗位优先级：  
    技术 / 产品 / 研究 / 社区 / 内容 / 运营  
    （尽量避开资金、交易、激励模型设计）
    
-   薪资结构：  
    **法币为主 + 代币为辅**  
    全 U / 全 Token = 风险明显上升
    
-   入职前 3 问：
    

1.  是否面向中国大陆用户？
    
2.  是否涉及募资、合约、杠杆？
    
3.  是否有清晰的海外主体与业务隔离？
    

**必要时候咨询律师！**

# **Web3安全与合规**

## 一、Web3 合规性要求与常见法律风险

**个人理解：** 在中国参与 Web3 行业，需要明确合规底线，避免参与代币融资和发币等高风险行为；在实际工作中，若涉及薪酬结构，包含法币部分、以代币作为补充的方式相对更稳妥，有助于降低合规和出金风险。

### 1\. 中国监管环境概览

-   自 2017 年以来，监管核心思路是**封堵虚拟货币的金融属性**，对区块链技术本身有限容忍
    
-   明确禁止 ICO、交易所运营、虚拟货币作为支付工具
    
-   严打涉币的非法集资、传销、洗钱、赌博等行为
    
-   技术可讨论，一旦涉及 Token、融资、交易即进入高风险区
    

### 2\. Web3 项目的主要法律风险

-   **代币发行与交易风险**
    
    -   不论叫积分、凭证或治理 Token，只要具备融资或流通属性，就可能构成非法金融活动
        
    -   参与代币模型、空投设计、合约部署、运营推广，可能被认定为共同行为人
        
-   **赌博与链游风险**
    
    -   “充值—抽奖/随机收益—提现或转售”的结构，容易被认定为赌博
        
-   **传销风险**
    
    -   多级返利、邀请奖励、团队计酬、算力挂靠等模式具有高度风险
        
-   **出金、换汇与洗钱风险**
    
    -   C2C/OTC 出金、用虚拟币作为换汇通道，可能构成非法经营
        
    -   若资金来源涉诈涉赌，即使不知情也可能被牵连
        
-   **民商事风险**
    
    -   虚拟货币交易合同通常不受司法保护，风险自担
        
    -   委托代投可能引发赔偿责任甚至更高风险
        

### 3\. 全球监管背景与趋势

-   各国总体趋势为**监管趋严但规则更清晰**
    
-   FATF 推动反洗钱与反恐融资标准统一，Travel Rule 对 Web3 提出更高合规要求
    
-   稳定币成为重点监管对象，要求储备透明、风险可控
    
-   欧盟 MiCA、美国稳定币立法、香港 VASP 制度提高行业准入门槛
    
-   合规成本上升，但长期有利于行业健康发展
    

### 4\. Web3 就业与用工风险

-   多数项目为境外注册，可能无法签署有效劳动合同
    
-   普遍缺乏社保、公积金保障，纠纷维权难度高
    
-   薪酬常以 USDT 或 Token 形式发放，存在法律与价值波动风险
    
-   出金环节最容易出现冻卡、涉案等问题，需格外谨慎
    

### 5\. 高发刑事风险类型

-   赌博类：可提现、随机收益、投注平台属性明显
    
-   非法经营类：虚拟币作为跨境换汇或支付媒介
    
-   非法集资类：矿机、返利、投资承诺吸收公众资金
    
-   传销类：拉人头、多层级返佣结构
    
-   洗钱类：缺乏 KYC/AML、资金来源不明
    

## 二、常见网络安全风险与防护措施

-   Web3 世界中，技术门槛不高，但安全门槛极高；
    
-   **不轻信、不乱装、不乱点、不乱签名**，是最基本也是最重要的安全原则。
    

### 1\. 为什么 Web3 学生更容易成为攻击目标

-   Web3 环境本身遵循“黑暗森林”法则，只要暴露在网络中就可能被攻击
    
-   学生群体普遍经验不足、警惕性不高，容易轻信“面试”“奖学金”“内推”“空投”等信息
    
-   一旦中招，钱包资产和账号权限往往**不可逆丢失**
    

### 2\. 常见网络安全风险类型

-   **钓鱼攻击**
    
    -   伪造面试通知、奖学金、官方活动
        
    -   诱导点击链接、连接钱包或签名
        
-   **恶意软件与木马**
    
    -   伪装成面试软件、学习资料、插件
        
    -   剪贴板劫持、远程控制、盗取钱包与账号
        
-   **社交工程攻击**
    
    -   假冒 HR、导师、学长学姐或好友
        
    -   利用信任关系诱导操作或转账
        
-   **供应链 / 插件攻击**
    
    -   恶意浏览器插件、被投毒的第三方依赖
        
    -   即使来自“看似正规渠道”也可能存在风险
        
-   **账号与隐私安全问题**
    
    -   密码复用、邮箱或手机号被劫持
        
    -   未开启 2FA 导致全平台连锁失守
        

### 3\. 主要攻击后果

-   钱包资产被盗，几乎无法追回
    
-   邮箱、社交账号、交易所账号被接管
    
-   电脑被长期监控，成为后续攻击的跳板
    

### 4\. 学生版核心防护原则

-   **面试与实习**
    
    -   只使用常见视频会议工具，拒绝安装“专用面试软件”
        
    -   安装程序前必须核实来源
        
-   **空投与福利**
    
    -   任何要求连接钱包、签名或输入助记词的行为都要高度警惕
        
    -   主钱包与测试钱包严格分离
        
-   **软件与插件**
    
    -   只从官网或官方商店下载
        
    -   浏览器插件尽量少而精
        
-   **账号与钱包**
    
    -   重要账号开启 2FA
        
    -   助记词与私钥只做离线保存
        
    -   转账前核对地址关键信息，防止剪贴板劫持
        

## 三、骗局实际案例

### 1\. 很多“被钓鱼”，其实是被偷偷装了木马

-   骗子常用套路：  
    给你发一份**看起来很专业、很有吸引力的文件**（行业报告、面试资料、内部文档）
    
-   你一打开：
    
    -   电脑可能就被装了木马
        
    -   你自己**一点感觉都没有**
        
-   木马常干的事：
    
    -   偷你浏览器里存的账号、登录状态
        
    -   偷你临时输入的密码、验证码
        
    -   偷文件、截图、录屏
        
-   所以很多人是：  
    **先丢邮箱 → 再丢社交号 → 最后钱包被清空**
    

**中招后不是“删文件就行”，而是电脑要清、账号要全改**

### 2\. 最容易骗到人的，是“看起来像官方”的东西

-   骗子特别会装官方：
    
    -   域名只差一个字母
        
    -   用“长得像英文字母”的字符冒充官网
        
-   看着是官网，其实是假的  
    **人眼很难分出来**
    
-   有时候邮件看起来也是“官方发的”，但其实是伪造的  
    **看到“官方”两个字，不代表就安全**
    

### 3\. 最防不住的一种情况：内部人作恶

-   如果项目内部有人安全意识差，或者故意搞事
    
-   用户基本很难防
    

项目靠谱不靠谱，**人比技术重要**

_首先，社区在以太坊里非常重要。这里的社区不是“用户群”，而是一个能参与讨论、影响方向的整体。作为运营，我不只是负责活跃气氛，而是要理解社区在关心什么、争论什么，把这些声音整理出来，反馈给项目和开发者，这其实是“共识运营”。_

_其次，开发生态让我意识到，Web3 运营必须和产品、技术走得很近。很多重要信息都来自 GitHub、开发者会议、黑客松等场景，而不是传统意义上的市场渠道。运营要能把这些偏技术的内容，用普通用户能理解的方式讲清楚，降低参与门槛。_

_另外，这一章也让我明白，Web3 的增长不是靠“平台投放”，而是靠工具、文化和参与感。黑客松、文档、开源工具、DAO 协作，本身就是拉新的方式。运营能做的，是把“参与路径”设计得足够清晰，让新人知道从哪一步开始。_

_最后，我也意识到以太坊社区强调的是“管理而不是控制”。对运营来说，这意味着不能用强管控思路去做社区，而是要给空间、给规则、给工具，让社区自己运转，而运营更多是引导和协调。_

_整体来看，这一章让我更确定：我想从运营岗位切入，是因为这个角色正好处在“社区、产品和开发”的中间位置，能真正参与到生态内部，而不是只在外围做传播。_

-   **为什么结果不能撤回？**  
    已写入区块，区块链不可逆

## 四、以太坊 dApp 的概念与应用

**_dApp 是把核心逻辑和资产交给区块链执行的应用，它用智能合约替代传统后端，在获得去中心化、可验证和抗审查优势的同时，也必须面对性能、成本和用户体验上的现实权衡。_**

### 1\. 什么是 dApp

**dApp（去中心化应用）= 链上的智能合约（后端逻辑） + 链下的前端界面（用户交互）**

它不是“完全没有服务器”，而是：

-   关键逻辑和资产放在**链上**
    
-   前端只是用来和用户交互
    

### 2\. dApp 在以太坊上的基本结构

**_dApp = 用智能合约当后端，用以太坊做数据库和结算层_**

**① 链上部分（后端）**

-   由智能合约构成
    
-   运行在以太坊主网或 L2（Rollup）上的 **EVM**
    
-   由全网节点共同执行和记录
    
-   一旦部署，逻辑公开、透明、长期在线  
    

**_“一组任何人都能调用、无法随意关闭的 API”_**

而且：dApp 可以直接复用别人已经部署好的合约（如 Uniswap、Aave、NFT 市场）

**② 链下部分（前端）**

-   使用普通 Web / App 技术开发
    
-   可以部署在：传统服务器 / IPFS、Arweave 等去中心化存储
    
-   主要负责：
    
    -   页面展示
        
    -   发起交易
        
    -   调用合约接口
        

和传统 App 最大的不同在于：**后端不在公司服务器，而在区块链上**

### 3\. dApp 的核心技术特征（为什么要用以太坊）

**① 去中心化**

-   没有单一公司或机构能控制整个应用
    
-   不存在一个“总机房”可以直接关停
    

**② 确定性**

-   同一合约在任何节点执行
    
-   相同输入 → 相同输出
    
-   全世界用户看到的结果一致
    

**③ 图灵完备**

-   EVM 支持复杂逻辑（不只是转账）
    
-   在 Gas 足够的前提下，几乎可以实现任意计算逻辑
    

**④ 隔离性（沙盒）**

-   单个合约出问题，只影响自身调用
    
-   不会把整个以太坊网络拖垮
    

### 4\. dApp 的优势（和传统 Web 对比）

**① 零停机**

-   合约一旦部署，全网节点都会执行
    
-   只要以太坊存在，后端逻辑就存在
    

**② 相对匿名 / 隐私**

-   使用地址和私钥即可交互
    
-   不需要传统实名注册（“伪匿名”，非完全隐私）
    

**③ 抗审查**

-   没有单一实体可以阻止你部署或调用合约
    
-   封杀成本极高
    

**④ 数据完整性**

-   链上数据几乎不可篡改
    
-   历史交易无法被事后伪造
    

**⑤ 无需信任 / 可验证**

-   合约代码公开、行为可审计
    
-   不用“相信运营方”，而是“相信代码 + 共识”
    

### 5\. dApp 能做什么（应用场景）

-   **金融（DeFi）**  
    DEX、借贷、稳定币、衍生品、保险、收益聚合
    
-   **资产与内容（NFT / 游戏）**  
    NFT 市场、链游、虚拟土地、音乐 / 视频分账
    
-   **组织与治理（DAO）**  
    链上投票、国库管理、协议治理、公共物品资助
    
-   **基础设施类**  
    钱包、ENS、身份、预言机、跨链桥、L2 本身
    
-   **其他探索**  
    去中心化社交、链上存证、订阅 / 会员系统等
    

## 五、以太坊去中心化机制

**_以太坊通过共识、网络、实现、扩容和治理多层设计，共同构建出高去中心化和强抗审查的区块链系统。_**

以太坊的去中心化并非依赖单一设计，而是由多层机制共同实现：

-   **共识层**：采用 PoS，任何人质押 32 ETH 即可成为验证者，出块与投票权分散，作恶会被罚没质押。
    
-   **网络层**：开放的 P2P 网络，任何人都能运行节点，没有中心服务器。
    
-   **实现层**：多客户端、多团队实现同一协议，避免单点故障。
    
-   **扩容路径**：采用 Rollup-中心路线，L1 保持高度去中心化，执行负载交给 L2。
    
-   **治理方式**：通过 EIP 与社区共识推动升级，而非简单的“持币即治理权”。
    

## 六、网络结构的开放性与参与机制

以太坊在设计理念上是 **permissionless（无需许可）** 的：  
任何人都可以运行节点、发送交易、部署合约，协议本身不设准入门槛或白名单。

但在现实中，去中心化会受到 **硬件、带宽和成本** 的限制：

-   运行全节点需要较高的存储、带宽和持续在线能力
    
-   节点越多，数据传播和共识协调的成本也会增加
    
-   因此，“节点越多越好”并不绝对，需要在开放性和效率之间取得平衡
    

为了解决这个矛盾，以太坊采用了**分层设计**：

-   **Layer 1（主网）**  
    负责共识、安全、最终结算和数据可用性，优先保证去中心化和安全
    
-   **Layer 2（扩展层）**  
    承担大量高频交易和执行，提升速度、降低成本，最终结果回到 L1 结算
    

| 对比 | L1（以太坊主网） | L2 |
| --- | --- | --- |
| 作用 | 安全、共识、最终结算 | 执行、加速、降费 |
| 速度 | 慢 | 快 |
| Gas 费用 | 高 | 低 |
| 去中心化 | 高 | 相对低 |
| 是否依托以太坊安全 | — | 是 |

目前以太坊采取 **Rollup-中心路线**，并通过数据分片（如 Danksharding / PeerDAS）持续降低扩容成本，从而在不牺牲去中心化的前提下提升整体性能。

**面试用： 以太坊的网络结构 = 理论上对任何人开放（permissionless），现实中受限 于硬件和带宽，但通过 Layer 2 + 数据分片等方案，把“更多人参与”和 “系统还能跑得动”两件事尽量同时做到。**

## 七、行业应用案例：金融、游戏与社交

一句话概括：  
**以太坊最成熟的应用赛道集中在金融（DeFi / 稳定币 / RWA）、NFT & 游戏，以及社交 / 内容 / DAO。**  
大量应用已迁移到 L2 执行，但**结算、安全和资产标准仍主要依托以太坊生态**。

### 1\. 金融：DeFi / 稳定币 / RWA（最成熟赛道）

这是以太坊体量最大、商业化程度最高的方向。

-   **DeFi**：DEX、借贷、质押等协议（如 Uniswap、Aave）  
    用户直接用钱包交易或借贷，无需托管资产
    
-   **稳定币**：以 DAI 为代表，是 DeFi 中的重要计价与结算工具
    
-   **质押与再质押**：Lido、EigenLayer 等协议，将 PoS 安全转化为流动性和收益
    
-   **RWA**：国债、基金等现实资产上链，传统金融机构开始深度参与
    

### 2\. NFT、游戏与虚拟世界

NFT 和链游最早在以太坊生态中爆发。

-   **早期探索**：CryptoKitties、Decentraland  
    证明了 NFT 和链上确权的可行性，也暴露了扩容问题
    
-   **现阶段发展**：  
    多数链游和元宇宙项目将高频交互迁移到 L2 或侧链  
    但资产标准、钱包体系和基础设施仍与以太坊深度绑定
    
-   **NFT 市场**：  
    OpenSea、Blur 等以以太坊为核心，支持 ERC-721 / ERC-1155
    

### 3\. 社交、内容与 DAO

近两年增长较快，多采用“协议 + L2”的形态。

-   **去中心化社交**：Lens、Farcaster 等  
    把社交关系、内容和互动变成可组合的链上数据
    
-   **SocialFi**：  
    将社交关系和注意力代币化，探索新的激励模式
    
-   **内容创作**：  
    Mirror 等平台用以太坊账户和合约实现发布、众筹和会员制
    
-   **DAO 与公共物品**：  
    协议治理 DAO、Gitcoin 等，通过链上规则进行治理和资助
    

## 八、生态系统创新：DeFi、NFT 与 DAO

**_以太坊通过 DeFi、NFT 和 DAO 三大创新，把金融、资产与组织规则程序化，构建了一个可组合、无需许可的链上生态系统。_**

可以把以太坊生态理解为三层结构：  
**DeFi（金融操作系统） + NFT（数字资产与身份层） + DAO（组织与治理层）**。  
它们不是简单把传统东西“搬上链”，而是引入了全新的机制设计。

### 1\. DeFi：重塑链上金融体系

DeFi 是以太坊最成熟、规模最大的创新方向。

-   **DEX / AMM**（如 Uniswap）：  
    用自动做市商取代传统订单簿，任何人都可提供流动性、赚取手续费，实现无许可交易。
    
-   **去中心化借贷**（如 Aave）：  
    超额抵押 + 程序化利率 + 自动清算，资金全程由合约控制，无人工干预。
    
-   **稳定币**（如 DAI）：  
    通过链上抵押与清算机制维持价格稳定，成为 DeFi 的计价与结算基础。
    
-   **扩展与 Rollup**：  
    大量交易在 L2 执行，结果回到 L1 结算，在保证安全的同时降低成本、提升性能。。
    

### 2\. NFT：从数字藏品到身份与权限系统

NFT 不只是图片，而是**唯一性 + 所有权 + 可编程性**的组合。

-   **基础标准**：  
    ERC-721 管理唯一资产，ERC-1155 适合批量资产与游戏道具。
    
-   **应用扩展**：  
    NFT 被用于艺术、收藏、游戏资产、门票和会员体系（甚至与实体商品绑定）。
    
-   **新方向**：
    
    -   ERC-6551：让 NFT 拥有自己的账户，能持有资产、参与交互
        
    -   ERC-7432：为 NFT 引入角色与权限，支持授权、租赁和精细化管理
        

### 3\. DAO：把组织规则写进代码

DAO 是以太坊孕育的新型组织形态。

-   **协议 DAO**：  
    用于治理 DeFi 和基础设施协议（参数调整、金库管理、升级决策）。
    
-   **公共物品 DAO**（如 Gitcoin）：  
    通过链上机制资助开源项目和公共基础设施。
    
-   **创意型 DAO**（如 Nouns）：  
    将 IP、社区治理和资金管理结合在一起，探索新的协作模式。

## 1\. 区块链是什么

区块链不仅是“区块连成一条链”，更关键的是它背后的去中心化网络：全球大量节点点对点通信，共同维护一份账本数据。即使部分节点失效，网络仍能继续出块与运行。

### 区块链的核心特征

-   去中心化：没有单一机构掌控全网
    
-   公开可验证：交易与状态可由任何节点验证
    
-   抗审查与高冗余：任意节点宕机不影响全网继续工作
    
-   规则一致：全网节点都遵循同一套协议规则
    

## 2\. 信任、托管与智能合约

在不互相信任的环境里，交易难点是“先交钱还是先交货”。传统解决方式是找第三方托管（Escrow）做中介。

智能合约可以被理解为“可编程的托管机器/自动售货机”

-   只要条件满足就执行规则（if this then that）
    
-   参与者不需要互相信任，只需要信任合约规则按代码执行
    
-   合约部署后具有强约束性：大家都必须按规则玩
    

## 3\. 激励机制（为什么有人愿意维护网络）

去中心化网络需要节点长期提供算力/带宽/存储来维持运行，因此必须有激励。

### 比特币的激励逻辑

-   节点（矿工/验证者）负责验证交易、打包区块
    
-   成功出块可获得奖励（区块奖励 + 手续费）
    
-   形成正反馈：更多节点参与 → 网络更安全 → 系统更可信 → 价值更稳固
    

### 比特币为何具备“货币属性”

-   供应量有限（稀缺性）
    
-   可自由转账（可流通性）
    
-   规则公开、无需许可即可参与网络（开放性）
    

## 4\. 区块链如何运行（交易流程）

### 区块链生态系统通常按以下流程运作

1.  用户通过钱包发起交易（转账/调用合约）
    
2.  交易广播到全网节点
    
3.  节点验证交易合法性（余额、签名、规则等）
    
4.  通过共识机制将交易打包成新区块
    
5.  新区块链接上链，全网账本状态更新
    
6.  出块节点获得奖励与手续费
    

## 5\. 区块链的核心组成

从整体看，区块链系统由两部分构成

1.  去中心化网络：由大量节点共同提供服务，保障网络持续运行
    
2.  区块链账本：用“链式结构”记录全网状态与交易历史
    

两者通过激励机制绑定

-   节点提供验证与出块服务
    
-   系统通过代币奖励与手续费激励节点持续参与
    

## 6\. 公链 vs 联盟链 vs 私链

按去中心化程度从高到低可大致分为三类

### 公链（Public Blockchain）

-   任何人都可以加入并参与维护
    
-   数据通常公开透明
    
-   通过共识机制进行去中心化决策
    
-   缺点：效率较低、成本较高
    

### 联盟链（Consortium Blockchain）

-   由多个组织共同治理，加入通常需要许可
    
-   数据对联盟成员可见或分级可见
    
-   规则修改需多方协商
    
-   优点：效率与隐私折中，适合跨机构协作
    

### 私链（Private Blockchain）

-   由单一主体控制，成员严格准入
    
-   数据仅对内部可见
    
-   优点：效率高、隐私强
    
-   缺点：中心化强、透明性弱
    

适用场景概览

-   公链：加密货币、DeFi、公共存证
    
-   联盟链：供应链、金融协作
    
-   私链：企业内部管理、审计
    

## 7\. Web2 vs Web 3.0 vs Web3

### Web2（当前互联网）

-   平台集中控制数据与规则
    
-   用户产出内容但不拥有数据
    
-   常见商业模式：广告、平台抽成
    

### Web 3.0（语义网）

-   通过标准化结构描述数据关系，便于机器理解
    
-   代表技术：RDF、OWL、SPARQL、链接数据
    
-   重点是“数据组织升级”，不等同于区块链
    

### Web3（去中心化互联网）

-   数据主权与资产归用户（私钥/钱包）
    
-   智能合约自动执行规则，减少中介信任成本
    
-   应用可组合（协议像积木一样被复用）
    
-   典型组件：钱包、签名、Dapp、智能合约、区块链（以及 IPFS 等分布式存储）
    
-   Web3 ≠ Web 3.0
    
-   Web3 不是万能：在高频社交等场景并不一定优于 Web2

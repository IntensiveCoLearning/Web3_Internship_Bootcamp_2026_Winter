### **零知识证明必须满足的三个性质**

形式上，一个零知识证明系统需要同时满足三个核心性质：

1.  **完备性（Completeness）**
    
    如果语句是真的，并且证明者与验证者都按照协议诚实地执行，那么验证者最终一定会被说服。
    
2.  **reliable（Soundness）**
    
    如果语句是假的，那么任何试图伪造证明的「作弊」证明者，都几乎不可能骗过诚实的验证者。
    
3.  **零知识性（Zero-Knowledge）**
    
    验证者在整个证明过程中，除了“语句是真的”这一事实之外**不会获得任何额外信息**。
    
    零知识证明通常用两个角色来描述：
    
    -   **证明者（Prover）**：掌握某个秘密信息，希望向他人证明自己「确实知道」这个秘密；
        
    -   **验证者（Verifier）**：希望确认某个断言是真的，但不想（或不允许）知道该秘密本身。

## 一、 Uniswap V2：AMM 界的“极简主义者”

V2 是所有自动做市商（AMM）的鼻祖，它的逻辑简单到一张纸就能写完。

-   **⚡️ 恒定乘积公式 (CPMM)**：核心就是
    
    XxY = k
    
    不论你怎么换，池子里两种代币数量的乘积必须是个常数 。
    
-   **📈 套利者是“搬运工”**：当池子里的价格和市场价不一样时，套利者会冲进来买低卖高，直到把价格“撞”回市场价 。
    
-   **💸 无常损失 (IL)**：LP（流动性提供者）的噩梦 。只要价格波动，你存进去的钱就可能比你单纯握着不动要少，目前只能靠手续费收入来“回血”补偿 。
    
-   **🔮 TWAP 预言机**：通过时间加权平均价格，让链上价格不容易被瞬间“插针”操纵 。
    

* * *

## 二、 Uniswap V3：资金效率的“榨汁机”

V2 虽然好，但资金利用率太低了。V3 就像是给 LP 加了“狙击准星”。

-   **🎯 集中流动性**：你可以自定义价格区间提供流动性 。比如你想在 2000-2500 刀之间做 ETH 的生意，那你的钱就全集中在这里，效率直接起飞 。
    
-   **间隔系统 (Tick)**：V3 把价格空间切成了一个个小格子（Tick） 。价格每跳一个格子，流动性就会发生变化 。
    
-   **🧮 $L$ 与 $\\sqrt{P}$ 公式**：为了合约算得更快、更省 Gas，V3 引入了流动性（L）和价格开方 的简化逻辑 。
    
-   **📊 手续费分档**：不再是一刀切。稳定币对选 0.01% 或 0.05%，主流币选 0.3%，山寨币波动大，可以选 1% 。
    

* * *

## 三、 Uniswap V4：万物皆可“勾” (Hooks)

V4 还没完全普及，但它是未来的大趋势，核心就是一个字：**灵活性** 。

-   **🪝 钩子机制 (Hooks)**：这是 V4 的灵魂 。它允许你在交易的前后、或者加减流动性的时候，“插播”一段你自己的逻辑 。
    
-   **能做什么？**：比如动态手续费、限价单、或者自动化的流动性策略调整 。甚至可以用它来防止抢跑攻击 。

## 一、 EVM 存储全家桶：哪里贵，点哪里

在以太坊虚拟机（EVM）这个大厨房里，东西乱放是要交巨额“保护费”的 。

-   **Storage (链上硬盘)**：这是你的“传家宝”柜子，数据永久上链 。**贵得飞起！** 初始化写一次要 20,000 Gas，改一次也要 5,000 Gas 左右 。
    
-   **Memory (运行时内存)**：像是个“临时工作台”，函数跑完就清理，不上链 。
    
-   **Calldata (外卖窗口)**：这是别人调用你时传进来的参数，**只读不改** 。
    
-   **Stack (计算栈)**：最底层的计算区，像是个叠盘子的架子，1+1 之类的基础运算都在这儿完成 。
    

> **💡 省钱秘籍：** 优化合约的核心就是**少碰 Storage**。能用 Memory 解决的，绝不动用链上存储 。

* * *

## 二、 数据类型：省钱才是硬道理

Solidity 的数据类型直接决定了你的合约是“豪车”还是“五菱宏光”。

-   **值类型 (Value Types)**：
    
    -   **uint8 ~ uint256**：虽然 uint256 最常用，但在 Storage 中，如果能用更小的位宽（比如 uint8）合并存储，能帮你省不少 Gas 。
        
    -   **address**：这是用户的“身份证号” 。记得千万**别用 string 存地址**，那是 Gas 杀手 ！
        
-   **引用类型 (Reference Types)**：
    
    -   包括 `string`、`mapping` 和 `动态数组` 。它们会开辟新的存储空间，成本偏高 。
        
    -   **Mapping (映射)**：它是做“账本”的神器，比如存每个人的余额或授权额度 。
        

* * *

## 三、 ERC20：发币的“通用协议”

想发自己的代币？你得按规矩办事。ERC20 就是那个让钱包、交易所都能认出你代币的“普通话” 。

-   **五大件存储**：代币名称 (Name)、符号 (Symbol)、精度 (Decimals)、总供应量 (Total Supply) 和管理员地址 (Owner) 。
    
-   **两本账本**：一本存“余额”，一本存“授权（我允许你动我多少钱）” 。
    
-   **关键动作**：
    
    -   `transfer`: 转账 。
        
    -   `approve`: 授权，这是玩 DeFi 的第一步 。
        
    -   `Mint`: 铸币，通常要设为 `internal`，不然谁都能给自己印钱了 ！
        

* * *

## 四、 合约的“保安”与“变身术”

### 1\. 谁能点我的函数？(可见性)

-   **external / public**：欢迎外面的世界来互动 。
    
-   **internal / private**：关起门来自己玩，防君子不防小人 。
    

2\. Modifier：最省心的“保安”

这是你笔记里的**重难点**！它就像是函数门口的保安：

-   **复用性**：不用在每个函数里都写 `require(msg.sender == owner)` 。
    
-   **Gas 友好**：只编译一次，到处复用，比散装的 `require` 更高效 。
    

* * *

## 五、 江湖险恶：避坑指南

solidity笔记最后那个 Q&A 揭露了 Web3 的暗黑面：

-   **钓鱼钱包**：看到有人故意泄露私钥，账上有钱却转不出来？那可能是个**合约账户**，转账函数被 `modifier` 锁死了 。
    
-   **抢跑机器人 (Front-running)**：你刚往钓鱼包转 Gas，Bot 瞬间就以更高 Gas Fee 把你的 Gas 抢走了 。
    
-   **重入攻击**：虽然 EVM 是单线程的，但如果你在更新余额前先给别人转账，对方可能趁机反复调用你，把钱取光 。
    

* * *

## 六、 开发利器：Remix IDE

笔记里提到了 Remix，这是你的“实验台”：

-   **编译阶段**：在虚拟 EVM 里找 Bug 。
    
-   **部署阶段**：代码变字节码上链，执行 `constructor` 初始化 。
    
-   **只读函数 (View/Pure)**：不消耗 Gas，随便点

## 二、 关键技术架构与特性

该协议通过引入零知识证明（ZKP）原理和解耦设计，重构了资产交互逻辑 4：

-   **哈希标识所有权**：资产的所有权标识不再是钱包地址，而是公钥的哈希值（$KeyHash$），这使得链上无法直接追踪到具体的持有者地址 555。
    
-   **所有权与发起地址解耦**：交易发起方（Transaction Initiator）与资产所有者（Owner）可以分离 6。任何人都可以作为中继者（Relayer）代付 Gas 并提交交易，而所有权仅由签名控制 7777。
    
-   **UTXO 风格的一次性 Key 机制**：
    
    -   **强制 Key 轮换**：由于交易时公钥会暴露，为了保证隐私，每笔交易后剩余资产会转移到新的 $KeyHash$ 8。
        
    -   **增强不可关联性**：通过一次性 Key 机制，外部索引无法建立起完整的资产流转图谱 9999。
        
-   **取消批准（Approve）机制**：为了防止在链上过早公开 $KeyHash$ 与公钥的关联，该协议去除了 ERC-721 中常见的授权步骤 10。

-   **合规出金**：建议走香港交易所等合规渠道，并主动向税局报税。

​​2025-2026年市场观点​​：

​​预测市场​​：Calibration（CAL）和Polymarket将保持主导地位，且预计将迎来新一轮增长。

​​Web3与传统金融融合​​：行业正在经历从“币圈”到“融入主流金融体系”的转型，重点在于合规化、拥抱机构资金。

Web3 公共物品 = 所有人都能用，但没人能独占收益的基础设施或资源

它和现实世界里的公共物品完全一样，只是搬到了链上。

为什么 Web3 特别强调「公共物品」？

因为 Web3 的目标不是：

让某一家平台垄断

而是：

让整个生态长期可持续

所以：

公共物品 ≈ Web3 的「地基」

没公共物品 → DeFi / NFT / L2 全部玩不下去

二、那「国库 / 军火库（Treasury）」又是什么？

本质上就是 DAO 的钱袋子。

一句话定义

国库（Treasury） = 用来长期资助公共物品和生态建设的资金池

它通常是：

多签钱包

或 DAO 管理的智能合约

钱从哪来？

常见来源：

原生代币（如 OP、ETH）

手续费收入

早期代币预留

生态合作收入

钱花到哪？

乎都花在这几类事情上：

资助公共物品开发者

给开源项目发 grant

支持研究、教育、社区

生态激励（但不是拉盘）

所以你会看到：

DAO 不发分红，却天天给人发钱

为什么叫「军火库」？

因为在 Web3 的叙事里：

公共物品 = 基础设施

开发者 = 工程兵

黑客松 / Grant = 军费

国库 = 军火库

谁的国库能持续供血，谁的生态就能打长期战争

# 害怕、坚持、坚持、坚持

首先是今晚的例会，我已经无法用语言表述了，大家都很厉害，我很害怕，特别是韩关潮同学分享的IPO上链的投研分析，我之前也很关注这个还买了点哈哈哈，就是但是那个SpacX计划于2026年进行IPO，当时引起了很强烈的市场反应，以下是我对关相关项目的分析，用AI润色过，

有没有风险。 当然有，而且 风险不小。

Jarsy 把 “私募股权 + 区块链代币化” 结合，本质上风险来自两个部分：

传统私募市场的风险 + 新型加密平台的风险叠加。

**Jarsy 的主要风险**

1）私募股权天生流动性极低（最大风险）

Pre-IPO 私募股权通常：

• 不能随时卖出

• 退出方式只有 公司上市（IPO） 或 被收购

• 可能需要 3-10 年 才能退出

• 公司可能永远不上市，甚至失败（你钱就没了）

虽然 Jarsy 用 token 让你“看起来可以交易”，

但真实情况是：

你的代币是否能卖掉完全取决于有没有接盘者（流动性池深度）。

如果没人买，代币会成为 死币。

2）法律风险（监管灰色地带）

许多国家对“证券代币化”监管非常严格：

• token 实际上代表股份，会被认定为 证券（security）

• 如果该平台没有得到牌照，未来可能 被限制或被封

• 你可能无法获得法律保护，也不能真正成为“股东”

一句话：

链上代币 ≠ 法律意义上的股份

（可能没有投票权、股东权）

⸻

3）平台风险（最现实）

Jarsy 是 2024 才成立的初创项目。

早期平台都有如下风险：

• 管理不善

• 财务管理失误（或跑路）

• 资产托管不透明

• 代币背后的股权是否真的存在？能否核查？

他们说自己有 “1:1 股权托管” + Proof-of-Reserve，

但你无法直接核实。

这是一个 信任平台，不是比特币这种无需信任的体系。

⸻

4）估值风险（Pre-IPO 反而可能比上市后更贵）

注意一个很反直觉的点：

现在流行的 Pre-IPO 股份，往往比上市后价格还贵

因为独角兽私募估值通常被 VC/机构吹得很夸张。

你买入的价格常常是：

• 最高点（IPO 前最后一轮估值）

• 如果公司上市破发，你就直接亏损

简单说：

Pre-IPO ≠ 早期投资

有些反而是“最晚进场、最贵的时候”。

⸻

5）信息不对称（你几乎不知道公司运营情况）

上市公司财报公开、受监管。

但 Pre-IPO 公司基本不披露详细财务。

你面对的是：

• 对公司了解极少

• 无法判断是否值得投资

• 完全依赖平台提供的信息

所以信息透明度远远低于股票市场。

⸻

6）美元清算 / 加密资产结算风险

你可能用 USDT / USDC 或信用卡支付。

风险包括：

• USDT、USDC 本身有稳定币合规风险

• 资金清算路径不透明

• 税务不明确

跨国操作，出现问题处理成本高。

⸻

适合什么类型的人？

适合：

• 对 Pre-IPO / 私募股权理解很深的人

• 有极强风险承受能力

• 把它当作长期投机玩法（赌运气）的人

• 有闲钱、愿意被锁 3-10 年的人

不适合：

• 想短线赚钱的人

• 想买了随时能卖的人

• 风险承受力弱的人

• 想要类似 BTC/ETH 那种高流动性的投资的人

**一句话总结风险**

Jarsy 是把“高风险、长周期、低透明度”的私募股权，用加密方式包装成 10 美元就能买到的代币。门槛变低，但风险没有降低。流动性、监管、退出方式都不确定。

# 运行原理、Dune、数据规划

## 布老师 web3 运行原理

印象比较深的有私钥的社会学意义，任何人都可以随机创建私钥，即刻拥有一个无需许可的账户与资产权力，对比传统的金融，开设银行账户时kyc非常麻烦，要考虑很多因素

-   布老师给出的背后原因：这是密码学提供的”可验证的个人主权“(我非常认同这个观点）
    

还有助记词和私钥的关系，我之前看dioweb3的介绍视频，说什么有一个私钥池抽出一个私钥然后再生成助记词再通过助记词散列函数计算得出更多的公钥即更多的账户，或者也可以说助记词生成更多私钥但是刚开始的私钥才是唯一钥匙 又或者说助剂词池子后面代替了私钥池 通过助剂词生无数个私钥选一个私钥作唯一钥匙 或者生出一个私钥其他直接是公钥 不管怎么样 其实这个点结合今天布老师的课程增加了我的疑问 不过这个应该不重要 我姑且认为我的这几个理解都是对的吧 还有地址是公钥的一部分

在交易这一节，印象比较深的是防重放序号，之前没太注意这个点，Nonce，其实签名也不太懂 我姑且认为是钱包为我生成了一段看不懂的代码 ，等下等下，刚刚我又复盘了一遍 我觉得就是就是私钥根据交易信息经过哈希计算得到一个值 你再按一个按钮 让这个值变附加你按上去的信息 这就是签名

然后到rpc,RPC（Remote Procedure Call）是钱包与区块链节点通信的方式，本质是远程调用节点能力,虽然区块链共识是去中心化的，但 RPC 作为访问入口往往集中在少数服务商，形成了“使用层的中心化风险”。rpc相当于于一个喇叭，因为你不可能每个节点去通知，成本太大，钱包一般自带一个npc

Chain ID 是区块链的唯一标识，用于防止交易在不同链之间被重放

Chainlist 是社区维护的第三方工具，用于快速配置链的 RPC 和参数，并非官方。(收集各条链 RPC 的网站，方便一键加网络)

# 今日思考

那一个大学生想在web3做运营类工作是不是也面临诸多风险（又可能被捉），那怎么办呢

要满足形式合规/深度合规/持续风控...

而深度合规运营要隔离中国市场（那交易所几乎都不行）

# 安全与合规分享会总结

-   安全四不要，不要点陌生链接 ，不要随便签名，不要装别人给的任何东西，不要随便转钱
    

**Web3 合规**

**第一层面：形式合规**

**目标：**在文本、合同、协议、白皮书等法律文件上做到形式合法合规

**方式：**

使用符合监管要求的术语和表述

避免使用敏感词汇（如“融资”“投资回报”“保本”等）

明确风险提示和免责条款

遵循国际标准（如 FATF、MiCA 等）的披露要求

**局限：**仅能降低表面风险，无法完全规避实际业务中的法律风险

**第二层面：深度合规（实际运营层面）**

**目标：**在业务模式、资金流向、用户交互等实际运营中规避法律风险

**关键点：**

避免非法募资：绝不涉及 ICO/IEO/IDO 等公开融资行为

隔离中国大陆用户：不向境内居民提供交易、兑换、支付等服务

防范刑事风险：远离赌博、传销、洗钱、非法经营等行为模式

建立真实业务闭环：不以“纯金融”为目的，具备真实技术或服务支撑

**挑战：**

需持续跟踪监管动态和司法判例

需建立内部合规体系和风险控制流程

需考虑跨境法律冲突和属地管辖问题

**“深度合规不是 Web3 特有”**

**给 Web3 从业者的建议**

| 合规层面 | 具体建议 |
| 形式合规 | 1. 聘请专业律师起草法律文件 2. 白皮书/官网避免承诺性表述 3. 明确用户协议和风险提示 4. 遵循国际合规标准（如 KYC/AML） |
| 深度合规 | 1. 业务设计避开“金融属性”红线 2. 技术架构支持合规审计与追踪 3. 运营中隔离中国大陆市场 4. 建立内部合规培训与监督机制 |
| 持续风控 | 1. 定期进行合规自查与风险评估 2. 关注监管政策与司法判例动态 3. 准备应急预案（如监管调查、用户投诉） |

**Q＆A：**

**二、为境外持牌项目方远程工作是否存在风险？如何规避？**

**风险筛查步骤**

**1.牌照真实性核查：**

仅持有美国牌照 ≠ 深度合规，可能只是“合规入门”

警惕项目方以“全球牌照”为名夸大合规范围

**2.是否面向中国大陆展业：**

需深度沟通，确认是否服务大陆用户

**实操验证：**

能否在不翻墙情况下访问官网？

能否用大陆邮箱/手机号注册账户？

若仅做 IP 隔离但仍允许大陆身份注册，仍可能被认定“向大陆展业”

**3.业务内容风险等级：**

若涉及合约、期权等衍生品业务，风险较高

技术、行政等支持岗位风险相对较低

**合同与薪酬风险**

**1.合同签约主体：**

纯境外主体签约：发生劳动争议时，在大陆维权极难

国内主体签约（包括第三方派遣）：劳动关系清晰，但若该公司涉非法业务，员工易被牵连

**2.薪酬支付方式：**

虚拟币支付工资存在法律风险：

中国劳动法要求以人民币支付工资

已有仲裁案例判决：虚拟币支付不被支持，公司需补发人民币工资

**风险规避建议**

**1.入职前尽职调查：**

验证牌照真实性、业务边界、用户来源

自行测试注册与访问限制

**2.合同审查：**

明确签约主体、薪酬结构、争议解决机制

**3.业务隔离：**

优先选择完全不服务大陆用户的项目

避免参与高风险业务板块（如合约、期权）

**三、若已为问题项目方打工，被调查时如何自保？**

**1.无统一标准答案：**

需根据具体岗位、业务内容、证据材料等“具体情况具体分析”

**2.超前合规预案：**

部分合规意识强的项目方会提前为核心员工安排授权律师

律师可协助员工梳理：

业务模式的法律风险点

可能面临的调查路径

应对策略与沟通话术

**3.资源博弈本质：**

最终可能演变为“资源博弈”，即律师凭借专业材料与当地资源进行协调

项目方若能提前部署法律支持，可显著降低员工个人风险

**四、Web3 新人入行有哪些不易察觉的合规风险？**

**1.行业本身风险密集：**

Web3 行业“处处是风险”，从个人钱包安全到业务开展均有隐患

举例：钱包可能收到伪装合约地址，授权后资产即被盗

**2.动态风险意识是关键：**

风险随行业经验积累而显化，需持续学习、更新认知

加入专业社群、跟踪行业知识是重要的风险规避方式

**3.“不作恶”是保命符：**

最大的风险往往来自“人心”，如合伙人卷款、内部背叛等

坚守底线、选择可信合作伙伴比文本合规更重要

**4.风险与收益的平衡：**

多数人通过简单出入金管控即可平稳参与

遇重大决策时可咨询专业律师

**五、Web3从业者推荐哪些合规出金方式？若遇账户冻结/调查如何应对？**

**合规出金方式推荐**

**1.香港合规出金：**

最安全路径：通过香港持牌交易所或合规渠道出金，税务成本较高但风险最低

**2.小额消费场景：**

使用合规U卡（需严格筛选发卡方）

**3.熟人/亲友网络：**

在信任圈内进行OTC交易，降低涉诈风险

**4.无统一安全方案：**

出金安全是“反洗钱”老难题，暂无100%安全产品

**遇账户冻结/调查应对策略**

**1.首次涉案且证据清晰：**

如实说明情况，提供交易记录、资金来源证明

配合调查，通常记录后即可解封

**2.多次涉案或证据复杂：**

建议先咨询律师，避免被推定为“以兑换维生”或“明知故犯”

**3.地区差异明显：**

南方地区沟通相对文明，北方地区可能更直接

若遇强势要求“到场”，可评估风险后决定是否配合

**4.退赔谈判技巧：**

若决定退赔，可要求公安同步解冻关联账户（微信、支付宝等）

退赔可作为换取生活账户正常化的筹码

5.  Stuck Transactions, Gas Limits, Multisigs, L2s, Lending...md
    

-   gas费
    
    -   一个糟糕的交易：Gas 大概是 33 Gwei。“编辑 Gas 费”，点“高级。把 Max Fee 设置为 10 。他们不会打包我们的交易。交易“漂（Float）”进了内存池
        
    -   每笔交易都有一个序号。 如果你现在的这笔卡住的交易 Nonce 是 6。那么在你这笔交易完成之前，Nonce 为 7、8、9 的交易全部都不会执行。 就像排队一样，6 号没过，后面的都得等着
        
    -   解决；加速。本质上是发一笔新的交易，拥有相同的 Nonce（6），但是更高的 Gas 费。选钱多的那笔，丢弃钱少的那笔。交易瞬间就成功了
        
-   重置账户
    
    -   不会删除你的私钥或资金。它只是清空了 MetaMask 本地的交易历史记录
        
-   取消交易
    
    -   我们再发一笔低 Gas（10 Gwei）的必死交易（Nonce 7）。它卡住了。
        
    -   现在，我们给自己发一笔 0 ETH 的交易。 在“自定义 Nonce”里填上 7。 把 Gas 费设高。 原理是：我们用一笔“转给自己 0 元”的高价交易，覆盖掉了那笔“转给 Austin 5 DAI”的低价交易。
        
    -   取消”的底层逻辑
        
-   去 Layer 2 (Optimism)

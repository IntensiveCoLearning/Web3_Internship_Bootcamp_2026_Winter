# ERC-7962

在 ERC-721 的基礎應用上結合 ZK 技術，向第三方保證其歸屬，但不暴露其真實地址。

舉例：星巴克會員在機場能夠取得貴賓室優惠，但星巴克不能把會員資料共享給機場，機場也不想知到，只需要證明屆可以了。

-   核心概念：以 Key Hash （公鑰哈希）而非地址作為所有權的標記
    
    -   發起交易的地址，不一定要是真實持有資產的地址
        
-   不在鏈上存儲地址：所有權與交易發起地址解耦
    
    -   不用地址本身發起
        
-   包含：ERC-20 ERC-721 兩種代幣種類
    
-   能夠做到鏈抽想：滿足 Web2 用戶體驗
    

解決什麼問題

-   隱私問題：避免地址追蹤問題、降低身份與資產的關聯性
    
-   UX 問題：無序持幣、處理 Gas，Web2 的用戶體驗一致
    
-   擴展性問題：支持批量交易、Relayer（簽完名之後隨便給誰都可以）、帳戶抽象場景

# 助记词的延伸脉络：

-   註记词 → 唯一 种子
    
-   种子 → 多个 私钥
    
-   私钥 → 单一公钥
    
-   公钥 → 单一 地址
    

**范例： OKX Web3 钱包只要纪录一个助记词就可以生成很多帐户（Address）**

# 私钥

社会学意义：个人主权的起点

-   密码学提供「可验证的个人主权」
    
-   可以自己生成「权力入口」，无需他人许可
    

最原始的样貌是 256 位元（256 个 0 或 1），相当佔据空间，如果以 16 进位制来表达，可缩短为 64 个字串，其中每一个字串都是由组字 0-9 和大小写英文字母 A-F 随机生成。

# 种子（Seed）

一段随机生成的字串序列，用来生成加密钱包的公私钥对（Keypairs）

一颗种子可以生成多个私钥

# 2026-01-14

第三章讲了一下合约账户的原理，以前只知道私钥不能外传。。。

以太坊内只有两类账户：

1.  EOA：由公私钥确定控制，eg：钱包
    
2.  智能合约：无密钥
    

其实也可以理解为人类控制的钱包和代码控制的钱包

它们的共性是有nonce & address& balance

-   地址的存在确保了每个账户在网络中是唯一且可识别的，从而能够接收和发送交易
    
-   balance是以太坊能够作为去中心化的支付系统的基础，确保每个账户可以拥有、转移资产
    
-   智能合约账户虽然没有密钥（公私钥对），但智能合约仍然会有自己的nonce（由外部触发的交易），用于保证智能合约的操作也有顺序
    

钱包的衍生品有多签钱包，以多数持有者签名同意为执行标准

智能合约中也可以设置时间锁，在延时到期后才可以被执行，但是也正是因为时间差，需要高度审计

在以太坊中，所有的地址都以0x开头，主要是为了标识这些地址是 **16进制** 格式，并且区分它们是以太坊地址而非其他类型的地址。

# 2026-01-13
<!-- DAILY_CHECKIN_2026-01-13_START -->


“第二章主要围绕着节点展开：在以太坊中，**区块**是实际存储交易和状态的账本数据，**节点**是一台接入网络、负责下载、验证和保存这些区块的计算机，而**客户端**是运行在节点上的软件，实现以太坊协议并决定节点如何同步、验证和通信。PoS 之后，一个完整节点必须同时运行**执行客户端**和**共识客户端，其中共识客户端负责决定区块的共识性，执行客户端负责计算“这些区块执行后状态是否正确”，两者通过 Engine API 连接以保证一致性并隔离风险。节点之间先通过发现与加密连接组成网络，再通过 Gossip 传播区块和交易；区块之间则通过父区块哈希形成链，并在共识层通过投票和最终确定性解决分叉。所谓节点同步**，本质就是先跟随共识层对齐全网公认的进度，再由执行层从已确定的位置开始逐块执行，直到追上当前链的状态。”

首先辨析一下节点和客户端，客户端是基于节点之上的，节点也不是区块链中的区块。区块是实际存储交易数据的地方，而节点是通过读取和验证这些数据来保持区块链正常运行的实体。

### 节点 (Node)

-   在以太坊网络中，节点指的是连接到以太坊区块链的计算机或服务器。每个节点都有自己的职责，可以参与区块链的共识过程、验证交易、存储区块链数据等。
    

### 客户端 (Client)

-   客户端是指运行在节点上的软件，它实现了与以太坊网络交互的协议。每个客户端可以实现不同的功能，但它们都遵循相同的以太坊协议。
    
-   重要分为：执行、共识和验证者
    
-   其中的共识客户端其实不仅仅是为了DAO而服务的，其必不可缺是因为主要用于维护区块链的特性：验证交易和生成新区块，这也是PoS的一部分。执行和共识通过engine API链接起来，使整个区块的安全性和隔离性增强
    

节点之间的链接方式是：先找好友再基于安全协议扩散

当一个新的以太坊节点上线时，大致会经历这样一个过程：

**① 发现阶段（UDP + Kademlia）**：

先通过引导节点 + Ping/Pong/FindNode，逐步认识一群“距离 ID 较近”的邻居，把自己的“通讯录”填满。

**② 连接阶段（TCP + RLPx/devp2p）**：

对选中的邻居建立加密的 TCP 连接，在同一条线路上多路复用区块同步、交易传播等子协议。

**③ 传播阶段（Gossip + 请求/响应）**：

新交易和区块通过 Gossip 像办公室八卦一样在全网扩散；缺失的历史数据则通过点对点请求-响应按需拉取。

而区块之间是通过父区块哈希相连接的链式结构，但实际上，区块链中的区块也存在一些特殊的情况，如 **分叉**（fork）和 **最终确定性**（finality）的问题。为了解决这一问题，区块链系统（尤其是以太坊 2.0）采用了 **有向无环图（DAG）** 的思想来帮助共识客户端快速选择“主链”，以避免分叉造成的不一致。

-   **分叉**：如果两个矿工（或验证者）同时挖出（或生成）一个新区块，网络会暂时形成分叉。此时，网络中的其他节点会选择最长的有效链作为主链，最终合并分叉。
    
-   **最终确定性**：在以太坊 2.0 的 PoS 中，区块在达到一定数目的确认后（如 2/3 的验证人确认），就会被认为是最终确定的，不会再出现分叉。
    

节点又根据不同的使用用途及经济上的权衡分为了三类：

-   全节点：提供网络的完全验证功能，但需要大量存储和计算资源，维护了网路中节点的安全性和去中性化
    
-   轻节点：仅存储区块链的头部，仅验证交易和区块
    
-   归档节点：存储以太坊完整历史数据
    

在PoS下，一个完整的区块链节点要同时运行下面两种客户端，隔离了共识or执行的bug：

-   **执行客户端（Execution Client，EL）：**关心被接受的区块做了什么事情
    
-   **共识客户端（Consensus Client，CL）：**关心哪个区块是被以太坊承认的
    

因为在 PoS 之后，“区块存在” 和 “区块执行” 是两条独立因果链，节点同步就是把这两条因果链重新锁在一起的过程。本质上就是先找到大家公认的帐本速度，然后从高一个不会改的地方开始审查，直到追齐了进度。

晚上听bruce老师的分享，更正了以下误区：

1.  钱包在交易中做：组装交易内容——用私钥签名——广播到区块链网络
    
2.  查看交易的status: unfinalized 表示区块是需要不可篡改时间的，在十分钟之后才是不可篡改的，这是在PoS下的机制
    
3.  钱包并不直接与区块链网络交互，后面链接的是PRC，所以钱包签名的交易会由rpc转发到区块链网络中。钱包只是充当一个用户工具，又要保存好私钥又要交易难度很高，加上钱包很多样，通过统一的接口来与网络交互会方便很多。

我之前有看过中本聪发布的关于去中心化的论文，以太坊十周年的时候梳理过时间轴，但是都是只停留在了了解这个概念的层面，做copypaste，对区块链底层的原理和相关概念仍然没有打通。

学习最大的体会是无论对于以太坊本身、还是项目方，在选择上都在进行物理与经济的权衡。学校擅长教我们要学会辩证的看待问题，但是从未教过我们如何在现实世界中运用辩证的思想进行选择。

以太坊早期的官方路线图，主要将网络开发划分为四个阶段：边境（Frontier，也有“前锋”的意思）、家园（Homestead）、都会（Metropolis）和宁静（Serenity）。整个过程是符合物理规律的。

边境阶段是以太坊发展过程中最基础的阶段。由于以太坊的目标是建立一个去中心化的平台，第一步是先提供基础设施和矿工的支持，以确保网络能够开始运行。没有这个基础，后续的功能拓展就无法顺利进行。

家园阶段是一个过渡阶段，旨在增强网络的稳定性、可用性和可扩展性，为后续更加复杂的功能和技术做准备。

都会阶段的核心目标是解决以太坊在发展过程中暴露出的性能瓶颈和可扩展性问题。随着去中心化应用的增长和网络需求的增大，必须通过优化协议和引入新技术来保持以太坊网络的高效运作。只有不断提升性能和安全性，才能支撑更广泛的应用场景。

宁静阶段的目标是解决以太坊网络面临的扩展性。随着用户和DApp的增多，单一链的处理能力无法满足需求。采用分片技术和PoS是对以太坊未来需求的必然响应。

接着我重新的明确了以太坊的定位，有如下两个方面：

1.  layer 1 blockchain：共识机制/EVM/ETH
    
2.  二级交易平台：支持Dapp、DEC
    

更明晰了dapp的概念，理解了很多项目方本身就是Dapp，这样就把以太坊、rollup、dapp穿起来了：

· dApp 的**后端代码** （智能合约）运行在以太坊主网或其 Rollup （二层网络）之上的 EVM 中，由全网节点共同执行和记录；

· dApp 的**前端**可以用任意 Web / App 技术编写，既可以放在传统服务器上，也可以托管在 IPFS、Arweave 等去中心化存储上

其中值得一提的是：分片是为了缓解区块链的“不可能三角”（即在去中心化、安全性和可扩展性之间难以同时实现高水平），通过并行处理来提升整体吞吐量和效率。以太坊早期路线图确实计划通过“执行分片”来实现，比如设想引入64条分片链，每条链独立处理部分交易和状态，从而分散负载。但后来转向了Danksharding方案，这是一种更注重“数据分片”的设计。主要原因是执行分片会加重主网的中心化程度和计算量，再加上rollup技术的兴起，让以太坊意识到没必要在L1上直接分片执行——可以让L2承担执行负载，主链只需提供廉价、安全的数据存储。

不过我发现我没有分清楚ETH和NTF的关系：以太坊以ETH（Ether）作为其原生资产，但它同时是一个强大的区块链平台，支持通过智能合约创建和铸造各种其他资产，包括NFT。

智能合约虽然不是以太坊创立的，但是以太坊是第一个大规模实现和普及智能合约的区块链平台。由Vitalik Buterin在2013年提出，并在2015年正式上线，以太坊通过其虚拟机（EVM）允许开发者编写和部署智能合约代码。

**一些辨析的概念：**

· 可替代的（ERC-20）：例如游戏里的“金币”、治理代币、稳定币等； · 不可替代的（ERC-721 / ERC-1155）：对应独特资产，如艺术品、门票或游戏角色。

“图灵完备”这个术语本质上就是：只要给足时间和资源，就能表达几乎任何可计算的逻辑。

# 2026-01-20
<!-- DAILY_CHECKIN_2026-01-20_START -->

### 1\. 行业地位：无可撼动的生态统治力

Solidity 是 EVM（以太坊虚拟机）的母语。尽管 Solana (Rust) 和 Move 生态（Aptos/Sui）在高性能领域发力，但 Solidity 的地位反而因为 **Layer 2 (L2)** 和 **zkEVM** 的普及而得到了巩固。

-   **标准制定者：** 从 ERC-20 到 ERC-721，再到复杂的流动性挖矿逻辑，Solidity 拥有最成熟的合约库（如 OpenZeppelin）。
    
-   **跨链通用语言：** 几乎所有新兴的链（如 Base, Arbitrum, Scroll, 甚至最近升级支持 Solidity 的 Polkadot 2.0）都必须优先支持 EVM。学会了 Solidity，就意味着你掌握了通往 90% 以上 DeFi 锁仓价值（TVL）的钥匙。
    

### 2\. 技术设计：极简与极险的平衡

Solidity 的设计深受 JavaScript 和 C++ 的影响，这导致了它具有明显的“两面性”：

-   **优点（易学性）：** 语法直观，开发者可以快速从 Web2 转向 Web3。它采用了面向合约（Contract-oriented）的设计，非常符合金融业务的直觉。
    
-   **缺点（安全陷阱）：** 它的底层设计（如重入漏洞、整数溢出、Gas 限制等）对开发者极其不友好。在 Solidity 中，**每一行代码都可能涉及真金白银的损失。**
    
-   **改进：** 随着 0.8.x 版本的迭代，语言本身内置了溢出检查等安全机制，编译器变得越来越聪明。
    

### 3\. 工具链的进化：从“简陋”到“工业化”

早期的 Solidity 开发者受困于 Remix 或 Truffle 等原始工具，但现在生态已发生了质变：

-   **Foundry 的统治：** 现在的开发者大多使用 Foundry（基于 Rust 开发），它极大地提升了测试速度和调试效率。
    
-   **形式化验证：** 2026 年，通过 AI 辅助审计和形式化验证（Formal Verification）来检测 Solidity 代码逻辑漏洞已成为大厂标配，这弥补了语言本身的某些不安全性。

# 2026-01-15
<!-- DAILY_CHECKIN_2026-01-15_START -->





1\. AI 的演变趋势（2023-2026）

-   **2023：LLM（大语言模型）**：专注于预测下一个 Token，被动响应用户查询。
    
-   **2026：Agentic AI**：结合推理、记忆和工具使用，实现自主执行任务。
    
-   预测数据：Gartner 预计到 2028 年，33% 的企业软件将嵌入 Agentic AI（2024 年 <1%）。Baker Donelson 称 2025 年为“AI 问责之年”，焦点转向合规责任。
    
-   转变核心：AI 从“对话”转向“行动”，面临法律与经济责任挑战。
    

2\. AI 智能体的解剖结构

-   **大脑（The Brain）**：LLM 负责推理与规划（如 GPT-5.2、DeepSeek V3.2、Claude 4.5）。
    
-   **记忆（The Memory）**：向量数据库（Vector DB）支持 RAG（Retrieval-Augmented Generation）流程，包括数据抓取和上下文存储，实现“持久化”记忆。
    
-   **工具（The Tools）**：API 接口连接外部世界，支持执行代码、查询数据库、发送指令。
    
-   关键词：传统 LLM 是“无状态”的，智能体通过向量 DB 和 API 获得持久身份与执行能力。
    

3\. 基础设施缺口：身份与信任危机

-   **身份欺诈（Identity Spoofing）**：传统网络无法区分人类与机器人，易被模仿执行未授权操作（来源：Rippling）。
    
-   **不可审计（Untraceability）**：缺乏不可篡改日志，AI 签署合同后责任难追溯（来源：Baker Donelson）。
    
-   **影子 AI（Shadow AI）**：59% 组织缺乏统一治理框架（来源：Truyo）。
    
-   问题本质：AI 行动缺乏可验证的身份和审计机制。
    

4\. 支付摩擦：机器经济需求

-   **旧模式**：信用卡订阅（$20/月）、人工 KYC、对微任务昂贵。
    
-   **新需求**：机器对机器（M2M）微支付、按 API 调用/推理付费、毫秒级结算。
    
-   预测：2025 年，机器人处理 70% 的稳定币转账。
    

5\. Web3 AI 协议栈：赋予自主权

-   **区块链作用**：提供不可篡改的身份证明和原生银行账户。
    
-   **层级结构**：
    
    -   **应用框架层**：SpoonOS Framework，作为开发者操作系统。
        
    -   **身份层**：ERC-8004 标准，链上护照与声誉系统。
        
    -   **交易层**：x402 协议，HTTP 原生机器支付。
        
-   核心：连接 Web2 AI 开发能力与 Web3 基础设施。
    

6\. ERC-8004：智能体的链上护照

-   **身份注册表（Identity Registry）**：基于 ERC-721，提供唯一、可转移的链上 ID。
    
-   **声誉系统（Reputation Registry）**：记录反馈评分（0-100 分），仅付费客户可评价，防止垃圾信息。
    
-   **验证机制（Validation）**：使用 TEE（可信执行环境）或 zkML 证明，解决“你是谁”和“你可信吗”的问题。
    

7\. x402 协议：HTTP 原生机器支付

-   **机制**：请求资源时需支付证明（402 Payment Required），自动签名交易（USDC），成功后返回 200 OK。
    
-   **应用**：按文件付费（Pay-per-file）、按算力付费（Pay-per-inference），无需订阅，无需人工干预。
    

8\. 安全与合规：从黑盒到透明审计

-   **合规需求**：持续监控（来源：Truyo）；欧盟 AI 法案要求高风险系统透明可追溯。
    
-   **Web3 解决方案**：不可篡改的审计日志，所有推理、交易、验证永久存证，满足严格监管。
    
-   优势：全生命周期链上记录，确保责任追溯。
    

9\. SpoonOS：Web3 AI 时代的操作系统

-   **定位**：桥梁框架，一站式集成身份验证与支付协议。
    
-   **赋能开发者**：专注模型能力，SpoonOS 处理信任与变现。
    
-   **架构**：
    
    -   输入层：数据流与向量 DB。
        
    -   核心层：SpoonOS Engine、统一 LLM APIs、智能体通信。
        
    -   协议层：Identity Manager (ERC-8004)、Payment Gateway (x402)。
        
    -   行动层：DeFi、主网交互、Web API 操作。
        
-   益处：无需从头构建基础设施。
    

10\. 基于 SpoonOS 的未来应用

-   **DeFi 交易智能体**：链上声誉资产管理器，自动套利，历史业绩可查。
    
-   **数据市场代理**：使用 Firecrawl 抓取数据，通过 x402 微支付出售数据集。
    
-   **自主服务代理**：自动修复代码 Bug，每 PR 收取 USDC。
    

11\. 2026：机器经济的崛起

-   **数据**：稳定币月结算量 $1.39 Trillion（来源：Coinbase）。
    
-   **愿景**：经济主体从人类转向亿计 AI 智能体，拥有自主钱包；Web3 承载高频、低额机器交易。
    
-   **增长**：AI 智能体普及指数级上升。
    

12\. 核心要点总结

-   **趋势**：AI 转向行动，面临法律经济挑战。
    
-   **协议**：ERC-8004（身份）+ x402（支付）构建信任基础。

今天的分享会主题是Web3运行原理

大部分之前学过，所以感觉没什么好记录的，都是很重要的基础知识

对于最后Bruce留下的问题很有兴趣，但是都是很宏观困难的话题，我纯主观意见哈

**1、如何提高安全性、降低管理私钥的复杂度？**

对于私钥管理，我能想到的就是本地加密了，我在编写合约的时候，如果用到私钥，会通过终端输入私钥，并直接加密处理，将加密后的文件存储到本地，使用的时候调用再输入密码，才能调用密钥。

但是针对大量私钥，密码设置就成了问题，全设置一个密码太不安全，随即生成的话，密码的存储又变成了棘手的问题。

因此对于大量私钥，最安全莫过于本地化存储。

如果要和互联网连接，就需要十分高等级的安全服务器，不仅要防止黑客攻击，还要防止管理不当私钥泄漏。

**2、没有中心化机构/税收时，公共基础设施谁来维护？如果有税收，又如何分配？**

这个我一眼想到了区块链矿工这一工作，没人对矿工的工资负责，但是矿工依旧乐此不疲地帮我们打包区块，干这些“力气活”，因为得到的收益是完全可预见而且可观。

公共基础设施也可以使用类似方式，不禁想到一句讽刺意味拉满的笑话：如果扫大街能发家致富的话，那么环卫工人将无地可扫。天下熙熙皆为利来，天下攘攘皆为利往，只要使用公共基础设施的人强制给维护的人一笔费用，总会有人来干这份工作。

如果有税收，我将全力支持POW共识，按劳分配，就这么简单，区块链真是GOAT。

**3、没有审查且隐私很强时，如何界定并治理有害信息/黑产？**

交易所和各国政府给出了不太令人满意的答案，比如在链上追踪黑U，收到黑U的人直接封锁账号，但关键是这个纯是单向操作，我有100w黑U，我随便散播这些黑产，我不会受到任何损失，但是那些不知情的人会直接被交易所拉黑，被警察找上门。

理想状态是涉及黑产/有害信息的交易根本不会发送出来，就是没人会去打包这些区块，根本不可能实现。

怎么界定，怎么处理，这个问题太庞大了，这估计整个Web3都在想办法，我答不上来。

**4、去中心化协作下，如何实现公平、可信的分配？**

这太简单不过了，在去中心化的前提下，所有人的工作都能被量化，而且透明可见，这还不能满足公平可信的分配吗？问题在于如何构造一个透明的预言机，让数据能透明的在链上链下传递，不会被内幕篡改。

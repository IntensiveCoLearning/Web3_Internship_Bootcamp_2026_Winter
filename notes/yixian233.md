# 2026-01-14
<!-- DAILY_CHECKIN_2026-01-14_START -->

这次晚间分享最大的收获，让我终于意识到：在国内做 web3，风险不是来自技术，而是来自“定性”。

同一件事，技术上自洽、商业上合理，但在大陆语境里，可能被直接理解成赌博、非法经营或共犯行为。这让我第一次真正放下“技术中立”的幻想；现实里，没有“我只是写代码 / 做运营”，只有“你在不在这条链条里”

我也对“合规”彻底祛魅了。

合规不是正确性，而是一种止损策略，是为了在不可控环境里，给自己留谈判空间，而不是保证安全。

另一个重要转变是：

“在哪里做”比“做什么”更重要。地理位置、司法管辖权、资金路径，往往比赛道和叙事更决定个人风险。

这次分享没有让我否定 web3，但让我放下了很多浪漫想象。清楚自己站在哪里，本身就是一种成熟。

# 2026-01-13
<!-- DAILY_CHECKIN_2026-01-13_START -->


资产自拖管其实就是银行的性质，web3把金融主权还给个人的同时，也把金融机构的安全风险全部转嫁给了用户，本质矛盾是去中心化不一定对用户友好。而私钥的复杂度是为了解决安全的问题，but却导致用户使用的门槛较高，个人感觉这是产品层面和基础层面的问题，需要从多方面去解决

而税收的分配问题，需要通过DAO来进行解决，web3的分配很难量化，所以说有了声誉系统和链上行为评分等等，但是个人感觉目前是个很难的问题，分配制度还不够成熟防止所谓链上民主被架空，以及低参与率的问题

界定有害黑灰产是个价值性的问题，Web3 理念都强调隐私保护和抗审查。but治理问题不会因为上链而消失，这需要我们进行审查。

当然这种审查也必须有足够权威和民主性

# 2026-01-12
<!-- DAILY_CHECKIN_2026-01-12_START -->



在今天的行业会议之前，我对于web3运营板块的工作还停留在web2阶段。

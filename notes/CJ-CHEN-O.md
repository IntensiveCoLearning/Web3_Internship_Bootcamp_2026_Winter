# 2026-01-18
<!-- DAILY_CHECKIN_2026-01-18_START -->
在学习 Web3 安全相关内容后，我逐渐意识到，Web3 的最大风险并非技术本身，而是**用户对风险的低估**。私钥即资产，一旦泄露或误签恶意合约，几乎不存在追回的可能，这与传统金融“可冻结、可申诉”的机制完全不同。因此，Web3 安全首先是认知安全，而非工具安全。

大量安全事故并非源于复杂攻击，而是钓鱼链接、假空投、授权滥用等基础问题，说明很多风险来自人为操作失误。通过学习我认识到，参与 Web3 必须建立最小权限原则，区分主钱包与交互钱包，谨慎授权和签名，并定期清理合约授权。只有在充分理解规则和风险的前提下参与，才能避免将“去中心化的自由”变成不可逆的损失。

「3 个人 + 3 个信封」的故事

背景

你有一笔一旦丢了就永远拿不回来的钱。

你很清楚一件事：

人是会犯错的。

你不想把“全部命运”压在自己一个人身上。

第一种做法（最常见，也是最危险）

你把所有信息写在 1 个信封里，

自己保管。

搬家丢了

被水泡了

忘记放哪

人出事了

👉 一次失误 = 全没

第二种做法（你现在这个想法）

你把这份重要信息 拆成 3 份，

装进 3 个信封：

信封 A：你自己

你保管一份。

信封 B：我（你信得过、但也不完全信的人）

我保管一份。

信封 C：第三方（律师 / 家人 / 规则）

一个不由你单独控制的人或机制。

⚠️ 规则是：

必须同时拿到至少 2 个信封，才能把信息拼出来。

“如果律师那份也忘了，不也是完蛋？”

答案是：

是的，有可能完蛋。

对比两种“完蛋方式”

世界 1：1 个信封

你忘了

👉 100% 完蛋

世界 2：3 个信封

要完蛋，必须同时发生：

你忘了

我这边也出问题

第三方也出问题

👉 不是不可能，但明显更难

这就是全部意义。

# 2026-01-16
<!-- DAILY_CHECKIN_2026-01-16_START -->


通过对 Web3 合规问题的学习，我逐渐认识到，监管所限制的并非区块链技术本身，而是失控的金融行为。中国内地明确禁止 ICO、虚拟货币交易所和挖矿，本质上是为了防范非法融资、金融风险扩散以及资源错配。这让我意识到，Web3 不能简单等同于炒币或投机，而应回归技术和应用层面。

合规并不意味着创新受阻，而是要求创新具备边界和责任。真正可持续的 Web3 形态，应集中在底层技术、数据分析、数字身份、协作工具等非金融或弱金融领域，而不是绕开监管进行资本运作。对个人而言，这次学习促使我调整方向，从追逐短期机会转向积累长期可迁移的能力，以更加理性、现实的视角看待 Web3 的发展路径。

# 2026-01-15
<!-- DAILY_CHECKIN_2026-01-15_START -->



以前我一直以为，链上交易就是点一下“发送”，等几秒到账。真正开始系统学之后才发现，一笔交易背后，其实是一整套流程。

交易首先发生在钱包里。钱包做的不是转账，而是用私钥给交易签名，本质上只是表达“我同意这笔操作”。签完名后，交易通过 RPC 发到节点，再被广播到整个网络。

接下来交易会进入 mempool。这里不是简单排队，而是一个博弈场。谁给的 gas 高、谁对别人更有利，谁就更可能被选中。这也是很多抢跑、夹子、MEV 发生的地方。

区块并不是自动生成的，而是由 builder 从 mempool 里挑交易，再由 validator 确认。标准并不是公平，而是收益和规则允许范围内的选择。

一旦交易进了区块，状态就被永久写入，无法撤回。我们平时用的区块浏览器，只是把这些结果展示出来，并不参与任何决策。

理解这条链路后，才发现“会用钱包”和“真正理解区块链”，差得其实很远。

# 2026-01-13
<!-- DAILY_CHECKIN_2026-01-13_START -->




这是我的第一条笔记。  
我是一名加密货币爱好者。

最早接触这个领域，是从最表层的炒币开始——价格、涨跌、情绪驱动的交易。后来逐渐意识到，这只是最末端的结果，而不是核心。再往下走，我开始真正去理解比特币和区块链本身。

比特币吸引我的，不是短期暴涨的故事，而是它背后的设计逻辑：  
去中心化、不依赖单一权威、规则写进代码、共识高于个人意志。它试图在一个低信任环境中，构建可验证、可持续的协作体系，这一点在现实世界中并不常见。

我并不认为区块链是万能答案，也不否认这个行业充满泡沫、投机和噪音。但正因为混乱，才更值得被认真拆解。技术、机制、博弈、人性，这些都真实地交织在一起。

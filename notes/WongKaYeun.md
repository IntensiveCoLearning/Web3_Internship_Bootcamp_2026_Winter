Web3 运行原理 笔记

Private Key一对一绑定 Public Key

Address 取最后的20位开头加上0x作为钱包地址（公开收款账号）

1.  助记词/私钥：拿到就有了资产控制权
    

-   助记词和私钥一对多，可以派生出多个私钥
    

2.  交易 Transaction/TX = 要做的事 + 手续费+ 防重放序号
    
    -   钱包在这个过程中组装交易内容，用私钥签名（证明是由你授权的），然后广播到区块链网络
        

-   **签名：** 交易的起点。当你点击“发送”或“确认”时，钱包（如 MetaMask）会使用你的**私钥**对交易数据进行**数字签名**，来证明这笔交易是由账户拥有者本人授权的，且内容未被篡改。
    

-   **传播：** 签名后的交易通过 **RPC**（远程过程调用）接口发送到区块链节点。节点会对交易进行初步检查（如格式是否正确、余额是否足够），然后将其广播给网络中的其他节点。
    

-   **排队：** 交易进入**内存池（Mempool）**。这是一个“待处理交易池”，所有通过验证但尚未被打包进区块的交易都在这里排队。这里的交易是按 **Gas Fee（手续费）** 高低进行排序的，给的小费越高，被处理的速度就越快。
    

-   **挑选：** **Builder（构建者）**负责从 Mempool 中挑选盈利最高的交易，并将它们排列组合成一个完整的区块。**Validator（验证者）：** 负责审核 Builder 提交的区块。这是共识机制的核心步骤，决定了哪些交易能够进入下一个区块。
    

-   **落盘：** 当区块被验证者选定并达成共识后，它会被正式挂在区块链的末尾。交易状态从“待处理”变为“成功”，此时资产已经完成转移，数据被永久记录在分布式账本中（即“落盘”）。
    
    -   一个区块其中可以包括多个交易，以太坊并不直接限制交易的“数量”，而是限制一个区块能消耗的 **Gas 总量**（即计算资源的总和）
        
    -   如果是复杂的智能合约交互（如 Uniswap 交易或铸造 NFT）：每笔交易可能消耗几十万 Gas，那么一个区块可能只能容纳 **几十到一百多笔** 交易。
        

-   **可查：** 区块链浏览器（如 Etherscan）抓取到最新的区块数据并进行索引。用户可以在浏览器上通过交易哈希（TxID）查到这笔交易的详细状态、手续费和时间戳。

入门导读部分学习：

1.  **区块**
    

在区块链中，所有数据最终都表现为十六进制字符串（由 **0-9** 和 **a-f** 组成的数字字母组合），可以把一个区块想象成一封**挂号信**

-   **区块摘要：** 相当于**信封**。它包含了这封信的元数据（谁寄的、信的编号、上一封信的指纹）。它的长度通常是**固定**的**前 X 个字节：** 协议规定前 80 个字节（以比特币为例）一定是区块头。程序读取到这里，就知道这是**区块摘要**、时间戳和难度系数。
    
-   **交易信息（区块体）：** 相当于**信件内容**。里面记录了密密麻麻的转账明细（A给B转了5块）。它的长度是**可变**的，取决于这单生意有多少笔交易。
    

2\. **交易过程：**

1.  交易将通过一个随机生成的钱包地址完成，与这个钱包地址相关的交易等在网络上全部是公开透明的。匿名是因为别人不知道你跟这个钱包有关联。
    
2.  组成部分：
    
    -   去中心化的网络
        
    -   代币激励（维持网络运行会有代币奖励，使用这个网络进行交易、转账、铸造NFT都需要支付代币）
        
3.  Web 3 vs Web 3.0
    

-   Web 3.0：语义网（The Semantic Web），**核心理念是结构化数据**：信息按照标准格式组织，让机器能够像人一样**理解数据的含义**。
    
-   Web3：去中心化网（The Decentralized Web），构建一个不依赖中介的互联网。例如，在 Web3 中，你的账号不需要向平台申请，而是存在你自己的数字钱包里，谁也无法封号或窃取。
    

以太坊概览学习

1.  **介绍**
    
    -   原生加密货币以太币（Ether，简称 ETH）
        
    -   核心创新在于 智能合约（Smart Contracts），是“区块链 2.0”的代表，有点类似于无人值守的自动售货机，不需要中介，只要预设条件达成（比如日期到了、或者收到货了），合约就会自动转账。
        
        -   案例：把手中的 ETH 存入 Aave 的智能合约作为抵押，合约会自动计算你能借多少稳定币，并立即放款。如果不还钱，合约会自动扣除抵押品。全程不需要人工审核，不看你的征信报告。
            
2.  **Proof of Work—Proof of Stake: The Merge (油车换成电动车）**
    

-   PoW(工作量证明):比拼谁家“**锄头多、体力好**”的体力劳动。攻击者需要控制全球 51% 的算力（极难）。
    
-   **PoS**(权益证明):比拼谁家“**股份多、责任大**”的股东大会。攻击者需要买下 67% 的流通币（会让币价暴涨，自毁身价）。
    
    -   一旦以太坊（或其他区块链）完成从 PoW 到 PoS 的升级，**“挖矿”这个行为就在这个网络上永久消失了。**
        

3.  **数据分片-Sharding（扩建车道）**
    

-   以太坊主网（Layer 1）比作一条**极其拥堵、收费昂贵的高速公路**，那么 L2 就是在旁边修建的**高架桥或快速路**
    
-   通过 **Dencun 升级** 引入了 **“数据外挂包”（Blobs）**。这好比在公交车（区块链）后面挂了一个临时储物箱，专门放 L2（如 Arbitrum, Optimism, Base）的交易数据，使得 L2 的手续费瞬间降低了 10 倍。
    
-   **2025-2026 年的目标是将这个“储物箱”升级为“物流中心”：**
    
    -   **增加 Blob 数量：** 2025 年的升级（如 **Fusaka 升级**）计划将每个区块能携带的 Blob 数量从目前的 3-6 个提升到更多。
        
    -   **PeerDAS（点对点数据可用性抽样）：** 这是 2025 年的核心黑科技。它让网络节点不需要下载整条路的视频，只需要随机“抽查”几个片段，就能确认整条路的数据是真实的。这极大地减轻了节点的负担。
        

4.  以太坊核心机制
    

-   **EOA = 个人钱包**：你用私钥“签名”来证明身份，可以主动发起任何交易
    
-   **CA = 智能保险箱**：按照预设程序自动运行，只有满足条件才执行操作（如：收到正确密码后自动转账）
    
-   **Gas = 计算资源费用**：
    
    -   Gas Limit：你愿意支付的最大计算量
        
    -   Gas Price：每单位计算的价格
        
    -   如果设置 1000 单位 ×10Gwei，实际用了 800 单位，退还 200 单位的费用
        
-   **EVM = 全球共享计算机**：所有节点同时运行相同的程序，确保结果一致且可信
    

**DeFi Intro**

-   智能合约取代了交易中的金融机构
    
-   **Flash loans:** It works on the basis that the loan is taken out and paid back within the same transaction. If it can't be paid back, the transaction reverts as if nothing ever happened.
    

**Development's guide to build on Ethereum**

1.  **what is blockchain:** a network, all nodes participate in a work following the same set of rules.
    
2.  **how it works: escrow & trust**
    

**Escrow（第三方托管/托管）** 是一种财务安排，指的是由**中立的第三方**暂时持有资金或资产，直到交易双方满足了合同中约定的特定条件。

你可以把它想象成一个“公平的裁判”或“保险箱”：在买家没有收到货、或者卖家没有收到钱之前，这笔钱由一个双方都信任的第三方管着，谁也跑不了，谁也不能乱动 --- allow people who don't know each other can make transactions or other activities. **The escrow is how the small contract works!!**

-   **This trust layer allow people to coordinate even though they don't trust each other.**
    
-   **The code is the law, you only need to trust the code.**

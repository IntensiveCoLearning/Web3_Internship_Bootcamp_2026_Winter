## 核心概念：Web3 社交的“金三角”

在开始之前，你需要理解 Web3 社交的三个层次：

1.  **钱包（身份基石）：** 你的唯一数字地址（如 `0x...`），是所有行为的终点。
    
2.  **域名/DID（可读身份）：** 将复杂的地址转化为人类可读的名字（如 `yourname.eth` 或 `yourname.lens`）。
    
3.  **社交协议（应用平台）：** 像 Farcaster 或 Lens，它们是承载内容的“操场”。
    

* * *

## 🛠 建立步骤详解

### 第一步：准备“数字指纹” —— 建立加密钱包

这是进入 Web3 的第一步。钱包是你登录所有 Web3 应用的“钥匙”。

-   **推荐工具：** MetaMask, OKX Wallet, Phantom (Solana 系)。
    
-   **关键笔记：** \* **助记词**是最高权限，永远不要告诉任何人，不要截图保存在手机里。
    
    -   建议准备一个小额资金的“社交专用钱包”，与存放重资产的钱包隔离。
        

### 第二步：申请“链上名片” —— 注册 DID 域名

直接用钱包地址交流很冷冰冰，你需要一个名字。

-   **主流选择：**
    
    -   **ENS (.eth)：** 以太坊生态的黄金标准。
        
    -   **SPACE ID (.bnb/.arb)：** 多链域名服务。
        
-   **操作：** 访问官网（如 `ens.domains`），搜索心仪名字，支付少量 Gas 费（手续费）即可铸造。
    

### 第三步：入驻“社交协议” —— 选择你的阵地

目前 Web3 社交已经形成了两大主流阵地，建议根据兴趣选择：

| 平台 | 特点 | 登录方式 |
| Farcaster | 目前最活跃，极客、开发者多。主应用是 Warpcast。 | 手机下载 App，需支付约 $5/年的存储费。 |
| Lens Protocol | 基于 Polygon 链，生态应用丰富（如 Orb, Phaver）。 | 钱包连接官网，符合条件的地址可免费领取 Profile。 |
| Bluesky | 推特创始人发起，体验极接近 Twitter，入门门槛最低。 | 邮箱注册或通过 AT Protocol 迁移。 |

一、 TG (Telegram) 运营与管理

  

• **频道与群组管理**：

  

  • 熟悉私有/公开频道、超级群组的区别。

  

  • 掌握管理员权限细分（匿名、删除消息、封禁用户、邀请链接管理等）。

  

• **Bot 功能应用**：

  

  • **基础管理**：使用 Rose 或 MissRose 等 Bot 进行入群验证、关键词过滤、自动回复。

  

  • **高级功能**：集成支付 Bot、投票 Bot 或自定义 API 机器人。

  

• **数据统计**：

  

  • 利用 Telegram 自带的 **Channel Statistics**（频道统计）查看关注者增长、消息阅读量。

  

  • 群组端使用第三方统计 Bot 监测发言排行和活跃时段。

  

二、 Discord 社区建设

  

• **频道结构设计**：

  

  • 设置公告区、规则区（Rules）、身份组领取区（Reaction Roles）、讨论区及语音频道。

  

• **权限与身份组 (Roles)**：

  

  • 熟练设置 **Permission Hierarchy**（权限继承与覆盖）。

  

  • 配置私密频道，针对不同等级身份组开放权限。

  

• **Bot 集成**：

  

  • **MEE6 / Dyno**：自动化欢迎语、等级系统、关键词屏蔽。

  

  • [**Collab.Land**](http://Collab.Land)：针对 Web3 社区的资产持有验证。

  

  • **Ticket Tool**：建立工单系统，处理用户申诉或咨询。

  

三、 数据面板与运营指标

  

• **核心指标**：

  

  • **活跃用户数 (DAU/MAU)**：每日/每月在群内发言或互动的唯一用户。

  

  • **参与率 (Engagement Rate)**：互动人数占总人数的比例，衡量内容吸引力。

  

  • **留存率 (Retention)**：新用户入群后在特定周期（如 7 天）后的留存情况。

  

• **数据工具**：

  

  • **Discord Server Insights**：查看详细的成员留存率、活跃度热力图和消息转化。

  

  • **自定义看板**：通过 API 抓取数据，形成可视化运营报告。

## 💡 核心原则：空格的使用

空格是中文排版的“留白”艺术，能显著提升阅读舒适度。

-   **中英文之间：** 必须增加空格。
    
    -   ✅ 在 LeanCloud 上
        
    -   ❌ 在LeanCloud上
        
-   **中文与数字之间：** 必须增加空格。
    
    -   ✅ 花了 5000 元
        
-   **数字与单位之间：** 建议增加空格（如 `20 TB`）。
    
    -   **例外：** 度数（`90°`）与百分比（`15%`）不需要加空格。
        
-   **全角标点前后：** **不加**空格。
    
-   **特殊情况：** 专有名词（如“豆瓣FM”）应遵循官方定义的格式。
    

## ✍️ 标点符号规范

标点符号的使用决定了句子的节奏与美感。

-   **使用全角中文标点：** 中文句子中一律使用全角符号（如 `，` `！` `？` `（）`）。
    
-   **不重复使用：** 避免 `！！` 或 `？？` 的堆叠，保持界面美观。
    
-   **英文专有名词/整句：**
    
    -   若在中文中引用英文整句，该句子内部使用**半角标点**（如 `「Stay hungry, stay foolish.」`）。
        
    -   英文书名、报刊名应使用**斜体**，而非中文书名号。
        

## 📋 文字与名词处理

-   **数字：** 一律使用**半角**字符（`123`），除非在设计海报为了对齐才偶尔使用全角。
    
-   **专有名词大小写：** 严格遵循官方写法（如 `GitHub`、`iPhone`），不可随意改为全大写或全小写。
    
    -   _提示：若网页视觉需要全大写，应通过 CSS (_`text-transform`_) 处理，而非修改原始文字。_
        
-   **拒绝不地道的缩写：** 避免使用 `h5` (应为 HTML5) 或 `Ts` (应为 TypeScript) 等非标准简称。
    

## ⚖️ 具争议但推荐的用法

这部分属于“进阶审美”，可根据个人或团队风格决定：

-   **超链接前后加空格：** 访问我们网站的最新动态，请 `点击这里` 进行订阅！
    
-   **使用直角引号：** 即使是简体中文，也推荐使用 `「」` 与 `『』`，视觉上比 `“”` 更整齐。

## 1\. 核心查询：如何搜寻信息？

在 Etherscan 首页顶部的搜索框中，你可以输入以下三类信息：

-   **钱包地址 (Wallet Address):** 查看该地址的资产余额、交易历史、持有的 NFT。
    
-   **交易哈希 (TxID/Hash):** 类似于快递单号，输入后可查看某笔转账的具体细节。
    
-   **合约地址 (Contract Address):** 查看某个代币（如 USDT）或 DeFi 协议的源码和交互记录。
    

## 2\. 详解交易详情页 (Transaction Details)

当你点开一笔交易时，以下几个字段最为关键：

| 字段 | 含义 |
| Status | Success 表示成功；Failed 表示失败（通常是因为 Gas 不足或合约逻辑错误）。 |
| Block | 该交易被打包进的区块高度。确认数越多，交易越不可篡改。 |
| From / To | 发送方和接收方。如果 To 旁边有个小图标，说明它是智能合约。 |
| Value | 转账的 ETH 数量。如果是代币转账（如 USDC），此项可能为 0，需查看下方的 Tokens Transferred。 |
| Transaction Fee | 实际消耗的手续费 = $Gas Used \times Gas Price$。 |

## 3\. 如何追踪资金流向？

追踪资产通常有两类路径：

### A. 查看钱包资产明细

进入钱包地址页面，你可以看到：

-   **Transactions:** 普通的 ETH 转账记录。
    
-   **ERC-20 Token Txns:** 查看稳定币（USDT/USDC）或其它代币的进出记录。
    
-   **NFT Transfers:** 追踪 ERC-721 或 ERC-1155 标准的 NFT 转移。
    

### B. 使用“内部交易” (Internal Transactions)

这是初学者最容易忽略的地方。有些资金转移是由**合约触发**的（例如你在 Uniswap 交易），它们不会出现在普通的 `Transactions` 列表中，而是在 `Internal Transactions` 选项卡里。

## 4\. 进阶功能：Gas 追踪与合约交互

-   **Gas Tracker:** 在网络拥堵时，可以查看当前的平均 Gas 价格（以 Gwei 为单位），避免因手续费设置过低导致交易长时间处于 `Pending`（挂起）状态。
    
-   **Read/Write Contract:** \* **Read:** 无需连接钱包，直接查询合约数据（如代币总量）。
    
    -   **Write:** 连接 MetaMask 后，可以直接在 Etherscan 上调用合约函数（例如在项目官网挂掉时紧急撤出资金）。
        

## 5\. 警惕安全风险

-   **代币钓鱼:** 任何人都可以创建虚假代币并转入你的钱包。如果在 `Token Transferred` 看到不认识的代币，不要轻易去其官网进行“解冻”或“授权”操作。
    
-   **标签系统:** Etherscan 会给已知的黑客地址、交易所地址打上标签（如 "Binance 1"），这能帮你快速识别资金去向。

-   **TVL (Total Value Locked, 总锁仓价值):** 衡量一个协议或公链规模的核心指标。它代表用户存入智能合约的资产总价值。可以类比为传统银行的**存款总额**或基金的 **AUM（资产管理规模）**。
    
-   **Mcap / TVL (市值/锁仓比):** 用于评估协议估值是否过高。
    
    -   比值 **< 1**：通常被认为协议价值被低估（锁仓资产比公司市值还多）。
        
    -   比值 **\> 1**：可能存在估值过高，或者该协议具备极强的“品牌溢价”或手续费捕获能力。
        
-   **FDV (Fully Diluted Valuation, 全稀释估值):** 假设代币全部释放后的总市值。
    
-   **Double Counting (重复计算):** 这是一个陷阱。例如，你在 Aave 存入 ETH 得到 aETH，再把 aETH 存入另一个协议，TVL 就会被算两次。DeFiLlama 默认会提供**排除重复计算**后的数据，以保证真实性。
    
-   **Stablecoin Dominance (稳定币占有率):** 某个生态中稳定币的比例，反映了资金的避险情绪或流动性活跃度。
    

* * *

## 如何通过 DeFiLlama 的数据发现机会？

### 1\. 寻找“资金流入”趋势

在 **Chains** 页面，不要只看 TVL 总量，要看 **1d / 7d / 1m Change**（增长率）。

-   **逻辑：** 资金总是先于币价波动。如果某条链的 TVL 在短期内剧增，通常意味着该生态有新的矿（高收益机会）或大额空投预期。
    

### 2\. 分析协议的盈利能力

点击左侧的 **Fees / Revenue**（费用/收入）。

-   **Fees:** 用户付给协议的总费用。
    
-   **Revenue:** 协议本身赚到的钱（扣除给流动性提供者的奖励后）。
    
-   **分析点：** 一个 TVL 很高但 Revenue 很低的协议，往往是靠补贴支撑的“虚胖”，不可持续。
    

### 3\. 利用 Yields 工具找“矿”

在 **Yields** 页面，你可以根据 APY（年化收益率）和 TVL 规模筛选收益池。

-   **重点关注：** 稳定币收益率。如果某个协议的稳定币 APY 远超市场平均水平（如 > 15%），请务必检查其风险评分。

DeFiLlama 是运营分析的“晴雨表”，重点在于理解资金流向。

### 1\. 核心指标与专有名词

-   **TVL (Total Value Locked):** 总锁仓量。衡量项目（或公链）吸引资金的能力。

-   **Yields (收益率):** 监控各个池子的 APY，分析竞品如何通过高收益吸引流动性。
    
-   **Raises (融资情况):** 追踪哪些项目拿到了钱，判断当前资本热点赛道（如 AI、DePIN、L2 等）。
    
-   **Categories (赛道分类):** 学习 Dexes, Lending, Liquid Staking (LST) 等名词，观察哪个赛道的份额在增长。

## Etherscan：微观行为追踪

区块浏览器是真相的终点，用于验证事实和监控竞品动态。

-   **Internal Transactions:** 这里的“内部交易”通常涉及智能合约自动触发的转账，是分析复杂 DeFi 操作的关键。

1\. 核心概念：什么是 ERC-20？在 ERC-20 标准出现之前，每个人发行的代币代码都不一样，这导致钱包和交易所很难适配。ERC-20 的出现规定了一套“通用语言”，只要你按照这个标准写代码，所有的以太坊钱包（如 MetaMask）和去中心化交易所（如 Uniswap）都能自动识别并支持你的代币。同质化：1 枚 USDT 永远等于另 1 枚 USDT。可分割性：通常可以精确到小数点后 18 位（类似 1 元可以分成 100 分）。2. 接口解析：它能做什么？一个标准的 ERC-20 智能合约必须实现以下 6 个核心函数。我们可以通过这张图直观地看它们是如何互动的：查询类函数（只读）totalSupply(): 返回代币的总发行量。balanceOf(address): 查看某个钱包地址里有多少币。操作类函数（写入）transfer(to, amount): 最常用。你直接把币转给别人。approve(spender, amount): 授权。告诉合约：我允许某人（或某个协议）动用我钱包里的 $X$ 个币。transferFrom(from, to, amount): 被动转账。由被授权的人（spender）调用，把币从你的钱包转走。allowance(owner, spender): 查看你还剩下多少额度授权给了对方。3. 授权机制：新手最容易踩的坑在 Web3 中，你会频繁遇到 "Approve"（授权） 操作。例如你想在 Uniswap 上卖出代币，你需要先执行 approve，允许 Uniswap 的合约操作你的币。💡 贴士： 很多钓鱼网站会骗你点击 approve 授权“无限额度”，一旦你点下去，对方就能不需要你的确认直接转走你所有的币。这是 Web3 安全中最需要防范的一环。

## 第一部分：法律红线——哪些事千万不能碰？

在境内参与 Web3 项目，要记住一个原则：**“技术创新可以，碰钱（金融）极险”。**

1.  **关于代币（Token/NFT）：**
    
    -   **不能公开筹资：** 无论叫“空投”、“白名单”还是“积分”，只要本质是向公众收钱发币，就涉嫌“非法集资”。
        
    -   **不能当钱花：** 虚拟货币在我国不具法律地位。如果跟人做生意用币结算，出了事法院可能不受理。
        
2.  **关于项目类型（刑事雷区）：**
    
    -   **链游（GameFi）：** 只要有“充钱→抽奖/概率产出→提现”的闭环，极易被定性为“开设赌场罪”。
        
    -   **推广返利：** 只要有“拉人头、多层级返佣”，就是\*\*“传销罪”\*\*。
        
    -   **跨境换汇：** 用虚拟币把人民币换成美金，属于“非法经营罪”（地下钱庄）。
        

## 第二部分：职场避坑——入职 Web3 的自我保护

Web3 公司多在海外，作为员工要防止自己变成“背锅侠”或“受害者”。

1.  **“写代码也要负责”：** 如果公司项目是搞诈骗或洗钱，程序员不能以“我只是打工写代码的”为由免责，可能会被定为“共同犯罪”。入职前一定要看白皮书。
    
2.  **薪资发放的猫腻：**
    
    -   **发 U（USDT）还是发人民币？** 法律规定工资必须发人民币。如果全发 U，你的社保公积金会断缴，且由于币价波动，你的收入没保障。
        
    -   **出金（换成人民币）风险：** 收到 U 后去卖掉，如果对方的钱是“脏钱”，你的**银行卡会被冻结**（俗称“冻卡”），甚至涉嫌“帮信罪”。
        
    -   **建议：** 尽量要求一部分法币收入，或者通过合规的 EOR（名义雇主）签合同。
        

## 第三部分：安全防护——如何不让钱包归零？

攻击者最喜欢的对象就是经验不足的“新人”。

### 1\. 致命的“面试/实习”陷阱

-   **场景：** HR 发给你一个“专用面试软件.exe”或“[测试代码包.zip](http://测试代码包.zip)”。
    
-   **后果：** 运行的一瞬间，你的浏览器 Cookie（登录状态）和钱包私钥就被偷走了。
    
-   **对策：** 只用 Zoom、腾讯会议、Google Meet。**绝不安装**任何对方传给你的不明软件。
    

### 2\. 钱包操作的“生死线”

-   **剪贴板木马：** 你复制了转账地址 A，但在粘贴的一瞬间，木马把它改成了黑客地址 B。
    
    -   **保命动作：** 确认地址时，必须核对**前 6 位和后 4 位**。
        
-   **签名陷阱：** 网页弹出“连接钱包并签名”。
    
    -   **保命动作：** 没看清权限前不要乱签名。这可能是一个“授权合约”，一旦点下去，黑客可以不经过你同意直接划走你所有的币。
        

### 3\. 基础工具安全

-   **2FA（双重验证）：** 别只靠手机短信验证码（会被拦截）。**必须用 Google Authenticator（谷歌验证器）** 或硬件密钥。
    
-   **助记词管理：** 永远**不要截图、不要存网盘、不要发微信**。老老实实抄在纸上，藏在物理世界。

Web3 协议的财务分析应关注其“价值捕获”能力，而非仅仅看代币价格：

-   **TVL (Total Value Locked)：** 协议的资产管理规模 (AUM)。
    
-   **Protocol Revenue：** 协议的毛利润。
    
-   **Incentives (Token Emission)：** 获客成本。需观察协议是否过度依赖增发代币来维持增长。
    
-   **Real Yield (真实收益)：** Protocol Revenue - Incentives。这是衡量项目是否具备可持续盈利能力的金标准。
    

**反思**

-   **去中心化账本的优势：** Web3 实现了“实时审计”。协议的每一分收入和支出都透明可查，这极大降低了传统审计中的信息不对称成本。
    
-   **估值模型的局限性：** 传统的 P/E 模型在 Web3 中需结合“代币效用（Token Utility）”进行修正。

**跨界实验：一位会计师眼中的 LSD 双雄（Lido & Rocket Pool）**

今天，我们用数据穿透流动性质押（LSD）赛道的两个巨头：**Lido (LDO)** 与 **Rocket Pool (RPL)**。

### 一、 财务透视：规模与效率的绝对博弈

在 Web3，**TVL（锁仓量）** 是我们的资产负债表总量，而 **Revenue（收入）** 则是我们的损益表核心。

1\. Lido (LDO) —— 稳健的现金流巨兽

-   **TVL：$277.4 亿**（市场占有率约 28%-30%）。
    
-   **年化总营收 (Annualized Fees)：$7.55 亿**。
    
-   **协议净利润 (Earnings)：约 $4,952 万**。
    
-   **会计洞察：** Lido 展现了惊人的**经营杠杆**。它只需花费约 **$2,600 万** 的年化激励成本（占收入比仅约 3%），就能维持其庞大的帝国。这种极高的获客效率，是其作为行业龙头的核心护城河。
    

2\. Rocket Pool (RPL) —— 牺牲短期利润的叙事者

-   **TVL：$18.1 亿**（约为 Lido 的 1/15）。
    
-   **年化总营收 (Annualized Fees)：$2,987 万**。
    
-   **协议净利润 (Earnings)：-$306 万 (亏损)**。
    
-   **会计洞察：** 从损益表看，RPL 目前处于**战略性亏损期**。它的毛利几乎为 0，因为其商业模式是将 100% 的佣金分给了底层的 **4,035 名** 节点运营商。它不仅不抽成，每年还要倒贴约 **$306 万** 的 RPL 代币作为激励。
    

### 二、 估值对账：谁被高估了？

作为会计师，我们不仅看赚多少，还要看市场给了多少倍数。

| 估值指标 | Lido (LDO) | Rocket Pool (RPL) |
| FDV (完全稀释市值) | $6.2 亿 | $4,300 万 |
| P/S Ratio (市值/营收) | 0.82x | 1.44x |
| 国库储备 (Treasury) | $1.93 亿 (含 $1.06 亿 Majors) | - (主要依靠代币增发) |

**分析结论：**

-   **Lido** 像是一家市盈率极低的传统蓝筹股，甚至已经开始讨论通过 **Buyback（回购）** 机制分配每年约 **$4,500 万** 的 DAO 收入。
    
-   **Rocket Pool** 虽然目前亏损，但其估值溢价来源于\*\*“Saturn 升级”\*\*。该升级计划将节点门槛从 8 ETH 进一步降至 1.5 ETH 左右，预计能带来 1.5-2 倍的 TVL 增长。
    

### 三、 风险审计（会计师的碎碎念）

1.  **合规性风险：** Lido 的分红讨论（LDO Staking）可能触及证券化红线，这是目前估值受压制的财务主因。
    
2.  **坏账风险 (Slashing)：** 关注节点罚没情况。Lido 的节点主要由 30 余家专业机构组成，而 RPL 则拥有 **4,000+** 散户节点。从审计视角看，RPL 的去中心化程度更高，系统性风险更分散。
    

### 结语

会计的本质是识别价值。Lido 卖的是“效率”**，它已经是一个高效的盈利机器；Rocket Pool 卖的是**“安全与情怀”，它正在用短期的亏损换取长期的以太坊生态地位。

作为一名新晋 Web3 分析师，我更倾向于关注**拐点**：当 RPL 开启抽成之日，或许就是其估值重构之时。

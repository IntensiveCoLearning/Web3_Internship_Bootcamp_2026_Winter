    -   社会工程：伪装成员工或招聘人员
        
    -   洗钱工业化：洗钱链条高度自动化，约45天完成从链上混币到非法平台法币结算

-   私钥窃取：针对性狩猎，基于病毒木马、社工和供应链渗透

    -   获取私钥并授权给恶意合约 → 诱导用户接触这些私钥 → 转入token，资产流失 → 洗钱

-   AI 智能合约漏洞挖掘：自动化攻防的军备竞赛

## 钱包、私钥和个人主权

在学习的过程中，这里的每一个概念都很有意思，但越深入越发现这里面包含的内容是无穷尽的，即使我很喜欢被好奇心驱使着前进，但我的注意力是有限的，所以我打算先搭建框架，后续补充细节，搞清楚里面的逻辑先

过去，我一直认为私钥是先于助记词出现的，但后来发现，在现在的主流实践中，大部分情况是，助记词是先于私钥出现

以下是用 Gemini 生成的流程图（从随机熵到钱包地址）

![钱包图片.jpg](https://raw.githubusercontent.com/IntensiveCoLearning/Web3_Internship_Bootcamp_2026_Winter/main/assets/marcosoccer211/images/2026-01-14-1768387163636-____.jpg)

### 第一阶段

**目的**：生成助记词

**过程**：钱包软件先从系统获取128~256位的真正随机数（熵），通过BIP-39标准算法将其转换成12~24个英文单词的助记词序列。这个助记词就是钱包的“备份”。

**举例**：

```
（随机数）0111000011111000…10011101
```

```
（助记词）witch collapse practice feed shame open despair creek road again ice least
```

### 第二阶段

**目的**：将助记词转化为一个固定长度二进制“根种子”（Root Seed）

**过程**：助记词（加密码短语）作为输入，通过PBKDF2算法进行2048次迭代的HMAC-SHA512计算，最终输出一个512位的二进制种子。

**种子的作用**：种子是所有私钥和地址的数学根源，种子让“只需备份助记词就能恢复一切”成为现实

**举例**：

```
种子（十六进制显示）cdcf85bea269b5b04ea4cf1fc66fbbd1bfe99e500ef0316dfd5aaeb7379b5e1fbdb002dc9e3de87ecbd4583041700b626793492712fa9fabb1c17c62a5e297ad
```

### 第三阶段

**目的**：从512位根种子构建一个分层确定性钱包（HD Wallet），让一个钱包能无限派生出无数个子私钥和地址。

**过程**：将512位根种子作为唯一输入，通过 BIP-32 标准的一次性计算生成主扩展私钥（Master Extended Private Key，记作 m），它包含主私钥和主链码。然后，使用预定义的路径（path）机制，一层一层地派生子扩展密钥（Child Extended Key），直到路径末端得到最终叶子私钥（Final Leaf Private Key）。

**关键定义**：

-   **主私钥**：核心私钥，能派生所有子私钥。
    
-   **主链码**：用于后续派生的“伪随机源”，确保派生看起来随机但其实确定性。
    
-   **HD 路径派生**：从主扩展私钥开始，通过一个预定义的“路径字符串”（path）逐层派生子密钥的过程。这个路径像文件系统的目录结构（例如 m/44’/0’/0’/0/0），允许钱包软件生成无限个子私钥、公钥和地址，而一切都基于单一根种子
    
-   **子扩展密钥 & 最终叶子私钥**：中间层是子扩展密钥（包含子私钥 + 子链码），可继续派生。路径末端（通常 index 层）是叶子节点，只输出普通 256位私钥（Final Leaf Private Key），不再派生子级，用于 Stage 4 生成单个地址。
    

**举例**：

```
主扩展私钥 ：xprv9s21ZrQH143K3QTDL4LXw2F7HEK3wJUD2nW2nRk4stbPy6cq3jPPqjiChkVvvNKmPGJxWUtg6LnF5kejMRNNU3TGtRBeJgk33yuGBxrMPHi
```

```
主私钥 ：e8f32e723decf4051aefac8e2c93c9c5b214313817cdb01a1494b917c8436b35
```

```
主链码：873dff81c02f525623fd1fe5167eac3a55a049de3d314bb42ee227ffed37d508
```

```
其他私钥链码类似
```

### 第四阶段

**目的**：将最终叶子私钥转换成用户实际能使用的的钱包地址，这个地址就是别人给你转账时复制粘贴的那个“收款码”。

**过程**：从叶子私钥开始，经过一系列不可逆的密码学计算和编码，最终生成一个固定长度的地址字符串。

**详细过程**：

```
Final Leaf Private Key → 使用椭圆曲线数字签名算法（ECDSA），在 secp256k1 曲线上进行点乘运算 → Public Key
```

```
Public Key → 先对公钥做 SHA256 哈希，再对结果做 RIPEMD160 哈希 → Public Key Hash
```

```
Public Key Hash → 在 PubKey Hash 前加版本字节（version byte），后加校验和（checksum），然后编码成 最终地址
```

**举例**：

```
叶子私钥：2b4be7f19ee27bbf30c667b642d5f4aa69fd169872f8fc3059c08ebae2eb19e7
```

```
公钥：03501e454bf00751f24b1b489aa925215d66af2234e3891c3b21a52bedb3cd711c
```

```
公钥哈希：bef5a2f9a56a94aab12459f72ad9cf8cf19c7bbe
```

```
Base58Check 最终地址：1JQheacLPdM5ySCkrZkV66G2ApAXe1mqLj
```

## 学习区块链基础概念

实习手册关于区块链基础概念手册的部分，写得太棒了，图文结合，以尽可能最小的信息量让我们理解区块链

为 [吃汤圆](https://x.com/web3_cty)，[Echo](https://x.com/Echo_liuchan)，[Bruce](https://x.com/brucexu_eth) 老师打call

但我仍旧想以我自己的方式来讲述区块链和比特币的概念，有点像费曼学习法吧

### 为什么会出现比特币？

比特币的出现绝不是凭空产生的，一定是人类为了解决什么问题，被赋予了一些使命而诞生的（中二过头了）

解决什么问题呢？如何在没有可信第三方的前提下，进行价值转移

关键是真正可绝对信任的第三方并不存在，让我们回到2008年，亦是金融危机，也是比特币闪亮登场的时候，一个叫中本聪的人公布了比特币白皮书《比特币：一种点对点的电子现金系统》，像是在抗议政府超发货币救市，将这场灾难均匀地分摊到每个普通人的购买力上，但更像是对传统金融体系的宣战

比特币在传统金融秩序近乎崩溃、人们对“中心化信用”产生严重怀疑的背景下诞生。那么如何去信任，如何去中心化地记账，便是比特币的使命

### 为了实现“去信任转移价值”，需要什么呢？

中心化依赖信任，怎么去信任呢，那就直接去中心化

我们需要一种全世界所有人共同维护、任何人都无法单独篡改的公开账本。传统数据库达咩，因为：

-   中心化数据库 → 管理员可以改
    
-   普通分布式数据库 → 还是有控制节点，或容易被51%攻击
    

所以必须发明一种全新的数据库来记账。这个数据库或者数据结构或者账本就叫区块链

### 什么是区块链呢？

从字面意义上理解，就是把一个个区块串成一条链，便是区块链

区块上有什么？有一笔笔交易。怎么判断交易真伪？交易由私钥签名。区块怎么串起来？每一个区块都有上一个区块的哈希值，每一个区块都有上一个区块的痕迹

```text
创世块 ← 块1（含创世哈希） ← 块2（含块1哈希） ← … ← 最新块
```

这样的设计有什么用呢？让这个“账本”具有了不可篡改的性质。

```
修改任何一个历史交易 → 它所在块的哈希变了 → 后面所有块的哈希都要重算 → 全网立刻发现不一致
```

### 谁来记账？怎么防止作恶？

解决了可篡改的问题，那如果坏人记账且不可篡改不就蛋糕了，放心，作为科幻小说都写不出来的发明才不是那么不堪一击

那么谁有权记账，或者说添加下一个区块？如果谁都可以加，那么攻击者可以无限发假区块洗交易，这个记账系统将直接瘫痪。

比特币给出了他的解决方案 → 工作量证明（Proof of Work）

为什么是 POW ？这个世界上不是所有的人都是善良的人，让不怀好意的人去记账，当然会出现记假账的可能。但记假账是要付出代价的，POW 就是为了奖励诚实记账人，惩罚假账哥而出现的，当诚实记账的奖励大于记假账的成本，人们自然会乖乖记账，这个证明的高明点在于它是一种人性的博弈，对抗贪婪

什么是POW ？就是一个决定谁记账的机制，谁最快完成一定的工作量，谁就有权记账，但最快完成这些工作，需要付出大量的电力，算力

当大多数人都是诚实记账，诚实节点占多数算力，诚实人几乎总是最先出块。当然也会出现，记假账的人出块更快的情况，一时快不代表一直快，一时快 = 幸运，一直快 = 大于51%算力，记账的过程是一场马拉松，跑赢前50米，没人会承认你是冠军

```
攻击者想追上主链并篡改历史 → 需要超过51%的全网算力 → 成本天文数字（目前比特币网络算力相当于全球前几百台超级计算机的总和）
```

这就形成了最长链共识：谁的链最长（工作量最大），谁代表真相。

这样，就解决了谁来记账的问题，当然是诚实人来记账啦！

### 比特币是什么？

理解了区块链为什么是比传统记账更好的记账方式，接下来就是比特币的出现时间啦

每个人记账都有自己记账的习惯和方式，如果给我们的记账方式加入以下规则：

-   总量固定2100万枚
    
-   每10分钟左右产生一批新币（区块奖励）
    
-   奖励每四年减半
    
-   记账单位叫“比特币”（BTC）
    
-   最小单位：1 satoshi = 0.00000001 BTC
    

于是，比特币，作为这种特殊记账方式的记账单位诞生了

```
比特币 = 用工作量证明保护的、总量固定、去中心化发行的、点对点可转移的数字稀缺资产 + 全球不可篡改的公开账本系统
```

### 区块链 ≠ 比特币

当然，大家不要搞混了，区块链是一类记账方式，比特币是采用区块链这类记账方式但做了些改进的新的记账方式的记账单位

-   比特币：第一个、也是最成功的“用区块链记账的数字货币应用”
    
-   区块链：一种技术架构（分布式不可篡改账本 + 共识机制）
    

### 简单总结一下

比特币：用数学 + 电力 + 经济激励，第一次在人类历史上实现了不需要信任任何人、就能全球点对点转移稀缺价值的系统

区块链：为了保护这个系统而发明的一种只能append、不能篡改、由最多诚实算力共同维护的分布式账本数据结构

有时觉得区块链蛮浪漫的，不可篡改，只要有电就会一直运行下去，上面记录的一切便实现了“赛博永生”，当人类灭绝时，其他行星的文明重新翻看区块链这本日记，用新的能源续写这本日记，文明便重新开始延续

所有人都会忘记你，但未来不会，宇宙也不会

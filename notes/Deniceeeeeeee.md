# 2026-01-26

这次硬男老师主要是从他过往参加黑客松的经验，聊了什么是黑客松、为什么要参加，以及怎么在有限时间里把事情做好。他提到像 ETH Global 这种黑客松，本质不是创业，更不是做一个完整产品，而是给你一个高强度的环境，逼你在很短时间内把想法变成一个能跑、能展示的东西。很多时候，光是完成这一点，本身就已经很有价值了。

他也讲到黑客松里团队配置的重要性。比较理想的组合是两名开发、一个产品、一个负责 Pitch 的人，Pitch 这件事真的不能忽视。评委通常不会一开始就深挖代码，而是先听你在做什么、解决什么问题，如果讲不清楚，再好的技术也很容易被略过。

在项目方向上，他给了很多实际可操作的思路，比如把其他链上已经跑通的模式搬到新链上，或者干脆围绕赞助商的 SDK 拼项目，这类方向对黑客松来说非常友好。另外，他也很鼓励大家做和现实生活有关的事情，比如解决当地的实际问题，或者关注宠物、女性安全、植物这些相对小众但真实的需求。甚至硬件项目，虽然在区块链黑客松里不多见，但一旦能 Demo 出来，存在感会非常强。

他还反复提醒了一些“血泪教训”，比如千万不要踩着 deadline 才提交，先交一个不完整版本都比什么都没交强；要分清 12pm 和 12am 这种看起来很低级、但每年都会有人搞错的时间点；提交资料不等于提交成功，一定要确认点了 submit。最重要的一点是，别把黑客松当创业，不用追求完美，36 个小时里想把所有细节都打磨好，最后大概率只会把自己搞崩。

工具方面，他推荐的思路也很实在，核心就一句话：能省时间就行。比如用 AI 做配音、用剪映快速剪个 Demo 视频，用 Cursor 辅助写代码，PPT 可以用 AI 起稿，但逻辑一定要自己想清楚，工具只是帮你快一点，而不是替你思考。

在 Vibe Coding 这一块，他讲得比较直白：这不是不用学代码，而是换一种方式跟代码打交道。真正重要的不是你会不会背语法，而是你能不能把需求讲清楚，把上下文给对。你目标不明确，AI 只会把事情越写越乱。会用 AI 编程的人，往往对系统结构和逻辑是有基本感觉的。

他也提醒不要被“几天做出产品”这种说法骗了，如果完全没有基础，很容易卡在 Bug 上走不出来。Vibe Coding 更像是一种放大器，把你原本就有的能力放大，而不是无中生有。

听下来最大的感受是，黑客松其实是在练一种综合能力：快速判断、取舍、协作和表达。而 Vibe Coding 则是让你在这种高压场景下，能更快把想法变成现实。最后拼的从来不是谁代码写得最多，而是谁更清楚自己在干嘛，以及能不能把一个还不完美的东西，讲成一个值得被听的故事。

# 2026-01-22
<!-- DAILY_CHECKIN_2026-01-22_START -->




今天我参加了两场看似不同、但内核高度相关的学习课堂：一场从技术层面拆解 Web3 与 DApp 的底层逻辑，另一场则以「南塘 DAO」为案例，讨论区块链如何进入真实的乡村与社区协作场景。它们分别回答了两个问题：Web3 是什么？以及，Web3 为什么有必要存在？如果用一句话总结今天的收获，那就是Web3 的目标不是更酷的技术，而是更公平、更可持续的协作方式。

在 DApp 研讨会中，最重要的一个观点非常清晰：Web3 真正的革命并不只是“去中心化”，而是所有权结构的改变。在 Web2 世界里，用户更像是平台的租客; 账号属于平台，数据归公司所有，规则可以被随时修改，一旦平台倒闭，资产与关系也会随之消失。而 Web3 试图重构这种关系，让用户成为真正的“房东”：钱包即身份，无需注册和许可；资产与数据写在链上，不再依附于单一平台；智能合约自动执行规则，减少人为裁量与不透明空间。

这种所有权逻辑的背后，依赖的是三大基础组件：智能合约、区块链与代币。智能合约是写进代码里的规则，自动执行、公开透明；区块链是不可篡改的分布式账本，用来记录资产与行为；而代币不仅是“钱”，更是激励、治理与协作的工具。在此基础上，Web3 才得以发展出 DeFi、NFT、GameFi 以及 DAO 等应用形态，让无需身份认证的金融、真正私有化的数字资产，以及去中心化的集体行动成为可能。

但研讨会也反复提醒：Web3 并非乌托邦。高昂的 Gas 费、私钥一旦丢失无法找回、智能合约漏洞与黑客风险，以及整体仍然不友好的用户体验，都说明它仍处在非常早期的阶段。它更像是 1995 年的互联网；原始、混乱、门槛高，但方向已经出现。

如果说 DApp 研讨会让我理解了 Web3 的“能力上限”，那么南塘 DAO 的分享，则让我看见了 Web3 的“现实必要性”。南塘所面对的并不是技术问题，而是一个非常现实的协作困境：在乡村与社区工作中，情感劳动难以被量化，付出往往模糊、不可见；决策与分配规则不透明，协作成本极高；很多冲突并非来自恶意，而是来自“没有一套明确的规则”。他们提出了一个很关键的判断；乡村真正缺的不是项目创意，而是一套可以长期运转的协作机制。

在这个背景下，南塘并没有把区块链当成噱头，而是当成工具。他们尝试用区块链记录每一次参与、关系与行动，把模糊的付出转化为可追溯的贡献记录，用透明规则降低误解，而不是靠信任硬撑。无论是生态农业、社群活动，还是联合开发的网页钱包与任务平台，背后都指向同一个目标：让协作过程被看见、被记住、被尊重。

其中让我印象最深的一点，是南塘将“故事”引入链上记录。每一次使用「南塘豆」兑换，都需要备注发生了什么，这些故事会被绑定并记录在链上。交易不再只是冷冰冰的数字，而是“关系发生过什么”的痕迹。这让我意识到，区块链记录的，不只是资产，而是关系史。

南塘的发展过程本身也非常现实：早期经历过无序探索，有成员因争执离开，后来才逐渐建立规则。而他们并非凭空开始，而是有十多年前罗伯特议事规则的实践基础。这也再次说明，DAO 不是一套代码，而是一种需要时间、经验与反复修正的治理能力。

对我而言，这两场课堂的最大启发并不在技术细节，而在于一个更大的判断：\*\*Web3 的价值，不在链上，而在它是否能改善链下的协作方式。\*\*技术本身不会自动带来公平，去中心化也不等于没有冲突，真正困难的从来不是写合约，而是设计规则、面对人性。

也正因为如此，像南塘这样把区块链用于情感劳动、协作关系与长期治理的尝试，才显得格外珍贵。如果说 Web2 解决的是“规模化连接”，那么 Web3 正在尝试解决的，是如何在没有中心权威的情况下，让陌生人能够长期、稳定、相对公平地协作。

这条路一定很慢，也一定会犯错。但或许，正如今天课堂中反复提到的那样：我们正站在 1995 年的互联网现场。而真正重要的，是现在就开始参与、思考与构建。

# 2026-01-21

今天这节课让我真正意识到，Uniswap 并不是一个“被动报价”的交易平台，而是一个依靠数学规则和市场参与者共同维持价格平衡的系统。价格并不是由某个中心化角色决定的，而是在套利者、流动性深度和手续费机制的共同作用下不断被“校准”。套利在这里不再是一个灰色行为，而是维持去中心化市场效率的关键力量。

通过恒定乘积公式 X × Y = K，我开始理解“价格”本质上只是池子中两种资产比例的体现，而不是一个人为设定的数值。这也让我明白了为什么流动性深度如此重要：K 值越大，市场越稳定，交易者承受的价格冲击越小；反之，在浅池中交易，看似简单的一笔 swap，其实会付出隐形的成本。

同时，我对「价格影响」和「滑点」的区分有了更清晰的认知。价格影响是机制内、可预期的结果，而滑点更多来自链上环境的不确定性，比如 MEV 和抢跑。这让我意识到，在链上交易中，用户不仅是在和一个合约交互，也是在和整个区块链环境博弈。滑点设置看似是一个小参数，背后却涉及安全性、成功率和交易成本之间的权衡。

总体来说，这堂课让我从“会用 Uniswap”走向“理解 Uniswap”，也让我对 DeFi 的核心精神有了更直观的感受；用代码和经济激励，而不是信任，来构建一个自我运转的金融系统。

今天连续上了两节课，主题围绕“Web3 公共物品资金”以及 Solidity 智能合约入门。这次课程让我对 Web3 生态中基础设施的价值与其背后的资金问题有了更系统的理解。Marcus 老师首先介绍了公共物品的概念，即具有非排他性与非竞争性的资源。在 Web3 世界中，许多支撑整个生态运作的要素，如开源代码、Solidity 语言本身、开发框架以及 DeFi 协议，都属于公共物品。虽然这些项目被广泛使用，但由于缺乏直接盈利模式，往往面临“公地悲剧”和搭便车问题，导致开发者和维护者难以获得合理回报。针对这一困境，课程进一步介绍了多种公共物品资助机制，包括决策效率高但权力集中的委员会制、鼓励广泛参与但存在女巫攻击风险的二次方捐助，以及由 Optimism 推动、根据项目实际影响力进行奖励的回溯性公共物品资助。通过 Gitcoin、Optimism 和 Octant 等案例，我逐渐理解到 Web3 并非没有公共财政，而是在探索一种更透明、去中心化且可验证的资助方式。课程中还将会有互动游戏“良心杀”也让我期待对 DAO 治理会有更直观的感受，想象着在模拟 50 人共同分配 10 万美元资金的过程中，即使规则清晰，决策依然会受到信息不对称与个人动机的影响，这让我意识到治理机制设计的重要性。

随后进行的 Solidity 教学从 EVM 的底层原理讲起，逐步过渡到语法基础，并以 ERC-20 代币合约作为实战案例，将理论与应用有效结合。我特别认同课程中对 Gas 优化、数据结构设计以及安全编码习惯的强调，例如理解 storage 读写成本、mapping 授权结构以及使用较新版本 Solidity 的必要性。整体而言，这一天的课程不仅让我对 Solidity 开发有了更清晰的入门认知，也让我意识到 Web3 不只是技术实现，更涉及激励机制、治理设计与公共价值的长期建设。接下来，我希望能进一步深入理解 ERC-20 的核心函数实现，并学习常见的智能合约安全漏洞，在实践中持续加深对 Web3 生态的理解。

# 2026-01-18
<!-- DAILY_CHECKIN_2026-01-18_START -->








在传统的 NFT 或代币世界里，资产是直接绑定到一个以太坊地址上的。这意味着任何人都可以在链上看到你拥有多少 NFT、做过哪些交易，以及哪些地址与你频繁互动。虽然地址本身不显示真实姓名，但通过交易历史分析，很容易推测出身份或行为习惯。即使生成多个地址，钱包之间的资产转移和交互仍然可以被分析；一旦某个地址与现实身份产生关联，链上资产就可能被持续追踪。因此，单靠“多地址”并不能真正解决隐私问题。

这正是 ERC-7962 想要解决的核心问题。它不再用地址表示资产所有权，而是引入 KeyHash（公钥的哈希）作为新的所有权标识。链上只存储 KeyHash，而不是地址本身。只要你持有对应的私钥，就可以通过签名证明自己是该 KeyHash 的控制者。链上只验证“你是否有权操作”，而不关心你是谁，也不要求你自己支付 Gas，从而同时提升了隐私性和用户体验。

ERC-7962 的关键不只是隐藏地址，而是彻底改变了资产与身份的绑定方式。通过一次性 KeyHash 和轮换机制，每次资产操作后，剩余资产都会迁移到新的 KeyHash，历史交易之间无法被可靠关联。这种设计类似比特币的 UTXO 模型，使链上行为不再天然等同于“个人轨迹”，而更像一次次独立、不可关联的授权行为。

从技术上看，所有权证明完全依赖签名。当你想执行操作（例如转移 NFT）时，会在本地生成一条包含 tokenId、接收方 KeyHash、nonce 和 deadline 的消息，并用私钥进行签名。签名和公钥可以由你自己提交，也可以交给 Relayer 提交。合约只做三件事：验证公钥哈希是否匹配 KeyHash、验证签名是否合法，以及检查 nonce 和 deadline。验证通过，操作即被执行。

为了更直观地理解这一模型，可以用 Starbucks 会员卡来类比。传统模式下，会员身份往往与账号或地址绑定，商家知道“你是谁”，隐私暴露风险较高。而在 ERC-7962 模型中，链上只记录 KeyHash = 会员编号。你是否是会员，只由你能否用私钥生成有效签名来证明。系统只确认“你有资格”，而不知道你的真实身份。

Gas 费用问题也是 ERC-7962 改善用户体验的重要一环。用户只需要签名，不需要持有 ETH。任何人或服务都可以作为 Relayer 代付 Gas 并提交交易。由于 Relayer 不掌握私钥，它无法伪造身份或篡改操作内容，资产安全完全由密码学保证。这让链上交互第一次真正接近 Web2：用户掌控权力，却不必承担复杂性。

从更宏观的角度看，ERC-7962 的意义在于，它第一次把资产控制权从地址中抽离出来。资产不再属于某个公开地址，而是属于一个可以被密码学证明控制的 KeyHash。你只需要证明“你有权”，而不是暴露“你是谁”。

如果说过去的 Web3 是“透明优先”，那么 ERC-7962 所代表的，是 Web3 走向成熟必须补上的另一块拼图：隐私、抽象，以及真实可用的用户体验。ERC-7962 让 Web3 第一次做到：你只证明“你有权”，而不必暴露“你是谁”，资产安全来自签名，而不是地址。

由 Spoon OS 的 Rick 老师主讲，主题为 **“AI 及其基础概念：机器人导致新经济体系”**。

### 一、 核心概念：从 LLM 到 AI Agent 的进化

Rick 老师指出，2023 年的大语言模型（LLM）本质上是 **“自回归的 token 预测器”**，虽然博学但有三大局限：**无状态（没记忆）、没工具（不能干活）、没身份（不能管钱）**。

-   **AI Agent 的三个核心组件**：
    
    1.  **大脑（LLM）**：负责推理、计划和理解意图，生成行动指令。
        
    2.  **记忆（Vector Database / RAG）**：通过向量数据库实现“长期记忆”，让 AI 记住你是谁。
        
        -   **例子**：像是一头**大象**，它能通过搜索历史信息，在新的对话窗口中依然记得你三小时前聊过的人生规划。
            
    3.  **手脚（API / MCP / Function Call）**：让 AI 能够调用外部系统、发邮件或执行代码。
        
        -   **例子**：以前的 AI 像个\*\*“被绑住手脚的天才”\*\*，只能动嘴；现在的 Agent 有了手脚，可以真正去订机票或执行交易。
            

### 二、 为什么 AI 需要 Web3？（痛点与解决方案）

AI Agent 目前处于“黑户”状态，传统互联网基础设施无法支持其运行。

1.  **身份验证（Identity）**：
    
    -   **痛点**：传统 HTTP 协议无法验证请求是人还是 AI 产生的。
        
    -   **例子**：就像路上一个没穿制服、没工牌的\*\*“外卖小哥”\*\*，你没法确认他的真实身份，容易出现“商业间谍”伪装成合法 Agent 窃取数据。
        
    -   **方案（ERC-8004 标准）**：为每个 Agent 铸造一个\*\*“链上护照” (NFT)\*\*，具有唯一性且可验证。
        
2.  **经济与支付（Money）**：
    
    -   **痛点**：AI 没身份证、没脸，无法通过 KYC（身份验证）办银行卡；传统信用卡无法处理金额极小（如 $0.001）、频率极高（每秒万次）的结算。
        
    -   **例子**：AI 调用一次 API 可能只要 0.001 美元，但信用卡的固定手续费都要 0.3 美元，**“手续费比交易额还贵”**。
        
    -   **方案（X402 协议）**：将支付能力嵌入 HTTP 协议，实现**原生微支付**，无需登录和绑定信用卡。
        
3.  **审计与合规（Audit）**：
    
    -   **痛点**：传统服务器日志可被篡改或删除。
        
    -   **方案**：利用区块链**不可篡改**的特性，记录 AI 的每一个重要决策和交易哈希。
        
    -   **例子**：当监管机构询问 AI 为何在某个时刻做决策时，企业无法通过删改日志来逃避责任，因为记录在链上是\*\*“全程透明、改不了”\*\*的。
        

### 三、 Spoon OS：AI Agent 的“安卓系统”

为了降低开发难度，Spoon OS 将复杂的底层逻辑（私钥管理、链上签名、支付协议）封装成简单的 API。

-   **例子**：开发一个 Web3 AI App 不再需要自己从头写“操作系统”，只需调用 Spoon OS 的 SDK，**两天内就能让 AI 拥有管钱和链上身份的能力**。

### 四、 QA 环节要点总结

**1\. 为什么有必要为 AI 专门创建一个账户？**

-   **解答**：如果是自己私下玩量化，可以共用账户；但如果你要做一个“量化大师”基金让别人投钱，就需要\*\*“链上可验证”\*\*的身份和逻辑，以证明你不会跑路，解决信任问题。
    

**2\. Web3 的效率（TPS）真的比 Web2 高吗？**

-   **解答**：传统 Web2 支付在跨境转账时，因为涉及多方中介和复杂的合规审查，其实非常慢且贵。Web3 的优势在于**跨境结算效率**和减少人工审查带来的延迟。
    

**3\. 用户使用 Web3 AI Agent 必须懂 Web3 吗？**

-   **解答**：不一定。开发者可以将 Web3 能力封装。Web3 主要是为了实现**透明、可审计和高效支付**。用户端可以根据场景选择支付方式，甚至 Web2 用户也能享受到这些信息服务。
    

**4\. 如果 AI 决策出错，责任归谁？**

-   **解答**：责任属于**开发者或部署者**。通过链上归属权记录，可以清晰追踪到这个“邪恶”或“犯错”的 Agent 是谁部署的。
    

**5\. AI 真的能帮我在 DeFi 里赚钱吗？**

-   **解答**：AI 只是工具。如果你是交易小白，AI 也会学到 90% 亏钱人的逻辑。**AI 的上限取决于人设定的策略**。
    
-   **建议**：先拿 100U 让 AI 跑跑看，如果它真能按你的逻辑赚到钱，再给它更多资金。
    

**6\. 名誉系统（Reputation）如何防范刷分（女巫攻击）？**

-   **解答**：通过\*\*“经济成本”\*\*限制。只有真实在链上产生过交易（调过该 Agent）的地址才有资格评价。抹黑一个 Agent 的代价是必须先花钱调用它，提高了作恶成本。目前该系统主要针对单个 Agent 的 ID，如果开发者做恶导致一个 Agent 名声臭了，目前确实可以换个 ID 重来，未来会持续优化。

### 第一部分：Web3 合规性深度解析（主讲人：邓律）

邓律师（漫坤律师事务所深圳合伙人）结合其办理大型交易所（如 CMC 前十交易所）永续合约案件的实务经验，剖析了中国大陆语境下的法律边界。

1\. 核心法律逻辑：法律的滞后性

-   **要点**：法律具有天然的滞后性，而 Web3 是激进创新的行业。律师的任务是利用相对滞后的法律，为新业务形态划定行为边界。
    
-   **例子**：针对 **DePIN（去中心化硬件网络）** 项目，法律上并没有专门的“DePIN法”，但办案单位会套用现有的法律规定来限制其业务。
    

2\. 中国大陆的三大“负面清单”

-   **ICO（公开融资）**：打击重点是“借发币名义面向公众募资”。
    
    -   **例子**：如果发一个 **Meme 币**只是为了好玩而不涉及公开筹款，并不违法。但如果像某些传统项目（如“认养一头牛”模式）那样，在没有牌照的情况下引入传销分润机制，风险极高。
        
-   **虚拟货币交易所与永续合约**：在大陆语境下，永续合约常被认定为\*\*“开设赌场”\*\*。
    
    -   **例子**：办案单位逻辑直接，会问从业者是否知道“94公告”或“924公告”。他们认为合约开多/开空就是“压大小”的赌博游戏，技术人员为这些软件写代码会被视为提供犯罪支持。
        
-   **虚拟货币挖矿**：虽然此前有禁令，但目前出现了一些“变通”模式。
    
    -   **例子**：以 **“AI 算力中心”** 或“超算中心”的名义与政府签协议，享受地方政策福利，实际上将算力以“外贸”形式卖给海外客户，至于海外客户是用它跑 AI 还是挖矿，则属于其个人行为。
        

3\. 个人与企业的实务风险

-   **“镀金攻击” (Dujin Attack)**：这是一种新型洗钱/攻击手段。由于海外资金盘洗钱成本增高，黑客会故意向特定人员（如正在晋升的体制内人员）的账户打入“黑钱”。
    
    -   **例子**：黑客只需付 5 万黑钱，就能往你账户打 10 万。这会导致你的账户不仅被冻结，甚至可能直接被立案，目的是在关键时刻精准打击受害者。
        
-   **U卡（加密货币银行卡）风险**：
    
    -   **例子 1**：发卡方可能在后台将你列为某虚假公司的“员工”以规避监管。一旦该公司被查，卡片立即冻结。
        
    -   **例子 2**：有些卡由私人组建，资金存在其掌控的多签钱包中。如果合伙人闹矛盾或跑路，用户的资产便无法找回。
        

4\. QA 环节与深度见解

-   **合规的两个层面**：一是法律文本上的\*\*“形式合规”**；二是**“中国特色深度合规”\*\*，即与地方办案人员建立良性协作关系，这在大陆展业中至关重要。
    
-   **Meme 币风险**：再次强调发币不等于犯罪，核心看是否涉及非法吸收公众存款（面向上百人不特定对象募资即为“底线”）。

### 第二部分：Web3 安全趋势与实战（主讲人：Adam）

Adam（GoPlus 安全专家）发布了 2025 年度安全分析报告，指出 2025 年 Web3 损失总额超过 35 亿美元。

1\. 两大攻击趋势：精细化与产业化

-   **精准猎杀 (Precision Hunting)**：针对交易所或高 TVL（锁仓量）协议的长期渗透。
    
    -   **例子**：**Bybit 遭窃 15 亿美元**。黑客渗透了多签服务商 Safe 的开发人员或前端，使得管理人员在操作时，看到的页面一切正常，实则签发了恶意交易。
        
-   **广洒网 (Mass Phishing)**：针对普通用户的自动化黑产。
    
    -   **例子**：**“地址投毒” (Address Poisoning)**。黑客向你转账 0.01U，该地址首尾号与你常用地址一模一样。如果你下次从历史记录里复制地址，就会转错，曾有用户因此损失 5000 万美元。
        

2\. AI 时代的新型威胁

-   **深度伪造 (Deepfake)**：利用 AI 模拟语音和视频。
    
    -   **例子 (UXLink 案例)**：黑客在 Telegram 冒充合作伙伴约开视频会议。管理人员在视频里看到了真实的“合作脸”，从而放松警惕点开了恶意链接（如伪造的 Zoom 安装包），导致私钥被窃。
        
-   **AI Coding 风险**：AI 辅助写代码可能引入恶意插件或后门。
    

3\. 针对新技术的“降维打击”：EIP-7702 骗局

-   **要点**：EIP-7702 是以太坊的一项重大升级，赋予了普通地址（EOA）智能合约的能力。但黑客利用信息差设计了新骗局。
    
-   **例子**：黑客公开一个私钥已泄露且存有资产的地址。你以为捡到了钱，但当你充入 Gas 费准备转账时，该地址预置的恶意合约（Delegator）会瞬间自动将你的 Gas 费转走，你永远跑不赢合约。
    

4\. 安全建议：“四不”大法

-   **不点**：不点击任何链接，尤其是会议链接、求职链接。
    
-   **不签**：拒绝\*\*“盲签”\*\*。在钱包弹出签名时，必须核对内容（例如：明明是领空投，内容却显示是 USDT 转账，则绝对不能签）。
    
-   **不装**：不随意安装插件或软件，应通过官方渠道独立下载。
    
-   **不转**：转账前必须核对完整地址。
    

5\. QA 环节要点

-   **如何识别真假代币？**：不能只看名字，要看合约地址。建议使用 GoPlus 的 Token Security 平台或 DexTools 查看风险提示（如是否为“貔貅盘”）。
    
-   **Web2 与 Web3 漏洞挖掘的区别？**：
    
    -   **类型不同**：Web3 侧重重入攻击、预言机操纵；Web2 侧重 XSS、SQL 注入。
        
    -   **共性**：目前都是“人工 + 静态/动态工具 + AI”的博弈过程。
        

### 总结

两场视频共同传递了一个核心信息：Web3 不仅是技术的革新，更是权力的重新分配。在享受个人主权的同时，必须具备极高的**法律风险防控意识**（敬畏规则、深度合规）和**资产安全自律意识**（独立验证、拒绝盲从）。

第二集 web3底层运行原理

### 1\. 钱包与个人主权：Web3 的起点

视频强调 Web3 的核心是无需许可的个人主权。

-   **私钥、助记词与地址**：
    
    -   **概念**：私钥是控制资产的“密码”或“印章”；助记词（12或24个单词）是派生无限私钥的“种子”；地址则是公开的“收款号”。
        
    -   **对比例子**：在**传统银行**开户需要身份证、去柜台、经审核且可能被拒；但在 **Web3**，任何人只需随机生成一个私钥，就立刻拥有了一个全球通用的账号，无需任何人授权。
        
-   **安全警示**：私钥一旦丢失或泄露，资产将无法找回。
    
    -   **例子**：有学员刚在黑客松拿了奖金，因授权环节没处理好（被钓鱼），奖金立刻被盗走，这是一次惨痛的教训。
        

### 2\. 交易、签名与 Gas 费：运行的经济动力

-   **数字签名**：钱包通过私钥对交易（如转账、调用合约）进行签名并广播。
    
    -   **原理例子**：签名机制能实现在**不暴露私钥**的前提下，让全网验证这笔信息确实是由你发出的，且内容不可篡改。
        
-   **Gas 费（手续费）**：这是为了防止垃圾交易淹没网络并激励节点。
    
    -   **加油站例子**：Bruce 老师将 **Gas Price** 比作加油站当时的“油价”（如1.2元），将 **Gas Usage** 比作路程消耗的“油量”。
        
    -   **路程例子**：从**上海去苏州**的路费（油耗）是相对固定的，但如果要去更远的**北京**（执行更复杂的智能合约逻辑），消耗的油（Gas）就会更多。如果大家都抢着上路，油价（Gas Price）就会上涨。
        

### 3\. 区块链网络：出块、共识与最终性

-   **出块流程与“公交车”类比**：
    
    -   **类比例子**：Bruce 老师展示了一个动画，将**区块**比作每12秒发一班的**公交车**，交易是排队的小人。
        
    -   **排队例子**：如果你只愿付10元路费，而别人愿付100元，那么付100元的人会**优先上车**；付得太少的人可能会被挤下车，等下一班。
        
-   **不可篡改性与最终确认**：
    
    -   **原理**：区块通过 Hash 值首尾相连，修改一个区块需要修改其后所有区块。
        
    -   **最终性例子**：由于网络延迟或分叉可能，刚成功的交易并不绝对安全。就像**电商发货**，商家不应在交易刚打包时就发货，而应等待约12-13分钟（以太坊达到 Finalized 状态），确保钱真正到账且不可逆转。
        
-   **共识机制**：对比了 POW（解难题、高耗能）和 POS（交押金质押、防作恶）。
    

### 4\. 智能合约：“代码即法律” (Code is Law)

智能合约是存储在区块链上、不可篡改的可执行代码。

-   **NFT Mint（铸造）例子**：在合约逻辑中可以设定“每个地址只能 Mint 一个”。当请求指向合约时，系统会自动检查你的地址是否铸造过，若有则直接拒绝，无需人工介入。
    
-   **去中介化的交换例子**：在**传统世界**买房需要律师、中介和法院来保证交易；但在 **Web3**，可以编写一个智能合约，规定“一旦收到 B 的钱，就自动把 A 的 NFT 转给 B”，从而消除了对第三方的信任依赖，极大降低了信任成本。
    

### 5\. 协议升级与网络治理

-   **EIP（改进提案）**：以太坊的升级通过社区反复讨论形成标准。
    
    -   **例子**：这并非某家公司说了算，而是像坎昆升级这样，需要经过长达一年的讨论、评审和社区共识才能实施。
        
-   **客户端多样性**：
    
    -   **例子**：网络由多种语言编写的不同软件（客户端）支持。这种多样性确保了即使某个软件（如 Geth）出 bug，其他客户端仍能运行，确保网络十多年来从未宕机。
        

### 6\. Web3 的核心价值与实战技巧

-   **核心特性**：去中心化、无需许可、抗审查、开放开源。
    
-   **安全技巧——“抢跑”（Front-running）例子**：如果你的私钥泄露了，为了保住资产，你可以和黑客比赛。如果你愿意支付 1000 元手续费，而黑客只肯付 100 元，节点会优先打包你的转账，从而在黑客动手前把钱转移到安全地址。
    

**总结**：Web3 不仅仅是技术，更是一种**权力的重新分配**，它将信任从人、机构转向了可验证的代码和数学共识。

**QA**

### 1\. 关于安全与私钥保护

-   **私钥被盗后的“抢跑”（Front-running）救金**：
    
    -   **问题**：如果私钥泄露了，真的能跑赢黑客吗？
        
    -   **解答**：原理是利用 **Gas 费激励机制**。如果你的私钥泄露，你需要立刻发出一笔转账，并支付极高的手续费（Gas）。
        
    -   **例子**：就像在路上拦车，黑客愿意付 100 元路费，而你愿意付 1000 元，**验证者为了赚更多钱会优先打包你的交易**。目前已有安全服务能监测异常转账，在黑客动手的一瞬间自动“抢跑”将资金转入安全地址。
        
-   **私钥生成的可信度**：
    
    -   **问题**：生成私钥的程序本身安全吗？
        
    -   **解答**：确实存在风险。过去曾有钱包因为**随机数算法有缺陷**，导致私钥被他人推算出来。
        
    -   **例子**：Bruce 老师建议不要使用随机的网页工具生成私钥。顶级玩家甚至会通过物理世界的随机性（如**手算、观察熔岩灯**等）来确保生成的私钥绝对随机且不可预测。
        
-   **MetaMask（小狐狸）的存储机制**：
    
    -   **问题**：如果不记助记词，只靠小狐狸的登录密码行吗？
        
    -   **解答**：不行。小狐狸将私钥加密后存储在电脑的\*\*本地缓存（Local Storage）\*\*中，登录密码只是解锁本地存储的钥匙。
        
    -   **例子**：如果你重装了系统或删除了浏览器插件，而你又没记下助记词，你的资产将**永远找不回来**，因为钱包服务器不存你的任何数据。
        

### 2\. 关于网络运行与技术细节

-   **区块大小与交易金额的关系**：
    
    -   **问题**：区块动画里的小方块大小代表交易金额吗？
        
    -   **解答**：不代表。小方块的“权重”主要与 **Gas 费（手续费）**有关，而非交易本身的金额。
        
-   **地址碰撞的概率**：
    
    -   **问题**：如果两个人的地址后 20 位一模一样会怎样？
        
    -   **解答**：这叫“哈希碰撞”。在数学和密码学的保护下，这种概率极低，**几乎可以认为直到宇宙毁灭都不会发生**。如果真的发生了，说明整个区块链的加密基石已经崩溃了。
        
-   **质押中心化风险**：
    
    -   **问题**：大家都把 ETH 放在交易所质押，权力不就被交易所垄断了吗？
        
    -   **解答**：确实存在风险。Bruce 老师建议有条件的同学**自己运行节点（Solo Staking）**，或者选择像 Lido 这样相对分散的协议，而不是交给中心化交易所，因为交易所作为中间商还会抽取一部分利润。
        

### 3\. 关于 Web3 的价值与未来

-   **Web3 是否会颠覆 Web2？**
    
    -   **解答**：Bruce 老师认为 Web3 不会彻底干翻 Web2，而是作为一种**关键的基础设施补充**，解决 Web2 解决不了的问题。
        
    -   **例子**：Web2 无法解决**无端的封号和审查**（如社群突然被封），而 Web3 提供的抗审查社交工具能让用户真正拥有自己的账号。
        
-   **“Web3 美团”是伪命题吗？**
    
    -   **解答**：很多生搬硬套的项目（如 Web3 版美团）是在“**拿着锤子找钉子**”。Web3 真正的价值在于**跨境支付和信用缺失场景**。
        
    -   **例子**：在非洲等汇率崩盘的国家，人们无法开立银行账户，但通过 Web3 钱包，他们可以轻松使用 **USDC（稳定币）**进行交易和保存资产。
        
-   **Degen（投机者）与 Regen（建设者）**：
    
    -   **解答**：行业内分为两类人：**Degen** 侧重投机、炒作和发币；**Regen** 则关注可持续性、去中心化治理和解决现实问题。Bruce 老师强调其社区更倾向于后者，思考如何保护个人主权。
        

### 4\. 资产追回的现实困境

-   **私钥泄露后钱能找回来吗？**
    
    -   **解答**：严格来讲，**几乎无法追回**。由于区块链的抗审查特性，如果钱还在黑客的钱包里，任何人都无法冻结它。
        
    -   **例子**：即便报案，警察往往也难以立案，因为很难证明私钥是谁泄露的。只有涉及巨额资产时，安全公司（如慢雾）才可能通过追踪资金流向，在黑客将钱转入交易所的一瞬间联系平台进行拦截。

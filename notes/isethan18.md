《021学习以太坊》开源教材

**第1章：认识以太坊 (What is Ethereum)**

起源与愿景：以太坊由Vitalik Buterin于2013年提出，2015年正式上线 。它不仅是类似比特币的账本，更是“全球可编程区块链” 。其核心是以太坊虚拟机 (EVM)，使区块链具备了运行智能合约的能力 。

**核心特性**

1)智能合约：存储在链上的自动执行代码，像“自动贩卖机”一样无需中介 。

2)dApps (去中心化应用)：不依赖单一服务器，具有零停机、抗审查、数据完整等特点 。

3)以太币 (ETH)：系统的“燃料” (Gas)，用于支付计算费用，也是PoS机制下的安全保证金 。

共识演进：2022年通过“合并” (The Merge) 从PoW（工作量证明）转向PoS（权益证明），能耗降低约99.95% 。

**第2章：网络结构与节点类型**

节点构成：合并后的完整节点由执行客户端 (EL) 和共识客户端 (CL) 组成，两者通过 Engine API 通信 。

**节点分类**

全节点 (Full Node)：验证所有交易，保存区块数据，但会裁剪老状态 。

1)归档节点 (Archive Node)：保存自创世以来所有历史状态，是链上的“时间机器”，适合审计和深度分析 。

2)轻节点 (Light Node)：仅保存区块头，通过Merkle证明验证数据，适合手机等受限设备 。

3)通信机制：基于UDP的Kademlia算法发现邻居，通过TCP上的RLPx协议建立加密连接，利用\*\*Gossip（八卦协议）\*\*在全网传播交易和区块 。

**第3章：账户类型与结构**

1)EOA (外部账户)：由私钥控制（如MetaMask钱包），用于发起交易，不包含代码 。

2)CA (合约账户)：由代码控制，包含逻辑和状态存储，不能主动发起交易，只能被EOA或其他合约调用 。

3)账户抽象 (AA)：以太坊正在推行的一种趋势，旨在让账户行为更灵活（如Gas代付、社交恢复），提升用户体验 。合规与刑事风险防范总结

Web3 合规与刑事风险防范：从理论架构到实操避险

一、 合规的两个维度：从“形式”到“深度”

形式合规（文本安全）：

1)核心： 在白皮书、合同及协议中，严格使用符合国际监管（如FATF、MiCA）的术语。

2)规避： 严禁出现“融资”、“投资回报”、“保本”、“二级市场操盘”等敏感词汇。

3)底线： 明确风险提示与免责声明。但需注意，形式合规仅能降低表面风险，无法对抗实质性的刑事穿透。

深度合规（业务安全）：

1)物理隔离： 绝不向境内居民提供服务。实操中不仅要屏蔽大陆IP，更要通过KYC拦截大陆证件注册，确保大陆用户“不翻墙无法访问、有证件无法注册”。

2)业务去金融化： 建立真实的业务闭环，项目应具备真实的技术支撑（如DePIN的硬件、AI的算力），避免沦为单纯的“代币炒作”工具。

二、 核心刑事风险点与法律定性

非法经营罪： 针对“法币与虚拟币”的兑换业务。在境内从事OTC撮合、资金池业务或变相买卖外汇，是目前执法部门打击的重点。

1)组织、领导传销活动罪： 重点排查项目是否存在“层级返利”、“拉人头机制”或“无限代收益”。即便有真实产品，若激励机制涉传，依然面临全案收缴的风险。

2)开设赌场罪： 针对具备“杠杆对赌、输赢不与涨跌幅挂钩、抽水渔利”特征的永续合约。辩护重点在于证明其属于“金融衍生品”而非“赌博工具”（参考146号指导案例）。

3)帮信罪/掩隐罪： 针对开发者或运营者。即便项目合法，若未履行反洗钱义务导致大量犯罪资金流入，个人亦可能被推定为“明知”而入罪。

三、 远程办公与求职者的避险指南

牌照误区： 警惕项目方宣传的“全球持牌”。持有美国或离岸牌照不代表具有全球合规性，更不能作为在境内展业的护身符。

**入职与薪酬风险：**

1)合同主体： 优先选择境内有实体公司的岗位（便于劳动维权），但需确认该公司不直接参与核心发币业务。

2)发薪方式： 法律要求以人民币发薪。以虚拟币发薪存在“账户冻结风险”和“劳动法支持缺失风险”（仲裁机构常判决公司补发人民币）。

3)身份管理： 避免在境内主体中担任法人、财务或关键决策人，技术岗位应保留日常代码提交记录，以证明自己仅从事非决策性的技术工作。

四、 实操自救建议：遇风险如何处置？

1)证据隔离： 养成保留链上记录、KYC信息、合规审查报告及非恶意沟通记录的习惯。在调查中证明自己是“善意第三人”。

2)配合调查原则： 遇到“冻卡”或警方询问，初犯、偶犯且能清晰说明资金来源的，通常可通过退赔涉案款换取解冻或不予立案。

3)最高原则： “法律具有滞后性，不作恶是最大护身符”。远离面向大陆用户的法币通道、高倍杠杆合约及具有传销特征的增长模式。

1\. 钱包与私钥系统

在理解了私钥和助记词后，需要区分钱包的存储形式，这直接决定了资产的安全级别。

热钱包 vs. 冷钱包：

热钱包 (Hot Wallet)： 联网状态，如手机 App (MetaMask, OKX Wallet)。优点是方便交易，缺点是私钥存在联网设备，容易被黑客攻击。

冷钱包 (Cold Wallet)： 硬件设备，私钥永不联网。交易时在硬件内签名，再传输结果。安全性最高。

EOA 钱包 vs. AA 智能合约钱包：

EOA (Externally Owned Account)： 你目前笔记中描述的传统钱包，完全依赖私钥。一旦丢失无法找回。

AA (Account Abstraction)： 账户抽象钱包。可以实现“社交恢复”（找回账号）、“免 Gas 费”等功能，是 Web3 大规模普及的方向。

2\. 交易与签名机制（深度补充）

理解了 Gas 和签名后，初学者最容易在“权限授权”上吃亏，这是资产被盗的高发区。

Approve（授权）机制： 在 Web3 中，如果你要用 DEX（去中心化交易所）卖出代币，你需要先执行 Approve。这代表你授权该智能合约“有权动用你钱包里一定数量的代币”。

风险提示： 很多钓鱼网站会骗取你的“无限授权”，从而搬空你的钱包。

EIP-1559 费用模型： 现在的 Gas 费由 Base Fee（基础费，会被销毁） + Priority Fee（小费，给矿工） 组成。这使得手续费预测更加准确。

3\. 区块链网络运行（架构补充）

除了你提到的 Layer 1（如以太坊主网），目前的 Web3 生态是多链并行的。

Layer 2（二层网络）： 由于以太坊主网（Layer 1）又慢又贵，出现了 Layer 2（如 Arbitrum, Optimism, Base）。

原理： 在二层处理交易，将成千上万笔交易打包成一个证明提交给 Layer 1，从而极大降低 Gas 费并提高速度。

跨链桥 (Bridge)： 资产在不同链之间转移的通道。

4\. Web3 核心概念补完：与 Web2 的区别

为了完整理解 Web3，建议在笔记中加入这一部分，明确“为什么需要 Web3”。

| 特性 | Web2 (传统互联网) | Web3 (价值互联网) |
| 数据所有权 | 归大公司所有 (Google, Facebook) | 归用户自己所有 (通过私钥控制) |
| 身份系统 | 手机号、邮箱、第三方登录 | 钱包地址 (无需许可，匿名性) |
| 透明度 | 后台逻辑黑箱 | 链上代码开源、数据公开可查 |
| 中心化 | 平台可以封禁你的账号 | 抗审查，没人能阻止你使用区块链 |

5\. Web3 常见应用形态 (DApp)

Web3 不仅仅是转账，它由以下几个核心板块构成：

DeFi (去中心化金融)： 无需银行，通过智能合约进行借贷、交易（如 Uniswap, Aave）。

NFT (非同质化代币)： 证明图片、音乐或游戏道具的唯一所有权。

GameFi (链游)： 玩家在游戏中获得的资产可以真实变现 (Play-to-Earn)。

SocialFi (去中心化社交)： 数据归用户，不再担心被平台删帖或限流。

6\. 安全建议

助记词永远不要存手机相册或收藏夹： 手写在纸上或存在离线设备。

不随意见签名： 尤其是看不懂的交互请求。

小额分仓： 交互不确定的项目时，使用存有少量资金的新钱包

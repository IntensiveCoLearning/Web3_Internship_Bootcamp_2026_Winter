# 2026-01-14
<!-- DAILY_CHECKIN_2026-01-14_START -->
今天在实习训练营中学习的主题是安全与合规，这一部分的内容明显更贴近现实风险，也与我自身的专业背景形成了更直接的连接。相比前几天更多是在理解 Web3 是什么、如何运行，今天的学习更像是在回答一个更为现实的问题：在这个体系中，风险具体来自哪里，边界又在哪里。

在安全部分的分享中，老师首先介绍了加密世界中几种常见的攻击方式。虽然并未深入到代码层面，但通过对攻击路径与结果的讲解，我逐渐意识到，加密世界中的安全问题并不仅仅是“技术漏洞”，而是高度依赖于机制设计、权限分配以及用户行为的系统性风险。与传统互联网相比，Web3 的一个显著特点在于，一旦攻击成功，损失往往是即时且不可逆的，这种“无法回滚”的特性，使得安全问题不再只是运维层面的瑕疵，而是直接关系到资产归属与责任承担。

这也让我第一次从法律视角重新审视“安全”这个概念。在 Web3 语境下，安全并不只是防止黑客入侵，更是对规则边界、操作权限以及风险告知的整体设计。如果规则本身存在缺陷，那么即便行为符合代码逻辑，也可能在现实世界中引发严重的法律后果。这种技术执行与法律评价之间的错位，是 Web3 特有、且极具挑战性的风险来源。

在合规部分，邓律师的分享则为这一整套技术与风险提供了清晰的现实坐标。他从中国境内的监管立场出发，明确说明了在现行法律与政策环境下，哪些行为是被严禁从事的。这一部分并非泛泛而谈，而是帮助我们建立起一个非常重要的认知：无论技术如何演进，执业活动与商业行为都无法脱离具体法域的监管边界。

通过邓律师的讲解，我更加清楚地认识到，Web3 并不存在所谓的“灰色安全区”。很多看似技术中立的行为，在特定法域下可能已经构成明确的违法风险。对于从业者而言，忽视这一点，往往不是因为不懂法律条文，而是因为过度沉浸在技术叙事中，误判了现实规则的适用范围。

更让我受益的是，邓律师从律师同行的角度，系统地分享了法律人在 Web3 领域可以如何开展业务。这并不是鼓励盲目进入一个“风口行业”，而是强调要在合规前提下，结合自身专业能力，找到可持续的服务切入口。无论是合规咨询、风险评估，还是为项目提供结构设计层面的法律支持，其前提都不是追逐热点，而是对监管环境、业务模式和技术机制的综合理解。

这一部分的分享，也让我对“Web3 法律业务”有了更加现实的认识。它并不是简单复制传统金融或互联网法律服务的模式，而是要求律师在理解技术逻辑的基础上，提前识别可能出现的合规冲突。这对法律人而言，既是门槛，也是机会。

整体来看，今天关于安全与合规的学习，让我第一次非常清晰地感受到 Web3 的另一面。它并非只有创新与效率，也伴随着高度集中的风险与责任。理解这些风险，并不是为了远离这个领域，而是为了在进入时保持足够的清醒。

这一天的学习，也在某种程度上把前几天关于技术、运行机制和行业结构的内容重新串联了起来。技术如何运作、规则如何被写入代码、安全如何被设计、合规如何被评估，这些问题不再是彼此割裂的模块，而逐渐构成了一幅完整的现实图景。

# 2026-01-13
<!-- DAILY_CHECKIN_2026-01-13_START -->

今天在实习训练营中继续学习了 Web3 的基础课程，内容由 Bruce 老师主讲，主题是 Web3 的运行原理。相比前一天偏行业与岗位认知的分享，这一节课明显更偏向底层逻辑与技术原理，但讲解方式并不追求工程细节，而是试图帮助非技术背景的学员建立一个能够“看懂 Web3 是如何运转”的整体框架，这对我而言收获非常明显。

课程一开始，Bruce 老师从区块链的基本原理讲起，并没有直接进入复杂的算法或代码，而是围绕“为什么区块链可以被信任”这个问题展开。通过对分布式账本、去中心化记账以及不可篡改性的讲解，我逐渐意识到，区块链并不是简单意义上的“新数据库”，而是一种通过技术手段重新分配信任成本的制度设计。对法律背景而言，这种设计本身就具有强烈的制度意义，因为它在一定程度上替代了传统由中心化机构承担的信任与证明功能。

在以太坊部分，Bruce 老师重点讲解了 ETH 的共识机制以及其与比特币的区别。通过对以太坊从工作量证明向权益证明转变的介绍，我开始理解这一转型背后的现实考量：并非单纯为了技术先进性，而是围绕效率、能源成本以及网络可持续性所做出的制度选择。这让我第一次较为直观地感受到，共识机制并不只是技术问题，而是一套关于参与资格、激励方式与责任分配的规则体系，本质上具有非常明显的“准制度”属性。

随后课程进入智能合约的基本原理部分，这也是本次学习中让我产生最多联想的内容。Bruce 老师强调，智能合约并不是“智能的合同”，而是将预设规则以代码形式固化在链上的自动执行程序。它并不会判断公平与否，也不会处理例外情形，只会在触发条件满足时按照既定逻辑执行。这一点让我不断将其与传统法律合同进行对照：如果说传统合同依赖解释、裁量与救济机制，那么智能合约更接近一种高度形式化、几乎不存在模糊空间的规则执行工具。

这种差异也让我开始意识到 Web3 世界中“规则即代码”的核心特征。规则不再主要通过事后解释与司法介入来实现，而是在事前通过技术结构直接限定行为边界。这对法律与合规提出了全新的挑战：法律人的角色可能不再只是事后判断是否合法，而是需要在规则被写入代码之前，参与其逻辑设计。

整体来看，这一节关于 Web3 运行原理的课程，虽然技术含量较高，但并未让我产生距离感，反而逐渐帮助我建立起一种“能够理解但不必成为工程师”的认知状态。我开始明白，对于非技术背景的参与者而言，真正重要的并不是掌握具体实现方式，而是理解这些技术机制在制度层面上解决了什么问题，又引入了哪些新的风险。

这堂课也让我更加确信，Web3 并不是一个可以简单用“去中心化”“创新金融”来概括的领域，而是一整套融合了技术、激励与规则的新型运行体系。对于法律背景的人来说，理解其底层原理，既是进入行业的前提，也是避免停留在表面合规讨论中的关键一步。

今天的学习对我而言，是一次从“知道它存在”到“开始理解它如何运作”的过渡，也为后续更深入的学习打下了必要的认知基础。

2016.1.12学习笔记

今天参加了实习训练营的行业分享环节。由于本职工作较为繁忙，能够投入系统学习 Web3 的时间并不算多，但这次训练营的分享内容本身信息密度较高，对我理解 Web3 行业与自身职业背景之间的关系仍然产生了比较清晰的启发。

本次分享由 Emily 老师主讲，整体并未从具体技术细节切入，而是从加密资产与 Web3 行业的发展历史讲起，逐步过渡到招聘市场的演变、岗位结构的变化以及当前较为真实的就业图景。

在加密资产的历史回顾部分，Emily 老师强调了一个非常重要的转变：早期加密资产更多是技术实验与理念驱动，而随着市场规模扩大、资金体量上升以及现实世界不断介入，合规与监管已经从“外部限制”逐渐转化为行业自身必须面对的结构性问题。**这一点对我这样具有法律背景的参与者而言尤为关键。Web3 并不是逃离法律的空间，而是在既有法律框架尚未完全适配的情况下，率先运行的一套新型经济与治理实验。**

在招聘市场的历史回顾中，老师也提到 Web3 行业用人逻辑的明显变化。从早期极度依赖技术与关系网络，到后来资本驱动下的岗位扩张，再到当前周期中更加理性、强调真实能力与长期价值的阶段，行业正在经历一次明显的去泡沫化。

这也是我在整个分享过程中感受最深的一点。对于法律背景而言，Web3 并不需要转行式的自我否定，而是要求对行业运行逻辑本身有更深入的理解。**合规、法务与风控岗位已经不再只是事后审查规则的角色，而是逐步前移到业务结构设计、产品合规路径选择以及风险预判之中。**这意味着，法律专业在 Web3 语境下的价值，并不取决于条文记忆，而取决于是否真正理解链上资产、交易机制和项目运作方式。

在求职渠道与职位画像的分享中，我也更加清楚地意识到，Web3 行业并不遵循传统投简历、面试、入职的线性路径。信息主要分布在社群、项目官方渠道以及长期互动形成的信任网络之中。某种意义上，是否被视为圈内人，本身就是一种筛选机制。这也意味着，学习 Web3 并不仅仅是知识输入，而是一种持续参与生态的过程。

关于薪酬与激励体系的介绍，则让我进一步意识到 Web3 行业的风险与回报是高度绑定的。薪资结构更加灵活，稳定性相对较弱，但与此同时，对专业能力的回报空间也更大。尤其是在行业不确定性上升的阶段，合规与法律能力反而可能成为项目稳定运行的重要支点。

整体而言，虽然今天在时间投入上并不充分，但这次实习训练营的分享对我而言，更像是一种方向校准。它让我更加确认，Web3 并不是一个需要抛弃原有专业的领域，而是一个要求在原有专业基础上重新理解技术、金融与规则关系的场域。接下来，比起盲目补技术细节，我更需要做的是系统理解加密资产的业务结构与合规逻辑，为后续更深入的学习和实践打下基础。

这次记录，更像是一个阶段性的起点，而不是总结。

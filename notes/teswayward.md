## **Web3 岗位全景图 笔记**

### 一、技术类岗位（Tech Roles）

这是 Web3 里最“核心”的一部分，适合有编程背景的同学。

**1\. 前端工程师**

-   负责去中心化应用（DApp）界面开发。
    
-   用 React、Vue 等开发钱包连接、交易签名等界面功能。
    
-   需要会与智能合约交互的 Web3 技术栈。
    
-   技能侧重 JavaScript、TypeScript、框架、区块链客户端库等。
    

**2\. 后端工程师**

-   支撑 DApp 的后台服务，比如处理链上链下数据交互、API 接口、事务处理。
    
-   需要用 Node.js / Go / Python 等搭建服务，和数据库连接。
    
-   会与智能合约开发团队协作。
    

**3\. 智能合约工程师**

-   用 Solidity 等编写并部署智能合约。
    
-   关注安全性、Gas 优化和逻辑正确性。
    
-   对 DeFi、NFT、DAO 等核心协议逻辑有深刻理解。
    
-   常用工具：Foundry、Hardhat、Remix 等。
    

* * *

### 二、非技术类岗位（Non-Tech Roles）

Web3 不只有写代码的工作，还有很多面向业务与生态建设的岗位。

**1\. 产品与运营**

-   负责产品生命周期管理、用户增长、运营策略和数据分析。
    
-   需要理解用户行为、设计增长策略、推动产品落地。
    

**2\. 社区管理**

-   构建 Telegram、Discord、Twitter（X）等社区生态。
    
-   组织线上活动、提升用户活跃度、解答用户问题。
    
-   社区是 Web3 项目核心资产之一，需要精准的沟通与内容输出能力。
    

**3\. 行业研究与数据分析**

-   跟踪市场趋势、用户行为、协议发展和竞争项目。
    
-   编写市场分析报告或白皮书，支持策略与产品决策。
    
-   需要熟练使用数据工具和链上数据分析平台。
    

* * *

### 三、技能与职业发展建议（补充）

虽然不同岗位要求不同，但整体上 Web3 求职有一些共通点：

-   技术岗位更看重实际项目经验与代码能力（如 dApp 作品、智能合约部署实践）。
    
-   非技术岗位则要有扎实的沟通、内容产出与理解行业动态的能力。
    
-   简历要突出对 Web3 的理解和实在贡献，不只是写技术名词或流行术语。

## Web3 合规与安全学习笔记

### 一、为什么 Web3 需要“合规 + 安全”一起看

在 Web3 中，**安全解决的是“会不会被偷”**，  
**合规解决的是“能不能长期存在”**。  
如果只有技术安全但不合规，项目可能被监管叫停；  
如果只有合规但安全薄弱，资产和用户仍然高风险。

* * *

### 二、Web3 合规主要在管什么

-   **反洗钱（AML）与反恐融资（CTF）**  
    监管重点是资金来源和去向是否可疑。  
    虽然链上是匿名的，但并非不可追踪，因此合规逐渐向链上延伸。
    
-   **KYC 与身份问题**  
    许多中心化入口（交易所、法币通道）必须进行 KYC，  
    DeFi 虽然强调去中心化，但现实中很难完全脱离监管体系。
    
-   **监管框架来源**  
    当前全球监管主要参考 **FATF** 的旅行规则（Travel Rule），  
    要求虚拟资产服务商对转账双方进行一定程度的信息留存。
    
-   **合规的现实矛盾**  
    Web3 追求去信任与匿名性，  
    合规强调可识别、可追责，  
    二者之间长期存在张力，而不是非黑即白。
    

* * *

### 三、Web3 安全的核心问题

-   **私钥与资产安全**  
    Web3 中“账户即资产”，  
    私钥一旦泄露，资产几乎无法追回，  
    这与传统金融“可冻结、可申诉”完全不同。
    
-   **智能合约安全**  
    合约漏洞、权限错误、逻辑缺陷，  
    都可能被攻击者利用并造成系统性损失。  
    因此审计、测试和权限设计非常关键。
    
-   **DeFi 与机制安全**  
    即便代码没问题，  
    经济模型、预言机、流动性结构也可能被攻击，  
    安全不仅是技术问题，也是金融设计问题。
    

* * *

### 四、合规与安全的交汇点

-   **链上透明性**  
    区块链天然可追溯，  
    这既有利于合规调查，也有助于事后分析安全事件。
    
-   **中心化与去中心化的边界**  
    合规通常从中心化节点入手：
    
    -   交易所
        
    -   托管钱包
        
    -   法币出入口  
        而真正“完全去中心化”的协议，目前仍是监管难点。
        
-   **安全是合规的前提**  
    如果协议频繁被攻击、资产大规模被盗，  
    即使形式上合规，也难以获得长期监管容忍。
    

* * *

### 五、Web3 项目常见风险

-   合规不清晰，随时面临政策风险
    
-   安全措施不足，容易成为攻击目标
    
-   用户误以为“去中心化 = 无风险”
    
-   权责模糊，出事后难以界定责任主体

## Web3 安全学习笔记

-   **Web3 安全在保护什么**  
    Web3 安全的核心不是“账号”，而是**私钥和合约逻辑**。一旦私钥泄露或合约有漏洞，资产几乎不可逆地损失。
    
-   **私钥与钱包安全**  
    在 Web3 中，钱包即身份。  
    常见风险包括：
    
    -   私钥/助记词被钓鱼或恶意网站获取
        
    -   签署了自己没看懂的交易
        
    -   使用不安全的浏览器插件或设备  
        本质问题是：**用户是自己的银行**。
        
-   **钓鱼与社会工程攻击**  
    很多攻击不靠技术漏洞，而是靠“骗”：
    
    -   假官网、假空投、假客服
        
    -   恶意签名请求（Approval Drain）
        
    -   仿冒 NFT / 项目方社群  
        Web3 安全中，人往往是最脆弱的一环。
        
-   **智能合约安全**  
    智能合约一旦部署，代码很难修改。  
    常见风险包括：
    
    -   重入攻击（Reentrancy）
        
    -   权限控制错误
        
    -   数学溢出或逻辑缺陷  
        因此合约审计在 Web3 中非常重要。
        
-   **协议与 DeFi 风险**  
    即使合约没漏洞，也可能存在：
    
    -   经济模型被攻击（套利、闪电贷）
        
    -   预言机被操纵
        
    -   流动性不足导致极端价格波动  
        安全不只是代码问题，也是**机制设计问题**。
        
-   **跨链与 Layer 2 风险**  
    跨链桥和 Layer 2 是高风险区域：
    
    -   依赖复杂的验证机制
        
    -   一旦桥被攻破，损失通常巨大  
        历史上很多重大安全事件都来自跨链环节。
        
-   **不可逆性的双刃剑**  
    区块链的“不可篡改”提高了可信度，  
    但也意味着：
    
    -   交易错了很难追回
        
    -   被盗资产恢复成本极高  
        Web3 更强调**事前防护，而非事后追责**。
        
-   **安全的基本原则**  
    一些简单但有效的习惯：
    
    -   助记词永不联网
        
    -   大额资产使用硬件钱包
        
    -   不随意授权、不随意点空投
        
    -   合约交互前尽量了解项目背景

## NFT（Non-Fungible Token）学习笔记

-   **NFT 是什么**  
    NFT 是一种**不可替代的区块链代币**。不可替代的意思是：每一个 NFT 都是独一无二的，不能像比特币那样 1:1 互换。
    
-   **和普通代币的区别**  
    普通代币（如 ETH、USDT）是“同质化”的，彼此没有差别；  
    NFT 更像是**数字世界里的“产权证明”**，强调唯一性和所有权。
    
-   **NFT 存的到底是什么**  
    NFT 本身通常不直接存图片或视频，而是：
    
    -   链上记录：所有者、合约地址、Token ID
        
    -   链下链接：指向图片/音频/元数据（如 IPFS）  
        可以理解为：**链上是证书，内容在链下**。
        
-   **技术基础**  
    大多数 NFT 构建在 **Ethereum** 上，常见标准包括：
    
    -   ERC-721：一对一、完全唯一
        
    -   ERC-1155：支持批量和半同质化资产
        
-   **NFT 能用来做什么**  
    NFT 的价值不只是“图片”：
    
    -   数字艺术与收藏品
        
    -   游戏道具与虚拟资产
        
    -   会员凭证 / 门票
        
    -   IP 授权、品牌权益、身份标识
        
-   **为什么有人愿意为 NFT 付费**  
    核心原因不是文件本身，而是：
    
    -   可验证的所有权
        
    -   可追溯的来源（provenance）
        
    -   稀缺性 + 社群共识  
        NFT 更像是一种**社会与技术共同背书的资产**。
        
-   **争议与问题**  
    NFT 也存在明显争议：
    
    -   价格波动大，投机属性强
        
    -   价值高度依赖叙事与社群
        
    -   版权 ≠ 所有权，容易被误解

-   **以太坊是什么**  
    以太坊是一条“可编程的区块链”。如果说比特币主要是“数字货币”，那以太坊更像是一个**去中心化应用的平台**。
    
-   **核心创新：智能合约**  
    以太坊最大的价值在于**智能合约**。它本质是部署在链上的代码，只要条件满足，就会自动执行，不依赖人工或中介。
    
-   **ETH 的角色**  
    ETH 是以太坊的原生代币，用来：  
    1）支付交易和合约执行的手续费（Gas）  
    2）作为 PoS 机制下的质押资产  
    3）在 DeFi、NFT 等生态中充当基础资产
    
-   **账户体系**  
    以太坊有两种账户：
    
    -   外部账户（EOA）：由私钥控制，人用
        
    -   合约账户：由代码控制，程序用
        
-   **共识机制演进**  
    以太坊已从 PoW 转向 **PoS（权益证明）**，核心目标是：
    
    -   降低能耗
        
    -   提高网络安全性
        
    -   为后续扩容打基础
        
-   **生态系统**  
    以太坊承载了大量应用：
    
    -   DeFi（去中心化金融）
        
    -   NFT
        
    -   DAO
        
    -   Layer 2 扩容方案  
        可以理解为“Web3 的底层操作系统”。
        
-   **现实中的局限**  
    以太坊也有问题，比如：
    
    -   主网交易费在高峰期偏高
        
    -   性能受限，需要 Layer 2 辅助  
        这些问题正通过技术升级逐步缓解。

理解赛道逻辑、技术趋势、经济模式

### **核心步骤**

**1.赛道认知**

-   DeFi / NFT / Gaming / Layer1/Layer2 / Web3基础设施
    
-   学习资源：DefiLlama、DappRadar、Messari、CoinGecko
    

**2.宏观故事梳理**

-   市场规模 / 成熟度 / 渗透率
    

**3.逻辑链条**

-   技术 → 产品 → 用户 → 收入 → Token 价值

-   绘制“生态图谱”
    
-   练习用一句话解释每个赛道的存在理由
    

* * *

## **二、生态理解（赛道 / 协议类型）**

### **目标**

-   能快速判断协议类型与价值逻辑
    
-   学会识别基础设施型 vs 消费型 vs 投机型产品
    

### **核心步骤**

1.  **分类协议**
    
    -   交易类 DEX（Uniswap、Sushi、Curve）
        
    -   借贷类（Aave、Compound）
        
    -   衍生品（GMX、Perp）
        
    -   稳定币协议（Terra、Maker）
        
    -   Web3 基础设施（Chainlink、The Graph）
        
    
2.  **赛道特征分析**
    
    -   用户类型（散户 / 机构 / 套利者 / LP）
        
    -   产品核心价值（交易、借贷、数据提供、资产流通）
        
    -   收入来源（手续费、利息、协议收益）
        
    
3.  **实践训练**
    
    -   用今天拆 Uniswap 的方法，选一个新的协议做“快速拆解”
        
    -   输出：协议类型 / 核心价值 / 用户 & 使用场景
        
    

* * *

## **三、产品分析（痛点 / 使用场景 / 用户）**

### **目标**

-   从用户和产品角度理解协议
    
-   能识别“真正驱动行为的机制”
    

### **核心步骤**

**真实体验**

-   Swap / 添加流动性 / 使用前端
    
-   注意操作：理所当然 vs 需要思考 / 信息提示策略
    

1.  **用户与痛点分析**
    
    -   谁在用？（散户、机器人、套利者）
        
    -   用户为什么用？（痛点 / 使用场景 / 收益动机）
        
    
2.  **运营观察**
    
    -   用户接触协议的路径（教程、钱包默认、社区、空投）
        
    -   协议是主动吸引还是被动筛选？
        
    -   风险提示与收益提示设计策略
        
    
3.  **今日经验总结**
    
    -   前端简单 → 减少认知负担
        
    -   链上复杂 → 支持专业用户 / 机器人
        
    -   护城河 → 行业默认共识，而非用户忠诚度
        
    

* * *

## **四、链上数据 & 财务指标验证**

### **目标**

-   用链上数据验证产品直觉
    
-   判断协议收入、用户行为、Token 使用
    

### **核心步骤**

1.  **链上行为分析**
    
    -   Etherscan 查看交易方法、频繁地址
        
    -   判断参与者：EOA vs 合约
        
    -   分析使用模式：人类 vs 自动化策略
        
    
2.  **协议数据**
    
    -   DefiLlama / Dune / Token Terminal
        
    -   TVL、收入、费用分成、交易量、用户增长
        
    
3.  **验证直觉**
    
    -   产品体验 → 链上使用 → 数据指标
        
    -   目标：确认对用户行为和价值捕获理解
        
    

* * *

## **五、风险识别 & 投资判断**

### **目标**

-   能把产品理解、生态定位、链上数据结合，做投资分析
    

### **核心步骤**

1.  **风险识别**
    
    -   技术风险：合约漏洞、MEV、治理安全
        
    -   替代风险：基础设施可被替代 → Token 价值可能受限
        
    -   市场风险：流动性不足、TVL 下跌、宏观加密环境变化
        
    
2.  **投资逻辑**
    
    -   产品价值稳定 → 基础设施型
        
    -   Token 捕获价值 → 是否有收入分配 / 治理红利
        
    -   护城河强度 → 行业默认共识 vs 用户忠诚 vs 补贴/社区
        
    
3.  **投资输出模板**
    
    -   产品核心价值
        
    -   用户与场景
        
    -   链上行为验证
        
    -   收入 / Token 捕获分析
        
    -   风险与不确定性
        
    -   投资结论（买 / 不买 / 观望）
        
    

* * *

## **六、实操训练循环**

每次分析协议，循环以下步骤：

1.  生态理解 → 赛道 & 协议类型
    
2.  产品拆解 → 痛点 / 使用场景 / 用户 / 运营
    
3.  链上验证 → Etherscan / DefiLlama / Dune
    
4.  投资分析 → Token 捕获价值 / 风险 / 投资结论
    
5.  总结复盘 → 哪些直觉正确？哪些数据不符？
    

* * *

## **七、Uniswap 拆解经验总结（模板）**

-   产品体验：极简前端，链上复杂
    
-   用户类型：散户 + 套利机器人
    
-   运营策略：教程 & 默认示例 → 行业默认共识
    
-   护城河：行业默认共识，而非用户忠诚
    
-   投资思考：基础设施型产品价值稳定，但 Token 捕获机制需验证
    
-   数据验证工具：Etherscan、DefiLlama、Dune
    
-   投资输出：产品核心价值 + 用户行为 + Token 捕获 + 风险评估

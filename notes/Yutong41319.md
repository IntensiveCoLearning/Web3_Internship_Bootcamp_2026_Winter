# **Day 5｜Convex 奖励路径与“真正的奖励”识别**

> 目标：在 Curve + Convex 体系中，**准确区分「存入 / 锁仓 / 记账行为」与「真实奖励发放」**，并学会从交易流中定位奖励合约。

* * *

## **一、Day 5 的核心问题**

在 Day 4 你已经能看懂：

-   用户 → Router → Pool → Router → 用户 的**Swap 路径**
    
-   Curve 中「操作发生在 Curve，但执行可能由第三方 Router 完成」
    

Day 5 进一步回答三个更关键的问题：

1.  **奖励是从哪里来的？**
    
2.  **哪些 ERC20 转账是真正的“reward”？**
    
3.  **Convex 中多个地址各自扮演什么角色？**
    

* * *

## **二、Convex + Curve 的基础结构（一定要分清）**

### **1️⃣ Curve Gauge**

-   Curve 中**真正产生奖励的地方**
    
-   你把 LP Token（如 3Crv）存入 Gauge
    
-   Gauge 负责按时间计算奖励（CRV 为主）
    

👉 Gauge ≠ 发钱地址，而是**记账 + 计算器**

* * *

### **2️⃣ Convex Voter Proxy**

-   Convex 用来 **代替用户与 Curve Gauge 交互** 的代理合约
    
-   所有：
    
    -   deposit to gauge
        
    -   withdraw from gauge
        
        都通过它完成
        

👉 作用：

-   集中投票权
    
-   集中 boost
    
-   集中管理 LP
    

* * *

### **3️⃣ Convex Booster**

-   Convex 的**核心调度合约**
    
-   负责：
    
    -   mint Convex 体系内的“记账 token”（ERC20 \*\*\*）
        
    -   把奖励拆分、分发到用户
        
    

⚠️ 你看到的：

```
From Null → Convex Booster
ERC20 ***
```

不是钱凭空出现，而是：

> 「记账 token 被铸造」

* * *

## **三、你看到的典型交易链，其实在做什么？**

以你多次贴的结构为例：

### **Step 1｜用户存入 LP**

```
User → Convex Voter Proxy → Curve Gauge
3Crv
```

✔️ 这是 **存入行为**

✔️ 数量 = 存入 LP 数量

* * *

### **Step 2｜Convex 内部记账**

```
Null → Convex Booster
ERC20 ***
```

✔️ 这是 **mint 记账 token**

❌ 不是奖励

❌ 没有市场价值

* * *

### **Step 3｜Booster → 中间地址**

```
Convex Booster → 0x6894...
ERC20 ***
```

✔️ Convex 内部账户或池子

✔️ 用来追踪你“应得多少奖励”

* * *

### **Step 4｜真正的奖励（重点）**

```
0x6894... → 用户地址
ERC20（非 3Crv / 非 ***）
```

✅ **只有这一类，才是你真正赚到的奖励**

可能是：

-   CRV
    
-   CVX
    
-   额外激励 token
    

* * *

## **四、为什么你看到：几乎都是 3Crv？**

这是一个**非常重要的观察**，而且你是对的。

原因：

-   你看的大多数交易是：
    
    -   存入 / 取出 LP
        
    -   Convex 内部调度
        
    
-   **奖励发放往往：**
    
    -   频率低
        
    -   批量
        
    -   由 getReward() 触发
        
    

👉 所以：

-   多数时间看到的是 **3Crv 流转**
    
-   少数时候才看到 **CRV / CVX 流向用户**
    

* * *

## **五、如何判断「这是奖励，而不是操作行为」**

你可以用这个判断表：

| 观察点 | 是否奖励 |
| --- | --- |
| From = Null | ❌ |
| Token = 3Crv | ❌ |
| From = Convex Booster | ⚠️ 不一定 |
| To = 用户地址 | ✅ |
| Token = CRV / CVX | ✅ |

一句话记忆：

> **奖励一定是：有市场价格 + 直接到用户的钱**

* * *

## **六、Day 5 的能力总结（你已经掌握的）**

到这里，你已经可以：

-   正确区分：
    
    -   LP 存入
        
    -   Gauge 记账
        
    -   Convex 内部 mint
        
    -   真正奖励
        
    
-   不会再被「ERC20 \*\*\*」误导
    
-   明白：
    
    -   Convex Booster ≠ 发钱机器
        
    -   Gauge ≠ 钱包

# **Day 3 学习笔记（Curve & Uniswap 对比）**

## **一、今日学习目标**

1.  理解不同 AMM 协议对价格波动的处理方式。
    
2.  学会识别不同协议的风险分配逻辑。
    
3.  能够将机制理解与投资逻辑结合，判断协议适用的资产类型与用户群体。
    
4.  为 Day 4 链上行为观察和投资判断打下基础。
    

* * *

## **二、今日核心内容**

### **1️⃣ Uniswap 的机制回顾**

-   核心 AMM 模型：**常数乘积公式** x \\cdot y = k
    
-   特点：
    
    -   价格随资产比例自由偏离
        
    -   LP 承担价格波动和无常损失
        
    -   适合波动资产交易
        
    
-   结论：
    
    > Uniswap 是面向广泛交易者与策略的开放市场，提供流动性和价格连续性，但价格容易波动。
    

* * *

### **2️⃣ 链上行为识别（复盘 Day 3 链上分析）**

判断池子交易主体的方法：

1.  **交易频率**
    
    -   高频交易 → 自动化策略
        
    -   人类交易 → 随机、间隔较长
        
    
2.  **调用方式**
    
    -   合约调用 → 策略主导
        
    -   钱包直接调用 → 人类用户
        
    
3.  **地址类型**
    
    -   Contract → 自动化策略
        
    -   EOA → 人类用户
        
    

**总结判定**：

> 高频交易 + 合约调用 + 大量 Contract 地址 → 池子主要参与者为自动化策略。

* * *

### **3️⃣ Curve 的机制理解**

-   目标资产：**价格锚定资产（稳定币）**
    
-   核心设计：
    
    -   在目标价格附近，曲线极度平坦 → 小交易几乎不改变价格
        
    -   边缘区间曲线陡峭 → 大额交易成本高
        
    
-   风险分配：
    
    -   小交易 → LP 风险几乎为零
        
    -   大交易 → 交易者承担价格偏离成本
        
    
-   对比 Uniswap：
    
    -   Uniswap LP 承担价格波动风险
        
    -   Curve 将风险推给交易者
        
    
-   投资与产品理解：
    
    > Curve 更像“银行间市场”，服务专业资金，追求低滑点和价格稳定，而非面向普通用户。
    

* * *

### **4️⃣ 直觉与机制总结**

| 协议 | 定位 | 风险承担 | 用户类型 | 适用资产 |
| --- | --- | --- | --- | --- |
| Uniswap | 开放市场 | LP 承担价格波动 | 人类 + 策略 | 波动资产 |
| Curve | 稳定币银行间市场 | 交易者承担价格偏离 | 策略化资金 | 稳定币 |

> 金句总结：

> **Curve 是稳定币的“银行间市场”，通过压平 AMM 曲线，将价格偏离成本转嫁给交易者，保护大额 LP，实现低滑点交易。**

* * *

### **5️⃣ 今日思维闭环**

1.  从 Uniswap 到 Curve，理解 **不同 AMM 的价格弹性设计**。
    
2.  理解 **协议风险分配逻辑** → 谁承担风险，谁获得收益。
    
3.  能够将机制理解映射到 **用户群体和资产类型选择**。
    
4.  为链上数据观察和投资判断做好铺垫。
    

* * *

## **三、今日心得 / 重点提醒**

-   **Curve 不追求用户量，而是保证稳定性** → 理解协议“性格”比公式更重要。
    
-   **风险再分配是理解 AMM 核心的钥匙** → 谁承担风险 = 谁捕获价值。
    
-   **直觉升级为机制理解** → “抗拒价格偏离” = “将成本转给交易者”。
    
-   **投资判断从机制出发** → 不是看价格，而是看协议适用的资产、风险和用户类型。

## **一、Web3 的整体结构理解（Day 1 核心）**

### **1️⃣ Web3 的三层结构**

我目前对 Web3 的理解可以拆成三层：

**🔹 底层：链的架构（Layer 1 / Layer 2）**

-   决定：
    
    -   安全性
        
    -   去中心化程度
        
    -   交易成本
        
    -   可扩展性
        
    
-   本质作用：
    
    > **为规则提供不可篡改的执行环境**
    

它不是直接创造用户价值，而是**约束整个系统的可能性边界**。

* * *

**🔹 中间层：机制与金融基础设施**

-   包括：
    
    -   DEX
        
    -   Lending
        
    -   Stablecoin
        
    -   Oracle
        
    
-   解决的问题：
    
    -   钱如何流动？
        
    -   风险如何定价？
        
    -   价值如何结算？
        
    

关键认知：

> **价值不一定归中间层所有，但离不开中间层的机制设计。**

* * *

**🔹 应用层：用户可感知的产品**

-   离用户最近
    
-   最容易被叙事 & 情绪驱动
    
-   最容易被高估，也最容易泡沫化
    

📌 我的判断：

> **应用层承担“想象空间”，但不一定承担“价值沉淀”。**

* * *

### **2️⃣ 关于估值与情绪的初步判断**

-   应用层：
    
    -   情绪驱动强
        
    -   易高估
        
    
-   基础设施 / 机制层：
    
    -   增长慢
        
    -   但更接近长期价值

# **Uniswap 投资与产品分析报告**

* * *

## **一、项目概览（Executive Summary）**

Uniswap 是以太坊生态中最核心的去中心化交易协议（DEX），也是自动化做市商（AMM）机制的代表性实现。其核心价值不在于提供复杂功能，而在于成为**链上资产定价与流动性的默认基础设施**。从产品、用户行为、链上数据与经济模型综合来看，Uniswap 在协议层面极其成功，但其原生代币 UNI 的价值捕获能力仍存在结构性争议。

* * *

## **二、宏观背景与赛道定位**

### **2.1 行业背景**

中心化交易所长期面临托管风险、监管不确定性与黑箱操作问题，而 Web3 的发展催生了对「无需信任、无需许可、可组合金融市场」的真实需求。DEX 因此成为 DeFi 体系中最基础、最刚需的模块之一。

### **2.2 赛道定位**

-   所属赛道：DeFi → DEX
    
-   协议类型：AMM（自动化做市）
    
-   网络位置：以太坊金融基础设施层
    

Uniswap 的本质并非一家“交易平台”，而是**一套去中心化市场机制的标准实现**。

* * *

## **三、产品与机制分析**

### **3.1 核心产品机制**

Uniswap 通过 AMM 模型（x\*y=k）实现无需订单簿的资产定价与交换，使任何 ERC-20 资产都可在无需许可的情况下建立流动性池并参与交易。

### **3.2 产品体验特征**

-   前端设计极简，仅保留 Swap 与流动性管理核心功能
    
-   几乎不进行用户教育与风险提示
    
-   假设用户已具备基本 Web3 操作能力
    

该设计显著降低了认知负担，同时将复杂性留在链上，服务专业用户与自动化交易系统。

* * *

## **四、用户结构与行为逻辑**

### **4.1 用户类型**

-   普通用户：用于换币、参与新资产
    
-   套利者 / MEV Bot：高频调用合约，维持价格效率
    
-   流动性提供者（LP）：承担无常损失以获取手续费收益
    

### **4.2 行为动机**

Uniswap 的主要交易深度并非由“忠诚用户”贡献，而是由套利者与机器人驱动，其存在保证了 Uniswap 成为链上价格发现的核心场所。

* * *

## **五、生态地位与护城河**

### **5.1 护城河来源**

-   深度流动性形成正反馈
    
-   钱包与聚合器默认集成
    
-   成为 DeFi 协议定价基准
    

Uniswap 的护城河并非用户忠诚度，而是**行业默认共识**。

### **5.2 可替代性分析**

AMM 机制本身高度可复制，但流动性共识与定价中心地位难以迁移，这使得 Uniswap 在长期竞争中保持优势。

* * *

## **六、链上数据与财务逻辑**

### **6.1 核心观察指标**

-   交易量与 TVL（DefiLlama）
    
-   协议产生费用与收入归属（Token Terminal）
    
-   高频地址与合约调用行为（Etherscan / Dune）
    

### **6.2 关键结论**

-   Uniswap 长期保持稳定的交易量与使用频率
    
-   协议收入主要分配给 LP
    
-   UNI 与现金流之间缺乏强绑定
    

* * *

## **七、Token 经济模型分析（UNI）**

### **7.1 UNI 的功能定位**

-   治理投票权
    
-   协议参数与未来费用分配决策权
    

### **7.2 价值捕获问题**

当前阶段，UNI 并不直接分享协议手续费收益，其价值更多来自于对未来治理权与潜在费用开关的预期。

这使 UNI 更接近于：一项对公共基础设施治理权的期权资产。

* * *

## **八、风险分析**

### **8.1 技术风险**

-   智能合约漏洞（历史较低）
    
-   MEV 对普通用户的潜在影响
    

### **8.2 市场与竞争风险**

-   新型 DEX 机制创新
    
-   Layer2 与跨链环境中流动性分散
    

### **8.3 Token 风险**

-   治理权价值难以量化
    
-   长期可能面临估值锚缺失问题
    

* * *

## **九、投资判断与结论**

### **9.1 协议层面判断**

Uniswap 是 Web3 金融体系中不可替代的基础设施，其产品与机制已被市场充分验证。

### **9.2 Token 层面判断**

UNI 是否具备长期投资价值，取决于：

-   治理是否启动真实的价值分配机制
    
-   社区是否形成明确的 Token 捕获共识
    

* * *

## **十、核心总结**

Uniswap 证明了 Web3 世界中「协议成功」与「Token 成功」可以并不等价。它是一个极其成功的公共市场机制，但 UNI 是否能够持续捕获这一公共价值，仍是一个开放性问题。

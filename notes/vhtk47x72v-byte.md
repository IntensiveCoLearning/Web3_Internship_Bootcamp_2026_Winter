# 2026-01-16

回顾这一周的学习轨迹，我感觉自己的认知曲线在逐步抬升：从理解以太坊的底层机制、合约执行逻辑、再到安全与合规风险，最后开始思考：我可以在Web3里做什么？作为一个学习者，我到底在追求什么？

我最早并不清楚Web3的核心到底是什么，经过这一周的学习我才认识到，它不是单纯的去中心化记账，而是一种所有参与者都能共识执行规则、共担风险的共享计算平台。而在这个平台上，智能合约就是规则，它不受某一方控制，而是受全网共识驱动。这个设计紧密关联了安全、身份、信任和激励机制，每一个技术选择背后都有现实世界的权衡。过去我对Web3的认识有点理想化，觉得去中心化就代表自由，但现在我终于知道，自由不是不受约束，而是在无中心控制的前提下，仍然能设计出可以被现实社会接受和约束的系统。安全事件和合规风险也不断提醒我，你在链上写的每一段代码、发起的每一次交易，背后都可能涉及法律边界、资金流动监管、身份认证等复杂问题。这些不是可选的，而是必须要理解并尊重的。

今天我特别去了解了一下本次实习计划的发起者ETHPanda。ETHPanda是一个致力于把全球以太坊生态与中文社区桥接起来的组织，它在教育、活动和技术co-learning方面做了很多基础设施层的工作。通过他们在全球举办的活动、与各类基础设施平台合作，可以看到一个事实：Web3 不再是“单打独斗”的小圈子，而是一个真实存在的全球协作网络。这种协作不仅体现在技术上，更出现在思想和行动方式上。Builder经济不是单纯做产品，而是在不确定性中打造价值流、生产知识和激励系统，让别人也可以站在你的肩膀上继续构建。这种说法和我之前对 Web3 的认识有很大不同。过去我总觉得 Web3 是技术、是币价、是热点，现在我开始理解，Web3的真正机会不在于投机，而在于构建可持续的价值理念。

在Builder的推文里，我也看到很多关于生产力工具、协作流程、AI + Web3的讨论（例如如何让 AI 帮忙读文档、写智能合约、自动化任务等）。这些内容虽然分散在各个Builder的主页，但我认为它们传达了同样的内容：未来的Web3 Builder越来越像是“协作者 + 工具设计者 + 社区生态推动者”三者合一。 

那么，我怎么融入这个世界？我能为这个系统贡献什么？不仅是写代码，更包括建立信息结构、构建学习系统、参与社区互动、推动合作关系。Builder的能力不是简单写代码，而是在于设计流程、降低别人的进入门槛、把碎片知识转化成可复用的方法论。对我而言，Web3不再是一个抽象的名词，它已经成为一个需要用行动、思考、协作、输出去回应的生态。作为一个还在起步的小白，我第一次觉得自己不只是一个看热闹的人，而是可以慢慢成为这个生态的一部分。

# 2026-01-15

今天根据每日计划读了实习手册和021学习以太坊，对Web3产生了一些新的认识和感悟：

Gas本质上是为计算和存储付费。不是你给某个人钱，而是你在为整个网络的算力和共识买单。每一次调用合约，其实都是在请求全网节点一起帮你算一遍，所以贵不是因为坑，而是因为它真的在消耗公共资源。

关于以太坊的账户体系，我搞清楚了EOA和合约账户的区别。钱包不是装币的地方，而是一个由私钥控制的外部账户，而真正复杂的逻辑、规则、资产管理，其实都写在合约账户里。

接着是节点、网络结构这些内容。虽然技术细节有点硬，但一个观念让我很震撼：哪怕你只是一个普通用户，只要你愿意，你也可以运行节点，成为网络的一部分，而不是被服务的人。

此外，今天还听了AI及其基础概念的分享会。

以前我把AI和Web3当成两条完全不同的赛道，但现在发现，它们其实在解决同一个问题，如何在没有中心化信任的情况下，让“非人主体”参与经济活动。AI在Web2里很强，但它没有身份、没有钱包、没有责任边界，而Web3恰好提供了身份、支付、审计和可追责的基础设施。当我看到“AI 智能体+链上身份+微支付协议”这一整套逻辑时，突然觉得Web3不只是金融工具，而更像是下一代数字社会的底层操作系统。未来可能不只是人在链上交易，而是大量 AI 在链上协作、调用服务、付费、结算。这让我第一次对为什么Web3还没死有了一个比较理性的答案，它不是为了取代Web2，而是在为人和机器都能参与的经济系统做准备。

明天继续加油。

# 2026-01-14
<!-- DAILY_CHECKIN_2026-01-14_START -->


如果说之前更多是在理解Web3的理想和结构，那今天几乎全部围绕着一个现实问题展开：Web3的风险究竟是怎么真实地发生的，又是怎么落到具体个人身上的？

一开始看Web3合规的内容时，我以为会是偏程序化、偏条款的学习，但真正读下来才发现，所谓合规远不只是写几份看起来合法的文件。形式上的合规，更多像是一层防护壳，能在表面上降低风险，却并不能真正保证安全。真正决定一个项目、一个用户是否安全的，是业务本身在现实法律体系中的位置。

Web3并没有一套独立于现实世界之外的法律系统。无论技术多新，监管最终审视的，仍然是金融、资金流、获利模式和社会危害性。这种认知让我对“深度合规”这个词有了更清晰的理解。它不是口号，而是从业务设计阶段就要不断问自己：如果把区块链这层技术拿掉，这件事在现实世界里算不算问题？以前我会觉得刑事风险离自己很远，好像只有做项目和搞资金盘的人才需要担心。但今天的内容让我意识到，Web3的刑事风险并不是少数极端案例，而是源于去中心化的技术逻辑与中心化的法律体系之间的错位。很多行为在技术层面是中性的，但一旦进入现实监管视角，就可能被倒推结果、评价社会危害性，进而演化成刑事问题。尤其是涉及出入金、撮合交易、拉新激励、合约交易等场景时，边界并没有想象中清晰。这让我第一次意识到，Web3的创新并不会自动获得理解，它往往需要在事后被解释。

在某些环境下，问题不一定出在行为本身，而是出在解释权上。同一套交易逻辑，在不同法域、不同监管阶段，可能会被赋予完全不同的定性。这种不确定性，是我以前完全低估的。而当我把这些内容和Web3安全的数据、案例放在一起看时，心里的警惕感更强了。安全事件的规模、频率和损失金额，都让我意识到，在Web3世界里，风险不是偶发的，而是常态。无论是黑客攻击、钓鱼和私钥泄露，还是Rug项目和社会工程，都不是因为技术不够先进，而是因为人性的弱点被反复利用。尤其是在AI加持下，攻击变得更隐蔽、更精准，很多骗局已经不再是低级错误，而是足以欺骗有一定经验的人。安全不应该被当成事后补救，而应该是整个系统的前提条件。如果连资产安全、身份安全都无法保证，再宏大的叙事也站不住脚。

Web3并不是一个高风险换高收益的简单公式，而是一个对认知、边界感和自控力要求极高的环境。技术只是工具，真正决定风险大小的，往往是人做决策时的判断方式。我们应该对新事物保持谨慎，但这种谨慎并不是恐惧，而是一种更接近现实的清醒。真正能在Web3长期待下去的人，可能并不是最激进、最敢赌的，而是那些对风险有长期认知、对底线有清晰判断的人。

学会不作恶、少幻想、保护自己，可能和学技术、学概念一样重要。

# 2026-01-13
<!-- DAILY_CHECKIN_2026-01-13_START -->



今天，我第一次把书里和视频中的以太坊和我自己的认知慢慢对齐了。一开始读《021学习以太坊》第一章的时候，我会以为这是一章偏eth发展历史的内容，可能有点枯燥。但真正读进去之后，我发现它其实在帮我回答一个很基础、但我之前一直没认真想过的问题：以太坊到底是为了解决什么问题而存在的？

以前我不清楚为什么在有比特币的情况下还需要以太坊，现在才意识到，以太坊的起点并不是货币，而是程序。Vitalik想做的事情本质上是把区块链从一个只能转账的系统，变成一个可以运行复杂逻辑的公共平台。这个出发点，让我第一次觉得以太坊不像是某个产品，而更像一套基础设施的实验。书里反复提到“全球共享的状态机”、“世界计算机”这些说法，一开始看得我有点抽象。但当我把它和视频里讲到的智能合约执行过程放在一起理解时，突然就通透了。原来所谓的“世界计算机”，并不是说它很快、很强，而是说全世界有无数节点，在共同维护同一份状态，并且对状态的变化达成共识。你写的代码，不是跑在自己的服务器上，而是跑在这个共识系统里。

拓展材料两段视频中从Web2到Web3的对比，对我冲击还挺大的。以前的人写程序，默认前提是“我控制服务器”。而在Web3里，这个前提被彻底拿掉了。智能合约一旦部署，就像规则被写进了一个所有人都能验证、却几乎无法篡改的系统里。讲到Gas的时候，我一直把它理解成手续费，但结合视频里对执行过程的讲解，我才意识到它更像是这台世界计算机的资源定价机制。你想让全世界的节点帮你算点东西，就必须为消耗的算力和存储付出成本。不然系统要么被滥用，要么无法维持。这让我开始明白，很多看起来不友好的设计，其实都是在现实约束下的理性选择。在对比以太坊和比特币那一部分，我的心态也发生了变化。以前我会下意识地站队，觉得一定要分出谁更好。但现在更像是接受了一件事：它们本来就不想做同一件事。比特币选择了极度克制，只专注于价值转移，而以太坊选择了开放，把复杂性留给开发者和社区。这条路显然更难，也更容易出问题，但它的上限确实更高。提到实际开发流程时，我并没有完全听懂所有细节，但我已经大概能想象到一个轮廓：钱包不只是装币的，而是你这个人的替身；测试网是为了让错误先发生在可控范围内；每一次部署，都是一次不可撤销的发布。

这一整天下来，我慢慢意识到，以太坊真正厉害的地方，并不在于某个具体应用，而在于它允许任何人，在无需许可的情况下，把规则写进一个全球可验证的系统里。DeFi、NFT、DAO只是这些规则自然长出来的形态，而不是终点本身。现在的我，依然算是 Web3小白，很多技术细节还没吃透，很多名词还需要反复消化。但和之前不同的是，我不再只是知道eth很厉害，而是开始理解它重要的原因，并不在于价格，而在于它试图解决的问题本身。今天的学习，转变了我看待区块链的视角，不是急着学怎么赚钱，而是先搞清楚这套系统到底在干什么、为什么要这样设计。

# 2026-01-12
<!-- DAILY_CHECKIN_2026-01-12_START -->




今天花了不少时间听了两场分享会。说实话，一开始的感受并不轻松，信息密度很高，很多地方需要边听边检索信息才能勉强跟上。但当我慢慢把内容在脑子里串起来的时候，反而有一种很微妙的感觉——我好像第一次不是站在“看热闹”的位置，而是开始试着理解这个行业本身在经历什么。

过去我对 Web3 的认知非常简单，甚至有点刻板：涨跌很快、风险很高、项目跑路、黑客横行，更多像是一个投机者聚集的地方。但在今天的学习之后，我意识到这个行业正在发生一种很明显的转向。它不再只依赖币价和叙事往上冲，而是被一步步拉进传统金融和监管体系之中。ETF、稳定币、机构资金、合规框架，这些原本在我印象中属于传统金融行业的词，正在频繁地和区块链绑在一起出现。这种变化让我产生了一些复杂的情绪。一方面，它好像不再那么自由和理想主义了；另一方面，我也意识到，如果没有这些东西，Web3可能永远只能停留在一个小圈子里自嗨。而现在，它正在被现实世界重新塑形。

技术层面的内容对我来说依然有难度，比如以太坊的升级、Layer2的扩容方案、区块传播、宕机和验证机制等等。有些细节我还没完全理解，但有一个观点让我印象特别深刻：并不是所有东西都要追求极致效率。以太坊真正看重的不是快，而是长期稳定、抗审查，以及在极端情况下依然可信的能力。这让我第一次意识到，不同公链之间的竞争，本质上是价值取舍，而不是谁的技术更酷。

相比技术，我对行业介绍分享会的内容感触更深。之前我会下意识地觉得，Web3机会多、门槛低，可能是普通人弯道超车的地方。但现实比我想象得残酷得多。真正值钱的不是对Web3的想象力，而是你能为这个生态创造什么价值。没有作品、没有真实贡献、没有长期积累，仅仅靠热情是很难留下来的。Emily老师的一段话让我印象特别深刻：“在这个行业里，简历本身并不重要，重要的是你在链上、在社区、在真实项目里留下过什么痕迹。哪怕是很小的参与，只要是真实的、持续的，都会比空泛的自我介绍更有说服力。”这让我意识到，在Web3行业，你想要被看见，需要脚踏实地做出成绩，而不是先包装、再入场。

听完分享会后，我更加清楚自己现在处在一个非常基础的位置。但这种基础不再是迷茫，而是知道自己哪些地方不懂，也知道如果继续学下去，大概该往哪里走。这次实习计划让Web3对我而言不再只是一个遥远又混乱的概念，而是一个正在被不断拉扯、重塑的认知。它不完美，有很多问题，但正因为如此，才值得慢慢看、慢慢学，而不是急着下结论。

# 2026-01-13
<!-- DAILY_CHECKIN_2026-01-13_START -->


以太坊的创新核心：智能合约（存储在区块链上可自动执行的代码）

The Merge: 从pow到pos（最重要的一次升级）

合并后以太坊的新架构：1. 执行层：处理交易，智能合约（原主网）

2.共识层：管理验证者，确定区块顺序（信标链，Beacon chain）

分片和layer 2：提升效率和可扩展性

以太坊是一个基于区块链的去中心化计算平台，目标是成为“全球计算机”

验证者的奖励：新发行的eth和交易费用；作恶者惩罚：质押的eth销毁（slashing）

数据可用性分片：为rollup提供更多更便宜的数据存储空间；以太坊主网专注于：结算层和数据可用性层

layer 2：链下处理大批量交易，定期把交易数据打包提交到以太坊主网，主网验证数据正确性，提供最终安全保障

EIP-4844（Cancun升级）: 引入专门的blob交易，blob数据会在一定时间内自动删除，不占用主网存储。代表项目：Arbitrum、Optimism

ZK-Rollup: 在链下一次性处理上百笔交易，生产一个正确性证明，以太坊主网只需要验证这个证明，无需重新执行所有交易。因此，多笔交易可以分摊验证成本。同时继承了以太坊主网的高安全性。代表项目：zkSync Era、Polygon zkEVM、Scroll

**以太坊升级里程碑**

1.  **The Merge（2022 年完成）**：从高能耗 PoW → 低能耗 PoS
    
2.  **EIP-4844（2024 年完成）**：让 L2 更便宜，普通人用得起（引入了blob）
    
3.  **数据分片（2025-2026 年）**：进一步降低 L2 成本，提升整体扩展性
    
4.  **Layer 1（L1）**
    
    -   **以太坊主网**：核心区块链，负责最终安全性与共识。
        
    -   **EVM**：以太坊虚拟机，执行智能合约代码。
        
    -   **账户系统**：外部账户（EOA）与合约账户（CA）共同构成网络基础。
        
5.  **Layer 2（L2）**
    
    -   **Rollup**：通过将交易批量处理后提交至 L1，降低 Gas 费。
        
        -   **Optimistic Rollup**：假设交易合法，仅在争议时验证。
            
        -   **ZK Rollup**：通过零知识证明验证交易，无需链上争议。
            
6.  **侧链（Sidechains）**：独立运行的链，通过桥接与主网交互。
    

以太坊生态分层架构：1. 应用层

2\. 协议层

3\. 扩展层

以太坊的三个关键机制：账户系统，gas模型，以太坊虚拟机（EVM）

每个账户都包含四个关键字段：

-   **Nonce**：防止重复交易的计数器（EOA 记录发送次数，CA 记录创建合约次数）。
    
-   **余额**：账户持有的 ETH 数量（单位为 Wei）。
    
-   **CodeHash**：EOA 为空哈希，CA 存储合约字节码的哈希值。
    
-   **StorageRoot**：记录账户数据的 Merkle 树根哈希（如 NFT 归属关系）。
    

Gas price组成：Base fee+ tip, base fee 会被销毁，维持eth通缩

EVM: 以太坊的大脑，专门用来运行智能合约的虚拟计算机。它运行在每个节点上

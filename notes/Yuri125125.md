## **2.区块链“牵一发而动全身”**

区块链数据进行哈希运算并随后附加到区块引用的历史记录的视频演示，哈希函数（如 SHA-256）有一个关键特性：输入的微小变化会导致输出的哈希值发生巨大且不可预测的变化。这是一个 **“牵一发而动全身”** 的机制。

一个区块主要包含两部分：

1.  **区块头**：包含前一个区块的哈希值（父哈希）、本区块交易的默克尔根、时间戳、随机数等。
    
2.  **交易数据**：这个区块打包的所有交易列表。
    

**关键点在于：当前区块的哈希值，是通过对“区块头”进行哈希计算得到的。而区块头里包含了“前一个区块的哈希值”。**

假设有一个恶意的矿工想篡改区块链历史中**第 N 号区块**里的某笔交易。

**步骤 1：修改数据**

-   他修改了第 N 号区块中的一笔交易。
    
-   这会导致该区块所有交易生成的**默克尔根**发生改变。
    

**步骤 2：区块哈希值改变**

-   因为默克尔根是区块头的一部分，所以第 N 号区块的**区块头**变了。
    
-   对新的区块头进行哈希运算，会得到一个**全新的、不同的第 N 号区块哈希值**。
    

**步骤 3：链断裂**

-   第 N+1 号区块的区块头里，记录着“父哈希”（即第 N 号区块的哈希值）。现在第 N 号区块的哈希值变了，导致第 N+1 号区块头中记录的“父哈希”指向了一个不存在的区块（因为旧的第 N 号区块已被修改，其旧哈希对应的区块已无效）。
    
-   **此时，从第 N+1 号区块开始，整条链就断开了**，因为它不再能链接到它本应跟随的有效父区块。
    

**步骤 4：修复链的代价（几乎不可能）**

为了让链重新有效，这个攻击者必须：

1.  为第 N 号区块重新计算一个有效的工作量证明（即挖矿，需要巨大的算力）。
    
2.  因为第 N 号区块哈希变了，他必须**接着重新计算第 N+1 号区块的工作量证明**，以更新其“父哈希”并使其指向新的第 N 号区块。
    
3.  然后，他必须**接着重新计算第 N+2 号区块的工作量证明**……以此类推，**一直重新计算到当前最新的区块**。
    
4.  而且，他做所有这些计算的速度，必须比世界上其他所有诚实矿工**共同延伸主链的速度还要快**，否则他修改的这条链永远追不上主链，不会被网络接受。
    

## **3.交易**

以太坊有几种不同类型的交易：

-   常规交易：从一个帐户到另一个帐户的交易。
    
-   合约部署交易：没有“to”地址的交易，数据字段用于合约代码。
    
-   执行合约：与已部署的智能合约进行交互的交易。 在这种情况下，“to”地址是智能合约地址。
    

![image.png](https://raw.githubusercontent.com/IntensiveCoLearning/Web3_Internship_Bootcamp_2026_Winter/main/assets/Yuri125125/images/2026-01-13-1768279609202-image.png)

燃料费是**用于执行某些操作的燃料数量，乘以每单位燃料的成本**。 无论交易成功与否，都要支付燃料费。简单的转账交易需要 21000 单位燃料。

燃料费必须用以太坊的本币支付，即以太币 (ETH)。 燃料通常以 gwei 计价，gwei 是以太币的一种计量单位。 一个 gwei 等于一个以太币的十亿分之一（0.000000001 个以太币，或 10-9 个以太币）。

比如，你可以说你的燃料费是 1 gwei，而不说 0.000000001 个以太币。

你支付的总燃料费分为两部分：`base fee` 和 `priority fee`（小费）

总费用等于：

`使用的燃料单位数 *（基础费 + 优先费）`

燃料是指验证者处理交易所需的计算。 用户必须为此计算支付费用。 `gasLimit` 和 `maxPriorityFeePerGas` 决定支付给验证者的最高交易费。

任何涉及智能合约的交易都需要燃料。

重要提示：区块本身的大小是有界限的。 每个区块的目标大小为 3000 万单位燃料，但区块的大小将根据网络需求而增加或减少，直至达到 6000 万单位燃料的区块限制（目标区块大小的 2 倍）。

区块包含的字段：

[https://ethereum.org/zh/developers/docs/blocks/](https://ethereum.org/zh/developers/docs/blocks/)

提交的交易包括下列信息：

[https://ethereum.org/zh/developers/docs/transactions/](https://ethereum.org/zh/developers/docs/transactions/)

-   `from` - 发送者的地址，该地址将签署交易。 这将是一个外部帐户，因为合约帐户无法发送交易
    
-   `to` — 接收地址（如果是外部帐户，交易将传输值。 如果是合约帐户，交易将执行合约代码）
    
-   `signature` – 发送者的标识符。 当发送者的私钥签署交易并确保发送者已授权此交易时，生成此签名。
    
-   `nonce` - 一个有序递增的计数器，表示来自帐户的交易数量
    
-   `value` – 发送者向接收者转移的以太币数量（面值为 WEI，1 个以太币 = 1e+18wei）
    
-   `input data` – 可包括任意数据的可选字段
    
-   `gasLimit` – 交易可以消耗的最大数量的燃料单位。 [以太坊虚拟机](https://ethereum.org/zh/developers/docs/evm/opcodes/)指定每个计算步骤所需的燃料单位
    
-   `maxPriorityFeePerGas` - 作为小费提供给验证者的已消耗燃料的最高价格
    
-   `maxFeePerGas` - 愿意为交易支付的每单位燃料的最高费用（包括 `baseFeePerGas` 和 `maxPriorityFeePerGas`）
    

### **Typed Transaction Envelope交易**

以太坊最初有一种交易形式。 每笔交易都包含 Nonce、燃料价格、燃料限制、目的地地址、价值、数据、v、r 和 s。 这些字段为 [RLP 编码](https://ethereum.org/zh/developers/docs/data-structures-and-encoding/rlp/)，看起来像这样：

`RLP([nonce, gasPrice, gasLimit, to, value, data, v, r, s])`

**Type 0 交易（传统交易）**

-   **核心特征**：原始交易格式，功能基础。
    
-   **主要区别**：不支持 EIP-1559 的动态费用市场机制。
    
-   **识别方式**：RLP编码后以 `0xf8` 开头，类型值为 **0x0**。
    

**Type 1 交易**

-   **核心特征**：在柏林升级中引入，主要增加了 `accessList`。
    
-   **主要区别**：包含预访问列表，可优化复杂合约交互的燃料费，但仍不支持 EIP-1559 费用机制。
    
-   **识别方式**：以字节 `0x01` 开头，类型值为 **0x1**。
    

**Type 2 交易（EIP-1559 交易）**

-   **核心特征**：伦敦升级引入，已成为标准交易类型。
    
-   **主要区别**：引入基础费与优先费的费用市场，燃料费更可预测。
    
-   **识别方式**：以字节 `0x02` 开头，类型值为 **0x2**。
    

**演进总结**：

以太坊交易从 Type 0 的基础格式，演进到 Type 1 的性能优化（`accessList`），最终在 Type 2 中通过 EIP-1559 实现了费用市场的根本性改革，提升了用户体验和网络效率。

## 4.以太坊虚拟机

以太坊虚拟机 (EVM) 是一个去中心化虚拟环境，它在所有以太坊节点上一种安全一致地方式执行代码。 节点运行以太坊虚拟机，以执行智能合约，利用“[燃料](https://ethereum.org/zh/gas/)”度量执行[操作](https://ethereum.org/zh/developers/docs/evm/opcodes/)所需的计算工作，从而确保高效的资源分配和网络安全性。

### **以太坊状态转换函数**

EVM 的行为就像一个数学函数：在给定输入的情况下，它会产生确定性的输出。 因此，将以太坊更正式地描述为具有**状态转换函数**非常有帮助：

`Y(S, T)= S'`

给定一个旧的有效状态 `（S）`\> 和一组新的有效交易 `（T）`，以太坊状态转换函数 `Y（S，T）` 产生新的有效输出状态 `S'`

## 1.Web3实习手册入门导读及行业知识

### **1.基础概念：**

**1.1.区块链基本概念**

1.  区块链定义
    

-   区块链是一种去中心化的分布式账本技术，确保在网络节点之间安全、透明且不可篡改地记录数据。
    
-   每条链由一系列依时间顺序相连的“区块”构成，每个区块包含交易数据及其哈希摘要。
    

* * *

2.  区块链特性
    

-   **不可篡改**: 利用前区块的摘要连接，修改历史信息需要重新计算后续全部区块。
    
-   **公开透明**: 所有交易记录对所有人可见，但钱包拥有者的身份保持匿名。
    
-   **快速交易**: 交易可在几分钟内完成，远快于传统跨国汇款。
    

* * *

3.  区块链类型
    

-   **公链**: 任何人皆可加入，数据公开透明，适用加密货币和公共存证。
    
-   **联盟链**: 由特定成员管理，数据半公开，适合金融合作和供应链管理。
    
-   **私链**: 仅限内部成员，数据完全私有，适用于企业内部管理和审计。
    

* * *

4.  Web3的区别
    

-   **Web2**: 中心化控制，用户缺乏数据主权，依赖广告收入。
    
-   **Web3**: 去中心化互联网，用户数据主权回归，智能合约自动执行，无需中介。
    

* * *

5.  去中心化优势与挑战
    

-   **优势**:
    
-   信任最小化，降低依赖中介的需要。
    
-   数据存储抗审查，高弹性。
    
-   用户自主管理资产与数据。
    
-   **挑战**:
    
-   可扩展性问题，公链在节点增多时效率低下。
    
-   安全与治理风险，潜在代码漏洞。
    
-   用户体验不友好，如复杂的私钥管理。
    

**1. 2 以太坊**

-   **定义**: 以太坊（Ethereum）是一个开源的去中心化区块链平台，支持智能合约，被称为“区块链 2.0”。
    
-   **核心创新**: 智能合约（Smart Contracts），实现自动化执行，无需人工干预。
    
-   **发展历程**: 由维塔利克·布特林于2013-2014年提出，2014年正式启动，成为继比特币之后的重要区块链项目。
    
-   **加密货币**: 原生代币为以太币（ETH）。
    

* * *

**以太坊与比特币的差异**

| 维度 | 比特币（Bitcoin） | 以太坊（Ethereum） |
| --- | --- | --- |
| 目标与定位 | 去中心化数字货币，强调安全与稀缺性 | 去中心化平台，支持智能合约与Dapps |
| 编程能力 | 限制较多，仅支持简单交易逻辑 | 图灵完备，支持复杂智能合约 |
| 共识机制 | 工作量证明（PoW） | 从PoW转向权益证明（PoS） |
| 交易速度 | 每10分钟一个区块 | 每约12秒一个区块 |
| 经济模型 | 总量固定 | 供应灵活，可能呈现通缩趋势 |

* * *

**以太坊的演进**

-   **以太坊 1.0阶段**:
    
-   采用PoW机制，存在能耗高、扩展性差等问题。
    
-   **以太坊 2.0与The Merge**:
    
-   2020年启动信标链，2022年正式合并，转向PoS，降低能耗和提升效率。
    
-   现在的架构分为执行层和共识层。
    

* * *

**关键机制**

-   **账户系统**:
    
-   包含外部账户（EOA）和合约账户（CA）。
    
-   EOA通过私钥控制，CA由智能合约代码驱动。
    
-   **Gas模型**:
    
-   交易需支付Gas费用，Gas费用=Gas Limit × Gas Price。
    
-   随着EIP-1559升级，Gas费用拆分为基础费用和小费。
    
-   **以太坊虚拟机（EVM）**:
    
-   运行智能合约，确保代码执行结果一致与可信。
    
-   具备图灵完备，支持各种逻辑判断与循环。
    

* * *

**未来升级与方向**

-   **分片技术**: 预计在2025-2026年实施，提供更便宜的数据存储空间。
    
-   **EIP-4844**: 通过Blob交易降低Layer 2的交易成本。
    
-   **ZK-Rollup技术**: 实现批量处理与低成本交易。

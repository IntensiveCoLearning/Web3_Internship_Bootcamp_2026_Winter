# 2026-01-18
<!-- DAILY_CHECKIN_2026-01-18_START -->
今天听了《ERC721 到 ERC7962 研讨》，对 Web3 资产协议的演进有了更深入的认识。

传统 ERC721 协议以钱包地址作为资产所有权标识，适合 NFT 等“独一无二的数字物品”，但在会员身份、隐私资产和商业应用场景中存在明显限制：用户需要理解钱包和 gas 机制，地址信息完全公开，容易形成行为画像，对 Web2 用户极不友好。

ERC7962 从底层重新设计了“所有权”的表达方式，不再以 ETH 地址作为资产归属，而是使用公钥哈希作为所有权标示，将资产所有权与交易发起地址解耦。用户只需通过签名表达意图，无需持币或直接上链操作，交易可由商家或 relayer 代为发起并承担 gas 费用，大幅降低使用门槛。

协议引入一次性 key 轮换和 UTXO 风格设计，每次交易都会生成新的哈希标识，避免资产关联和隐私泄露，同时取消 approve 机制，减少安全风险。结合零知识证明思想，系统可以验证交易合法性，但无法识别用户真实身份，在隐私保护与可验证性之间取得平衡。

# 2026-01-17
<!-- DAILY_CHECKIN_2026-01-17_START -->

这几天的学习让我对 Web3 不再停留在“概念拼图”，而是逐步形成了从底层运行机制 → 风险与安全 → 合规边界 → AI 与 Web3 融合 → 个人学习方法论的整体认知框架。

一、对 Web3 的理解

通过对 Web3 运行原理的系统学习，我第一次真正理解了 Web3 是一套以密码学为核心的“去中心化运行系统”，而不是单纯的币、NFT 或应用。

1.钱包不是账户，而是主权工具

2.私钥/助记词并非登录凭证，而是资产控制权本身。这让我意识到 Web3 的自由并非“更方便”，而是“不可逆的责任转移”——没有客服、没有重置，只有你自己。

3.交易与签名机制重塑了信任模型签名验证让我看到一个关键转变:信任不再来源于机构，而来源于“可验证的数学事实”。Gas 机制也让我意识到，区块链不是免费的，它只是把成本显性化、市场化。

4.智能合约不是程序，而是社会规则的表达“Code is Law”不再是一句口号，而是一种把信任、规则和执行压缩进代码的社会实验。它减少中介摩擦，但也放大了代码错误的代价。

二、安全认知的提升

Web3 安全部分给我最大的冲击是：安全不是技术细节，而是这个行业的生存底色。

数据让我直观意识到：攻击已高度产业化、国家化；普通用户和顶级机构面对的是同一批对手

Bybit、Blancer 等案例让我明白:真正危险的不是“看得见的钓鱼”，而是“看起来完全正常的流程被悄悄替换”。

同时，我也意识到 AI 正在改变攻击与防御的形态：AI 可以挖出多年未被发现的合约漏洞；Deepfake 已经开始绕过“信任熟人”这条心理防线

这让我对“四不原则”不再是背诵，而是理解为一种低认知成本但高收益的生存策略。

三、合规学习带来的现实感

合规部分让我对 Web3 的认知明显“落地”了。我开始理解 “技术中立在大陆语境下并不成立”，法律并不是是否“合理”，而是是否“被这样解释和执行”。合规的本质不是安全，而是 降低被选择性执法击中的概率。尤其是关于：个人出入金、项目方经营、KOL 推广责任。

四、AI × Web3：第一次看到“下一阶段形态”

AI 与 Web3 的结合部分，是这一周最具“未来感”的内容。我开始理解：AI 的核心短板：身份、支付、审计，Web3 恰好补齐了这些短板。X402、ERC-8004、Agent OS 等概念让我意识到：Web3 可能不是为“人”而生的下一代互联网，而是为“智能体”准备的经济与信任层。这让我第一次把 Web3 从“金融实验”重新定位为： 机器之间可信协作与结算的基础设施。

五、学习方法与个人反思

周会分享让我意识到，真正拉开差距的不是信息量，而是学习结构与输出机制。OKR、思维导图、AI 协作，本质都是在对抗“碎片化理解”“先完成，再完美”这句话击中了我：在高不确定领域，完成度本身就是一种竞争力。我也开始重新审视自己的位置：Web3 不要求“推翻过去”，而是要求重新组合能力；修心、抗噪音、长期视角，比短期情绪更重要。

# 2026-01-15
<!-- DAILY_CHECKIN_2026-01-15_START -->


在学习 AI与Web3 基础概念的相关内容后，我逐渐意识到，二者的关系可能并不是“AI 需要区块链来锦上添花”，而是当 AI 开始具备自主行动能力时，现有的互联网体系本身就难以继续承载它。随着 AI 从单纯的对话模型，演进为能够调用工具、执行任务、甚至代替人进行决策的 Agent，它事实上正在接近一种“行动主体”的形态。但在当前的 Web2 架构下，这种主体是模糊且不被真正承认的：AI 没有稳定可验证的身份，其行为记录掌握在中心化平台手中，也无法独立完成价值交换。这意味着，一旦 AI 的影响扩大，它的行为既难以被信任，也难以被追责。

从这个角度看，Web3 提供的价值开始变得具体而现实。链上身份、不可篡改的行为记录以及原生的价值转移机制，并不是为理想化的去中心化服务，而是恰好对应了 AI Agent 在真实运行中暴露出来的问题。它们让“这个 AI 是谁、做过什么、是否值得继续合作”成为一个可以被验证的问题，而不是完全依赖平台背书或中心化数据库。与此同时，我也并不认为 AI 与 Web3 的结合一定会自然发生。当前的链上交互成本、技术复杂度，以及安全风险，依然对普通开发者和使用者构成很高门槛。如果 Web3 不能在降低使用成本和提升体验上取得实质进展，那么它即便在逻辑上成立，也可能只停留在小范围实验中。这也让我对 Web3 的未来产生了一种更谨慎的判断：它未必会成为所有 AI 的底座，但很可能在“需要可信协作、自动结算和可审计行为”的场景中，率先体现出不可替代性。尤其是在多 Agent 协作、机器对机器交易逐渐增多的情况下，传统互联网的信任机制会越来越吃力。

对我个人而言，这种理解反而让我对 Web3 的关注更加理性。我不再单纯地把它视为一个充满投机与叙事的领域，而是尝试从“它解决了什么新问题”出发去判断其价值。AI 的发展正在逼迫我们重新思考信任、责任与主体性的问题，而 Web3 是否能成为其中的一部分，我认为仍值得持续观察与深入学习。

# 2026-01-14
<!-- DAILY_CHECKIN_2026-01-14_START -->



今天的学习主要围绕 Web3 的底层运行原理展开，并在此基础上结合邓律师关于合规与现实环境的分享，对技术能力与实际应用边界有了更完整的认识。

在技术层面，我系统理解了 Web3 的账户与资产控制机制。钱包的本质并非资产存储工具，而是私钥管理与交易签名工具。私钥是资产控制权的唯一凭证，公钥和地址只是其派生出的公开标识，助记词本质上等同于一组可恢复私钥。这让我从底层逻辑理解资产“归属”的含义，也明白了为什么私钥与授权管理是 Web3 安全体系中最核心的部分。

在交易执行机制上，我梳理了一笔链上交易从生成到最终确认的完整流程：钱包组装交易数据并进行私钥签名，通过 RPC 广播至节点网络，交易进入交易池排队，随后由验证者选择打包并通过共识机制写入区块。在这一过程中，节点无需获取私钥即可验证交易合法性，体现了密码学在身份验证中的关键作用。同时，我对 Gas 机制有了更清晰的理解，能够区分 Gas Price 与 Gas Usage 在成本构成中的不同角色，并理解网络拥堵与合约复杂度如何直接影响交易费用。

在区块链网络层面，我进一步理解了出块时间与最终性之间的关系。以以太坊为例，交易并非在出块后立即具备完全不可逆性，而是需要经过多轮共识确认进入 Finalized 状态。这让我对“到账”“确认数”等常见概念有了更深刻的技术认知，而不是简单以时间长短作为判断标准。

在智能合约方面，我将其理解为运行在区块链虚拟机中的确定性程序，其核心价值在于将规则以代码形式公开固化，并在满足条件时自动执行，从而减少对中心化中介的依赖。这也让我意识到，合约风险同样是代码层面的，一旦部署，错误逻辑或权限设计问题很难通过事后手段修复，因此合约审计与安全设计具有基础性意义。

在此基础上，听了关于web3合规的分享，我开始意识到技术逻辑与现实世界规则之间的差异。虽然 Web3 在技术上实现了去中心化运行和无需许可的交易，但在现实社会中，法律责任与监管定性仍然会围绕“行为结果”和“实际功能”展开。即便交易由代码自动执行、由私钥签名发起，相关参与者仍可能被纳入传统法律框架进行评价。

# 2026-01-13
<!-- DAILY_CHECKIN_2026-01-13_START -->




今天主要学习了两个方面：一是 Web3 行业全局介绍与求职路径，二是以太坊中文周会内容。对我来说，这两部分信息形成了一种很有意思、但也有点“割裂”的对照。从行业全景来看，我第一次比较系统地理解了 Web3 并不是一个线性发展的行业，而是经历了明显的阶段更替：从早期的极客实验，到 ICO 与 DeFi 带来的造富与乱象，再到应用爆发、机构入场，最后走向合规化和金融基础设施化。尤其是 2024–2026 这一阶段，监管趋于明确、ETF 打通传统金融、RWA 和 AI 融合，让 Web3 不再只是“圈内人的游戏”，而是正在被主流资本和制度认真对待。这一点在数据层面也很直观：机构资金入场、市场规模扩大、但真正高价值岗位却高度集中。但也正是这里让我产生了比较强烈的现实感。表面上看，Web3 是“高增长行业”，但实际的人才结构非常陡峭：初级岗位极度拥挤，而真正有议价能力的是 Rust 工程师、安全审计、合规和投研这类高门槛角色。再加上 AI 工具已经开始替代一部分基础工作，让我意识到，Web3 并不是“门槛低、机会多”，而是对个人能力要求更极端的行业。如果没有明确的能力积累方向，很容易被行业叙事本身迷惑。

相比之下，以太坊中文周会的内容让我看到了一种完全不同的节奏。无论是 ERC-4804 解决 NFT 元数据失效问题，还是 PeerDAS 为 blob 扩容、Layer2 增长铺路，这些内容几乎都不追逐短期热点，却在不断修补和强化底层结构。Vitalik 重申“世界计算机”的使命时，我明显感觉到，以太坊所代表的这一派叙事，更关心的是系统能否长期稳定运行，而不是下一轮市场情绪。这种长期主义和行业中常见的投机项目形成了很强的反差。这也让我产生了一种微妙的矛盾感：一方面，行业全景告诉我 Web3 正在变得更成熟、更主流；另一方面，真正推动这一切的，却是一些短期内几乎“看不到回报”的基础设施建设。这让我开始反思自己对“入行”的预期——我之前可能潜意识里还是把 Web3 当成一个“机会窗口”，但现在更像是在面对一个需要长期投入、持续学习、且容错率极低的体系。尤其是链上操作的不可逆性，也让我对安全和风险多了一层敬畏。总的来说，今天的学习让我对 Web3 多了一些现实判断。它不再只是一个靠叙事驱动的风口行业，而更像是一个正在成型的新型基础设施生态。机会确实存在，但并不平均分配，而是留给那些能在长期主义中找到自己位置的人。对我来说，接下来可能比“快速入行”更重要的，是想清楚自己究竟能在哪个维度持续积累，而不是被宏大的行业前景牵着走。

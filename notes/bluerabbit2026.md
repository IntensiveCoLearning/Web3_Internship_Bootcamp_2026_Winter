# Web3 运营与 KOL 合作学习笔记

1.  **KOL 合作核心**
    
    优先对接头部 KOL，看中粉丝量与影响力，尾部账号性价比低。Web3 领域 “黑红也是红”，部分项目方借争议 KOL 引流，用户抱着 “快进快出” 心态参与。头部 KOL 年费可达 30 万，多与大交易所签约。
    
2.  **Web3 运营要点**
    
    与 KOL 侧重宣传不同，运营聚焦项目本身。冷启动可买粉、对接撸毛网站发任务，任务设置避免过难。流量转化数据差，自然流量推文 200 + 浏览量即算优质，高数据多为刷量。社群运营可借兴趣小组形式，国内需注意合规，避免规模过大。
    
3.  **求职与发展建议**
    
    技术岗需补实战能力，多参与黑客松；复合型人才（懂金融 + 技术）更吃香，推荐量化、交易所岗位，优先考虑 OKX、BN 等大平台。入行可多参加线下活动获取内推机会。
    
4.  **工具与避坑**
    
    AI 可辅助作图、开发，但底层技术研发仍需人工。辨别项目靠白皮书与 API 接口，无相关内容的项目需谨慎。

1\. 传统LLM的三大致命限制

-   无状态：每次对话从零开始，上下文窗口有限（早期32K/64K），像“金鱼记忆”。
    
-   无工具：只能生成文本，无法调用API、执行代码，像“被绑住的天才”。
    
-   无身份：无法代表任何人签署协议、管理资产，像“黑户”。
    

2\. 2026 Agentic AI的突破  
三大组件让AI“有脑、有记忆、有手脚”：

-   大模型（LLM） → 推理引擎，理解意图、制定计划、输出指令（GPT系列、Claude、DeepSeek等）。
    
-   记忆（Vector DB + RAG） → 通过embedding → 向量数据库（Pinecone等）实现长期私有记忆，从“金鱼”进化到“大象”。
    
-   工具调用（Function Calling / MCP） → 支持ReAct循环（思考→行动→观察），让AI真正“说到做到”。
    

3\. Web3解决的三大痛点  
AI能力强但仍是“黑户”：身份不可验证、行为不可审计、支付颗粒度/速度/门槛不匹配（验证码、人脸、信用卡手续费高、结算慢）。

-   X402：HTTP原生支付协议（402 Payment Required状态码），嵌入支付地址/金额，支持稳定币微支付（0.001美元级），毫秒级结算，适合agent间高频交互。
    
-   ERC-8004：AI链上身份证（基于ERC-721 NFT），包含身份注册表（唯一ID、能力声明）、声誉注册表（付费客户评价+防刷分：需真实交易记录、时间衰减）、验证机制（TEE/zkML）。
    
-   Spoon OS：Web3 AI操作系统框架，封装钱包托管、身份一键上链、X402支付、统一LLM接口、多agent编排、DeFi工具等，让开发者快速构建有“身份证+银行卡”的AI agent。
    

4\. 未来展望

-   到2028年，33%企业核心应用将采用Agentic AI（Gartner）。
    
-   机器经济需要区块链作为“操作系统”：高TPS、低gas、24h不间断、不可篡改审计日志，解决合规与信任（欧盟AI法案、美国监管需求）。
    
-   畅想：数亿AI agent在线，每秒万级交易 → DeFi套利、数据市场、代码协作、自动化运营等新物种涌现。

合规笔记

大陆合规本质是“有规可合 + 风险封控”。法律滞后但执法态度明确，三大红线（负面清单）仍是主轴：  ICO公开融资（非法集资/诈骗）  

虚拟货币非法买卖外汇（非法经营）  

变相传销/资金盘

单纯发币（无公开募资）目前不直接入罪，但几乎所有后续玩法都极易触雷。最高刑事风险业务（目前办案主流）  永续合约 / 期权 / 预测市场 → 绝大多数被定性为开设赌场（公安最常用路径）  

境内法币USDT/USDC兑换（U3/优三/出金入金）→ 非法经营罪（非法买卖外汇）  

面向不特定对象募资 → 非法吸收公众存款 / 集资诈骗  

境内团队/个人为海外合约平台引流、推广、KOL带单 → 开设赌场帮助犯 / 共犯 / 帮信罪

当下最常见的个人雷区  出入金冻卡（已占咨询量70%+）  

被路径攻击（洗钱团伙反向利用合规通道送“脏钱”）  

莫名收到标记资产 → 秒变洗钱链条一环  

使用来源不明的U卡 → 资金丢失 + 卷款跑路双重风险

中国特色“深度合规”现实  文本合规（合同、流程、KYC文件）只是入门券  

真正能挡子弹的，往往是与地方建立深度协作关系（部分头部项目已做到异地公安难跨境抓捕）  

没有这种“地方默契”，合规文件再漂亮也只是“谈判筹码”，而非免死金牌

给转行/新人最硬建议  境内彻底远离合约期权、公开募资、境内兑换相关岗位与项目  

境外雇主必须真正不面向大陆展业（无VPN可注册、可下载App = 极高风险）  

出金首选香港持牌合规路径（接受税务主动申报）  

任何异常资金入账 → 第一时间留痕、自保优先（必要时主动报案留底）

1\. 出入金合规路径：没有“绝对安全”产品，动态+本地化是关键邓律师直言：完全无风险的出入金通道不存在（否则就是顶级投资机会）。支付行业千年反洗钱难题，在加密领域更难。

推荐优先级（结合个人实际情况排序）：香港持牌合规路径（最安全选项）：不在意税务成本的情况下首选。接受主动申报个税/资本利得税。香港路径已成主流，但回大陆后可能被税局要求补税。

合规U卡（小额日常使用）：找正规发卡方，避免来源不明或多签钱包跑路风险。

老乡/亲友OTC：律师本人福建背景，靠本地资源最稳；内蒙/北京用户需结合自身“人脉圈”评估。强调：千万别误解为推荐“黑OTC”——正规熟人兑换仍是主流，但需留痕（聊天记录、转账备注）。

核心原则：每个人生活场景不同（地域、资金规模、频率），本地资源+合规通道组合最实用。避免频繁大额不明来源兑换。

2\. 账户冻结/异常入账应对：环境已明显改善，但仍需主动+专业2025-2026年趋势：比前几年文明很多。南方公安多电话+微信沟通，要求提供材料；北方部分仍强势要求到场，但直接上门抓人已少见（出差成本高、需成绩）。

收到异常/赃款典型场景：当天多地公安同时联系（案例：300万一天6地）。

反应决定画像：越像“正经生意人”（着急生意受影响），越容易被认定不知情。

第一次冻结：如实说明+提供交易记录（来源、背景、对手KYC），基本做笔录就解。无需过度紧张。

多次/已留痕：明知推定风险大（被认定“以兑换维生”），先找律师聊再去。多次易被定性为“明知”，量刑不利。

学生/自由职业者（无实体生意背书）：最典型痛点（如大湾区学生香港兼职兑换被抓）。应对更被动，建议：主动留痕（交易备注、聊天记录）。

异常资金第一时间自查链上来源（OKLink等），必要时主动报案留底（证明不知情）。

3\. 退赔谈判核心：无立案前提下博弈空间大律师实操案例：材料扎实+多次沟通后，公安认定不知情，但被害人闹信访→当事人主动退钱换快速解封微信/支付宝/银行卡。

策略：未立案：退钱换解封关联账户（目标是尽快恢复正常支付）。

能忍冻结半年-1年：可硬扛（你是受害人，正常生意收款）。

关键：证明“不知情、上游黑钱非我引入”（交易记录+背景说明）。

底线：别一次性全退，先谈条件（公安帮忙点解封）。

1\. 安全第一原则

私钥绝对不能截图、完整复制、全文发送或存储在易被监控的地方。剪贴板易被恶意软件窃取，建议分段复制+手动输入。涉及真金白银时，安全意识必须极度严格（曾有学员因授权泄露被黑客快速转走奖金）。

2\. 个人主权与无许可特性

任何人可随机生成私钥 → 立即拥有一个无需KYC、无需审批的钱包地址与资产控制权。这就是“个人主权”的密码学起点。与传统银行（需身份证、柜台、可能被拒、跨境受限）形成鲜明对比。

3\. 助记词 vs 私钥  助记词（seed phrase） → 一对多，可派生无数个私钥  

每个私钥 → 一对一，对应唯一地址

派生规则由确定性加密算法保证（BIP-39 / BIP-32等）。

4\. 交易（Transaction / TX）核心组成  要做的事（转账、调用合约等）  

Gas费（网络资源付费 + 激励机制）  

Nonce（防重放序号）

钱包（如MetaMask、Rabbit）负责：组装交易 → 用私钥签名 → 广播至网络。

5\. 数字签名本质

用私钥签名交易信息 → 任何人用对应公钥（地址）可验证签名有效性，但无法伪造签名（除非拥有私钥）。

核心安全：不暴露私钥的前提下证明“我就是地址拥有者”。6. Gas机制类比  Gas Price ≈ 油价  

Gas Used ≈ 路程耗油量  

总费用 = Gas Price × Gas Used

复杂操作 / 网络拥堵 → Gas价格飙升。防止垃圾交易、激励验证者。

7\. 区块确认与最终性

交易广播 → 进入内存池 → 验证者挑选（优先高Gas费） → 打包进区块 → 共识确认。

以太坊PoS下：  出块 ≈ 12秒  

需等待约12–13分钟（多区块确认）达到“Finalized”状态，才认为几乎不可篡改。

8\. 去中心化程度关键点  节点地理分散（美国、德国、中国等）  

客户端多样性（Geth、Nethermind、Besu、Erigon等）  

RPC仍存一定中心化风险（可自选或研究去中心化RPC方案）

一句话总结运行原理

私钥签名证明身份 → 共识网络确保账本可信 → 智能合约自动执行规则。

# Web3 入门学习笔记 | 从钱包到区块链运行原理

> 📅 学习日期：2026年1月 🎓 课程来源：Web3训练营

* * *

## 前言

最近参加了一个Web3训练营，老师从最基础的钱包创建讲到区块链底层运行原理，整个课程信息量很大，脑子确实有点"爆炸"的感觉。但正如老师说的，这些内容会帮助我们构建一个完整的知识大图。趁着记忆还新鲜，把学到的核心内容整理成笔记，也方便以后回顾。

* * *

## 一、钱包与私钥：你的数字身份起点

### 核心概念

钱包的本质是**密钥对管理工具**，包含：

-   **私钥（Private Key）**：绝对保密，相当于你的"万能钥匙"
    
-   **公钥/钱包地址**：可以公开分享，相当于你的"收款账号"
    
-   **助记词（Seed Phrase）**：私钥的人类可读形式，一组12-24个单词
    

### 重要关系

```
助记词 → 私钥 → 公钥/钱包地址
（一对多）  （一对一）
```

助记词可以派生出无数个私钥（一对多关系），而每个私钥对应唯一的钱包地址（一对一关系）。

### ⚠️ 安全警示

这是老师反复强调的重点，之前有训练营学员因为私钥/助记词泄露，刚获得的奖金就被黑客转走了：

-   **绝不截图**保存私钥或助记词
    
-   **绝不发送**给任何人
    
-   **复制时要小心**——粘贴板容易被监控，建议复制一部分+手打一部分
    
-   涉及真金白银时，安全意识要拉满
    

### 社会学意义

Web3钱包代表着**个人主权的起点**——任何人都可以无需许可地创建账号和拥有资产。对比传统银行开户需要身份证、审批、可能被拒绝，而且还有地域限制，加密钱包提供了一种全新的金融自主权。

* * *

## 二、交易与签名：区块链上的"转账"是怎么工作的

### 交易（Transaction/TX）的组成

一笔交易基本包含：

1.  **你要做的事情**：转账、调用合约方法、投票等
    
2.  **手续费（Gas）**：为网络资源付费
    
3.  **防重放序号（Nonce）**：每笔交易递增，防止被重复执行
    

### 钱包的作用

钱包（如MetaMask、Rabby）做的事情：

1.  组装交易内容
    
2.  用私钥签名
    
3.  广播到区块链网络
    

### 数字签名的原理

这是一个很精妙的密码学设计：

-   我用**私钥**对交易信息签名
    
-   任何人都可以用**公钥**验证这个签名是否由我发出
    
-   但没有私钥，任何人都无法伪造签名
    

验证流程：`原始消息 + 声称的发送地址 + 签名` → 密码学算法 → 验证签署者身份

这意味着：**只要知道你的私钥，就能以你的身份发送任何交易**，所以私钥安全至关重要。

* * *

## 三、Gas费用：区块链的"汽油费"

### 什么是Gas

可以把Gas理解为汽油费的概念：

-   **Gas Price**：当时的"油价"，由网络拥堵程度决定
    
-   **Gas Used**：完成这笔交易消耗的"汽油量"
    
-   **总费用** = Gas Price × Gas Used
    

### 类比理解

从上海开车到北京：

-   路程固定（Gas Used基本固定）
    
-   油价波动（Gas Price随网络繁忙程度变化）
    
-   总油费 = 路程 × 油价
    

### Gas的作用

-   防止垃圾交易淹没网络
    
-   为网络维护者提供激励
    
-   构成区块链的经济闭环
    

* * *

## 四、区块链网络运行原理

### 交易的完整生命周期

```
钱包签名 → RPC/节点广播 → 进入交易池排队 → 验证者打包成区块 → 共识确认 → 上链完成
```

### 区块的结构

每个区块包含：

-   区块编号（Block Number）
    
-   若干笔交易（如312笔）
    
-   父区块哈希（Parent Hash）→ 指向上一个区块
    
-   当前区块哈希
    
-   验证者投票信息
    

### 链式结构的安全性

每个区块都通过Parent Hash指向上一个区块，形成一条"链"。要篡改某个区块，必须同时修改它之后的所有区块——这在计算上几乎不可能。

### 最终确认（Finality）

刚出的区块还不够安全！以太坊需要约12-13分钟才能达到Finalized状态。在此之前，交易理论上可能被回滚。

**实际应用**：如果你是电商卖家，收到加密货币支付后不能立即发货，需要等待交易最终确认。

## 五、共识机制：去中心化网络如何达成一致

### PoW（工作量证明）- 比特币使用

类比：大家比赛解一道超难的题，谁先解出来谁就能记账并获得奖励。

-   优点：安全性经过长期验证
    
-   缺点：非常耗能，算力竞争激烈
    

### PoS（权益证明）- 以太坊使用

类比：交押金（质押32 ETH）加入"记账员"名单，随机被选中出块并获得奖励；如果作恶，押金被没收。

-   优点：节能环保
    
-   缺点：可能导致"富者越富"
    

* * *

## 六、去中心化程度分析

### 钱包创建 ✅ 高度去中心化

任何人都可以随机生成钱包，无需KYC或许可

### 交易广播 ⚠️ 相对中心化

依赖RPC服务商（如Infura、Alchemy），存在单点风险。以太坊基金会正在研究去中心化RPC方案

### 网络运行 ✅ 相对去中心化

-   全球约13,700+个节点
    
-   约97万个验证者（Validator）
    
-   多种客户端软件（Geth、Prysm等），避免单点故障
    

### 为什么多样性很重要

如果所有节点都运行同一个软件，一旦该软件出现Bug，整个网络可能分叉或宕机。多种独立开发的客户端软件分散了这种风险。

* * *

## 七、智能合约：链上的可执行代码

### 什么是智能合约

把代码部署到区块链上，代码逻辑不可篡改、可追溯、自动执行。

> "Code is Law" —— 代码即法律

### 实例：NFT Mint逻辑

```solidity
// 简化示例
function mint() {
    require(!hasMinted[msg.sender], "已经Mint过了");
    hasMinted[msg.sender] = true;
    // 铸造NFT...
}
```

通过合约代码限制每个地址只能Mint一次。

### 开源的重要性

涉及资产的合约通常应该开源，方便审计。如果一个项目的合约不开源，可能是"心虚"的表现。养成习惯：交互前先看合约是否开源。

* * *

## 八、以太坊升级机制：EIP流程

### EIP（Ethereum Improvement Proposals）

以太坊的改进通过社区协作推进，不是某个公司单方面决定：

```
论坛讨论 → 形成EIP草案 → 社区审议 → Last Call → Final → 客户端升级
```

每年大约1-2次大版本升级，每次升级都是一次"硬分叉"。

### 最近的升级

2025年的Fusaka升级包含了多个EIP，如调整交易Gas Limit等。

* * *

## 九、Web3的核心价值总结

| 特性 | 说明 |
| --- | --- |
| 无许可 | 任何人都可以读写区块链，无需审批 |
| 抗审查 | 节点全球分布，钱包/RPC可替换 |
| 开放开源 | 客户端代码、交易记录全部公开 |
| 去中心化 | 没有单一控制点 |
| 隐私 | 地址与真实身份分离（但链上记录公开） |

## 十、值得思考的问题

老师最后留了几个开放性问题，我觉得很有价值：

1.  **资产托管**：如何提高安全性同时降低管理复杂度？对于不熟悉技术的人（如老年人），如何管理私钥？
    
2.  **公共物品**：没有税收的去中心化世界，谁来维护节点、开发客户端、提供教育？资金如何分配？
    
3.  **有害信息治理**：抗审查对正常用户有利，但对黑客也一样。如何在保护隐私的同时治理恶意行为？
    
4.  **去中心化协作**：在互不认识、互不信任的环境下，如何实现公平可信的分配，让合作持续下去？

# 今日学习笔记：初探 Web3 和 NFT 的世界

今天花了差不多一个小时看完了《我的第一个 NFT》这个教程，感觉对区块链和 NFT 这些之前一直听说但没搞懂的概念，终于有了一个比较清晰的认识。

* * *

**区块链原来是这么回事**

之前总觉得区块链很神秘，今天才知道它的核心概念其实挺直观的——就是把交易记录打包成一个个"区块"，然后像串珠子一样串起来。每个区块里还包含了上一个区块的摘要信息，所以如果有人想篡改历史记录，就得把后面所有的区块都改掉，这基本上是不可能的。

更妙的是分布式记账这个设计。全球有无数个节点都在记录同一份账本，除非你能控制超过一半的节点，不然你改的东西根本不会被承认。突然觉得这个设计真的挺聪明的。

**关于 NFT 的理解**

NFT 和普通代币的区别，教程里用的比喻很好懂：普通代币像钱，一块钱和另一块钱没区别，可以随便换；但 NFT 就像画作或者收藏品，每一个都是独一无二的。

让我印象比较深的是 NFT 的"完全控制权"这个特性——你真正拥有它，不像游戏里的装备，账号被封了就什么都没了。不过教程也提醒说，国内很多所谓的"数字藏品"其实不算真正的 NFT，因为不在公链上，还是受平台控制的。这点之前完全没意识到。

**几个实用的收获**

钱包那部分挺实用的。助记词是最重要的东西，丢了就真的找不回来了；钱包地址可以公开，但私钥绝对不能告诉任何人。感觉跟银行卡号和密码的关系差不多，但后果更严重，因为被盗了基本没法追回。

还学了几个 Web3 的"黑话"：DYOR（自己做研究）、FOMO（怕错过机会）、FUD（恐惧不确定）。以后看到这些缩写至少不会一脸懵了。

**一点个人感想**

说实话，看完之后既觉得有意思，又有点担心。有意思的是这套去中心化的设计确实解决了一些现实问题，比如跨国转账、数字资产的真正所有权这些；担心的是安全问题，教程里列举的那些被骗案例看得我挺紧张的，什么钓鱼网站、假客服、恶意授权之类的，感觉这个圈子的坑还挺多。

教程最后说的那句话挺实在的：在决定买任何 NFT 之前，先问问自己"愿不愿意随便给一个陌生人送钱"，如果不愿意，就好好做功课。这个心态挺重要的，不能光看别人说能赚钱就冲进去。

总的来说，今天算是入门了，对这个领域有了基本的认知框架。接下来可能会试着装一下 MetaMask 玩玩看，但暂时不打算投真金白银进去，先观察观察再说。

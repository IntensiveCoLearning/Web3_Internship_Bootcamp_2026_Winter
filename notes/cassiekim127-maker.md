# web3合规要求及法律风险

## 一、 中国监管基调与核心政策

1.  **监管总基调**
    
    -   全面封堵虚拟货币金融属性，有限容忍区块链技术创新。
        
    -   Web3 技术本身不违法，触及 Token 发行、交易、矿池运营等场景易涉刑。
        
2.  **核心限制政策（2017 年起）**
    
    -   禁止 ICO、IEO、IDO 等代币融资行为。
        
    -   禁止境内虚拟货币交易所运营，禁止境外交易所向境内居民提供服务。
        
    -   明确虚拟货币**不具有法偿性**，不得作为支付工具。
        
    -   严厉打击以虚拟货币为载体的赌博、传销、洗钱、非法集资等犯罪。
        

## 二、 核心法律风险梳理

### 1\. 代币发行与交易风险

-   任何具备融资功能或可流通性的代币发行，无论命名为 “积分”“治理 Token”，均属非法金融行为。
    
-   技术人员参与代币模型设计、合约部署，会被认定为共同行为人，不能以 “仅写代码” 免责。
    
-   空投、白名单分发代币若实质面向公众筹资，可能构成**非法吸收公众存款罪**。
    

### 2\. 刑事风险（赌博、传销、洗钱等）

| 罪名 | 风险行为特征 |
| --- | --- |
| 开设赌场罪 | 链游设置 “充值 - 抽奖 - 提现” 闭环，以虚拟货币为筹码，具备 “以小博大” 概率机制 |
| 组织、领导传销活动罪 | NFT、DAO、挖矿项目设置 “邀请返利”“多级推广”“团队计酬” 模式 |
| 洗钱罪 | 通过虚拟货币转换、跨境转移犯罪所得，未落实 KYC/AML 措施 |
| 非法经营罪 | 利用虚拟货币开展跨境换汇、资金支付结算业务，未经监管许可 |

### 3\. 民商事争议风险

-   虚拟货币买卖、委托代投合同**大概率被认定无效**，法院通常不予支持，风险自担。
    
-   委托代投若出现资金亏损、去向不明，受托方可能面临民事赔偿，甚至涉非法经营 / 非法集资刑责。
    

## 三、 全球监管趋势

1.  **主要地区监管框架**
    
    -   **欧盟 MiCA**：全球首个全面加密监管框架，要求稳定币发行商持牌，禁止算法稳定币。
        
    -   **美国**：SEC、CFTC 分工监管，推进稳定币立法，允许银行提供加密托管服务。
        
    -   **香港**：推行 VASP 牌照制度，2024-2025 年落地稳定币监管框架。
        
2.  **FATF 核心要求**
    
    -   虚拟资产服务提供商（VASP）需遵守 **Travel Rule**，转账超 1000 美元 / 欧元需收集收发双方信息。
        
    -   DeFi 协议、去中心化交易所面临合规挑战。
        
3.  **监管趋势总结**
    
    -   趋严化：强化稳定币、交易所、DeFi 监管，对标传统金融合规标准。
        
    -   协调化：FATF、G20 推动全球监管协同，跨境合作加强。
        
    -   合规成本上升：企业需配置专业法务团队，业务模式需适配监管要求。
        

## 四、 入职 Web3 行业风险防范指南

### 1\. 雇佣关系风险

-   境外项目无境内用工主体，无法签订有效劳动合同、缴纳五险一金，薪资争议、工伤维权困难。
    
-   EOR 名义雇主模式可部分解决合规问题，但纠纷时劳动关系认定仍是难点。
    
-   入职前需评估：是否接受灵活用工、无社保公积金的安排，建议咨询专业律师。
    

### 2\. 薪酬结构风险

-   法律规定工资应以人民币支付，**Token/USDT 发薪可能被认定无效**。
    
-   自发 Token 薪资存在价值归零风险，且会拉低社保、经济补偿计算基数。
    

### 3\. 虚拟货币出金风险

-   出金优先选择可信渠道，避免参与灰色交易。
    
-   留存资金合法来源记录，防止因交易对手资金涉黑产导致银行卡冻结、涉刑。
    

### 4\. 项目合法性审查

-   入职前查阅项目白皮书、Token 分发机制、收益模型。
    
-   重点关注：是否面向境内用户、是否承诺保本收益、是否含多级返利结构。
    

## 五、 其他风险提示

1.  **虚拟货币属性定性**：我国不认可其货币属性，但保护其财产价值；相关业务活动均属非法金融活动。
    
2.  **挖矿风险**：虚拟货币挖矿属于淘汰类产业，被全面禁止。
    
3.  **立法趋势**：2025 年中央政法工作会议提出研究虚拟货币立法建议，行业将逐步规范化。

# 以太坊基础了解

## 基本介绍

以太坊是一个开源的去中心化的区块链平台，**以太币是其原生加密货币**，以太坊是通过以太币提供去中心化的以太虚拟机（EVM）处理点对点合约。以太坊的核心创新在于 **智能合约**（Smart Contracts），智能合约是存储在区块链上的可执行代码，能够在满足预设条件时自动执行操作，无需人工干预。

## Ethereum与Bitcoin的主要差异

|   | 比特币 | 以太坊 |
| --- | --- | --- |
| 目标与定位 | 去中心化的数字货币，强调安全、稳定和稀缺性（总量 2100 万枚） | 去中心化平台，支持智能合约和 Dapps，定位为“区块链 2.0” |
| 共识机制 | 工作量证明（PoW） | 从 PoW 转向权益证明（PoS） |
| 交易速度 | 速度慢，每 10 分钟生成一个区块 | 速度快，区块时间约 12 秒 |

-   PoW工作量证明
    

矿工通过计算机算力“挖矿”，第一个得出哈希值的矿工可以确认区块并获得奖励。但矿工需要消耗大量电力资源，竞争激烈。

-   PoS权益证明
    

是通过质押加密货币来保护网络的系统，质押32个ETH可成为验证者，系统随机选择验证者来提议和验证区块，验证者获得新发行的 ETH + 交易费用，作恶者质押的 ETH 被销毁。无需大量电力和硬件，大大增加了攻击成本。

## L1和L2

-   **Layer1主网**
    

L1是以太坊的主干道，是核心区块链，负责最终安全性与共识，它非常安全、去中心化，但有个大问题就是堵车严重，且gas费贵，为了解决这一问题衍生出了L2.

-   **Layer2（二层扩展解决方案）**
    

L2相当于在主干道旁边修建的辅路或高架桥，与主路紧密相连以确保安全性，将大部分交易转移到辅路上进行，以提升速度。L2主要技术：

-   **Rollup：**通过将交易批量处理后提交至 L1，降低 Gas 费。就像是所有交易上了一辆大巴车，司机将所有人的目的地（交易数据）压缩成一张票根，大巴车定期将这张票根开回主路教给L1存档。
    
-   ZK Rollup：通过零知识证明验证交易，无需链上争议。
    
-   Optimistic Rollup：假设交易合法，仅在争议时验证。
    

# web3行业赛道

### DeFi：去中心化金融

-   Uniswap：去中心化交易所（DEX）
    
-   Compound：去中心化借贷协议。比如，如果你存入 1 个 ETH，系统会给予你相应的 cETH 代币。用户在借入资产时，必须提供超额的担保，如果市场波动导致抵押资产的价值低于借款要求，Compound 会触发强制清算，即强制卖出部分抵押资产来偿还贷款。
    
-   MakerDAO：稳定币系统。MakerDAO 是一个去中心化的借贷协议，它允许用户通过超额抵押资产来生成DAI，一种与美元挂钩的稳定币。DAI 的价值稳定在 1 美元左右，因此可以作为一个避险工具或支付手段。
    

### NFT：数字所有权革命

-   NFT的本质是数字资产的唯一性和所有权，每个NFT代表一个唯一的数字资产，并在区块链上有不可篡改的记录。
    
-   智能合约：自动化的所有权转移和交易。例如，在NFT 交易中，智能合约可以设定当你转售该 NFT 时，原作者会自动获得一定比例的转售收入（这被称为“版税”）。两大案例：CryptoPunks、OpenSea
    

### DAO：去中心化组织

和一群志同道合的朋友，不用任何老板，也不需要复杂的公司结构，通过投票、分享资源，共同决定一个组织的未来——这就是 DAO 的核心理念。

### MEME

MEME通常指以网络文化为基础的代币，特点是有趣、搞怪、社区驱动，代表有DOGE（狗狗币）、PEPE（青蛙币）、SHIB（柴犬币）

# 学习笔记

## 区块链基础概念

-   **区块链是什么**
    

区块链是一种去中心化的分布式账本技术，每条链是由一系列按照时间顺序相连的“区块”组成，每个区块内部包含了多笔交易数据及元数据，确保了数据记录的完整性与可追溯性。分布式网络记账确保了历史交易记录的可信性。

区块链是由链和区块组成：

区块：

1.  每一个区块包括两个部分：交易记录等一些信息, 和之前的区块摘要（哈希）。
    
2.  每一个区块只有有限的存储容量。
    
3.  每个区块会在一定时间内打包，例如大概 10 分钟打包生成一个。
    

链：

1.  一条链只可以链接两个区块。
    
2.  区块按照顺序依次串联。
    

区块链特性

1.  不可篡改
    
2.  公开透明、匿名
    
3.  快速交易
    
4.  去中心化
    

-   **比特币**
    

比特币最明显的属性就是货币属性。不同的网络服务提供商可以得到不同的代币奖励，比如：比特币。

-   **区块链核心组成部分**
    

1.  去中心化的网络和区块链
    
2.  维持网络运行的代币激励：去中心化的网络由无数节点提供服务来维持网络运行，这些操作统称为挖矿。维持这些服务的人一般称之为矿工。矿工们维持网络运行需要奖励，就像你工作需要工资，在区块链中对矿工的奖励一般是代币，你经常听说的燃料费（Gas Fee）就是矿工们的“工资”。
    

-   **一条链的运行**
    

用户发起交易→交易广播→节点验证→打包成块→链接上链→奖励发放

-   **公联、私链、联盟链**
    

| 区块链类型 | 节点加入方式 | 数据可见性 | 管理模式 |
| --- | --- | --- | --- |
| 公链 | 任何人自由加入 | 所有人可见 | 去中心化 |
| 私链 | 由老板严格审批 | 仅内部成员可见 | 中心化 |
| 联盟链 | 需联盟成员邀请/审批 | 仅联盟成员可见 | 多中心化 |

-   **web2、web3.0、web3**
    

1.  web2（当前互联网），如：阿里、腾讯，由平台垄断，中心化
    
2.  web3.0（语义网），不是区块链技术；主要不依赖AI，而是通过标准化数据格式实现
    
3.  web3（去中心化互联网）
    

数据主权归用户：用区块链存储身份和资产

无需中介：智能合约自动执行规则

典型应用：MetaMask、Uniswap、ENS

-   **注意**
    

1.  **Web3 ≠ Web 3.0**
    

-   Web3 是区块链驱动的革命
    
-   Web 3.0 是语义网技术驱动的数据组织升级
    

-   **去中心化的优势和挑战**
    

1、优势

①降低信任成本

②提升了系统的安全性和可用性

③用户自主管理

④区块链与智能合约构建了开放创新生态

2、挑战

①可扩展性瓶颈

②安全与治理难题

③用户体验与成本难以平衡

④存在法律与合规风险

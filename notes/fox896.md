# 2026-01-20

以下是 **Solidity（^0.8.x）** 的核心关键字分类讲解，搭配 **示例** 和 **安全注意事项**：

* * *

### **1\. 合约结构**

**合约/接口/库**

-   `contract`：定义合约。
    
    ```
    SOLIDITY
    ```
    
    ```solidity
    contract MyToken { ... }
    ```
    
-   `interface`：仅声明函数签名（无实现），用于交互。
    
    ```
    SOLIDITY
    ```
    
    ```solidity
    interface IERC20 {
        function transfer(address to, uint amount) external returns (bool);
    }
    ```
    
-   `library`：可重用代码（如 `SafeMath`），通过 `using for` 绑定到类型。
    
    ```
    SOLIDITY
    ```
    
    ```solidity
    library Math {
        function add(uint a, uint b) internal pure returns (uint) { return a + b; }
    }
    using Math for uint;
    ```
    
-   `is`：继承。
    
    ```
    SOLIDITY
    ```
    
    ```solidity
    contract Child is Parent { ... }
    ```
    
-   `abstract`：抽象合约（可包含未实现函数）。
    
    ```
    SOLIDITY
    ```
    
    ```solidity
    abstract contract Base {
        function mustImplement() public virtual returns (bool);
    }
    ```
    

* * *

### **2\. 函数修饰符**

**\*\*可见性**

-   `public`**/**`private`**/**`internal`**/**`external`：
    
    -   `public`：内部/外部均可访问（自动生成 getter）。
        
    -   `private`：仅合约内部。
        
    -   `internal`：合约及子合约（默认）。
        
    -   `external`：仅外部调用（如 `this.f()`）。
        
    
    ```
    SOLIDITY
    ```
    
    ```solidity
    uint private secret;
    function readSecret() public view returns (uint) { return secret; }
    ```
    

**\*\*状态修饰符**

-   `view`：不修改状态（不消耗 gas）。
    
-   `pure`：不读取/修改状态（如纯计算）。
    
    ```
    SOLIDITY
    ```
    
    ```solidity
    function addPure(uint a, uint b) public pure returns (uint) { return a + b; }
    function readBalance(address addr) public view returns (uint) {
        return addr.balance;
    }
    ```
    
-   `payable`：允许接收 ETH（⚠️ **风险**：重入攻击）。
    
    ```
    SOLIDITY
    ```
    
    ```solidity
    function deposit() public payable { balances[msg.sender] += msg.value; }
    ```
    

**\*\*特殊修饰符**

-   `virtual`：允许子合约重写。
    
-   `override`：重写父合约函数（必须匹配签名）。
    
    ```
    SOLIDITY
    ```
    
    ```solidity
    contract Parent {
        function foo() public virtual returns (uint) { return 1; }
    }
    contract Child is Parent {
        function foo() public override returns (uint) { return 2; }
    }
    ```
    
-   `modifier`：函数包装器（如权限检查）。
    
    ```
    SOLIDITY
    ```
    
    ```solidity
    modifier onlyOwner() {
        require(msg.sender == owner, "Not owner");
        _; // 继续执行函数体
    }
    function sensitiveAction() public onlyOwner { ... }
    ```
    

* * *

### **3\. 变量与数据位置**

**\*\*存储位置**

-   `storage`：合约永久存储（gas 高）。
    
-   `memory`：临时存储（函数调用期间）。
    
-   `calldata`：外部函数参数（不可修改，gas 低）。
    
    ```
    SOLIDITY
    ```
    
    ```solidity
    function processData(bytes calldata data) public {
        bytes memory copy = data; // calldata → memory
    }
    ```
    

**\*\*状态变量修饰符**

-   `constant`：编译时常量（如 `PI = 3`）。
    
-   `immutable`：部署时设置一次（gas 优化）。
    
    ```
    SOLIDITY
    ```
    
    ```solidity
    address public immutable owner;
    constructor() { owner = msg.sender; }
    ```
    

* * *

### **4\. 控制流**

-   `if`**/**`else`**/**`for`**/**`while`：与其他语言类似。
    
-   `revert`**/**`require`**/**`assert`：
    
    -   `require(condition, "error")`：输入验证（退还 gas）。
        
    -   `assert(condition)`：内部错误（如算术溢出）。
        
    -   `revert("error")`：手动回退。
        
    
    ```
    SOLIDITY
    ```
    
    ```solidity
    function transfer(address to, uint amount) public {
        require(amount <= balances[msg.sender], "Insufficient balance");
        balances[msg.sender] -= amount;
        balances[to] += amount;
    }
    ```
    

* * *

### **5\. 特殊关键字**

-   `emit`：触发事件。
    
    ```
    SOLIDITY
    ```
    
    ```solidity
    event Transfer(address from, address to, uint amount);
    emit Transfer(msg.sender, recipient, 100);
    ```
    
-   `try`**/**`catch`：外部调用错误处理（0.6.0+）。
    
    ```
    SOLIDITY
    ```
    
    ```solidity
    try otherContract.someFunction() returns (bool success) {
        // 成功逻辑
    } catch {
        // 失败逻辑
    }
    ```
    
-   `assembly`：内联 Yul/汇编（高级用途，⚠️ **风险**）。
    
    ```
    SOLIDITY
    ```
    
    ```solidity
    function add(uint a, uint b) public pure returns (uint) {
        assembly { result := add(a, b) }
    }
    ```
    
-   `this`：当前合约实例（用于外部调用）。
    
    ```
    SOLIDITY
    ```
    
    ```solidity
    this.someFunction(); // 通过外部调用触发 fallback/receive
    ```
    

* * *

### **6\. 安全注意事项**

1.  `delegatecall`：
    
    -   **危险**：保留上下文（如 `msg.sender`），但执行目标合约逻辑。
        
    -   **攻击向量**：可被恶意合约劫持执行流。
        
    
    ```
    SOLIDITY
    ```
    
    ```solidity
    (bool success, ) = target.delegatecall(abi.encodeWithSignature("malicious()"));
    ```
    
2.  `payable`：
    
    -   **风险**：结合 `transfer`/`sendValue` 易造成重入攻击。
        
    -   **解决方案**：使用 **Checks-Effects-Interactions** 模式或 `ReentrancyGuard`（OpenZeppelin）。
        
3.  `tx.origin`：
    
    -   **危险**：易受 **钓鱼攻击**（应使用 `msg.sender`）。
        
4.  `block.timestamp`：
    
    -   **风险**：矿工可轻微操纵（≈±15秒），避免用于随机数。

ERC-7962标准介绍：这是一个基于哈希的代币标准，利用零知识证明（ZK）技术来保护用户隐私。核心思想是通过哈希算法证明某些信息的真实性，但不泄露具体内容。

应用场景举例：Alex老师用星巴克会员和机场合作的例子说明：用户可以向机场证明自己是星巴克会员，但不需要泄露具体身份信息。星巴克可以部署合约并支付gas费用，用户体验类似Web2但交易在区块链上完成。

技术特点：

使用哈希算法实现简单的零知识证明

可以实现免gas费交易（由商家代付）

用户不需要持有传统钱包和私钥

保护用户隐私的同时保持交易透明性

主要问答：

关于隐私与透明的矛盾：通过数学方法保证隐私不被泄露

关于gas费：在商业场景中，有动力的一方（如星巴克）愿意支付

关于监管和审查：技术本身是为了解决现有隐私泄露问题，而非制造新问题

关于实现方式：相关代码已在GitHub和Magic论坛公开

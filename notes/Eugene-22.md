# 从零到一学习以太坊 - 第二章

## 以太坊节点

-   节点：运行以太坊客户端软件并连接到其他节点的电脑
    
-   客户端：对以太坊协议的具体实现
    
    -   执行客户端：负责链上的业务逻辑和状态
        
    -   共识客户端：负责权益证明 PoS 共识
        
    -   验证者客户端
        

以太坊节点 = 一台电脑 + 两个必备客户端 + （可选）一个验证者客户端：

-   执行客户端负责 “算交易、管状态、对外提供 RPC”
    
-   共识客户端负责 “跑 PoS、选区块、投票达成共识”
    
-   它们通过 Engine API 的 “专线电话”互相协作，整个节点再通过 P2P
    

## 以太坊节点的通信方式

### 核心组件

-   Web3 Library
    
    -   应用层与以太坊节点的通信链接，连接钱包、发交易、读状态、订阅事件
        
-   以太坊节点
    
    -   共识客户端
        
        -   共识层 PoS，提议区块、验证区块、投票、最终性
            
        -   决定哪个区块最终算数、链往哪条走
            
    -   执行客户端
        
        -   执行层EVM，执行交易/合约、更新账户状态、生成 receipts、计算 state root
            
        -   交易、执行、更新状态
            
-   Web3.eth 和 Web3.beacon
    
    -   Web3.eth：主要面向执行客户端，交易、合约调用、状态读取
        
    -   Web3.beacon：主要面向共识客户端，共识相关数据、Beacon 链信息
        
-   Engine API
    
    -   Merge 之后引入的一组内部 JSON-RPC
        
        -   共识客户端：给执行客户端下发区块提议/构建指令
            
        -   执行客户端：执行后返回结果
            
-   P2P
    
    -   共识客户端有自己的 P2P 网络，传播 beacon block / attestation 等
        
    -   执行客户端也有自己的 P2P 网络，传播交易、区块、同步数据
        
    -   节点靠协议把信息扩散到全网

# 从零到一学习以太坊

## 第一章：认识以太坊

### 以太坊的起源

-   时间：2013 年提出，2015 年正式上线主网
    

### 以太坊核心特征

-   核心组件
    
    -   以太坊虚拟机：能够执行智能合约代码
        
    -   ETH：支付交易费用和计算服务，即使资产也是燃料
        
-   特点
    
    -   智能合约：已经过渡到验证者（PoS）时代，验证者负责打包并执行合约，获取 Gas 作为手续费
        
    -   分布式应用程序 dApps
        
    -   代币 Tokens：供分布式应用程序使用
        
    -   权益证明（PoS）
        
    -   燃料费（Gas）
        
    -   Proto-Danksharding（EIP-4844）与 Dencun 升级
        
    -   分片与 Danksharding
        
    -   叔块（Uncle / Ommer Block）
        
-   协议层
    
    -   燃料费（Gas）
        
    -   Pos 机制质押保证金
        
    -   经济安全锚定
        
-   应用层
    
    -   记账单位
        
    -   流动性资产
        
    -   NFT
        

### ETH 的系统作用

-   以太币（ETH）是以太坊区块链上的原生数字资产
    
    -   支付 Gas 的燃料：Gas 费用会变化
        
    -   Pos 机制质押安全的保证金
        
    -   DeFi / NFT / Web3生态中抵押资产之一
        
        -   NFT（非同质化代币）， 每一枚 NFT 都代表某个特定、不可互换的资产，数字世界里的“所有权凭证”或“收藏证书”
            

### 如何理解**全球可编程区块链**

区块链、可编程、全球化

-   区块链
    
    -   去中心化
        
    -   不可篡改
        
    -   安全透明
        
-   可编程
    
    -   智能合约：部署在区块链上可以自动执行的代码
        
    -   图灵完备虚拟机（EVM），构建去中性化应用（dApps）
        
-   全球性
    
    -   由遍布全球的成千上万节点组成的全球共享计算机
        
    -   从技术视角看，以太坊维护的是一个全球共享的状态机（State Machine）
        

### 以太坊去中心化应用 dApps

-   基本概念
    
    -   dApp 的后端代码 （智能合约）运行在以太坊主网或其 Rollup 之上的 EVM 中，由全网节点共同执行和记录
        
    -   dApp 的前端可以用任意 Web / App 技术编写，既可以放在传统服务器上，也可以托管在 IPFS、Arweave 等去中心化存储上
        
-   技术特征
    
    -   运行的服务器去中性化
        
    -   确定性，同一份合约在任何节点上执行，给定同样的输入得到同样的输出
        
    -   图灵完备，可以实现任何的复杂逻辑计算
        
    -   隔离性
        
-   **dApp 开发的代价**
    
    -   维护困难
        
    -   性能开销与扩展难度
        
    -   网络拥堵与 Gas 成本
        
    -   用户体验门槛高

# Web3 实习手册 - 入门导读

## 区块链基础

### 区块链基础概念

-   去中心化的分布式账本技术
    
-   区块
    
    -   交易记录 + 之前的区块摘要（哈希）
        
-   链
    
    -   串联区块
        

### 如何确保历史记录不可篡改？

-   数据结构（哈希链）：篡改可检测
    
-   分布式网络共识机制：重写数据需要付出巨大成本
    

### 区块链的分类

根据访问权限与治理模式

-   公联
    
    -   自由进出成为节点
        
    -   行为透明化
        
-   联盟链
    
    -   受邀请或申请成为节点
        
    -   权限分为决策者（读写）和观察者（只读）
        
    -   行为半透明
        
-   私链
    
    -   成为节点审批严格，成员固定
        
    -   透明度低，但隐私性强
        

### Web3 核心组成

钱包 → Dapp → 智能合约 → 区块链

-   钱包（MetaMask 等）：管理私钥、发起签名、把 DApp 的请求发送到区块链网络以及确认交易。
    
-   Web 3.0 开发
    
    -   Python + RDFLib + SPARQL
        

## 以太坊概览

### 智能合约

-   存储在区块链上的可执行代码
    
    -   自动执行
        
    -   使用以太币（ETH）来支付相关费用
        

### **权益证明（PoS）机制**

-   质押 32 ETH 成为验证者
    
-   奖励机制、惩罚机制
    

### 架构组成

-   Layer 1，以太坊主网
    
-   Layer 2，二层扩展解决方案
    
-   侧链
    
-   以太坊生态分层架构
    

### 以太坊核心机制

-   以太坊账户分两类
    
    -   外部账户（EOA）：由私钥控制，用于发起交易
        
    -   合约账户（CA）：由合约代码控制，不能主动发起交易，只能被 EOA 调用触发执行
        
-   典型交互路径
    
    -   EOA ↔ EOA：转账
        
    -   EOA → CA：调用合约（如买 NFT、授权 Token、链上投票）
        

### Gas 模型

-   费用计算
    
    -   总费用 = Gas Used × Gas 单价
        
-   Gas 的作用
    
    -   经济激励：让验证者优先打包（给 tip 更容易被处理）
        
    -   防滥用：限制恶意消耗资源（Gas 用完则执行回滚）
        
-   EIP-1559 之后的费用结构
    
    -   Base Fee 基础费：协议自动调整，直接销毁
        
    -   Tip 小费：额外给验证者，用于加速确认
        

### 以太坊虚拟机 EVM

-   核心特点
    
    -   确定性执行：同样输入 → 全网得到同样结果，保证可验证
        
    -   隔离安全：合约运行在沙盒环境，不能随意访问外部系统
        

### 一般流程

-   钱包（EOA）发起交易 → 签名
    
-   交易进入网络 → 验证者打包
    
-   节点在 EVM 执行合约 → 消耗 Gas → 更新状态
    
-   结果写入区块链，可追溯记录

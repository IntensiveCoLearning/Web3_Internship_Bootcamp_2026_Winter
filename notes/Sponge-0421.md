# 2026-01-15

**Agentic AI × Web3：从聊天机器人到自主经济体**

**一、背景与趋势：AI 正在从“对话”走向“行动”**

1.  技术演进路径  
    
    -   2023：LLM 时代  
        本质：预测下一个 Token
        
    -   特点：被动响应、无状态、不可持续执行任务
        
    -   2026（趋势）：Agentic AI 时代  
        
    -   特点：具备推理、记忆、工具调用能力
        
    -   能自主执行复杂任务并参与经济活动
        

2.权威预测  

-   Gartner：到 2028 年，33% 的企业软件将嵌入 Agentic AI
    
-   法律界共识：2025 年是“AI 问责之年”，合规与责任成为焦点
    

**二、什么是 Agentic AI（代理式人工智能）**

**1\. 定义**

Agentic AI 是能够自主决策、自主执行任务、并持续迭代行为的人工智能系统。

**2\. 核心能力**

-   推理（Reasoning）
    
-   记忆（Memory）
    
-   工具使用（Tool Use）
    
-   自主执行（Autonomous Action）
    

**三、AI 智能体的解剖学结构（Anatomy）**

**1\. 大脑（Brain）**

由大语言模型（LLM）驱动

-   负责：  
    
    -   规划任务
        
    -   推理决策
        

-   示例模型：GPT-5.x、DeepSeek、Claude
    

**2\. 记忆（Memory）**

-   向量数据库（Vector DB）
    
-   RAG 流程：  
    
    -   数据抓取 → 向量化 → 上下文存储 → 检索调用
        

-   解决问题：  
    
    -   LLM 无状态、不可长期记忆的问题
        

**3\. 工具（Tools）**

-   API 接口
    
-   功能：  
    
    -   执行代码
        
    -   查询数据库
        
    -   发起交易或指令
        

关键洞察：

传统 LLM 是“无身份、无记忆、无执行权”的；

Agentic AI 通过工具 + 记忆 + API，获得“持续存在的执行身份”。

**四、Agentic AI 面临的核心风险**

**1\. 身份欺诈（Identity Spoofing）**

-   无法确认操作者是人类还是机器人
    
-   AI 可能被伪装执行未授权操作
    

**2\. 不可审计（Untraceability）**

-   缺乏不可篡改日志
    
-   AI 若签署错误合同，责任难以追溯
    

**3\. 影子 AI（Shadow AI）**

-   企业内部大量 Agentic AI 未被统一治理
    
-   安全与合规风险显著
    

➡️ 本质问题：

Agentic AI 缺乏“可信身份 + 可审计行为”的基础设施。

**五、机器经济的到来与支付问题**

**1\. 现实变化**

-   2025 年：约 70% 的稳定币转账由机器人或自动代理完成
    

**2\. 旧支付模式的局限**

-   信用卡订阅制（如 $20/月）
    
-   人工 KYC
    
-   不适合高频、低额、机器对机器交易
    

**3\. 新需求**

-   M2M（机器对机器）微支付
    
-   按 API / 按推理次数付费
    
-   毫秒级结算
    

**六、Web3 AI 协议栈：三层结构**

**1\. 身份层（Identity）**

ERC-8004：智能体链上护照

-   基于 ERC-721 的链上身份
    
-   包含：  
    
    -   唯一 ID
        
    -   可验证身份
        
    -   声誉评分（0–100）
        

-   验证方式：  
    
    -   TEE
        
    -   zkML**2\. 交易层（Transaction）**
        

x402 协议：HTTP 原生机器支付

-   利用 HTTP 402 状态码
    
-   支持：  
    
    -   按文件付费
        
    -   按算力 / 推理付费
        

-   特点：  
    
    -   无订阅
        
    -   自动签名
        
    -   稳定币（如 USDC）结算
        

**3\. 应用层（Application）**

SpoonOS Framework

-   Web3 AI 的“操作系统”
    
-   功能：  
    
    -   集成身份验证
        
    -   内置支付协议
        
    -   智能体编排
        

-   连接 Web2 AI 开发工具与 Web3 基础设施
    

**七、安全与合规：从“黑盒 AI”到“透明审计”**

1.  合规要求  
    
    -   全生命周期行为记录
        
    -   推理、交易、验证可追溯
        
    -   符合 EU AI Act 等高风险系统要求
        

2.Web3 的优势  

-   不可篡改审计日志
    
-   链上永久存证
    
-   原生满足监管需求
    

**八、SpoonOS 架构概览（逻辑理解）**

1.  输入层：用户或外部请求
    
2.  核心层：  
    
    -   向量数据库
        
    -   LLM API
        
    -   智能体编排引擎
        
3.  协议层：  
    
    -   ERC-8004（身份）
        
    -   x402（支付）
        
4.  输出层：  
    
    -   链上交易（DeFi）
        
    -   Web API 操作
        
    -   自动执行任务
        

**九、典型应用场景**

DeFi 交易智能体  

-   自动套利
    
-   可验证历史业绩
    
-   链上声誉背书
    

数据市场代理  

-   自动抓取高质量数据
    
-   通过微支付向其他 AI 出售数据
    
-   自主服务代理  
    自动修复代码
    
-   每次 PR 修复自动收费
    

**十结论**

AI 正在从“工具”演变为“经济主体”

Agentic AI 的规模化必须依赖：  

-   可验证身份（ERC-8004）
    
-   原生支付（x402）
    
-   操作系统级框架（SpoonOS）
    

Web3 是目前唯一能支撑高频、低额、机器交易的基础设施

未来经济主体不再只有人类，而是数以亿计拥有钱包的 AI 智能体

**Web3 安全与风险趋势学习笔记**

**主题：Web3 安全态势、典型攻击方式与应对策略**

一、**2025 年 Web3 安全态势总览**

1\. **整体损失规模巨大**

2025 年 Web3 领域共发生 **1200+ 起较严重安全事件**，累计损失金额 **超过 35 亿美元**，安全形势整体不容乐观  。

2\. **攻击趋势呈“双轨并行”**

• **精准猎杀（Targeted Attacks）**：针对头部协议、重点项目和高净值账户发动定向攻击，单笔损失巨大。

• **广撒网攻击（Mass Attacks）**：以数量取胜，通过钓鱼、私钥窃取、欺诈代币等方式获取收益  。

3\. **安全问题已成为行业系统性风险**

安全事件的频发对用户信心、项目可持续性及 Web3 行业整体发展构成严重威胁  。

二、主要攻击类型与高频风险点

1\. **私钥窃取（Private Key Theft）**

• 通过恶意链接、假钱包、木马程序等方式诱导用户泄露私钥

• 一旦私钥泄露，资产将被直接转移，几乎无法追回

2\. **钓鱼攻击（Phishing）**

• 利用假官网、假空投、假客服等方式诱导用户授权

• AI 技术的引入使钓鱼内容更加“真实可信”，识别难度显著上升  。

3\. **Rug Token / Rug Pull**

• 项目方或攻击者通过创建代币、制造热度后迅速撤资

• 呈现出**职业化、团队化、流程化**趋势，欺骗性更强  。

三、典型案例与攻击手法演变

1\. **攻击方式的技术融合趋势**

2025 年 Web3 攻击不再依赖单一漏洞，而是呈现以下特征：

• 新型协议特性滥用

• 供应链漏洞利用

• 自动化脚本与 AI 辅助攻击

• 多步骤、低成本、高隐蔽性组合攻击  。

2\. **攻击范式变化**

攻击者从“机会型攻击”转向“系统化攻击”，具备：

• 更高成功率

• 更低被追踪概率

• 更强的规模复制能力

四、**AI 时代下的 Web3 安全新挑战**

1\. **AI 放大攻击效率**

• 自动生成钓鱼文案与仿真页面

• 快速分析用户行为与弱点

• 降低攻击门槛，使非技术型攻击者也可参与

2\. **防御成本同步上升**

传统人工审核与经验判断难以应对 AI 驱动的攻击模式，安全防护需要向智能化、系统化升级  。

五、**GoPlus 提出的核心安全观点与建议**

1\. **安全是基础设施，而非事后补救**

Web3 安全不应被视为“成本”，而是决定行业能否走向大规模应用的**关键基础设施**  。

2\. **安全建设需前置化、系统化**

• 项目侧：安全审计、实时监测、风控系统

• 用户侧：风险识别、授权管理、私钥保护

• 行业侧：安全数据共享与协同防御

3\. **安全能力将成为 Web3 核心竞争力**

未来 Web3 项目的可信度、长期价值与安全能力高度绑定  。

六、结论

2025 年 Web3 安全形势严峻，攻击手段呈现智能化、自动化和规模化趋势。在 AI 技术加持下，传统防御模式面临挑战。安全已从“技术问题”升级为“行业发展前提”，只有将安全视为核心基础设施，Web3 才具备走向主流应用的可能性。

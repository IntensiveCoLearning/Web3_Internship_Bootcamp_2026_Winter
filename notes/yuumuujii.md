# Web3 公共物品

**什么是公共物品？**

根据经济学定义，公共物品具有两个核心特征：**“非排他性”** 和 **“非竞争性”**。

我们可以用“高速公路”这个类比来通俗地理解这两个特征：

-   **非排他性**：任何人都可以开车上高速公路，你无法轻易地阻止某个人使用它。
    
-   **非竞争性**：一个人在高速公路上行驶，并不会影响或减少另一个人行驶的权利和体验（在不考虑堵车的情况下）。
    

**Web3 中的公共物品示例**

在 Web3 生态中，这些“高速公路”通常是开源软件、协议和开发者工具。它们构成了整个生态的技术基石，为所有建设者提供了便利。

| 公共物品示例 | 为什么它很重要？ |
| Solidity 语言 | 智能合约开发的基础语言。 |
| 开源开发框架（如 Hardhat） | 帮助开发者更快、更方便地构建应用。 |
| 开源协议代码（如 Uniswap） | 确保透明度和安全性，让用户可以信任协议没有后门。同时，它也允许任何人在其之上自由地构建和集成，从而激发了无需许可的创新。 |
| 开发者文档和 API | 降低开发门槛，帮助开发者更快上手和集成。 |

这些公共物品构成了整个 Web3 生态的技术基石，保障了安全与开放。既然这些设施如此重要且免费，那么由谁来为它们的开发和维护买单呢？

\--------------------------------------------------------------------------------

### 一、 核心挑战：“公地悲剧”与市场失灵

“搭便车”问题（The Free-Rider Problem）

Web3 公共物品面临的核心困境，可以用经济学中的“公地悲剧”或“搭便车问题”来解释。开发者可以随意使用像 Solidity 或 Hardhat 这样的开源工具来构建自己的商业项目，而无需向这些工具的创造者支付任何费用。

这就导致了一个关键问题：如果每个人都能免费使用这些资源，那么谁还有动力来维护和改进它们呢？就像一片花园里的植物，如果人人都能免费欣赏和采摘，却没人愿意去浇水施肥，花园最终会变得荒芜。

市场失灵的后果

这种“搭便车”行为导致的直接后果是“市场失灵”。在公共物品领域，一个关键的洞察是：**高使用量并不等于高收入**。

尽管像 Hardhat 这样的项目可能有成千上万的开发者在使用，但这巨大的使用量并不能直接转化为可持续的收入来支持团队的持续开发。纯粹的市场规则无法为这些公共物品进行有效定价和资助，因为它们的设计初衷就是免费和开放的。

为了确保 Web3 生态系统的持续健康发展，社区必须探索新的资助机制来捕获和回馈这些公共物品创造的价值。接下来，我们将探讨 Web3 生态系统中为解决这一难题而涌现的三种主流资助模式。

\--------------------------------------------------------------------------------

### 二、 主流的 Web3 公共物品资助模式

2.1 委员会制：快速但集中的决策

**工作原理**

委员会制是最直接的一种资助模式。它效仿了现实世界中政府的运作方式：由一个专家委员会来审查提案，并决定如何分配金库中的资金。就像政府委员会决定“应该在哪里修路”一样，Web3 项目中的委员会决定应该支持哪个项目，以促进生态系统的发展。

**优缺点分析**

| 优点 (Pros) ✅ | 缺点 (Cons) ❌ |
| 简单快速 | 权力高度集中 |
| 决策效率高 | 个人偏好固化与偏见 |
|   | 成员知识有限，难以应对复杂多样的提案 |

2.2 二次方资助（Quadratic Funding, QF）：放大社区的声音

**核心理念**

二次方资助（QF）是一种更民主化的资助方式，它试图放大社区的集体智慧，而不是少数“巨鲸”（大额持有者）的意志。其背后的数学直觉可以用一句话来概括：

“100 个人为同一个项目投票，其影响力远大于 1 个人投出 100 票。”

这种机制的目标是鼓励更多普通用户参与，放大社区的真实共识，降低少数大户通过资本垄断决策的可能性。

**实践与挑战**

在以太坊生态中，**Gitcoin** 是最具代表性的二次方资助平台，一度因其资助了数千个公共物品项目而被誉为“以太坊的军火库”。然而，这种模式也面临着独特的挑战。

| 优点 (Pros) ✅ | 挑战 (Challenge) ⚠️ | 解决方案 (Solution) 💡 |
| 鼓励更多人参与 | 女巫攻击 (Sybil Attack)：一个人可以注册大量虚假账户进行投票，以骗取匹配资金。 | 推出 Gitcoin Passport 等去中心化身份验证机制，提高作恶门槛。 |
| 降低大户垄断效应 | 参与门槛提高：复杂的身份验证过程可能会降低部分普通用户的捐赠积极性。 | 生态系统仍在探索如何在确保安全性的同时，简化验证流程以找到最佳平衡点。 |

2.3 回溯性公共物品资助（Retrospective PGF, RPGF）：为已产生的影响力付费

**核心理念**

回溯性公共物品资助（RPGF）提出了一种全新的资助范式，其核心原则是：**“先创造价值，后获得奖励”**。

这与传统的风险投资（VC）模式截然相反。VC 投资的是对未来的承诺和预期回报，而 RPGF 是为项目**已经**对生态系统产生的积极影响和结果付费。这种模式奖励的是已经被验证的贡献，而不是未来的故事。

**实践与演进**

**Optimism** 是 RPGF 模式最主要的推动者和实践者，仅在 2023 年就通过多轮 RPGF 发放了超过 1 亿美元的资助。

-   **运作方式**：首先，通过一套可量化的指标（例如代码贡献量、用户数等）来计算项目已经产生的价值和影响力；然后，再引入一个由社区代表组成的委员会进行投票，决定具体的资助金额。
    
-   **模式演进**：Optimism 的 RPGF 模式也在不断迭代。在早期（如第三期），评判标准较为模糊，导致了混乱的局面——当时有超过 200 名评审员需要评估 600 多份来自不同领域的提案，认知负荷巨大。后来，该模式演进得更加精细化，将资助按领域（如开发者工具、治理工具、社区教育等）分轮次进行，并为每个领域设定更清晰的评判指标，使得资助分配更加公平和有效。
    

这三种模式各有侧重，从精英决策到社区共识，再到影响力驱动，共同构成了 Web3 公共物品资助的工具箱。那么，这些不断演进的机制对 Web3 的未来意味着什么？

# 什么是 Layer 1 和 Layer 2？

## 1\. 引言：为什么我们需要“区块链高速公路”？

### 1.1. 问题的提出：区块链世界的“交通堵塞”

想象一下，像以太坊（Ethereum）这样的区块链是一条非常安全、坚不可摧的公路。但这条公路目前只有一个车道，所有车辆（代表用户的交易）都必须在这条单车道上行驶。当用户数量不多时，一切都很顺畅。但随着越来越多的人涌入，这条公路很快就会变得拥挤不堪。

这种“交通堵塞”会带来两个严重问题：

1.  **通行费昂贵**：为了让自己的车辆能优先通过，司机们不得不支付更高的通行费（在区块链世界里，这被称为“Gas Fee”）。
    
2.  **速度缓慢**：即使支付了高昂的费用，交易确认仍然需要很长时间，就像在高峰时段堵在路上一样。
    

为了解决这个日益严重的问题，开发者们提出了两种核心的扩容方案：优化主干道本身 (**Layer 1**)，以及在主干道之上修建高架桥和快速通道 (**Layer 2**)。本文将为你详细解释这两种技术，并使用以太坊和 Arbitrum 等真实案例，帮助你理解它们是如何协同工作的。

### 1.2. 章节过渡

首先，让我们来了解一下作为这一切基础的“主干道”——Layer 1。

\--------------------------------------------------------------------------------

## 2\. 地基与主干道：详解 Layer 1 (L1)

### 2.1. L1 的核心定义

**Layer 1**（简称 L1）指的是区块链网络的基础层，也就是我们常说的主链。它是一个独立的、自给自足的系统，负责处理和最终确认所有交易，并独立维护整个网络的安全。

**以太坊（Ethereum）** 就是当今最著名、最核心的 Layer 1 区块链之一。

### 2.2. L1 的首要任务：安全与去中心化

与追求极致的速度和效率不同，像以太坊这样的 L1 网络，其设计的首要目标是稳固和可靠。根据以太坊创始人 Vitalik Buterin 的观点，L1 的核心是**韧性 (resilience)** 和**抗审查**。这意味着网络必须足够强大，能够在没有单一实体控制的情况下持续运行，并抵抗任何试图阻止或篡改交易的企图。

L1 的核心价值可以总结为以下几点：

-   **终极安全性** L1 必须依靠自身的力量来保障网络安全。正如源文中所强调的，"L1 最重要的任务就是保障自身的安全。" 这是整个生态系统信任的根基。
    
-   **去中心化** 确保网络不由任何单一的公司或组织控制，而是由全球成千上万的独立节点共同维护。这是抵御攻击和单点故障的关键。
    
-   **韧性与抗审查** 网络的目标是确保持续运行，并且任何符合规则的交易都不会被无理拒绝或审查。
    

### 2.3. L1 面临的挑战

正是因为 L1 对安全和去中心化的坚守，它不得不牺牲一部分效率，从而导致了前文提到的“交通堵塞”问题——即交易成本高、速度慢。

L2 的存在价值恰恰印证了 L1 的这一挑战。源文中提到一个有趣的现象：当以太坊（L1）通过自身升级，使得 Gas 费用显著降低时，市场对于以 Optimism（OP）为代表的 L2 的需求和看好程度就会下降。

根据源文观点，当以太坊（L1）的 Gas 费用大幅降低时，“市场对以 Optimism (OP) 为代表的 L2 的看好程度就会下降，因为 L1 本身已经变得足够便宜了。”

这反向证明了，L2 的主要使命就是为了解决 L1 在高峰时段的高费用和拥堵问题。

### 2.4. 章节过渡

为了在不牺牲 L1 安全性的前提下解决拥堵，开发者们想出了一个绝妙的办法：在主干道之上修建“高架桥”和“快速通道”。这就是 Layer 2 的由来。

\--------------------------------------------------------------------------------

## 3\. 高架桥与快速通道：详解 Layer 2 (L2)

### 3.1. L2 的核心定义

**Layer 2**（简称 L2）是构建在 Layer 1 主链之上的“第二层”网络。它旨在分担主链的计算和交易压力，从而提高整个生态系统的处理能力（可扩展性）。

继续使用我们的“高速公路”比喻：

-   **L1** 是那条单车道但无比安全的**主干道**。
    
-   **L2** 就像在主干道上方修建的**高架桥**。
    

大量车辆（交易）可以在高架桥上快速行驶，支付更低的费用。行驶结束后，高架桥会将所有车辆的最终通行记录打包，统一汇报给主干道进行最终的安全确认。这样一来，主干道不再需要处理每一笔单独的交易，极大地缓解了交通压力。

### 3.2. L2 的核心优势：继承 L1 的安全

L2 最重要的价值主张，也是它与其它独立的区块链（即新的 L1）最大的区别在于：**它在提供更快、更便宜交易的同时，继承了 Layer 1 的安全性和去中心化特性。**

L2 就像是 L1 安全性的“延伸”。它自己不负责最终的安全保障，而是将打包好的交易数据定期提交回 L1，由 L1 强大的共识机制来确保这些交易的最终性和不可篡改性。

知名券商 Robinhood 决定构建 L2 的核心原因，完美地诠释了这一点：

"他们选择 L2，是因为想利用以太坊的安全性、去中心化，以及庞大的 EVM 生态流动性。"

### 3.3. L2 生态概览

如今，L2 生态已经非常繁荣，涌现出许多不同的解决方案。以下是源文中提到的几个代表性项目：

-   **Arbitrum (ARB):** 目前被认为是整个 L2 生态中最活跃的网络之一，吸引了像 Robinhood 这样的大型项目在其之上构建。
    
-   **Optimism (OP):** 另一个知名的 L2 解决方案，与 Arbitrum 形成了双雄格局，但近期因 L1 费用下降而面临一些挑战。
    
-   **Starknet:** 一个著名的 L2 解决方案，但源文中提到它曾发生过宕机事件，也反映出 L2 技术仍在不断发展和成熟中。
    

### 3.4. 章节过渡

理论讲完了，让我们通过一个真实的商业案例，来更深刻地理解为什么一个大公司会选择 L2，而不是从零开始打造一条属于自己的 L1。

\--------------------------------------------------------------------------------

## 4\. 案例分析：为什么 Robinhood 选择 L2 而非从零构建 L1？

### 4.1. 问题的核心

一个引导性的问题是：像 Robinhood 这样拥有强大技术和资本实力的公司，为什么不创建一个全新的、完全由自己掌控的 Layer 1 区块链，而是选择在 Arbitrum (一个 L2) 的技术框架之上，构建自己的 Layer 2 网络？

### 4.2. L1 vs. L2 决策对比

这个决策背后的权衡，清晰地揭示了 L1 和 L2 的核心差异。我们可以通过下表进行对比：

| 特性 | 自建一个新的 Layer 1 | 在以太坊上构建 Layer 2 |
| 安全性 | 极具挑战。 需要从零开始建立共识机制、吸引验证者，并时刻防范潜在攻击，独立保障自身的安全。 | 继承。 直接享受以太坊提供的顶级安全保障，由全球最强大的去中心化网络之一提供保护。 |
| 生态系统 | 孤立。 需要自己从零开始吸引开发者、用户和资金，建立一个全新的生态和流动性。 | 互联互通。 可以立即接入以太坊庞大的用户基础、海量资产和深厚的“EVM 生态流动性”。 |
| 开发难度 | 巨大。 不仅要开发核心协议，还要负责维护整个网络的基础设施，并持续进行升级和治理。 | 相对简单。 可以利用像 Arbitrum 这样的现有框架来快速启动，让团队更专注于应用本身和业务逻辑。 |

### 4.3. 结论提炼

Robinhood 的选择揭示了 L2 的核心价值：**它让项目方可以站在巨人的肩膀上，专注于业务创新，而不必承担从零开始构建和保护一个区块链网络的巨大负担。** 通过选择 L2，Robinhood 既获得了以太坊的安全性与流动性，又享受了 L2 带来的低成本和高效率，这是一个明智的战略选择。

### 4.4. 章节过渡

当然，L2 内部也并非铁板一块，同样存在激烈的竞争和发展差异。但这恰恰说明，整个生态正在走向更加成熟和细分的阶段。

\--------------------------------------------------------------------------------

## 5\. 总结：L1 与 L2 的共生关系

### 5.1. 核心关系回顾

通过以上的分析，我们应该清楚地看到，Layer 1 和 Layer 2 并非替代关系，而是一种**共生互补**的关系。

-   **L1 是信任的根基**（主干道），它提供最终的安全保障和去中心化结算。
    
-   **L2 是可扩展性的引擎**（高架桥），它在 L1 的安全保障下，让区块链应用能够以更低的成本服务于更广泛的用户。
    

没有 L1 提供的坚实基础，L2 将成为无源之水；没有 L2 提供的扩容方案，L1 的应用场景将受到极大限制。

### 5.2. 核心观点提炼

以下是本文希望你掌握的三个最核心的观点：

-   **L1 是安全基石：** 像以太坊这样的 L1，其首要目标是去中心化和安全，这是整个生态系统的信任来源。
    
-   **L2 是扩容方案：** 像 Arbitrum 这样的 L2，通过继承 L1 的安全性，为用户提供更快、更便宜的交易体验，解决了 L1 的拥堵问题。这正是为什么像 Robinhood 这样的公司选择 L2，以便在享受以太坊安全性的同时，为用户提供高效、低成本的服务。
    
-   **协同发展是未来：** L1 与 L2 的协同工作，就像一个国家的公路网与高架桥系统协同运作一样，是推动区块链技术走向大规模应用的关键。

# ERC-7962：您在公共区块链上的私密通行证

### 1\. 核心问题：星巴克与机场贵宾室的故事

假设你是星巴克会员，而某个机场的贵宾室为星巴克会员提供免费进入服务。

**困境来了：** 你该如何向贵宾室证明自己的会员身份？

-   星巴克不能直接把完整的会员名单交给机场——这是严重的隐私侵犯。
    
-   机场需要一种可靠的方式来验证你的身份，但又不想获取完整的会员名单。
    
-   你希望顺利进入贵宾室，同时又不向机场里的所有人暴露你的个人身份。
    

这个场景凸显了对“选择性披露”的需求——即在不泄露所有底层私人数据的情况下，证明某个特定事实的真实性。这种看似神奇的、在不泄露秘密的情况下证明某事的技巧，是通过一种名为“零知识证明”的概念实现的。

\--------------------------------------------------------------------------------

### 2\. 解决方案的“魔法”：简单了解零知识证明（ZKP）

零知识证明（Zero-Knowledge Proofs, ZKP）是一种加密方法，它允许一方（证明者）向另一方（验证者）证明某个陈述是真实的，而无需透露除了“该陈述为真”之外的任何信息。

让我们用一个简单的数学类比来理解。假设你知道两个秘密数字 2 和 3，它们的和是 5。使用零知识证明，你可以提供一个特殊的“证明”，向他人表明你知道的两个秘密数字相加等于 5，但整个过程你完全不需要透露这两个数字具体是 2 和 3。对方可以检查这个证明并百分之百确信你的结论，但他们永远无法从证明本身反推出你的秘密数字。

回到星巴克的例子，借助零知识证明，你可以向机场贵宾室提供一个关于你会员身份的“证明”。他们可以验证这个证明是有效的并让你进入，而全程无需知道你的名字，也看不到完整的星巴克会员名单。ERC-7962 正是采纳了这一强大理念，并将其直接构建成一种新型的区块链代币。

\--------------------------------------------------------------------------------

### 3\. ERC-7962 简介：魔法如何在区块链上实现

ERC-7962 与标准 NFT 之间最根本的区别在于所有权的记录方式。

| ERC-721（标准 NFT） | ERC-7962（隐私代币） |
| 所有权被记录为一个公开的钱包地址。任何人都可以看到哪个地址拥有哪个代币。 | 所有权被记录为一个密钥哈希（K-Hash）——即某个公钥的加密指纹。这个哈希是匿名的，不会直接与任何人的主钱包地址关联。 |

ERC-7962 通过以下三个核心创新实现了这一目标：

1.  **通过密钥哈希拥有所有权** 取代你公开的钱包地址，ERC-7962 使用一个秘密的“一次性”公钥。只有这个公钥的哈希值（K-Hash）作为“所有者”被存储在区块链上。这意味着你的会员身份与一个匿名的加密指纹相关联，而非你的公开身份，这是其隐私保护的基础。
    
2.  **将所有者与支付者分离** 这个标准将交易的发起方（`msg.sender`，即发起链上行为的钱包地址）与资产的所有者解耦。这意味着其他人，例如被称为`中继者`（Relayer，即代用户提交交易的服务）或`支付主`（Paymaster，即同意为他人支付燃料费的账户）的实体，可以为你支付交易的燃料费。这正是允许星巴克为你支付费用，让你无需拥有加密货币就能进入贵宾室的技术魔法。
    
3.  **使用一次性密钥实现最大化隐私** 密钥必须只使用一次，这一点至关重要。因为当交易发生时，实际的公钥会被揭示。为了防止这个被揭示的公钥被追踪，任何剩余的代币余额或被转移的代币都会被分配给一个**全新的、未使用的 K-Hash**，从而有效切断了可追溯的链条。这确保了在你证明一次会员资格后，没有人能通过这个“数字足迹”追踪你未来的其他活动。这与比特币的 UTXO 模型类似。
    

这些技术创新共同解决了用户隐私和 Web3 技术普及方面的重大难题。

\--------------------------------------------------------------------------------

### 4\. 为什么说 ERC-7962 是一个颠覆者

ERC-7962 解决了三个关键问题：

-   **实现真正的链上隐私** 通过 K-Hash 和一次性密钥，该标准阻止了他人根据你的钱包地址构建一个关于你资产和交易历史的公开档案。**你的身份与你的资产之间的联系被切断了。**
    
-   **搭建从 Web2 到 Web3 的桥梁** 通过允许公司或其他实体代付燃料费（燃料费赞助），ERC-7962 为新手用户消除了最大的入门障碍。**新用户无需预先购买加密货币或理解像“燃料费”这样的复杂概念就能开始使用。**
    
-   **实现灵活高效的操作** 该模型天然支持批量交易和账户抽象等高级功能，**使其对企业和开发者来说功能强大。**
    

\--------------------------------------------------------------------------------

### 5\. 平衡的视角：ERC-7962 的权衡

为了提供一个公正和完整的画面，以下是 ERC-7962 的一些权衡之处：

-   由于额外的加密验证，其燃料费成本比标准代币更高。
    
-   公钥会在交易过程中暴露，这就是为什么密钥必须一次一换。
    
-   它不向后兼容现有的代币标准和基础设施。
    
-   它需要更复杂的密钥管理方法。
    

\--------------------------------------------------------------------------------

### 6\. 结论：兼具隐私的所有权

ERC-7962 利用密钥哈希和燃料费赞助的巧妙构想，让用户能够在不牺牲隐私的前提下证明其资产所有权。

这不仅仅是一个技术标准，更是一种向 Web3 用户中心化设计的哲学转变。它将隐私和易用性视为核心功能，而非事后弥补的选项。像 ERC-7962 这样的标准对于构建一个更安全、更私密、更容易被主流大众接受的 Web3 至关重要，它正在弥合技术力量与普通用户需求之间的鸿沟。

# Web3入门(第一周周会记录📝）

### 1\. 区块链是什么？(这台超级计算机的“分布式账本”)

首先，我们需要理解这台超级计算机的底层基础——区块链。你可以把它想象成是这台计算机的记忆系统或底层操作系统，它记录着发生的一切。

Vincent在梳理自己的认知时，将区块链拆解为几个关键元素，让我们逐一来认识它们：

1.  **账本 (Ledger):** 把它想象成一个公开、透明的记事本。网络上发生的每一笔交易，比如A转给B多少钱，都会被清晰地记录在这个账本上，所有人都看得到。
    
2.  **区块 (Block):** 为了方便管理，我们不能一笔一笔地零散记录。区块链的做法是，将一段时间内（比如10分钟）发生的所有交易打包在一起，形成一个数据包。这个数据包，就是一个“区块”。
    
3.  **哈希 (Hash):** 每个区块生成后，系统会给它盖上一个独一无二的“指纹”，这个指纹就是“哈希”。它的生成不仅取决于当前区块的内容，还紧密关联着**上一个区块的指纹**。因此，每个指纹都“记住”了它前面的那个指纹，这就像给文件打上了无法伪造的、且与上一份文件关联的印章。
    
4.  **节点 (Node):** 这个公开账本并不是存放在某一个中心服务器上，而是由成千上万个独立的计算机（也就是“节点”）共同维护。每个节点都保存着一份完整且一模一样的账本副本。
    
5.  **链 (Chain):** 最关键的一步来了。正是通过“哈希”这个独特的指纹，每一个区块都紧密地与前一个区块相连，一个接一个，形成了一条长长的链条。由于每个区块的指纹都依赖于前一个，只要有人想篡改其中任何一个区块的内容，它的指纹就会改变，后续所有区块的指纹也都会对不上，整个链条就会“断裂”。这就是区块链“不可篡改”特性的来源。
    

这个坚不可摧的账本本身已是安全领域的奇迹。但正如Vincent所发现的，一个安全的账本仅仅是个开始。真正让它实现飞跃的，是将这个“记忆系统”转变为一个真正的“大脑”。

### 2\. 以太坊有何特别之处？(让计算机变得“图灵完备”)

正如Vincent所理解的，早期的区块链技术（以比特币为代表），主要是一个“简单的记账技术”。它的功能很纯粹，但也非常有限。而以太坊的出现，则带来了一次革命性的飞跃，其核心在于实现了一个关键概念——**图灵完备 (Turing Complete)**。

“图灵完备”听起来很深奥，但Vincent用了一个非常形象的比喻来解释它，这个比喻源自科幻小说《三体》：

“如果看过《三体》就知道那个用秦始皇有很多人通过简单的‘是’或‘否’的逻辑最后形成了一个自我计算的一个工具。那图灵完备大概的意思就是它能够去能在上面跑程序了。”

这个比喻的核心启示是： **“图灵完备”意味着这台Web3超级计算机从此不再只能记账，而是可以运行任何我们能想象到的程序了。**

它从一个功能单一的“计算器”进化成了一台真正的“通用计算机”。这台计算机从此拥有了无限的潜力。但一台能运行程序的计算机，还需要有真正的“软件”才能发挥作用。这些“软件”在Web3世界里，又是什么呢？

### 3\. 智能合约是什么？(运行在计算机上的“小程序”)

如果说以太坊让Web3这台超级计算机变得可以编程，那么智能合约就是实际运行在上面的软件或应用。Vincent用了一个我们非常熟悉的概念来类比它：

“这个软件是什么呢？它就是合约，每一个合约你可以视作它就是一个小程序，这个小程序可以执行你想要的东西。”

这些“小程序”非常特别。它们不是由某家公司部署在自己的服务器上，而是被部署在整个区块链网络中。它们一旦设定好规则（比如“如果A完成任务，就自动将10个币转给A”），就会自动、永久地运行下去，无需任何人工干预，也无法被单方面停止或修改。

这些自动执行的“小程序”听起来强大到不可思议，但这也引出了一个核心问题：如果程序分布在整个网络中，由谁来保证它的安全？是什么力量让它坚不可摧，没人能轻易“拔掉电源”？

### 4\. 去中心化的魔力 (为什么没人能拔掉这台计算机的电源？)

问题的答案，正是Web3最核心的魔力所在——**去中心化 (Decentralization)**。

智能合约的运行方式与我们熟悉的传统软件完全不同。Vincent对此的解释直击要害：

“这些小程序它不是跑在某一台固定的电脑上或者是某一个服务器上的，而是跑在所有这个矿工的机器上的。”

这意味着什么？还记得我们在第一部分提到的成千上万个“节点”吗？要去关闭或恶意修改一个智能合约，你不能只攻击一台服务器，你必须同时控制全球网络中超过一半（即51%）的节点计算机，这在现实中几乎是一项不可能完成的任务。正是这种分布式的结构，赋予了系统前所未有的健壮性。

**这使得整个系统几乎无法被单一实体关闭或恶意修改，从而保证了其稳定和可信。**

没有人能单方面“拔掉电源”，也没有人能随意篡改规则。代码即法律，一旦部署，便会忠实执行。这股无法被单一实体控制的力量，正是Web3世界的基石。现在，让我们退后一步，看看这幅由不同部件拼成的完整图景。

## AI及其基本概念

**1\. 传统AI的“三不”局限**

传统的AI聊天机器人之所以只能说不能做，是因为它们面临着三个致命的限制。这些限制就像三道无形的墙，将它们困在了虚拟的对话框里。

• **无状态（没有记忆）** AI不记得你们之前的对话。每一次交流都是一个全新的开始，就像一个记忆很短的朋友。这背后的技术原因是其有限的**上下文窗口**（早期模型仅为4k或32k字符），一旦超出限制，它就会“失忆”。

• **无工具（无法行动）** AI无法调用API、执行代码或访问外部系统。它就像一个“双手被绑住的天才”，虽然满腹经纶，却无法与真实世界或数字世界中的其他服务进行互动。

• **无身份（无法担责）** AI在数字世界里没有合法的身份。它不能代表任何个人或实体，无法签署协议，也无法管理资产。它是一个没有身份的“幽灵”，无法承担任何法律责任。

然而，在2024年，新一代的Agentic AI（AI智能体）诞生了，它彻底突破了这三大限制。

\--------------------------------------------------------------------------------

**2\. AI智能体登场：能思考、有记忆、会行动的新物种**

AI智能体（Agentic AI）的出现，标志着AI从一个只会聊天的“嘴替”，进化成了一个能帮你干活的“打工人”。一个完整的AI智能体由三个核心组件构成，让它拥有了思考、记忆和行动的能力。

| 核心组件 | 功能比喻 | 关键技术 |
| --- | --- | --- |
| 大脑 (The Brain) | 推理和决策者 | 大型语言模型 (LLM) |
| 记忆 (The Memory) | 长期记忆库 | 向量数据库 (Vector Database) |
| 手脚 (The Tools) | 与世界互动的工具箱 | API调用 / Function Calling |

这三个组件协同工作：**大脑 (LLM)** 负责理解意图、制定计划；**记忆 (向量数据库)** 通过将文本转化为高维向量，并根据语义相似度进行搜索，从而找到最相关的信息，赋予AI长期记忆；**手脚 (API调用)** 则让AI能够执行指令，与外部世界互动。

有了大脑、记忆和手脚，AI智能体变得异常强大，但一个更深层次的问题也随之浮现：它在数字世界里依然是个没有身份的“黑户”。

\--------------------------------------------------------------------------------

**3\. 新的挑战：强大AI的“身份”与“信任”危机**

一个能力很强但没有合法身份的AI智能体，在当前的互联网架构下面临着非常尴尬的“黑户”处境。这引发了三个致命的问题，形成了一场信任危机。

1\. **身份欺诈风险** 在传统的HTTP协议下，没有任何机制可以验证一个请求究竟来自哪个AI。这带来了巨大的安全隐患，例如在**Meta商业间谍案**中，竞争对手就曾通过伪装成合法的自动化工具渗透到内部系统窃取数据，本质上就是冒充了一个合法的AI智能体。

2\. **行为不可审计** 当AI代表企业签署一份错误的合同并造成巨大损失时，我们该如何追责？法律专家指出，当前中心化服务器的日志如同一个“黑盒”（黑合），可以被随意篡改或删除。为此，知名律所 **breaker** 已将2025年定义为 **“AI问责元年” (Year of AI Accountability)**，凸显了这一法律漏洞的紧迫性。

3\. **支付体系不兼容** 传统的金融支付系统是为人类设计的，完全不适用于高频、自动化的AI。其障碍主要有三点：

    ◦ **需要人类验证：** 注册账户时需要填写验证码或进行人脸识别，AI无法通过。

    ◦ **交易成本高：** 信用卡交易有最低手续费，不适合AI之间低于1美元的微支付（Micro-payment）场景。

    ◦ **结算速度慢：** T+1甚至T+3的结算周期，无法满足AI在一秒内可能需要完成数十次交易并即时结算的需求。

这些看似无解的信任和支付难题，恰好是Web3技术能够大显身手的地方。

\--------------------------------------------------------------------------------

**4\. Web3的解决方案：为AI颁发“身份证”和“银行卡”**

Web3为AI智能体提供了解决信任和支付问题的两大利器：一张不可伪造的身份证和一个可编程的银行账户。它通过去中心化、不可篡改和可编程的特性，完美地填补了传统互联网的短板。

| AI面临的挑战 | Web3的解决方案 |
| --- | --- |
| 身份欺诈风险 | ERC-8004标准<br>通过NFT为每个AI提供独一无二、可验证的链上身份。该标准还包含一个防刷分声誉注册表（只有交易过的客户才能评价，且评分有时间衰减因子），以及一套验证AI能力的机制。 |
| 行为不可审计 | 区块链技术<br>提供不可篡改、公开透明的日志记录，确保AI的每一个决策和行动都有据可查，无法伪造。 |
| 支付体系不兼容 | HTTP 402协议与加密货币<br>通过 HTTP 402 协议（一种原生的网页支付协议）与加密货币，让服务器可以直接向机器客户端请求付款，从而实现无需人工干预、低成本、高效率的微支付和即时结算。 |

有了身份和支付能力，AI智能体终于可以名正言顺地参与经济活动，一个全新的“智能体经济”时代正拉开序幕。

**5\. 展望未来：一个由AI驱动的经济新纪元**

这并非遥远的科幻小说，而是一个正在发生的现实。根据Gartner的预测，到2028年，33%的企业应用将采用Agentic AI，与2024年不到1%的比例相比，将实现超过30倍的惊人增长。

这并非空想，数据已经揭示了机器经济的雏形：目前，**稳定币的年结算量已高达39万亿美元**，远超Visa的1万亿美元，而其中大部分交易由自动化程序驱动。同时，全球API经济（机器间的通信接口）正以**每年超过30%的速度增长**。

想象一下未来的场景：数以亿计的AI智能体在网络中不间断地运行，它们每秒钟都在进行成千上万笔交易——购买数据、支付算力、分配利润。这种规模和频率的机器间经济活动，是传统金融体系完全无法承载的。

只有基于Web3的Layer 2网络等基础设施，凭借其每秒数以万计的交易处理能力（TPS）和极低的手续费，才能支撑起如此庞大的机器经济。

AI与Web3的结合并非炒作，而是技术发展的必然。当机器开始学习思考和行动时，Web3正在为即将到来的智能体经济，构建那个不可或缺、值得信赖的基础设施。

## **Web3 运行原理**

**1\. 第一站：身份与钥匙 (私钥、助记词与钱包地址)**

**核心概念：您的数字主权**

在Web3的世界里，一切始于“个人主权”——一个具有深远社会学意义的概念。这意味着您可以随时随地创建属于自己的账户，无需任何机构的许可。这与传统银行形成了鲜明对比，在银行开户通常需要身份验证、背景审查，甚至可能被拒绝。

Web3的个人主权，建立在以下几个核心概念之上：

• **私钥 (Private Key):** 这是您在Web3世界中最重要的资产。您可以将它想象成一个极其复杂的密码，或是您个人保险箱的**唯一钥匙**。它的安全原则至关重要，请牢记：**“谁掌握私钥，谁就控制资产”**。

• **助记词 (Mnemonic Phrase):** 私钥是一长串随机字符，极难记忆。为了方便用户备份和恢复，助记词应运而生。它通常由12或24个常见的英文单词组成，可以看作是找回您保险箱钥匙的“秘密口令”。

• **钱包地址 (Address):** 这是您在区块链上公开的收款账号，由您的私钥通过一系列加密算法单向生成。您可以把它比作保险箱上对外公开的“门牌号”。任何人都可以向这个地址发送资产，但只有拥有私钥（钥匙）的您才能打开它。

**关系梳理：它们是如何运作的？**

这三者之间存在着清晰的派生关系，可以简化为如下流程：

`助记词 (一) -> (多) 私钥 (一) -> (一) 公钥 -> 钱包地址`

简单总结：助记词是“种子”或“树根”，可以派生出多个不同的私钥。而每一个私钥都唯一对应一个公开的钱包地址。这意味着，只要您安全地保管好助记词，就能随时恢复对所有相关账户的控制权。

**安全第一：您的资产守护原则**

绝大部分的资产被盗都是因为私钥和助记词的泄露。因此，请务必遵守以下几条铁律：

• **绝不截图或联网传输：** 永远不要将您的私钥或助记词截图保存在手机相册中，或通过微信、邮件等在线方式发送给任何人，包括您自己。

• **警惕复制粘贴：** 避免完整复制您的私钥或助记词。一些恶意软件会监控您的剪贴板，窃取信息。如果必须复制，可以分段操作。

• **警惕钓鱼授权：** 这并非危言耸听；许多资产失窃正是因为用户在不知情的情况下，将资金权限授予了恶意的智能合约，就像我们一位学员曾痛失刚刚赢得的黑客松奖金一样。与未知网站交互时，务必谨慎阅读授权请求。

拥有了身份和钥匙，下一步就是如何使用它们来发起一个动作——创建一笔交易。

**2\. 第二站：签署指令 (创建交易与数字签名)**

**什么是交易 (Transaction)？**

在Web3中，一笔交易的本质是一个发送给网络的“指令包”。这个包裹里清晰地写明了您想要做什么。它通常包含三个核心内容：

• **你要做的事：** 这是一个明确的指令，例如，“从我的地址转0.1 ETH到另一个地址”，或者“调用某个智能合约的特定功能”。

• **你愿意支付的费用 (Gas费)：** 这是您为了让网络处理并验证您的指令而支付的“邮费”或“小费”。我们将在下一站详细讨论它。

• **交易序号 (Nonce)：** 这是一个从0开始计数，每笔交易递增1的数字。它的作用是防止您的交易被恶意重复执行。您可以把它想象成一张支票的编号。如果您开出了一张编号为101的支票，银行绝不会兑现第二张同样来自您账户的101号支票。Nonce的作用与此类似，它能有效防止攻击者复制您的交易并重复广播，从而保护您的资金安全。

**数字签名的魔力：不可伪造的“印章”**

当您在钱包中点击“确认”或“发送”时，一个神奇的过程正在发生：**数字签名 (Digital Signature)**。

您可以把它想象成：您使用一个独一无二的、刻有您秘密信息的“私人印章”（即您的私钥），在一张写好了指令的纸（即交易内容）上盖章。这个盖章的过程就是数字签名。

这个“印章”具有三个关键特性：

• **身份验证：** 任何人都可以通过比对您公开的印章样式（公钥/地址），来**验证**这个盖章确实出自于您，但他们无法从印章中反推出您的秘密信息（私钥）。

• **不可否认：** 一旦盖章，您就**无法否认**这个指令是您发出的，这为交易提供了确定性。

• **不可伪造：** 除非有人偷走了您的私人印章（私钥），否则任何人都**无法伪造**您的签名。

**钱包的角色**

在此过程中，您的钱包（如MetaMask）扮演了两个重要角色：

1\. **助手：** 帮助您方便地组装好交易这个“指令包”，填好目的地、金额等信息。

2\. **管家：** 使用您安全存储在本地设备上的私钥，为这笔交易进行签名，而无需将私钥暴露在互联网上。

现在，指令已经签署完毕。但在把它发送出去之前，我们还需要了解一个让许多新手困惑，却又至关重要的概念：Gas费。

**3\. 第三站：支付路费 (详解Gas费)**

**为什么需要Gas费？**

Gas费的存在主要有两个目的：

1\. **防止网络拥堵：** 如果发送交易是免费的，攻击者就可以发送海量的垃圾交易来瘫痪整个网络。收取费用提高了作恶的成本。

2\. **激励验证者：** 区块链网络由遍布全球的验证者共同维护。Gas费是对他们付出计算资源、维护网络安全的一种经济激励。

**Gas费的构成：油价与油耗**

为了更好地理解Gas费是如何计算的，我们可以使用一个“开车去加油站”的比喻：

| 概念 | 比喻 | 解释 |
| --- | --- | --- |
| Gas Price (油价) | 加油站当时的汽油单价 | 您愿意为每个计算单位支付的费用。当网络拥堵时，就像交通高峰期，大家会“抬高油价”（出更高的Gas Price）来争取优先“加油”（被处理）。 |
| Gas Usage (油耗) | 完成一次旅程所需的汽油总量 | 执行一笔交易所需要的计算资源总量。简单的转账操作就像短途行驶，“油耗”很低；而与复杂的智能合约交互，就像长途跋涉，“油耗”就很高。 |

**核心公式**

最终您需要支付的交易费用，可以用一个非常简单的公式来计算：

`交易费用 = Gas Usage * Gas Price`

总结来说：您这趟旅程的总路费，等于您的车实际跑了多远（油耗），乘以当时的汽油单价（油价）。此外，当网络像交通高峰期一样拥堵时，许多人会争相“加油”（提交交易），这就会导致“油价”（Gas Price）上涨。

准备好了路费，您的交易“包裹”就被广播到了广阔的区块链网络中，正式开始了它的“上链之旅”。

**4\. 第四站：排队上车 (交易广播与打包成块)**

**进入“候车大厅” (Mempool)**

当您的交易被广播出去后，它并不会立刻被处理。它首先会进入一个被称为“Mempool”（内存池）的公共交易池。您可以把这里想象成一个巨大的火车站“候车大厅”，所有等待被处理的交易都在这里排队，等待登上下一班列车。

**“公交车”的比喻：打包区块**

接下来，我们可以用一个“公交车发车”的比喻来理解区块打包的过程：

• **区块 (Block) 是一辆公交车：** 在以太坊网络中，大约每12秒就会有一辆“公交车”发车。这辆车的容量是有限的，不能装下候车大厅里所有的乘客（交易）。

• **验证者 (Validator) 是司机：** “司机”的角色是决定让哪些乘客上车。为了最大化自己的收入，司机会优先选择那些支付了更高“车费”（Gas费）的乘客。这就是为什么提高Gas费可以显著加快交易确认速度的原因。

• **发车情景：** 发车前，验证者（司机）会从“候车大厅”（Mempool）中挑选一批出价最高的交易，将它们装进“公交车”（打包成一个区块），然后发车。那些出价较低、没能上车的交易，只能继续在候车大厅等待下一班车。

当您的交易成功“上车”（被打包进区块）后，旅程还未结束。这辆“公交车”还需要得到全城所有信使的共同认可，才能被正式地、永久地记录在城市的公告板上。

**5\. 第五站：全网共识与最终确认 (共识机制与最终性)**

**去中心化世界的挑战：如何达成信任？**

在一个由互不相识、甚至互不信任的匿名参与者组成的去中心化网络中，如何确保大家记录的都是同一本、真实有效的账本？这就需要一套所有人都必须遵守的规则，这套规则就是**共识机制 (Consensus Mechanism)**。

**两种主流的共识机制**

目前，最主流的共识机制有两种，我们可以用简单的类比来理解它们：

• **工作量证明 (Proof of Work - POW):** 就像一场全球性的解数学难题竞赛。无数“矿工”使用强大的计算机（“算力”）疯狂计算一道难题，谁最先解出答案，谁就获得了记账权（打包下一个区块的权利）并赢得奖励。这是比特币所采用的机制。

• **权益证明 (Proof of Stake - POS):** 更像一个选举系统。参与者需要抵押一定数量的代币作为“保证金”来获得成为验证者的机会。系统会从所有验证者中随机挑选一位来记账。如果这位验证者试图作恶（例如，打包虚假交易），他抵押的保证金就会被没收作为惩罚。这是当前以太坊所采用的机制。

**链式结构与不可篡改性**

区块链之所以安全，很大程度上得益于其“链式”结构。每个新生成的区块，都会包含前一个区块的唯一身份标识（`parent hash`），就像链条一样环环相扣。

这种设计带来了强大的不可篡改性。具体来说，每个区块的哈希值都是根据其自身数据**加上**前一个区块的哈希值计算得出的。因此，如果有人试图篡改历史中某个区块的哪怕一个字节，该区块的哈希值就会彻底改变。这个新的、错误的哈希值将不再与下一个区块记录的`parent hash`相匹配，从而导致下一个区块失效。为了“修复”这个问题，攻击者必须重新计算下一个区块的哈希值，但这又会使其与再下一个区块的记录不匹配……如此形成了一场算力上几乎不可能完成的多米诺骨牌效应，从根本上保证了数据的不可篡改性。

**最终性 (Finality)：从“待定”到“永久”**

一个刚刚被打包的区块，其状态是 `unfinalized`（未最终确认）。理论上，此时仍然存在被网络中的其他分支（即短时间存在的竞争链，或称“分叉”）替代的微小可能性。这是因为在一个去中心化的网络中，不同节点间可能存在微小的延迟，导致它们在同一时间生成了不同的区块。

为了达到100%的确定性，我们需要等待一段时间。在以太坊网络中，大约需要12到13分钟。当这个区块后面连接了足够多的新区块，网络就此单一链条达成了压倒性共识，它的状态就会变为 `finalized`（已最终确认）。到了这个时候，这笔交易才被认为是完全安全和不可逆转的。

这个过程，就像一份重要文件的草稿，在经过多轮审阅和层层盖章后，最终变成了不可更改的正式版本。您的交易，至此才算真正完成了它的奇妙旅程。

**结论：一场权力的重新分配**

回顾这趟旅程：我们通过**私钥**签名掌控了自己的数字主权，以**无需许可**的方式创建交易并广播到网络，经由**去中心化**的验证者网络打包确认，最终将记录**公开可验证**地写入了不可篡改的账本。

但这不仅仅是一次技术流程的展示。从更深层次看，Web3的运行原理本质上是**一种权力的重新分配**。它将数字资产的所有权、参与网络的权利、以及抵抗审查的能力归还给了个体。

理解Web3需要跨学科的视角，它融合了密码学提供的安全基础、经济学设计的激励机制，以及社会学探索的去中心化治理模式。它不仅仅是代码，更是一场关于未来数字社会如何构建的深刻实验。

当我们拥抱这个新范式时，也面临着全新的挑战：在这个资产自托管的新世界，我们如何才能降低安全管理的门槛，让每个人都能安心参与？在一个没有中心化税收的体系中，我们又该如何共同出资，维护我们赖以生存的公共基础设施？

理解一次交易的生命周期，是您开始思考这些宏大问题、探索更广阔去中心化世界的第一步。

## **Web3 安全**

**1.0 两大威胁模式：精准猎杀 vs. 广撒网 (Precision Hunting vs. Casting a Wide Net)**

Web3 领域的攻击行为呈现出两个截然不同的趋势：“精准猎杀”和“广撒网”。前者针对大型目标，而后者则与我们每个普通用户息息相关。

| 特征 | 精准猎杀 (Precision Hunting) | 广撒网 (Casting a Wide Net) |
| --- | --- | --- |
| 目标 (Target) | 交易所、高总锁仓价值 (TVL) 的协议等高价值实体。 | 广泛的普通 Web3 用户，尤其是刚刚入门的新手。 |
| 方法 (Method) | 高度复杂、精密布局、长期准备的攻击，常涉及国家级黑客组织（例如源头提到的“北朝鲜黑客”）。 | 产业化、自动化、大规模的攻击，通过钓鱼、诈骗代币等方式广泛传播。 |

虽然“精准猎杀”对整个生态系统构成巨大威胁，但本指南将重点关注“广撒网”攻击，因为这是每一位 Web3 用户在日常操作中都不可避免会遇到的挑战。

\--------------------------------------------------------------------------------

**2.0 深入了解“广撒网”攻击：每个用户都必须警惕的陷阱**

以下是每个 Web3 用户几乎肯定会遇到的三种最高发的攻击类型。了解它们的工作原理是保护您自己的第一步。

**2.1 钓鱼攻击 (Phishing): 窃取你的授权和签名**

钓鱼攻击的核心是**欺骗**。攻击者通过伪造的网站、邮件或消息，诱骗您签署一笔恶意交易，或者直接骗取您的私钥或助记词。这类攻击是 Web3 世界中最普遍的威胁，毫不夸张地说，每时每刻都在发生。

**高阶钓鱼手法：地址投毒 (Advanced Phishing: Address Poisoning)**

这是一种极其隐蔽且完全自动化的钓鱼技巧，也被称为“粉尘攻击”。

**攻击步骤如下：**

1\. **正常交易：** 您向一个常用地址（例如，朋友的钱包或交易所的充值地址）发送了一笔正常的交易。

2\. **伪造地址：** 攻击者创建一个新的钱包地址，这个地址的**开头和结尾几个字符**与您常用的那个目标地址完全相同。

3\. **投毒：** 攻击者从这个伪造的地址向您的钱包发送一笔零价值或极小价值的交易。

4\. **设下陷阱：** 这笔“投毒”交易会出现在您的交易历史中。下次，当您想再次向常用地址转账时，如果您习惯于从交易历史中复制地址，并且只核对了开头和结尾，您就极有可能复制到攻击者的伪造地址，从而将您的资金转入虎口。

这种攻击的自动化和产业化程度令人震惊。在一个真实的案例中，某用户因地址投毒被盗取了价值 5000 万美元的资产。然而，事件发生后，黑客的攻击机器人仍在持续不断地向该受害地址发送“投毒”交易。这说明攻击是如此自动化，以至于黑客本人甚至都还不知道自己已经得手。

**关键提醒：** 在转账前，请务必完整核对每一个字符，而不仅仅是检查地址的开头和结尾。

**2.2 欺诈项目 (Scam Projects): Rug Pulls 和貔貅盘**

在新的市场热点或 Meme 币流行期间，欺诈项目（也称“土狗项目”，即社区俚语中对高风险、低质量项目的称呼）会大规模涌现。攻击者甚至形成了“RP 工厂”（Rug Pull Factory），批量制造与热门项目同名的诈骗代币，引诱投资者上钩。

其中一种最常见的骗局是 **貔貅盘 (Píxiū Pán)**。

• **定义：** “貔貅”是中国神话中一种只进不出的神兽。因此，“貔貅盘”指的是一种您只能买入但**永远无法卖出**的诈骗代币。一旦您购买，您的资金就会被锁定在合约中，任由项目方抽走。

**核心防御方法：** 在购买任何代币之前，**务必通过官方渠道核对代币的合约地址**。您可以从项目官方网站、官方推特或 CoinGecko、CoinMarketCap 等可信的第三方平台获取最准确的合约地址。

**2.3 病毒与木马：直接盗取你的私钥**

除了诱骗您主动操作外，攻击者还会使用更直接的手段：通过病毒、木马或恶意的浏览器插件来感染您的设备。这些恶意软件被设计用来扫描和窃取存储在您设备上的私钥或助记词文件。

一旦私钥泄露，攻击者就能完全控制您的钱包。因此，养成良好的设备使用习惯至关重要：

• **不要安装**来源不明的软件或应用程序。

• **不要添加**未经官方验证的浏览器扩展程序。

• 始终从官方网站或官方应用商店下载软件。

\--------------------------------------------------------------------------------

**3.0 新型威胁：当 AI 深度伪造 (Deepfake) 进入 Web3**

随着技术的发展，攻击者的手段也在不断进化。如今，人工智能 (AI) 深度伪造 (Deepfake) 技术正被用于实施极其逼真的社会工程学攻击，即使是经验丰富的团队也可能中招。

以 Uxlink 项目遭受的攻击为例，我们可以清晰地看到这种新型威胁的可怕之处：

• **长期渗透 (9月开始)：** 攻击者早在 9 月份就开始行动，在 Telegram 上冒充 Uxlink 的商业合作伙伴，并与其团队建立初步联系。

**4.0 你的个人安全行动指南：牢记“四不”大法**

了解了风险之后，我们该如何行动？养成简单的安全习惯是保护自己的最强武器。请牢记以下“四不”原则，并将其融入您的每一次 Web3 操作中。

1\. **不点 (Don't Click)**

    ◦ **含义：** 不要轻易点击来自任何渠道（Telegram、邮件、私信等）的未知链接。即使是朋友发来的链接，也最好通过其他渠道交叉验证一下。

    ◦ **行动：** 访问项目时，请通过官方推特或收藏夹进入官网，而不是通过搜索引擎或聊天群里的链接。

2\. **不签 (Don't Sign)**

    ◦ **含义：** 避免“盲签”。在您的钱包弹出任何签名或授权请求时，务必仔细阅读并理解您将要批准的操作是什么。

    ◦ **行动：** 检查签名的具体内容——您是授权转出 USDT，还是仅仅领取一个空投？金额是否正确？如果您看不懂或有任何疑虑，请直接拒绝。

3\. **不装 (Don't Install)**

    ◦ **含义：** 不要随意安装来源不明的软件、App 或浏览器插件。

    ◦ **行动：** 始终通过官方网站或官方应用商店（Google Play, Apple App Store）下载钱包和其他应用程序。警惕搜索引擎广告中的假冒链接。

    ◦ **专家实例：** 即使是安全专家也时刻保持这种警惕。一位分析师曾分享，在准备一次线上分享会时，他收到了主办方发来的 Zoom 会议链接。尽管他相信链接是合法的，但出于严格的安全习惯，他没有点击该链接，而是选择自己去 Zoom 官网搜索并下载了应用程序。这个小小的举动，正是专业安全意识的体现。

4\. **不转 (Don't Transfer)**

    ◦ **含义：** 在转账前，务必仔细核对收款地址。同时，警惕任何承诺高额回报并要求您先转账的“投资计划”。

    ◦ **行动：** 完整地、仔细地核对收款地址的每一个字符。对于任何要求您转账才能获得更大利益的说法，都要保持怀疑。

## **Web3 合规**

**1\. “负面清单”：在中国大陆需要避开的三大高风险业务**

在中国大陆从事Web3业务，首先要明确一张“负面清单”。以下三个领域因其高风险性，是所有从业者必须谨慎对待甚至完全规避的。

**1.1. ICO与发币：问题不在于“币”，而在于“公开融资”**

1\. **核心误解澄清:** 首先需要明确，单纯为了技术实验或个人兴趣而“发币”本身，并不直接构成违法。如今，用软件在十分钟内创造一个新代币已非难事。

2\. **打击重点:** 大陆监管打击的核心是 **“借用发币来进行公开融资的行为”**。这才是真正的红线。

3\. **类比说明:** 这种监管逻辑并非只针对币圈。在传统行业，也存在类似的案例。例如，模仿“认养一头牛”的模式，向公众出售奶牛份额并承诺分润，如果没有获得相应牌照，同样会面临法律风险。其根本原则是明确的：无论行业外衣是乳制品还是加密货币，未经许可的公开募资都是监管的首要打击目标。

4\. **案例警示:** 近年国内一个Meme币案例极具警示意义。项目方在社区炒作、推高币价后，突然撤回流动性，导致投资者血本无归。尽管聘请了国内顶尖的刑事律师进行辩护，该项目方最终仍以“诈骗罪”被定罪。

**1.2. 虚拟货币交易所：永续合约为何被视为“开设赌场”？**

1\. **执法逻辑:** 大陆办案单位在处理交易所案件时，逻辑往往非常直接。他们并不深入理解虚拟货币的复杂性，而是会问当事人一个关键问题：“你听说过‘九四公告’吗？” 如果当事人回答“听过”，就可能被认定为明知故犯，具有“犯罪意图”。

2\. **视角对比:** 对于永续合约的定性，办案单位和律师的辩护视角存在巨大差异，具体如下表所示：

| 视角 | 对永续合约的定性与逻辑 |
| --- | --- |
| 办案单位视角 | 将其简化为“压大小”的赌博游戏。因此，交易所被视为“赌博网站”，而交易所的运营、技术人员则被视为“开设赌场”的共犯。 |
| 律师辩护视角 | 强调其与赌博有根本区别。语言习惯揭示了本质：人们说“炒股”、“炒期货”，但说“赌球”、“赌马”。其核心差异在于，金融活动中用户掌握**“风险控制权”，可以随时平仓止损；而赌博则是“买定离手”**，下注后便失去控制。永续合约更接近“炒期货”，因为风险控制权始终在用户自己手中，这与赌博的逻辑完全不同。 |

3\. **执法动机:** 在某些情况下，可能存在“逐利性执法”的现象。一些地方当局希望通过打击交易所，没收其资产，从而实现地方财政的盈利。

**1.3. 虚拟货币挖矿：在禁令下的“合规”路径**

1\. **政策波动:** 挖矿政策经历了一个动态变化的过程。从最初的全面禁止，到后来部分地方当局因财政需求，开始尝试联系项目方回归。

2\. **合规“伪装”:** 在这种背景下，一些项目方探索出了与地方政府合作的“合规”路径，即将业务包装成 **“AI算力超算中心”**。项目方以此名义与政府签订协议，甚至可以获得一些地方福利政策。

3\. **业务逻辑:** 其合规的核心在于业务模式的转换。项目方以“外贸”的形式，将超算中心的算力销售给海外客户。至于海外客户购买算力后是用于挖矿还是支持AI研发，则与境内公司无关，从而在形式上规避了风险。

**2\. 个人风险：从“出入金”到U卡的隐藏陷阱**

对于个人而言，即便不直接运营项目，在参与Web3的过程中也面临着诸多潜在的资金陷阱。

**2.1. 出入金：当心“收到脏款”与新型“入金攻击”**

1\. **主要风险:** 个人在进行法币出入金（On/Off-Ramp）时，核心风险始终是 **收到来自上游犯罪活动的“黑钱”或“脏款”**。一旦银行账户因此被冻结，需要耗费大量时间和精力去证明自己的清白。

2\. **新型威胁:** 随着从业者风险意识的提高，传统洗钱的成本也在增加。这催生了一种新型的恶意行为——“入金攻击” (Rùjīn Gōngjí)。其运作方式是：不法分子故意向特定目标（例如，正在竞争某个职位的两个人之一）的账户打入一笔脏款。这笔钱一旦到账，会立刻触发多个地区的公安风控系统，导致目标账户被冻结，甚至被立案调查，从而达到攻击目的。

3\. **行动建议:**

    ◦ **常规出金:**

        ▪ 与交易对手方核实身份信息。

        ▪ 向对方讲明交易背景。

        ▪ 以合理、接近市场价的价格进行交易。

    ◦ **应对意外收款:**

        ▪ 如果你的账户突然收到一笔来路不明的大额资产，必须提高警惕。

        ▪ 在必要时，应主动向公安机关报案并留下记录，这会成为未来保护自己的有力证据。

**2.2. U卡：数字游民生活方式背后的两种风险模型**

U卡（用USDT等稳定币充值的消费卡）因其便利性，受到了许多Web3从业者和数字游民的青睐。然而，看似便捷的背后，隐藏着发行方的结构性风险。以下是两种需要特别警惕的高风险模式：

• **“员工卡”模式:**

    ◦ **运作方式:** 发卡方在万事达等机构注册一个公司账户。当用户申请U卡时，后台系统会将用户的个人信息注册为这家公司的“员工”。

    ◦ **风险:** 这种模式经不起审查。一旦发卡公司被调查，所有关联的“员工卡”极有可能被集体冻结。

• **“多签钱包”模式:**

    ◦ **运作方式:** 几个资金雄厚的大户共同管理一个多签钱包地址，用来归集所有用户的充值资金。

    ◦ **风险:** 这种中心化的资金池模式风险极高：

        1. **内部风险:** 几个合伙人之间可能发生内讧，或者直接卷款跑路。

        2. **外部风险:** 如果他们因为没有严格执行KYC/KYT（了解你的客户/交易）流程，导致其在交易所的总账户因涉嫌洗钱等原因被冻结，那么所有用户的U卡都会立刻无法使用。

\--------------------------------------------------------------------------------

**3\. 终极解读：中国特色的“两层合规”**

在中国大陆的特殊环境下，Web3的“合规”并不仅仅是法律条文的堆砌，而是需要从两个层面去理解和实践。这套“两层合规”模型，是任何希望在中国大陆长期生存的Web3项目都必须掌握的核心方法论。

**3.2. 第二层面：中国特色的深度合规 (Deep Compliance with Chinese Characteristics)**

• **内容:** 这是 **“中国特色的合规”**，其核心是与项目所在地的当局建立并维持良好的协作关系。

• **作用:** 这种深度的合规至关重要。做得好的项目方，不仅能让自己的业务在当地顺利开展，甚至可以在面临外地公安的“跨省抓捕”时，得到本地当局的保护。更进一步，当他们自己成为受害者时（例如，内部员工侵占资产），还可以在大陆以受害人的身份进行报案处理。

-   **区块链：一个大型的、没人能赖账的“公家账本”** 我尝试理解那个叫“去中心化”的概念。
    
    -   过去我们要证明一个东西是我的，得找公证处、找银行。现在区块链让全世界的机器一起盯着，谁想偷偷改账本都不行。这种“信任”不再建立在权威机构上，而是建立在数学代码上。
        

2.  我第一次读 Web3 教程的真实体验是：**每个词都像一个“缩写宇宙”**，读着读着就会忘记主语是谁。下面是我目前用“人话”给自己做的临时解释：
    

-   **Wallet（钱包）**：不是装钱的软件，而是“身份 + 权限”的容器。核心不是余额，是“谁能签名”。
    
-   **Mint（铸造）**：把“某个 NFT 的记录”正式写进链上（像给一张证书盖章入档）。
    
-   **Smart Contract（智能合约）**：一套自动执行的规则；我更愿意把它理解成“写死在系统里的制度”。（制度被代码化）
    
-   **Contract Address（合约地址）**：这套制度本身在链上的门牌号。
    
-   **Chain ID（链的编号）**：你到底在“哪个世界/哪条链”上操作，避免走错片场。
    
-   **Signature（签名）**：不是签字好看，是“用私钥证明是你本人授权”。教程/代码里也会提到用私钥生成签名来做校验。
    
-   **IPFS**：我现在把它当成“去中心化的文件存放/索引方式”，NFT 可能不直接把图片塞进链里，而是存一个指向它的引用。项目的后端也提到会生成 NFT 图片并上传到 IPFS。
    
-   **Infura / Pinata**：我粗暴理解为“帮你接入链/帮你把文件传到 IPFS 的服务商工具”。
    

3.  作为第一天入坑的“门外汉”，我的真实感受是：**门槛真的不低。**当教程开始提到钱包、私钥、以太坊标准（比如那个叫ERC-721的东西）时，真的有点吃力。但我发现了一个很有意思的切入点：**NFT其实是在重新定义“价值”。** 为什么一个数字像素头像能卖高价？这不就是社会学里讨论的“共识”和“身份认同”吗？这种技术与人文的碰撞，是我坚持读下去的动力。
    
4.  我从这个项目“隐约读到的价值观”：它把自己称为 **public goods（公共物品/公共产品）**，并且把前端、后端、合约都开源出来（至少从仓库描述上能确认这一点）。对我这种社科背景的人来说，这挺有意思：一方面它是技术教程；另一方面它像在示范一种“Web3 文化姿态”：**用开源与公共教育来建立信任**，而不是只靠营销话术。

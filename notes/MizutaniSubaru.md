# 2026-01-16
<!-- DAILY_CHECKIN_2026-01-16_START -->
**1\. Web3 合规的宏观环境**

• **合规的必要性：** 在大陆语境下，政策收紧导致活动空间减少，合规的核心在于“控险”。

• **地域差异：** 海外（如香港）合规侧重于**持牌经营**（如稳定币条例通过后业务收缩）；而大陆侧重于**刑事合规**，即避免触犯刑法红线。

**2\. 三大核心红线领域**

• **ICO (公开融资)：** 打击的重点不在于“发币”本身（单纯发个 MEME 币不违法），而在于利用发币进行**公开募资**的行为。

• **虚拟货币交易所：**

    ◦ 办案人员往往不理解虚拟货币逻辑，仅通过《九四公告》等政策判定相关行为为非法。

    ◦ **风险点：** 运营人员和技术人员常被定性为“开设赌场”的共犯，即便技术人员主张“技术中立”，也难逃法律责任。

• **虚拟货币挖矿：**

    ◦ 目前大陆已禁止直接挖矿。

    ◦ **合规转型：** 许多项目方通过“AI 算力中心”或“超算中心”名义与政府签约，将算力作为外贸产品卖给海外，实现业务合规化。

**3\. 出入金与 U 卡风险**

• **出金风险：** 约 70% 的法律咨询涉及出金导致的账户冻结。目前建议通过香港合规交易所路径，但需注意大陆税局的报税要求。

• **U 卡陷阱：**

    ◦ **身份冒用：** 发卡方可能将你登记为海外公司的“员员”，存在动卡风险。

    ◦ **资金风险：** 许多发卡方并未做严格的 KYC/KYT，若合伙人矛盾或账户因其他非法行为被冻，用户资产可能随之“跑路”。

**4\. 项目方经营与 KOL 推广**

• **项目方：** 只要面向不特定对象募集资金，基本逃不开“非吸”（非法吸收公众存款）或“非法经营”罪名。

• **非法经营罪：** 这是一个“口袋罪”，涵盖了 70% 的 Web3 刑事场景，特别是涉及法币与外币对敲的行为。

• **KOL 风险：**

    ◦ 即便 KOL 只是拿返佣、没直接接触客户钱，只要其推广的项目被认定为“开设赌场”，KOL 极易被定性为共犯。

    ◦ **案例：** 某资讯平台因给一家后被认定为赌博网站的交易所打广告，导致大陆员工全被带走。

**5\. 律师的合规建议**

• **建立良好关系：** 在大陆展业，除了表面的法律文书合规（形式合规），建立与当地监管部门的良好协作关系（中国特色合规）至关重要。

• **风险隔离：** 对于高风险业务（如去中心化预测市场），建议完全放在海外，不要与大陆司法认定“硬刚”。

# 2026-01-15

1\. 钱包、私钥与个人主权

• **私钥 (Private Key)：** 相当于资产控制的最高密码，本质是一串随机生成的字符串，拥有者即拥有资产控制权。

• **助记词 (Mnemonic)：** 为了方便记忆，通过特定标准将私钥转化为 12 或 24 个单词序列；一个助记词可以派生出多个私钥（一对多关系）。

• **钱包地址 (Address)：** 由私钥推导出公钥，再经处理截取生成，类似于公开的收款账号。

• **个人主权：** Web3 允许任何人无须许可地创建账号，提供可验证的个人主权，不依赖银行等中心化机构的审批。

• **安全原则：**

◦ 私钥和助记词不可重置，一旦丢失资产无法找回。

◦ 严禁截图或在剪贴板复制完整内容，防止黑客通过木马盗取。

◦ 建议大额资产使用硬件钱包（冷钱包）进行物理隔离。

2\. 交易与数字签名

• **交易 (Transaction/TX) 的构成：** 包含执行动作、Gas 费用（手续费）和随机数 (Nonce，用于防重放攻击)。

• **数字签名机制：**

◦ 用户使用私钥对交易信息签名。

◦ 网络节点通过数学算法，利用公钥和原始信息验证签名是否由对应私钥发出，而无须暴露私钥本身。

• **Gas 机制：**

◦ **Gas Price (油价)：** 用户愿意支付的单价。

◦ **Gas Usage (耗油量)：** 任务复杂度决定消耗多少单位的 Gas。

◦ **作用：** 防止垃圾交易淹没网络，并作为验证者的激励机制。

3\. 区块链网络运行逻辑

• **交易生命周期：** 钱包签名 -> 通过 RPC 节点广播 -> 进入交易池排队 -> 验证者 (Validator) 打包成区块 -> 共识达成 -> 加入区块链。

• **共识机制：**

◦ **POW (工作量证明)：** 通过高能耗的算力竞赛（做题）获取记账权。

◦ **POS (权益证明)：** 质押代币 (如以太坊需质押 32 ETH) 进入验证者列表，随机挑选记账，造假会被没收押金。

• **区块链的最终性 (Finality)：**

◦ 新生成的区块存在被篡改的极小可能。

◦ 以太坊通常需要 12-13 分钟达到 “Finalized” 状态，此时交易才被认为不可逆转。

4\. 智能合约 (Smart Contract)

• **本质：** 存储在区块链账本中的一段可执行代码，具有不可篡改和可追溯的特性。

• **运行：** 节点接收到指向合约的请求时，会在虚拟机中拉取代码执行并改变链上状态。

• **社会学意义：** “代码即法律” (Code is Law)，减少了对中介（律师、银行）的依赖，极大降低了信任成本。

5\. 协议升级与网络安全

• **EIP (以太坊改进提案)：** 社区协作推进的标准，流程包含讨论、初稿、评审到最终确定。

• **客户端多样性：** 运行不同语言编写的软件（如 Geth 等）以减少单点漏洞导致的宕机风险。

• **去中心化 RPC：** RPC 是钱包连接区块链的接口，目前仍是相对中心化的环节，未来需要通过去中心化 RPC 技术进一步优化。

**一些概念**

| 概念 | 详细解释 |
| --- | --- |
| 助记词 (Mnemonic) | 虽然私钥是最终的控制权，但其 64 位 16 进制形式难以记忆。助记词利用 BIP-39 等标准将随机数映射为常见的单词，方便用户记录在纸上。助记词可以推导出私钥，但私钥不能反向推导助记词。 |
| Nonce (随机数/序号) | 在交易中，Nonce 是指该地址发出的交易序号。它有两个作用：1. 确保交易按顺序执行；2. 防重放攻击，确保同一笔交易不会在不同的链上或同一条链上被恶意执行两次。 |
| RPC (远程过程调用) | 钱包本身不存储区块链数据，它通过 RPC 接口向节点服务器发送指令。如果 RPC 服务商宕机或进行审查，用户的交易将无法广播到网络中。 |
| 最终性 (Finality/Finalized) | 区块链并非实时绝对安全。因为网络延迟可能产生短期的分叉，只有当后续区块足够长，使得修改历史的成本高到理论上不可能时，才称为达到最终性。在以太坊 POS 下，这通常需要 2 个 Epoch (约 12.8 分钟)。 |
| 抢跑 (Front-running) | 因为交易在打包前会进入公开的交易池，黑客或机器人可以监测到高利润交易，通过支付更高的 Gas 费让节点优先打包自己的交易，从而截获利润。 |
| 客户端多样性 (Client Diversity) | 如果全网 90% 的节点都运行同一种软件，一旦该软件出现 Bug，整个网络可能陷入停顿或被攻击。多样的客户端实现（如用 Go、Rust 等不同语言编写）能增强网络的韧性。 |

**一、 2025 年 Web3 安全数据概览**

2025 年 Web3 领域的安全事件在频次和金额上均呈现上升趋势，安全已成为 Web3 通往大规模应用的必经之路。

• **安全事件数量**：全年共发生约 **1200 起** 公开可查的安全事件。

    ◦ **B 端（项目方/协议）**：约 500-600 起。

    ◦ **C 端（个人用户）**：约 500-600 起资产品被盗案例。

• **经济损失**：总计经济损失超过 **35 亿美元**。

• **行业核心痛点**：由于 Web3 天然具有金融属性，风险管理（安全）是其核心。

\--------------------------------------------------------------------------------

**二、 核心安全趋势：精准猎杀与广洒网**

**1\. 精准猎杀 (Precision Hunting)**

针对高价值目标（如交易所、高 TVL 协议）的有组织攻击。

• **目标**：中心化金融机构 (Cfi/CEX) 和链上头部协议。

• **特点**：前期准备周期长、布局精密、挖掘漏洞深度高。

• **典型攻击者**：以\*\*国家级黑客（如北朝鲜 Lazarus Group）\*\*为代表，其 2025 年获利至少 20 亿美元。

**2\. 广洒网 (Wide Netting)**

针对广大普通用户和 Web3 新手的产业化攻击。

• **特点**：利用自动化工具和黑产链进行大规模渗透。

• **常见手段**：

    ◦ **貔貅盘/资金盘 (Rug Pull)**：通过“代币工厂”批量发行虚假代币，捕捉热点进行诈骗。

    ◦ **钓鱼服务平台 (Drainer as a Service)**：产业化的钓鱼工具链，每小时都在发生攻击。

\--------------------------------------------------------------------------------

**三、 典型案例深度解析**

**1\. Bybit 攻击事件（损失 15 亿美元）**

• **攻击逻辑**：黑客并非直接攻击交易所，而是渗透了多签服务商 **Safe** 的一名核心开发者。

• **手段**：黑客在 Safe 的前端植入针对 Bybit 特定账户的恶意代码（Payload）。当管理人员进行正常资金调拨时，看到的页面行为完全正常，但签名的却是被篡改的恶意交易。

• **启示**：即使使用了多签等高级安全措施，供应链环节的渗透依然致命。

**2\. UXLink 深度伪造攻击**

• **攻击逻辑**：结合 **AI 深度伪造 (Deepfake)** 与社会工程学。

• **过程**：黑客在 Telegram 冒充合作伙伴，通过 AI 伪造视频通话获取高管信任，随后发送带有后门的虚假会议链接或文档，最终渗透管理权限。

**3\. Balancer 漏洞事件**

• **攻击逻辑**：黑客利用 **AI 漏洞挖掘技术**，发现了已运行多年且人工审计未曾发现的深层智能合约漏洞。

\--------------------------------------------------------------------------------

**四、 新技术带来的新骗局**

**1\. EIP-7702 升级下的钓鱼**

• **技术背景**：**EIP-7702** 赋予了普通地址 (**EOA**) 智能合约的能力，允许批量交易、社交恢复等。

• **骗局形式**：黑客利用已泄露私钥的地址，提前授权恶意代理合约（Delegator）。随后诱导用户向该地址充值（如声称帮其取款），一旦资金转入，会立即被自动化合约转走。

**2\. AI Coding 与软件供应链攻击**

• **风险**：开发者过度依赖 AI 生成代码，可能引入恶意插件或带有后门的依赖包。

• **案例**：**Trust Wallet** 浏览器插件曾因供应链攻击被植入恶意后门。

\--------------------------------------------------------------------------------

**五、 专业术语解释**

| 术语 | 解释 |
| --- | --- |
| TVL (Total Value Locked) | 总锁仓价值。衡量一个 DeFi 协议规模的核心指标，指用户抵押在该协议中的资产总额。 |
| CEX / Cfi | 中心化交易所 / 中心化金融。由中心化机构运营的平台（如 Bybit、Binance）。 |
| 多签 (Multisig) | 需要多个私钥共同签名才能授权交易的安全机制。 |
| 地址投毒 (Address Poisoning) | 又称“粉尘攻击”。黑客向用户发送小额资产，生成一个首尾字符相似的恶意地址，诱导用户从历史记录中错误复制地址进行转账。 |
| 貔貅盘 (Rug Pull) | 开发者吸引投资者买入代币后，突然撤走流动性或利用合约限制用户卖出，导致资产清零。 |
| 零日漏洞 (Zero-day) | 尚未被公众、厂商或安全界知晓的软件漏洞，黑客利用此类漏洞攻击成功率极高。 |
| 重入攻击 (Re-entrancy) | 智能合约漏洞的一种，攻击者在合约更新余额前多次提取资金。 |
| EOA (Externally Owned Account) | 外部拥有账户。最常见的用户钱包地址，由私钥直接控制。 |

| 客户端多样性 (Client Diversity) | 如果全网 90% 的节点都运行同一种软件，一旦该软件出现 Bug，整个网络可能陷入停顿或被攻击。多样的客户端实现（如用 Go、Rust 等不同语言编写）能增强网络的韧性。 |

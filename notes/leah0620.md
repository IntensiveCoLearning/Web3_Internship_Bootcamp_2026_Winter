# Day17学习笔记2026.1.28 - 链上数据分析相关术语

### 一、 核心规模与增长指标

-   **总锁仓量 (TVL, Total Value Locked)**：指在协议中锁定的加密资产总价值，是衡量 **DeFi 协议市场占有率和吸引力** 的核心指标。
    
-   **活跃地址数 (Active Addresses)**：在特定时间段内执行过至少一次交易的唯一地址数量，类似于 Web2 的“活跃用户”，反映 **用户基础及增长趋势**。
    
-   **交易量 (Trading Volume)**：特定时间段内所有交易涉及资产的价值总额，反映了 **交易的资产规模与市场情绪**。
    
-   **交易数 (Transaction Count)**：指链上发生的交易频率，高交易数通常意味着 **网络需求增加**。
    

### 二、 经济与盈利指标

-   **协议收入 (Revenue)**：指协议在收取的费用（如管理费、平台费）中扣除成本后的收益，衡量协议的 **业务盈利能力**。
    
-   **协议费用 (Fees)**：用户为使用平台服务而支付的总价值。
    
-   **交易费 / Gas 费用**：用户支付给验证者的成本，反映了 **网络拥堵程度及用户交易成本**。
    
-   **完全稀释市值 (FDV, Fully Diluted Valuation)**：代币全部释放后的总市值，常用于项目价值对比分析。
    

### 三、 智能合约与技术术语

-   **ABI (应用二进制接口)**：解析合约数据的关键工具，能将十六进制（HEX）格式的原始数据 **解码并恢复成可读的结构化信息**。
    
-   **合约事件 (Events) 与日志 (Logs)**：智能合约执行时产生的非结构化数据，是进行 **特定业务（如质押、借贷）分析** 的数据源。
    
-   **交易哈希 (Transaction Hash)**：每笔交易的唯一标识，通过它可以查询交易状态、收发双方地址等详细信息。
    
-   **代币解锁 (Token Unlocks)**：涉及代币的 **释放比例及分发计划**，对市场供应量有重要影响。
    

### 四、 常用的数据应用工具

-   **RootData**：用于查找 Web3 项目库、投融资信息、团队背景及项目热度指数。
    
-   **DeFiLlama**：主要用于分析 DeFi 协议的 **TVL、协议收入及费用** 排行。
    
-   **Token Terminal**：提供协议的财务数据看板，包括 **活跃用户、FDV、成交量** 等聚合指标。
    
-   **Etherscan (区块链浏览器)**：用于查询实时交易列表、区块高度、最新 Gas 费用及验证智能合约代码。
    
-   **Dune**：通过 SQL 语言进行自由定制化的链上协议数据查询与可视化分析。

# Day11学习笔记2026.1.22 - Dapp

# 一、先搞清楚：Dapp 到底是什么（不讲技术版）

### 1️⃣ Dapp 是什么？

> Dapp = 跑在区块链上的产品 / 服务

| Web2 | Web3 |
| --- | --- |
| App / 网站 | Dapp |
| 公司服务器 | 区块链 |
| 公司说了算 | 规则写在合约里 |

👉 Dapp 不是“公司产品”，而是“**协议 + 前端 + 社区**”。

### 2️⃣ Dapp 的 3 个组成部分（非常重要）

| 模块 | 文科生理解 |
| --- | --- |
| 智能合约 | 写死的规则（谁能干嘛、钱怎么分） |
| 前端 | 用户看到的网页 / App |
| 链 & 生态 | Dapp 寄生的“平台”，像 App Store |

**市场 / BD 常犯错误**：

❌ 只盯着前端 UI

✅ 要理解 **“这个 Dapp 靠哪条链活着”**

# 二、市场 & BD 必须会的「基础术语」（高频）

## 🧱 区块链 & 生态

### 1️⃣ 公链 / L1 / L2

-   **L1（主链）**：Ethereum、Solana
    
-   **L2（扩容）**：Optimism、Arbitrum
    

👉 BD 视角理解：

-   L1 = 一线城市（贵、规范）
    
-   L2 = 新区（补贴多、机会多）
    

**市场用法**：

> “我们是 build 在 Optimism 上的 Dapp”

### 2️⃣ 生态（Ecosystem）

不是口号，是**钱 + 资源 + 流量池**

| 生态给你的 | 对市场的意义 |
| --- | --- |
| Grant | 启动资金 |
| 官方背书 | 信任度 |
| 联合宣发 | 冷启动 |

## 💰 钱 & 商业模式（BD 最重要）

### 3️⃣ Token（代币）

> Web3 的“股权 + 积分 + 激励工具”

你不用懂代码，但要分清：

| 类型 | 用来干嘛 |
| --- | --- |
| 治理 Token | 投票、提案 |
| 功能 Token | 使用产品 |
| 激励 Token | 拉新、留存 |

**市场常见话术**：

> “Token utility 还在设计中”
> 
> \= 商业模式还没完全想清楚

### 4️⃣ Tokenomics（代币经济模型）

一句话：

> 钱怎么发、谁拿钱、什么时候拿

你至少要能看懂这 3 个：

-   总量
    
-   分配比例
    
-   解锁周期（Vesting）
    

**BD 风险点**：

-   解锁太快 → 抛压
    
-   团队占比过高 → 不去中心化是假的
    

### 5️⃣ 收入从哪来？

| 模式 | 类比 |
| --- | --- |
| Gas / Fee | 手续费 |
| 协议抽成 | 平台服务费 |
| NFT / Membership | 会员制 |
| 企业合作 | ToB 授权 |

## 📈 市场增长 & 数据

### 6️⃣ TVL（Total Value Locked）

> 锁在协议里的钱

👉 市场视角：

-   TVL = 信任度 + 规模感
    
-   不等于真实收入
    

### 7️⃣ DAU / MAU

和 Web2 一样，但：

-   钱包 ≠ 人
    
-   一个用户可能很多钱包
    

### 8️⃣ User Journey（用户路径）

市场 & BD 必须画得出来：

**看到 → 进来 → 授权钱包 → 使用 → 留存**

👉 很多 Dapp 死在：

-   钱包授权太复杂
    
-   首次体验太差
    

# 三、合作概念

### 9️⃣ Grant（资助）

不是白拿钱，是：

> “用你的项目帮生态完成 KPI”

常见 Grant：

-   Gitcoin Grants
    
-   生态官方 Grant
    

### 🔟 Partnership（合作）

Web3 合作 ≠ 签合同

| 常见合作 | 实际含义 |
| --- | --- |
| 联合宣发 | 换流量 |
| 技术集成 | 换生态支持 |
| Co-marketing | 互相背书 |

### 1️⃣1️⃣ DAO

你可以理解为：

> 一个没有老板、靠规则运转的组织

BD 要知道：

-   谁是真正做决定的人
    
-   钱在哪个多签里
    

# 四、其他概念

### 1️⃣2️⃣ Narrative（叙事）

> “我们为什么存在”

Web3 非常吃叙事：

-   公共物品

市场工作 = 把抽象叙事翻译成人话

### 1️⃣3️⃣ Community（社区）

不是微信群，是**治理 + 共识 + 激励**

你要分清：

-   真社区（能投票、能共建）
    
-   假社区（只喊口号）
    

### 1️⃣4️⃣ Trust（信任）

Web3 的信任来自：

-   合约是否开源
    
-   团队是否匿名
    
-   资金是否透明
    

# 五、一句话速查表

# Day10学习笔记2026.1.21 - DeFi业态

## 一、TOP DEX

### 🔥 1. **Uniswap**（以太坊及多链）

-   是目前最具影响力的去中心化交易所之一，在 **以太坊生态及扩展链（如 Arbitrum、Base）** 上都有部署。
    
-   相比其他 DEX，**流动性深、交易对多、用户基础大**，长期位居 DEX 排名前列。
    

👉 适合：主流 ERC-20 代币 swapping 和流动性挖矿

**衍生：假如你作为一家DEX的BD，你需要掌握哪些知识？**

**1\. 核心交易机制与演进（“卖点”与“竞争力”）** BD 需要向潜在合作伙伴解释为什么选择你的协议，这要求你理解不同版本的技术差异：

• **V2 基础模式**：掌握**恒定乘积公式 x x y = k**，这是 AMM 的基石，确保池子永远有流动性，不会被“掏空” 。

• **V3 集中流动性**：这是 BD 谈合作的核心。要强调**资金效率**，即 LP 可以自定义价格区间，用更少的钱产生更多的手续费 。

• **V4 钩子机制 (Hooks)**：代表了未来的**定制化能力**。你可以告诉项目方，V4 支持在交易中插入自定义逻辑，如**动态手续费、限价单或特殊的流动性策略**。

**2\. 流动性与风险管理（给 LP 的“避坑指南”）** BD 在招募流动性提供者时，必须能够专业地解释收益与风险：

• **无常损失 (Impermanent Loss)**：必须坦诚告之这是 AMM 固有的风险 。你需要引导 LP 通过赚取手续费来对冲这种损失 。

• **滑点 (Slippage) vs 价格影响 (Price Impact)**：这是最容易混淆的。

BD 需解释：价格影响是池子深度决定的（可计算），而滑点是不可预测的偏差 。

• **手续费档位选择**：针对不同项目给出建议。例如：稳定币对建议 0.01%-0.05%，主流资产 0.3%，高波动资产则需 1% 。

**3\. 价格发现与安全（“平台信任背书”）** 合作伙伴会关心协议的安全性和价格的准确性：

• **预言机 (TWAP)**：解释协议如何通过**时间加权平均价格**提供抗操纵的链上价格数据 。

• **安全防御**：掌握防止“抢跑攻击”的方案，如建议用户使用隐私交易池（Flashbots）或利用 V4 的钩子机制进行逻辑保护 。

**4\. 生态扶持与资源（“ BD 的谈资”）** BD 手中的“筹码”不仅是技术，还有生态资源：

• **资金支持**：熟悉 **Uniswap 基金会**的资助流程 。

• **项目孵化**：可以为合作伙伴提供**审计补贴、孵化器入驻、黑客松**报名等额外价值 。

• **社区推动**：利用全球线上线下的活动资源，帮助合作伙伴提升品牌曝光 。

### 🍰 2. **PancakeSwap**（BNB Chain）

-   当前 DEX 市场交易量最大的单个平台之一，在 **BNB Smart Chain / 多链上运行**，手续费低、用户活跃度高。
    

👉 适合：低费率交易、AMM 流动性提供、收益农场

### 🌊 3. **Curve Finance**

-   专注于 **稳定币间兑换**（如 USDC/USDT/DAI 等）和极低滑点交易。
    

👉 适合：稳定币交易、低滑点兑换

## 二、DEX 入门：与中心化交易所（CEX）完全不同的核心术语

### 1\. 身份与资产控制（谁管钱？）

在 CEX 你是“开户”，在 DEX 你是“连接”。

-   **Non-Custodial (非托管)**：核心区别。CEX 帮你管币；DEX 下币永远在你的钱包里，交易是通过**智能合约 (Smart Contract)** 自动瞬间交换的。
    
-   **Seed Phrase / Private Key (助记词/私钥)**：DEX 的命根子。CEX 丢了密码能找回，DEX 丢了助记词，资金彻底归零。
    
-   **Permissionless (无需许可)**：没有 KYC（身份认证）。只要有钱包（如 **MetaMask**），任何人、任何时间都能交易。
    

### 2\. 交易引擎（怎么买卖？）

CEX 靠“订单簿（人与人撮合）”，DEX 靠“流动性池（人与合约交互）”。

-   **AMM (自动做市商)**：DEX 的核心数学引擎。你不是跟另一个交易者成交，而是根据数学公式直接从池子里“兑换”。
    
-   **Liquidity Pool (流动性池)**：由各种代币对（如 ETH/USDT）组成的“资金仓库”。
    
-   **Swap (兑换)**：DEX 特有的叫法。不像 CEX 分买入/卖出，DEX 是将 A 币直接兑换成 B 币。
    
-   **LP (Liquidity Provider - 流动性提供者)**：
    
    -   **定义**：把自己的币存入池子供人交易，赚取手续费分成的人。
        
    -   **注意**：在 DeFi 里是流动性提供者，在 VC 圈指**Limited Partner**有限合伙人（出资人），不要混淆。
        

### 3\. 成本与交易细节（多付了什么钱？）

在 DEX，除了手续费，你还面临以下“链上”特有的支出和概念：

-   **Gas Fee (燃油费/链上费)**：给区块链矿工的保护费。无论交易成败都要付，通常需预留该链的原生代币（如 ETH、BNB）。
    
-   **Approve (授权)**：**特有步骤**。第一次卖出某种代币前，必须先“授权”合约有权搬运你的币。这会消耗一次 Gas 费。
    
-   **Slippage (滑点)**：成交价与预期价的偏差。在波动大或池子小时，必须手动调高滑点容忍度，否则交易会失败。
    
-   **Price Impact (价格影响)**：你的这笔单子对池子价格的波动贡献。如果显示“Too High”，说明你想买的量太大，池子太小，你会买得非常贵。
    

### 4\. 风险黑话（怎么亏的钱？）

-   **Impermanent Loss (无常损失)**：**LP 特有风险**。如果池子里两个币的价格走势严重脱节，你赚的手续费可能还抵消不了币价波动的损失，不如直接持币不动。
    
-   **Rug Pull (地毯式撤资)**：项目方直接抽干流动性池跑路。这是 DeFi 最常见的骗局。
    

### 💡 给小白的防坑总结：

1.  **预留 Gas 费**：钱包里永远要留一点 ETH 或 BNB。很多人把币全换了，结果发现没 Gas 费转账或卖出了。
    
2.  **注意 Price Impact**：买入前盯着看一眼这个百分比，超过 2-3% 就要小心了，说明你被“宰”了。
    
3.  **Approve 安全性**：不要在没听说过的钓鱼网站点击 Approve，那是把你钱包的“钥匙”递给别人。

## 从具体工作场景看如何应用以上平台

### 场景一：BD 想确认“这个项目到底有没有真实用户？”

👉 用 **Etherscan**

### 目标

判断一个项目是不是“空气项目”

### 最简单操作

1️⃣ 打开项目官网 → 找到 Token 合约地址

2️⃣ 复制合约地址 → 粘贴到 **Etherscan 搜索框**

你重点看 3 个地方：

| 看什么 | 在哪看 | 说明 |
| --- | --- | --- |
| Holders | Holders | 是否只有几百人持币 |
| Transfers | Transfers | 是否每天有人转账 |
| Transactions | Txns | 是否长期活跃 |

### 市场判断逻辑

-   只有几个大户 + 几乎没转账 = ❌ 假繁荣
    
-   持币人数稳定增长 + 每天有交互 = ✅ 真实使用
    

📌 **适合：BD 背调、项目筛选**

### 场景二：市场要写一份“用户增长分析报告”

👉 用 **Dune**

### 目标

回答：这个协议最近 3 个月用户在涨还是在跌？

### 最简单操作（0 写代码版）

1️⃣ 打开 Dune

2️⃣ 搜索框输入：项目名 + dashboard

例如：`Uniswap user growth`

3️⃣ 点开点赞多的 Dashboard

重点看：

-   Daily Active Users
    
-   New Wallets
    
-   Transactions Count
    

4️⃣ 右上角 **Fork（复制）**

→ 变成你自己的仪表盘

### 你可以直接用于报告的句式

> According to Dune data, Uniswap’s daily active users have increased by 32% over the past 3 months, indicating steady organic growth.

### **📘 公共物品：文科生友好解读**

一种**对所有人开放、大家共同受益，且不会因为使用而被耗尽**的事物。这基于两个核心特性：

| 特征 | 比喻解释 | Web3中的例子 |
| --- | --- | --- |
| 非排他性 | 像路灯，无法阻止任何人享受它的光亮。 | 公链、开源代码、公开的研究报告。 |
| 非竞争性 | 像知识，我学会后，你依然可以完整地学习，我的使用不消耗它。 | 区块链上的数据，一个人查询不影响其他人使用。 |

在Web3世界中，公共物品的内涵有新发展：

-   **新视角：正外部性** - 更强调其“**利他属性**”。指某个事物在使自己受益的同时，也能**无偿给社会带来额外好处**。例如，一个优化的加密算法，在保护开发者自己产品的同时，也让整个网络生态更安全。
    
-   **新矛盾：免费与可持续** - 文章用“**收入-邪恶曲线**”提出了一个有趣矛盾：**完全免费的公共物品虽然社会效益最大，但创造者可能缺乏持续动力**。因此，适度的资助或商业化，是为了让创造者能活下去并持续创造价值，最终实现社会整体利益（即“帕累托最优”）的最大化。
    
-   **新范式：Web3的基石** - 在Web2中，企业通过建立壁垒（如专利、数据垄断）来获利。而**Web3的核心精神是开放与协作**，因此像公链、开源协议这类**非排他、非竞争**的公共物品，反而能催生更强的网络效应和信任，成为新商业模式的基础。

## 一、视频学习

### 一、 Web3的“过路费”逻辑

1.  **Nonce (随机数/计数器) - 关键知识点！**
    
    -   **概念：** 你的钱包发出的每一笔交易都有一个序号，叫 Nonce。比如第1笔是 Nonce 0，第2笔是 Nonce 1，必须按顺序执行。
        
    -   **卡单原理：** 如果你的 Nonce 6 的交易因为Gas给少了卡住了，那么你发的 Nonce 7、Nonce 8 无论给多少钱，都会排在后面等，全部卡死 \[[10:00](http://www.youtube.com/watch?v=11QTT6BK5j0&t=600)\]。
        
    -   **BD/运营场景：** 当你给合作伙伴打款或发放奖励卡住时，不要傻等，也不要疯狂重发。**发一笔新的交易，使用相同的 Nonce，但给更高的 Gas 费**，覆盖掉原来那笔卡住的交易
        

### 二、 L1 与 L2：解决拥堵的扩容方案

> BD/运营 必知： 为什么现在的项目都往 Layer 2 (L2) 跑？

1.  **L1 (以太坊主网) vs L2 (如 Optimism)**
    
    -   **L1 (Mainnet)：** 像繁华的市中心（曼哈顿），安全、资金量大，但极度拥堵、由于Gas费太贵（动不动几十美金），普通用户玩不起 \[[30:42](http://www.youtube.com/watch?v=11QTT6BK5j0&t=1842)\]。
        
    -   **L2 (Layer 2)：** 像市中心旁边的新开发区。由于使用了特殊技术（Rollup），交易速度极快，费用极低（几毛钱甚至更低），但安全性最终还是依赖于L1。
        
2.  **跨链桥 (Bridge)**
    
    -   操作：要把资产从 L1 搬到 L2，需要过“桥”。
        
    -   **痛点：** 过桥有“等待期”（Grace Period），资金会暂时消失几分钟，用户会很慌 \[[33:17](http://www.youtube.com/watch?v=11QTT6BK5j0&t=1997)\]。
        
    -   **运营启示：** 如果你的项目在 L2 上，新用户入金门槛是一个巨大的漏斗，需要教育用户如何使用 Bridge。
        

### 三、 DeFi (去中心化金融) 初体验

> BD/运营 必知： 钱是怎么生钱的？

1.  **借贷 (Lending/Borrowing)**
    
    -   **玩法：** 类似于银行抵押贷款。你把 ETH 存进去（抵押），借出稳定币（如 DAI）。
        
    -   **应用场景：** “做多”或“做空”。比如你看跌某个币，就借出来卖掉；或者你想用钱但不想卖掉手里的 ETH，就抵押 ETH 借钱花 \[[49:04](http://www.youtube.com/watch?v=11QTT6BK5j0&t=2944)\]。
        
    -   **Approve (授权)：** 这是一个大坑！在使用任何 DApp（去中心化应用）动用你的代币前，你必须先发一笔交易“授权”它扣款。所以往往是**两笔交易**（一次授权，一次执行），这会增加用户操作成本 \[[44:40](http://www.youtube.com/watch?v=11QTT6BK5j0&t=2680)\]。
        
2.  **DEX 聚合器 (如 Matcha)**
    
    -   **概念：** 帮你去所有去中心化交易所（Uniswap, SushiSwap等）比价，找到最划算的兑换路径 \[[41:45](http://www.youtube.com/watch?v=11QTT6BK5j0&t=2505)\]。
        

### 四、 多签钱包 (Multisig) - 这里的重点是 Gnosis Safe

> BD/运营 必知： 公司的钱该怎么管？这是企业级合规运营的核心。

1.  **为什么要用多签？**
    
    -   **单签风险：** 只有你一个人有私钥（助记词），如果丢了或者你出意外了，公司的钱就全没了；或者员工卷款跑路。
        
    -   **多签机制 (M-of-N)：** 比如设定 4 个人管理钱包，必须其中 2 个人同意（签名），交易才能发出去 \[[01:03:38](http://www.youtube.com/watch?v=11QTT6BK5j0&t=3818)\]。
        
    -   **DAO 的雏形：** 这就是最简单的去中心化组织管理方式。
        

### 五、 BD/运营视角的总结与话术

1.  **关于用户体验 (UX)：** Web3 目前的 UX 依然很糟糕（“Welcome to Web3”是视频里的口头禅，通常伴随着报错或等待）。在设计产品流程时，必须考虑到**授权(Approve)、Gas波动、跨链等待**这些劝退环节。
    
2.  **关于基础设施：** 现在的趋势是“L1 做资产结算，L2 做高频交互”。如果你的项目需要用户频繁操作（如游戏、社交），必须选 L2，否则用户付不起 Gas 费。
    
3.  **关于安全：** 对外合作或管理团队资金，**必须使用 Gnosis Safe (多签钱包)**。这显得你们专业且安全。不要用个人的 Metamask 地址收公款。
    
4.  **WalletConnect：** 这是一个连接协议，能让你用手机钱包扫码登录电脑网页，或者让多签钱包连接到 DApp 进行操作 \[[01:10:12](http://www.youtube.com/watch?v=11QTT6BK5j0&t=4212)\]。这是打通移动端和桌面端的重要工具。

## 一、NFTs视频学习

### 1、 NFT 的本质：数字世界的“房产证”

-   **文科生理解：** 想象买了一幅画。在现实中，你拥有那张纸；在 Web3 中，NFT 不是那张纸，而是**挂在区块链上的“所有权证书”**。
    
-   **BD/运营笔记：**
    

1.  **所有权与使用权分离：** 别人可以下载你的图片（使用），但链上记录只有你才是它的主人（所有权）。
    
2.  **可验证性：** 市场人员可以通过检查用户的钱包，确认他是否真的持有某个 NFT，从而决定是否发放粉丝福利（如：持有特定 NFT 才能进群或参加线下活动）。
    

### 2、 两种存储方式：决定了 NFT 的“保值期”

这是 BD 在对接项目时最需要关注的技术细节，因为它直接关系到产品的“寿命”。

1.  **IPFS（去中心化存储）：**
    
    -   **原理：** NFT 的证书在链上，但图片太重了，存放在 IPFS（类似分布式网盘）里。
        
    -   **关键词：内容寻址（Content Addressable）。** 如果图片内容改了一个像素，它的链接地址就会变。这意味着 NFT 的内容是**不可篡改**的。
        
    -   **运营注意：** 必须要有人持续“续费”（Pinning）存储，否则图片可能会丢失。
        
2.  **On-Chain（全链上存储）：**
    
    -   **原理：** 把图片代码化（SVG格式）直接塞进区块链。
        
    -   **优点：** 只要以太坊还在，这个 NFT 就永不消失。
        
    -   **缺点：** 极贵！像是在金条上刻字。
        

### 3、 市场 BD/运营的实战锦囊

1、Token Gating（代币门槛）—— 最实用的运营工具

-   **应用：** NFT 不仅仅是艺术品，它可以是**入场券**。
    
-   **场景：** “只有持有本系列 NFT 的用户才能参与闭门会议/获得空投/解锁独家内容”。这种逻辑是目前 Web3 社区增长的核心逻辑。
    

2、版税（Royalties）—— 创作者的新武器

-   **传统痛点：** 艺术家卖出一幅画后，这幅画在二手市场涨到 100 万，艺术家一分钱也拿不到。
    
-   **Web3 优势：** 可以在智能合约里写死：**“未来每次转手，都要分 10% 给原作者”**。
    
-   **BD 话术：** 在对接艺术家或 IP 方时，这是最能打动他们的点——**长期的被动收入**。
    

3、可组合性（Composability）—— “乐高”式营销

-   **例子：** 视频提到了 Loot 项目。它只发了一堆文字（属性），把“怎么玩”留给社区。[\[00:41:10\]](https://www.youtube.com/watch?v=NOdrEpnoCiM&list=PLJz1HruEnenAf80uOfDwBPqaliJkjKg69&index=5)
    
-   **运营策略：** 可以做一个 NFT，而其他开发者可以基于 NFT 开发游戏或穿搭。
    

### 4、 避坑指南（BD 风控必备）

-   **伪去中心化：** 有些 NFT 把图片存放在私人服务器上（如阿里云/亚马逊），如果公司倒闭，用户买的 NFT 就会变成一张“空网页”。BD 在选择合作伙伴时，应优先选择使用 IPFS 或 Arweave 存储的项目。
    
-   **审查风险：** 像 OpenSea 这样的平台是 Web2 运营模式，它们有权屏蔽某些 NFT。**“看不见不代表不存在”**，真正的所有权依然在用户的钱包里。
    

💡NFT 不是一张小图片，而是一种**自带金融属性、可编程、且能直接连接粉丝的“智能会员卡”**。

### 第一章：1. 核心逻辑：从 1 到 N 的跨越

-   **文科理解：** 比特币（BTC）像是一张“数字支票”，只能用来转账（功能单一）。而以太坊（ETH）则是一个“数字底座”或“全球共享的操作系统”，你可以在上面盖房子、开商店、签合同。
    
-   **BD视角：** 以太坊不仅能传输价值（钱），还能运行代码（程序）。这意味着你可以把合同、规则、投票逻辑直接写在链上，且**无人能篡改、无人能关停**。以太坊不是某种竞争币，它是**基础设施**。所有的 DeFi（金融）、NFT（文化）、GameFi（游戏）都长在它上面。作为 BD，要意识到卖的不是一个产品，而是一个**生态系统的入场券**。
    

**2\. 核心概念：智能合约 (Smart Contract**

-   **文科理解：** 想象一个“自动执行的法律合同”。一旦满足条件（比如你付了钱），货物（数字资产）自动归你，没有任何中间人可以反悔。
    
-   **市场/BD 卖点：** **低成本信任： 它是BD对外谈合作的核心卖点——减少对中介的依赖**
    
    ```
      去信任（Trustless）：合作伙伴不需要相信你的公司信誉，只需要相信代码。
    
      效率：省去了昂贵的律师、公证和银行结算成本。
    ```
    

**3\. 以太坊的终极目标：去中心化应用 (dApp)**

-   **文科理解：** 现在的 App（如微信、美团）数据在公司手里，公司想封就封。dApp 的数据在链上，用户自己掌握数据权。
    

### 第二章：**1\. 账户模型：外部账户 (EOA) vs 合约账户 (CA)**

-   **文科理解：**
    
    ```
    EOA（外部账户）： 就像你的“私人钱包”，由私钥控制，只有你能动里面的钱。
    ```
    

**CA（合约账户）：** 就像一个“无人值守的自动售货机”，它没有私钥，只按预设的程序办事。

-   **BD 应用：** 许多商务合作涉及到“多签钱包”或“自动利润分配”，就是利用合约账户来实现自动分润，解决合作中的信任难题。
    

**2\. Gas 费：生态系统的“汽油”**

-   **文科理解：** 在以太坊上做任何事（发公告、转账、签合同）都要交小费。网络越拥堵，小费越贵。**商业平衡：** Gas是为了防止有人恶意发送大量垃圾信息（让作恶变得昂贵）。
    
-   **市场视角：** 这是目前以太坊最大的“痛点”。在做市场推广时，必须考虑**用户交互成本**。如果一个 NFT 卖 10 块钱，但 Gas 费要 50 块，这项目就很难推。**在策划营销活动时（如空投），必须避开Gas高峰期，否则用户参与成本太高。**
    
-   **BD 话术：** 了解 **Layer 2**（二层网络，如 Arbitrum, Optimism），它们是用来降低 Gas 费的解决方案，是目前商务合作的主流战场。\*\*质押（Staking）\*\*本身就是一个巨大的金融市场，是BD寻找合作伙伴的重要赛道。
    

**3\. 状态机：全网公认的“真相”**

-   **文科理解：** 以太坊像是一本全球共享的“大账本”。每发生一件事，全网所有的电脑都会同步更新。
    

### 💡 针对岗位的实战建议（有用笔记总结）

-   **给BD/市场：**
    
    -   **卖点提炼：** 不要推销“区块链”，要推销“抗审查、去中心化带来的透明度”和“全球结算的效率”。
        
    -   **活动设计：** 考虑到Gas费，如果是针对小白用户，尽量选择在Layer 2（以太坊的二层网络）开展活动。

    ### 1\. ENS (以太坊域名服务)：Web3 的“名片”
    
    -   **文科理解：** 就像互联网早期的 IP 地址（一串乱七八糟的数字）变成了网址（[https://www.google.com/search?q=google.com），ENS](https://www.google.com/search?q=google.com），ENS) 把长达 42 位的以太坊地址（0x...）变成了可读的名字（比如 `sanfordstout.eth`）。
        
    -   **市场/BD 笔记：**
        
        -   **身份聚合器：** ENS 不只是一个名字，它是用户在所有 Web3 应用中的通用账号。用户只要改一次 ENS 的信息，所有应用都会同步更新，这极大降低了用户的操作门槛 \[[27:30](http://www.youtube.com/watch?v=wYSMNdIRoII&t=1650)\]。
            
        -   **商业溢价：** 简短好记的 ENS 域名具有稀缺性，就像早期的顶级域名（.com）一样。3-4 位的域名现在非常昂贵 \[[07:59](http://www.youtube.com/watch?v=wYSMNdIRoII&t=479)\]。
            
        -   **反向解析 (Reverse Record)：** 这是一个关键技术点。设置“反向解析”后，应用才能识别出你的地址对应哪个名字。作为 BD，如果你的合作项目能完美支持 **ENS 显示**，会显著提升用户体验 \[[23:11](http://www.youtube.com/watch?v=wYSMNdIRoII&t=1391)\]。
            
    
    ### 2\. 去中心化交易 (DEX) 与 Uniswap：不打烊的“无人售货机”
    
    -   **文科理解：** 传统的交易所像有收银员的超市，有上下班时间和中间商；去中心化交易所（如 Uniswap）更像是一台**自动售货机**，只要有代码规则，任何时候都能兑换资产。
        
    -   **市场/BD 笔记：**
        
        -   **流动性池 (Liquidity Pool)：** 为什么无人值守也能交易？因为背后有“池子”。池子里存着两种代币（比如 ETH 和 DAI），用户拿一种换另一种 \[[31:30](http://www.youtube.com/watch?v=wYSMNdIRoII&t=1890)\]。
            
        -   **BD 合作点：** 任何新项目发币，首要任务就是去 DEX 提供“**流动性**”。没有流动性，代币就无法交易，项目也就没有生命力。
            
        -   **套利 (Arbitrage)：** 价格是通过“套利者”来维持平衡的。如果某个池子的币便宜了，就会有人买走去别的平台卖，从而让价格回归正常 \[[31:51](http://www.youtube.com/watch?v=wYSMNdIRoII&t=1911)\]。
            
    
    ### 3\. **稳定币 DAI**：Web3 的“定海神针”
    
    -   **文科理解：** 比特币、以太坊价格波动太大，没法直接用来标价和买咖啡。DAI 是一种通过“抵押”产生的代币，目标是永远等于 1 美元。
        
    -   **市场/BD 笔记：**
        
        -   **用户体验 (UX) 的救星：** 作为一个 BD 或市场，如果想搞空投或奖励，发 ETH 风险很高（今天发 100 块，明天变 60 块），\*\*发稳定币（USDT USDC）\*\*是给用户最稳妥的体验 \[[37:36](http://www.youtube.com/watch?v=wYSMNdIRoII&t=2256)\]。
            
        -   **超额抵押机制：** DAI 为什么稳？因为它背后有价值更多的资产（如 ETH）锁在合约里作为支撑（就像当铺，你抵押 100 块的金子，只能借走 60 块的钱） \[[40:09](http://www.youtube.com/watch?v=wYSMNdIRoII&t=2409)\]。
            
        
        Q：“既然都有 USDT 和 USDC 了，为什么还要 DAI？”
        
        从去中心化（DAI）到中心化（USDT/USDC）的迁移
        
        **BD：** 在对接项目时，如果对方是一个极度追求“去中心化”的 DAO 组织，他们通常更偏好用 **DAI** 结算。如果对方是合规机构，他们一定首选 **USDC**。
        
        **产品经理：** 在设计支付功能时，USDT 是“必选项”（用户多），USDC 是“合规项”（风控好），DAI 是“极客项”（DeFi 原生支持好）。
        
        **市场运营：** 空投或奖励活动，选 **USDC** 显得正规且透明，选 **DAI** 显得专业且懂 Web3 文化，选 **USDT** 则是最接地气的做法。
        
    
    ### 4\. 身份与 Sybil Attack (女巫攻击)：Web3 的难题
    
    -   **核心逻辑：** 在以太坊上，创建新身份几乎零成本。
        
    -   **市场/BD 警告：** \* 由于身份太好伪造，市场活动（如抽奖、空投）经常会遇到**女巫攻击**——即一个人开几千个账号来薅羊毛 \[[37:09](http://www.youtube.com/watch?v=wYSMNdIRoII&t=2229)\]。
        
        -   **应对方案：** 现在的项目会通过检查“账号活跃度”（比如是否投过票、是否有交易记录）来判断这是否是一个**真实人类**。

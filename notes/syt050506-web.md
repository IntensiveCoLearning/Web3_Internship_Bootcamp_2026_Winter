# 2026-01-14
<!-- DAILY_CHECKIN_2026-01-14_START -->
刚入 Web3 的时候，注意力基本在三件事上：

-   有没有机会
    
-   能不能快点赚
    
-   别人都在冲什么
    

但很快就发现一个很现实的问题：**Web3 里的错误，大多数是一次性的。**

转错地址，钱没了

私钥泄露，资产被清空

授权点错，合约直接把你掏干

合规踩线，平台直接消失

没有客服兜底，也没有“撤回键”。  
从那以后，我开始把「安全 + 合规」当成一个项目是否值得参与的第一道门槛。

* * *

## 二、Web3 安全

### 1️⃣ 私钥和助记词：最基础，但翻车率最高

**谁掌握私钥，谁就拥有资产。**

但也是最容易被忽视的一条。

-   助记词截图存在手机
    
-   发到微信“临时保存”
    
-   连了一个假网站就全授权了
    

现在我给自己定的几条硬规则：

-   助记词只离线保存（纸、硬件）
    
-   不截图、不云端、不发任何人
    
-   主钱包和测试/空投钱包完全隔离
    

只要涉及私钥，我宁愿麻烦，也不图省事。

* * *

### 2️⃣ 智能合约：链上 ≠ 安全

合约在链上、不可篡改 = 安全

但真实情况是：

-   合约可能一开始就写错
    
-   管理员权限可能极大
    
-   升级逻辑可能留后门
    

关注：

-   有没有主流审计（不是万能，但至少是态度）
    
-   是否开源、文档是否清楚
    
-   项目方有没有“一键改规则”的能力
    

如果一个项目：

-   合约复杂
    
-   解释含糊
    
-   权限集中
    

那我通常选择观望

* * *

### 3️⃣ 钓鱼和社交工程：最常见、也最有效的攻击

Web3 里大量安全事故，和技术关系不大，和情绪关系很大。

典型套路都很熟：

-   “你被选中空投了”
    
-   “官方紧急公告，速点”
    
-   冒充管理员私聊你
    

它们本质上都在制造三种情绪：

-   贪
    
-   急
    
-   怕错过
    

**凡是主动来找你、催你操作的，先当骗子看。**

宁可错过机会，也不赌这一把。

* * *

### 4️⃣ 跨链桥：我最谨慎使用的基础设施

跨链桥几乎是 Web3 历史上被攻击最多的地方之一。

原因很现实：

-   资金量大
    
-   逻辑复杂
    
-   一旦出事，影响面极广
    

我现在的使用原则是：

-   能不跨就不跨
    
-   一定要跨就选头部
    
-   单次金额控制，不 all in
    

跨链对我来说是工具，不是收益来源。

* * *

### 5️⃣ 伪去中心化：名字是 Web3，控制权还是在少数人手里

很多项目打着去中心化的旗号，但实际上：

-   后端可控
    
-   管理员能冻结资产
    
-   数据根本不在链上
    

这类风险在：

-   中心化交易所
    
-   托管钱包
    
-   一些 Web3 外壳项目
    

* * *

## 三、Web3 合规：我从排斥到理解的过程

### 1️⃣ 合规并不等于背叛 Web3

早期会觉得：

-   合规 = 被收编
    
-   去中心化就该无监管
    

但慢慢发现：

-   想长期活下来的项目，一定绕不开监管
    
-   机构资金、主流用户，对合规极度敏感
    

现在我更倾向于认为：合规不是为了限制创新，而是筛掉明显的系统性风险。

* * *

### 2️⃣ 我重点关注的几个合规信号（非法律）

我不会去逐条研究法律，但会留意这些点：

-   **KYC / AML**：中心化平台基本绕不开
    
-   **合规牌照**：决定项目能否长期运营
    
-   **证券属性风险**：Token 是否可能被认定为证券
    
-   **地区限制**：不同国家政策差异极大
    

如果一个项目对这些问题完全回避，我会提高警惕。

* * *

## 四、我现在是怎么用「安全 + 合规」筛项目的

我会先问自己几个问题：

-   资产是不是在我自己控制下？
    
-   项目方有没有“一键改规则”的能力？
    
-   出问题时，最坏结果我能不能承受？
    
-   这是短期博弈，还是长期项目？
    

如果这些问题让我明显不安，那我大概率只会：看热闹，而不是下场。

* * *

## 五、写给自己的阶段性总结

Web3 从来不是一个“无风险高收益”的地方。  
它更像是一个：

-   奖励认知
    
-   惩罚侥幸
    
-   放大人性的市场
    

对我来说：

-   **安全是底线**
    
-   **合规是过滤器**
    
-   **能活下来，机会才会轮到你**

## 技术岗笔记

### 1\. Web3 前端工程师

我对 Web3 前端的理解是：  
**不是单纯写页面，而是负责“用户和区块链之间的交互层”。**

主要做的事情包括：

-   写 Dapp 的前端页面
    
-   实现钱包连接、交易签名、信息展示
    
-   调用智能合约，把链上数据展示出来
    
-   和后端对接 API
    
-   做性能优化和兼容性
    

需要的能力基本是：

-   HTML / CSS / JavaScript 基础
    
-   React 或 Vue（现在更多是 React + Next.js）
    
-   会用钱包（比如 MetaMask）
    
-   熟悉 Viem 和智能合约交互
    
-   对以太坊、Solana 这些链有基本认知
    

一个很重要的点：  
👉 **现在主流已经从 Ethers.js / Web3.js 转向 Viem 了**

常见技术栈：  
HTML、CSS、JavaScript、TypeScript、React、Next.js、Viem

* * *

### 2\. Web3 后端工程师

后端更像是一个“中枢”，负责把链上、链下和业务逻辑连在一起。

主要做的事情：

-   开发 Dapp 后端服务
    
-   处理链上数据、交易、事件
    
-   给前端提供 API（REST / GraphQL）
    
-   和智能合约工程师协作
    
-   保证系统稳定和性能
    

需要的能力：

-   Node.js / Go / Python 至少会一个
    
-   能用 Viem / Ethers 和链交互
    
-   会数据库（MySQL、PostgreSQL 等）
    
-   对智能合约的基本逻辑有理解
    
-   懂 Docker、CI/CD 会很加分
    

常见技术栈：  
Node.js / Go / Python、Viem、API、数据库、Docker

* * *

### 3\. 智能合约工程师

这是技术岗里门槛最高的一个。

我的理解是：  
**合约代码就是规则本身，写错就可能直接造成资产损失。**

主要做的事情：

-   用 Solidity 写智能合约
    
-   部署到以太坊或各种 L2
    
-   写测试、做安全审计
    
-   优化 Gas
    
-   和前后端一起完成产品逻辑
    

需要的能力：

-   Solidity 熟练
    
-   理解区块链底层原理（Gas、共识等）
    
-   熟悉安全问题（重入攻击、权限管理）
    
-   会用 Foundry / Hardhat
    
-   了解 ERC-20、NFT 等标准
    

这个岗位一般更偏向**有项目经验的人**，新手直接冲会比较吃力。

* * *

## 非技术岗笔记

### 1\. 产品与运营

本质是：**让产品被用户真正用起来**。

主要做的事情：

-   跟进产品上线和迭代
    
-   做用户增长、留存、活跃
    
-   协调技术、市场、合规
    
-   用数据判断哪些地方需要优化
    

需要的能力：

-   理解产品和用户
    
-   会看数据（Excel、SQL）
    
-   沟通和推进能力强
    

* * *

### 2\. 社区管理（Web3 很重要）

Web3 项目很多时候：  
**产品还没成熟，但社区一定要先有。**

主要做的事情：

-   运营 Telegram / Discord / Twitter
    
-   组织 AMA、活动、任务
    
-   管理社区情绪和节奏
    
-   输出公告和内容
    

需要的能力：

-   对 Web3 / DAO / NFT 有感觉
    
-   文案和沟通能力
    
-   能看社区数据和舆情
    
-   能独立策划活动
    

* * *

### 3\. 研究分析

更偏向理性和长期价值。

主要做的事情：

-   做行业和协议研究
    
-   分析市场和竞品
    
-   写研究报告或白皮书
    
-   支持代币经济模型设计
    

需要的能力：

-   数据分析（Excel / Python）
    
-   对 DeFi 和 Tokenomics 有理解
    
-   报告写作能力强
    
-   会用链上数据工具
    

## 我的一些阶段性想法

-   Web3 不是只有写代码一条路
    
-   从 JD 出发，比盲目学效率高很多
    
-   技术岗更看项目和实操
    
-   非技术岗更看理解力和表达力
    
-   这是一个非常“交叉”的行业
    
-   于我个人而言，我更倾向于运营向的岗位，未来想发展为懂一些技术的运营。

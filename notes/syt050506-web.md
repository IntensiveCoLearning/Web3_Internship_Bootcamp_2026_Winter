## 技术岗笔记

### 1\. Web3 前端工程师

我对 Web3 前端的理解是：  
**不是单纯写页面，而是负责“用户和区块链之间的交互层”。**

主要做的事情包括：

-   写 Dapp 的前端页面
    
-   实现钱包连接、交易签名、信息展示
    
-   调用智能合约，把链上数据展示出来
    
-   和后端对接 API
    
-   做性能优化和兼容性
    

需要的能力基本是：

-   HTML / CSS / JavaScript 基础
    
-   React 或 Vue（现在更多是 React + Next.js）
    
-   会用钱包（比如 MetaMask）
    
-   熟悉 Viem 和智能合约交互
    
-   对以太坊、Solana 这些链有基本认知
    

一个很重要的点：  
👉 **现在主流已经从 Ethers.js / Web3.js 转向 Viem 了**

常见技术栈：  
HTML、CSS、JavaScript、TypeScript、React、Next.js、Viem

* * *

### 2\. Web3 后端工程师

后端更像是一个“中枢”，负责把链上、链下和业务逻辑连在一起。

主要做的事情：

-   开发 Dapp 后端服务
    
-   处理链上数据、交易、事件
    
-   给前端提供 API（REST / GraphQL）
    
-   和智能合约工程师协作
    
-   保证系统稳定和性能
    

需要的能力：

-   Node.js / Go / Python 至少会一个
    
-   能用 Viem / Ethers 和链交互
    
-   会数据库（MySQL、PostgreSQL 等）
    
-   对智能合约的基本逻辑有理解
    
-   懂 Docker、CI/CD 会很加分
    

常见技术栈：  
Node.js / Go / Python、Viem、API、数据库、Docker

* * *

### 3\. 智能合约工程师

这是技术岗里门槛最高的一个。

我的理解是：  
**合约代码就是规则本身，写错就可能直接造成资产损失。**

主要做的事情：

-   用 Solidity 写智能合约
    
-   部署到以太坊或各种 L2
    
-   写测试、做安全审计
    
-   优化 Gas
    
-   和前后端一起完成产品逻辑
    

需要的能力：

-   Solidity 熟练
    
-   理解区块链底层原理（Gas、共识等）
    
-   熟悉安全问题（重入攻击、权限管理）
    
-   会用 Foundry / Hardhat
    
-   了解 ERC-20、NFT 等标准
    

这个岗位一般更偏向**有项目经验的人**，新手直接冲会比较吃力。

## 非技术岗笔记

### 1\. 产品与运营

本质是：**让产品被用户真正用起来**。

主要做的事情：

-   跟进产品上线和迭代
    
-   做用户增长、留存、活跃
    
-   协调技术、市场、合规
    
-   用数据判断哪些地方需要优化
    

需要的能力：

-   理解产品和用户
    
-   会看数据（Excel、SQL）
    
-   沟通和推进能力强
    

* * *

### 2\. 社区管理（Web3 很重要）

Web3 项目很多时候：  
**产品还没成熟，但社区一定要先有。**

主要做的事情：

-   运营 Telegram / Discord / Twitter
    
-   组织 AMA、活动、任务
    
-   管理社区情绪和节奏
    
-   输出公告和内容
    

需要的能力：

-   对 Web3 / DAO / NFT 有感觉
    
-   文案和沟通能力
    
-   能看社区数据和舆情
    
-   能独立策划活动
    

* * *

### 3\. 研究分析

更偏向理性和长期价值。

主要做的事情：

-   做行业和协议研究
    
-   分析市场和竞品
    
-   写研究报告或白皮书
    
-   支持代币经济模型设计
    

需要的能力：

-   数据分析（Excel / Python）
    
-   对 DeFi 和 Tokenomics 有理解
    
-   报告写作能力强
    
-   会用链上数据工具
    

## 我的一些阶段性想法

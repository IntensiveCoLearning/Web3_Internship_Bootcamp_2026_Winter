**1\. 大语言模型（LLM）** 通俗解释：AI的“聪明脑子”，专门处理人类语言的“超级大脑”，比如GPT系列、Claude、Deepseek都属于这类。 课程里的作用：负责看懂你的需求（比如“帮我做DeFi套利”）、定做事的步骤、下达行动指令，但它自己不会动手，得靠其他工具落地。 举例：就像公司里的“项目经理”，能听懂客户需求、制定执行计划，但需要程序员、设计师这些“下属”（工具）去把事情做完。 **2\. 自回归TOKEN预测器** 通俗解释：早期AI的“说话逻辑”，“TOKEN”就是“字、词或短语”，AI会根据你说的话，猜下一个最该出现的字/词，一步步拼出完整回答。 课程里的作用：解释早期AI“只会聊天不会干活”的本质——比如你让它写情书，它能逐字凑出内容，但让它订机票就不行，因为没法实际操作。 举例：类似玩“接话游戏”，你说“今天天气”，AI猜下一个词是“真好”，再接着猜下一个是“适合”，直到连成“今天天气真好，适合出门”。 **3\. Agentic AI（智能体AI）** 通俗解释：2026年进化后的“能干活的AI”，不光有聪明脑子（LLM），还有记性、手脚（工具）和身份证，能自己完成任务、赚钱。 课程里的作用：AI从“嘴替”变成“打工人”的关键，解决了早期AI没记性、不会动手、没身份的问题。 举例：就像一个全能管家，能记住你不吃香菜（有记忆）、帮你订酒店（会动手）、代表你签收快递（有身份），不用你全程盯着。 **4 . 向量数据库（Vector Database）** 通俗解释：AI的“长期记忆本”，专门存“数字化后的信息”（比如你和AI的聊天记录、文档内容），还能快速找到相关内容。 课程里的作用：解决早期AI“忘事”的问题——把重要信息存进去，AI下次回答时能快速调取，不用每次都“从零开始”。 举例：就像你的手机备忘录，把重要的事记下来，需要时直接搜关键词就能找到，AI靠它从“金鱼记忆”变成“大象记忆”。 **5\. Embedding（嵌入）** 通俗解释：把文字、文档这些“看不懂的信息”，转换成电脑能识别的“一串数字”的过程，相当于给信息做“专属编码”。 课程里的作用：向量数据库能存信息的前提——只有把文字转成数字，AI才能快速检索、记住。 举例：就像给每本书贴一个“条形码”，超市扫码能知道价格，AI扫这个“数字编码”能快速找到对应的信息。 **6\. Function Call（函数调用）** 通俗解释：AI的“手脚”，让AI能调用预设的功能（比如查数据、发邮件、执行代码），从“只说不做”变成“说到做到”。 课程里的作用：AI和外部工具沟通的关键——AI下达的不是普通文字，而是“执行指令”，工具收到后干活，再把结果反馈给AI。 举例：你让AI“查今天以太坊的收盘价”，AI通过Function Call调用数据查询工具，拿到价格后再用自然语言告诉你。 **7\. React结构** 通俗解释：AI完成复杂任务的“工作流程”，就是“想（Reason）→做（Act）→看结果（Observe）”的循环。 课程里的作用：让AI能持续推进任务，比如做套利交易——先想“哪个平台有差价”（Reason），再查数据（Act），最后根据结果调整策略（Observe），直到赚差价。 举例：就像你做饭，先想“要做番茄炒蛋”（Reason），再去切菜、炒蛋（Act），尝一口咸了就加点糖（Observe），循环调整直到做好。 **8\. ZKML（零知识机器学习）** 通俗解释：一种“保密又能证明”的技术——AI能证明“我确实按承诺的规则干活了”，但不泄露核心机密（比如模型参数、输入数据）。 课程里的作用：验证AI“没撒谎”，比如AI说自己用了高端模型，通过ZKML能证明它没骗你，还不泄露模型的核心技术。 举例：就像你考试得了100分，能证明自己会做这些题，但不用把解题思路、笔记全告诉别人。 **9\. TEE（可信执行环境）** 通俗解释：一个“安全隔离的小空间”（硬件层面），AI在里面运行时，数据和模型不会被篡改、窃取，还能生成“证明”（证明自己按规则来）。 课程里的作用：和ZKML类似，都是让AI的运行过程“可信”，比如AI执行交易时，在TEE里运行能确保没人中途改指令。 举例：就像一个带密码锁的保险箱，你把重要文件放进去，只有你能打开，别人看不到、改不了里面的内容。 **10.API （应用程序编程接口）** API 是 应用程序编程接口（Application Programming Interface）的缩写，简单说就是 “不同软件之间的沟通桥梁”。 核心作用：让两个完全不同的程序（比如 AI 和数据平台、APP 和服务器）能互相 “说话”、传递信息、调用功能，不用管对方内部是怎么工作的。 课程里的实际应用：AI 之所以能 “干活”，全靠 API 当 “手脚”：AI 想查链上交易数据，就通过 API 调用区块链的数据服务；AI 想执行套利交易，就通过 API 对接 DeFi 平台的交易功能；AI 想调用工具（比如发邮件、算数据），也是通过对应的 API 实现。 通俗举例：就像你去餐厅吃饭，不用进厨房自己做饭（不用懂餐厅内部运作），只要告诉服务员（API）“我要鱼香肉丝”（发送请求），服务员把需求传给厨房，最后把做好的菜（返回结果）给你 ——API 就是这个 “服务员”，连接你（调用方）和厨房（提供服务的程序）。 课程里的关键关联：API 是 AI 从 “只说不做” 变 “说到做到” 的核心：AI 生成的指令，通过 API 才能变成实际操作；API 经济规模在快速增长（每年 30%+），本质是机器之间（比如 AI 和 AI、AI 和系统）的协作越来越多，而 API 就是它们的 “沟通语言”。 **二、Web3相关技术名词** **1\. 区块链（Blockchain）** 通俗解释：一种“大家一起记的公开账本”，数据按顺序打包成“区块”，像链条一样连起来，所有人都有一份副本，没人能单独改。 课程里的作用：解决AI的“信任问题”——AI的身份、交易、决策记录上链后，改不了、删不掉，谁都能查，不用怕造假。 举例：就像班级里的“共同记账本”，谁借了谁的钱、借了多少，所有人都记下来，没人能偷偷改“借钱金额”。 **2\. 以太坊（Ethereum）** 通俗解释：最常用的“区块链平台”，不光能记交易，还能运行代码（智能合约），是Web3生态的“核心舞台”。 课程里的作用：AI的“身份证”（ERC8004）就是在以太坊上创建的，相当于给AI办身份证的“民政局”。 举例：就像一个大型商场，商家（开发者）能在里面开店（部署智能合约），消费者（用户）能在里面交易，AI的身份就在这里“注册”。 **3\. 智能合约（Smart Contract）** 通俗解释：部署在区块链上的“自动执行程序”，只要满足预设条件，就会自动干活，不用人管，结果改不了。 课程里的作用：AI执行交易、签协议的基础——比如AI套利时，满足“差价≥3%”的条件，智能合约就自动转账，不用人手动操作。 举例：就像自动售货机，你投币（满足条件），它就自动掉饮料（执行操作），没人能中途拦截、改规则。 **4\. SDK（软件开发工具包）** 通俗解释：开发者的“工具集合”，包含现成的代码、接口、说明书，不用从零写代码，直接调用就能实现功能。 课程里的作用：Sparkos提供的SDK，让开发者不用懂区块链、私钥管理，直接调用就能给AI办身份、开支付功能。 举例：就像做饭的“预制菜套装”，里面有切好的菜、调好的料，你不用自己洗菜、切菜，直接加热就能吃，SDK就是开发的“预制菜套装”。 **6\. 智能体编排引擎** 通俗解释：Sparkos里的“任务分配员”，能把复杂任务拆给多个AI，或者让多个AI协作，同步进度、汇总结果。 课程里的作用：让AI能完成复杂任务——比如“链上数据分析+套利”，编排引擎让“分析AI”找机会，“交易AI”执行，最后汇总结果。 举例：就像项目主管，把“做网站”的任务拆给“设计AI”（做界面）、“开发AI”（写代码）、“测试AI”（找bug），最后把成果汇总成完整网站。 **7\. 支付通道（Payment Channel）** 通俗解释：降低区块链交易手续费的“技巧”，双方先在链上存一笔钱当押金，之后多次交易在链下进行，最后一起结算，只付一次手续费。 课程里的作用：解决AI小额交易“手续费比本金高”的问题——比如AI每秒调用10次API，不用每次都付手续费，攒到最后一起结。 举例：就像你和朋友吃饭，每次AA都单独转账（每次付手续费），不如先记账，月底一起转（一次付手续费），省麻烦还省钱。 **8\. 批量结算（Batch Settlement）** 通俗解释：把多笔小额交易“攒起来一起付”，一次性上链结算，减少交易次数，降低手续费。 课程里的作用：和支付通道类似，都是为了降低AI高频小额交易的成本，让AI“赚钱更划算”。 举例：就像快递网点，把多个小包裹（小额交易）拼成一个大包裹（批量结算），一起发货，比单个发货省运费（手续费）。 **9\. SaaS（软件即服务）** 通俗解释：传统软件的“订阅模式”，比如视频会员、办公软件，按月/按年付费使用，不用自己安装、维护服务器。 课程里的作用：对比说明传统模式不适合AI——AI有时忙有时闲，订阅制会“高峰期不够用、低谷期浪费”，而X402的“用一次付一次”更适配。 举例：就像健身房年卡，你办了年卡但很少去，浪费钱；而AI需要“按次付费”，用一次付一次，不浪费。 **10\. RPA（机器人流程自动化）** 通俗解释：按固定规则自动干活的“机器人”，比如自动填表格、发邮件，但没有AI的决策能力，只能按预设步骤来。 课程里的作用：对比说明AI智能体的优势——RPA只能“按剧本演戏”，AI能自主决策、学习，还能赚钱，市场空间更大。 举例：就像自动扫地机器人，只能按预设路线扫地，遇到障碍物绕开，但不会自主判断“哪里脏先扫哪里”，而AI能做这个判断。 **四、其他高频名词** **1\. 女巫攻击（Sybil Attack）** 通俗解释：攻击者创建多个“假身份”（比如多个钱包地址），用来刷好评、刷差评，或者恶意攻击系统。 课程里的作用：声誉系统要防范的问题——ERC8004通过“只有真实交易用户能评分+付手续费”，提高攻击成本，减少假身份刷分。 举例：就像有人注册多个小号，给自家店铺刷好评，给竞争对手刷差评，女巫攻击就是这个意思。 **2\. KYC（了解你的客户）** 通俗解释：传统金融的“身份认证流程”，比如开银行账户要填身份证、人脸识别，证明你是“真实的人”。 课程里的作用：说明传统支付不适合AI——AI没有身份证、人脸，过不了KYC，而Web3钱包不用KYC，有地址就能支付。 举例：就像你去银行办卡，工作人员要查你身份证、拍照片，确认是你本人，这就是KYC，AI没法做这些步骤。 **3\. 无状态（Stateless）** 通俗解释：AI“记不住事”，每次对话都是全新开始，不记得之前聊过的内容、你的偏好。 课程里的作用：早期AI的致命短板——比如你跟AI聊了3小时人生规划，换个窗口它就全忘了，向量数据库解决了这个问题。 举例：就像一个没记性的店员，你刚跟他说“不要香菜”，下一次点单他又忘了，早期AI就是这样。 **4\. 上下文窗口（Context Window）** 通俗解释：AI能“记住的对话长度上限”，比如32K、64K，超过这个长度，AI就会忘记前面的内容。 课程里的作用：早期AI“忘事”的原因之一——对话太长，AI记不住前面的内容，向量数据库能扩展它的“记忆长度”。 举例：就像你的手机内存，内存小（上下文窗口小），打开多个APP就会卡顿、闪退；内存大（上下文窗口大），能同时运行多个APP，AI也是如此。 **5\. 套利（Arbitrage）** 通俗解释：利用不同平台、不同时间的价格差赚钱，比如A平台的代币卖1美元，B平台卖1.05美元，在A平台买、B平台卖，赚0.05美元差价。 课程里的作用：AI在DeFi中的常见应用——AI能实时监控价格差，快速执行交易，而且操作上链可查，可信透明。 举例：就像有人在批发市场以10元/斤买苹果，再以15元/斤在小区卖，赚中间的差价，AI做的就是类似的事，只是速度更快、范围更广

### 以太坊原理

以太坊可以理解成一条不仅能转账，还能运行程序的区块链。

-   ETH 是原生货币
    
-   支持智能合约（写好规则，满足条件自动执行）
    

以太坊是一个去中心化、所有人共同维护的计算系统。

### 从 PoW 换成 PoS（**从“消耗资源”变成“承担经济责任”**）

以前 PoW（挖矿）和比特币一样

-   拼算力、拼电
    
-   谁算得快，谁打包区块
    
-   安全，但浪费能源，也容易被大矿池垄断
    

现在改成 PoS：

-   拼“押钱”
    
-   想参与记账，先质押 ETH
    
-   作恶有惩罚机制，没收质押
    

### 区块产生原理

**验证者：**质押 32 ETH 就能成为验证者（更新后是32个及以上，可以不是刚好32）。负责参与投票、维护网络

**提议者：**每 12 秒随机选一个，负责提出新区块

**委员会：**随机选一批验证者，对新区块投票，超过 2/3 同意，区块才算通过

### 现在的以太坊两层分工合作

-   **执行层：**处理交易和智能合约，用 EVM 执行代码，交易要付 Gas，防止乱用资源
    
-   **共识层：**决定谁能出块，用 PoS 达成一致
    

### 防止作恶（**作恶成本高，有经济损失**）

-   想攻击网络，需要控制大部分质押的 ETH
    
-   一旦被发现，质押的 ETH 会被罚没
    
-   如果一群验证者一起犯错，罚得更重
    

### PS

-   矿工是 PoW 时代的概念，现在是验证者
    
-   质押多 ≠ 能控制网络
    
-   PoW 的难度调整是算法自动完成的，个人改不了
    
-   流程：用户发交易 → 执行层算结果 → 共识层投票 → 全网更新状态
